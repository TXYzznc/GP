# Unity Skills 操作记录规范

## 📋 概述

使用 Unity Skills MCP 工具执行 Unity 编辑器操作时,必须记录所有操作到指定的日志文件中,以便追踪和审查。

## 📁 记录位置

**记录文件夹**: `MCP工作区/Unity_MCP开发记录/`

**文件命名规则**: `Unity操作记录_YYYY-MM-DD.md`

**示例**: `Unity操作记录_2026-03-06.md`

## 📝 记录格式

每条操作记录必须包含以下信息:

```markdown
## [HH:MM] 操作简要描述

**工具名称**: `tool_name`

**调用参数**:
- 参数1: 值1
- 参数2: 值2
- 参数3: 值3

**执行结果**: ✅ 成功 / ❌ 失败

**关键输出**:
- 输出信息1
- 输出信息2

**备注**: 
- 相关说明或注意事项

---
```

## 🔧 记录时机

### 必须记录的操作

所有通过 Unity Skills MCP 工具执行的操作都必须记录,包括但不限于:

- **GameObject 操作**: 创建、删除、修改、重命名
- **Component 操作**: 添加、移除、设置属性
- **Material 操作**: 创建、设置颜色、设置纹理
- **Scene 操作**: 保存、加载、切换
- **Asset 操作**: 导入、删除、移动
- **其他所有 Unity Skills 工具调用**

### 记录流程

1. **执行前**: 明确操作目的
2. **执行中**: 调用 Unity Skills 工具
3. **执行后**: 立即记录操作详情

## 🎯 记录最佳实践

### 1. 及时记录
- 完成操作后立即记录,避免遗忘细节
- 不要等到一天结束才批量记录

### 2. 详细描述
- 记录足够的上下文信息
- 说明操作的目的和预期结果
- 记录遇到的问题和解决方案

### 3. 标准化格式
- 使用统一的 Markdown 格式
- 保持时间戳格式一致 [HH:MM]
- 使用 ✅ 和 ❌ 标识成功/失败

### 4. 关联信息
- 记录相关的 GameObject 实例ID
- 记录文件路径和资源引用
- 记录依赖关系和前置条件

### 5. 备注说明
- 记录操作的业务目的
- 记录后续需要的操作
- 记录注意事项和陷阱

## 🔍 记录查阅

### 查找特定操作
使用文本搜索工具在记录文件中查找:
- 按工具名称搜索: `**工具名称**: \`gameobject_create\``
- 按对象名称搜索: `name: Ground`
- 按时间范围搜索: 打开对应日期的文件

### 分析操作历史
- 统计每天的操作数量
- 识别常用的工具和操作模式
- 发现重复性操作,考虑自动化

## ⚠️ 注意事项

### 1. 不要使用自动钩子
- postToolUse 钩子会导致循环触发
- 无法准确识别 Unity Skills 工具
- 手动记录更可靠和灵活

### 2. 保护敏感信息
- 不要记录密码或密钥
- 不要记录个人身份信息
- 使用占位符代替敏感数据

### 3. 定期归档
- 每月归档旧的记录文件
- 保留最近 3 个月的记录在工作区
- 长期归档到项目文档库

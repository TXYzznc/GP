# 棋子手动移动后丢失索敌目标修复需求文档

## 介绍

修复棋子在玩家手动移动操作后丢失索敌目标，无法重新进行正常索敌的问题。该问题导致棋子在移动完成后持续出现"目标不在攻击范围内"警告，AI无法重新搜索和锁定新目标，严重影响游戏体验。

## Bug 分析

### 当前行为（缺陷）

1.1 当玩家手动移动棋子到新位置后，系统会恢复AI状态为Idle，但棋子会持续尝试攻击移动前的旧目标

1.2 当旧目标不在新位置的攻击范围内时，系统会持续输出"目标不在攻击范围内"警告，但不会重新搜索目标

1.3 当棋子移动到新位置后，目标搜索计时器可能没有正确重置，导致需要等待完整的搜索间隔才能重新搜索

### 期望行为（正确）

2.1 当玩家手动移动棋子完成后，系统应该立即清除旧目标并重新搜索适合的敌人目标

2.2 当棋子移动到新位置后，系统应该基于新位置重新评估攻击范围和目标优先级

2.3 当AI状态恢复为Idle时，系统应该立即触发目标搜索，而不是等待搜索计时器

### 不变行为（回归预防）

3.1 当棋子进行非玩家控制的AI移动时，系统应该继续保持现有的目标搜索和攻击逻辑

3.2 当棋子在攻击过程中未被玩家移动时，系统应该继续保持现有的攻击状态管理

3.3 当棋子的目标搜索间隔机制在正常AI行为中时，系统应该继续按现有逻辑工作

# 游戏测试管理窗口使用指南

## 打开测试管理窗口

在 Unity 菜单栏中：**Tools > Clash of Gods > Test Manager**

## 功能区划分

### 📦 背包系统（Inventory）
- **打开背包** / **关闭背包** - 显示/隐藏背包UI
- **添加物品** - 随机添加5-10种物品到背包
- **移除物品** - 移除第一个可堆叠物品3个
- **使用物品** - 使用背包中的第一个可用物品
- **可堆叠测试** - 测试物品堆叠功能
- **背包满测试** - 填满背包以测试满载状态
- **存档读档** - 测试背包数据保存和加载
- **边界测试** - 测试边界情况（无效ID、超量移除等）
- **打印状态** - 输出当前背包内容到控制台
- **清空背包** - 清空所有背包物品

### ⚔️ 战斗系统（Combat）
- **测试配置加载** - 加载战斗配置表并显示信息
- **生成战斗场地** - 生成战斗竞技场环境

### 👹 敌人系统（Enemy）
- **生成敌人** - 生成单个敌人进行测试
- **批量生成** - 批量生成敌人（数量、排列方式由Inspector配置）
- **清除所有敌人** - 销毁场景中所有敌人

### 🔫 投射物系统（Projectile）
- **发射投射物** - 发射单个投射物（随机方向）
- **重置统计** - 重置投射物发射统计数据

### 🎲 游戏状态（Game State）
- **切换到探索** - 切换到探索状态
- **切换到战斗准备** - 切换到战斗准备状态

## 注意事项

1. **组件依赖** - 每个功能区都依赖对应的Controller/Manager组件存在于场景中
   - 如果场景中缺少组件，会显示警告信息

2. **测试日志** - 所有测试功能都会输出详细的日志
   - 打开 **Window > General > Console** 查看日志

3. **性能考虑**
   - 批量生成敌人时会根据配置数量生成，数量过多可能影响性能
   - 投射物发射会消耗性能，建议在性能测试时关闭自动发射

4. **状态恢复** - 大多数测试操作是单向的
   - 例如"清空背包"后需要手动重新添加物品
   - "清除所有敌人"后需要重新生成

## 快捷键已迁移说明

以下快捷键已全部迁移到此Test Manager窗口：

| 原快捷键 | 功能 | 新位置 |
|---------|------|-------|
| Tab | 打开/关闭背包 | 游戏内快捷键（保留） |
| U | 切换到探索状态 | 游戏状态区 |
| O | 切换到战斗准备状态 | 游戏状态区 |
| Space | 发射投射物 | 投射物系统区 |
| A | 切换自动发射 | Inspector配置 |
| R | 重置投射物统计 | 投射物系统区 |
| H | 显示/隐藏UI | Inspector配置 |
| 左右箭头 | 切换投射物 | Inspector配置 |
| F1-F8 | 敌人动作测试 | 敌人系统区（需要扩展） |

## 扩展指南

如需添加新的测试功能：

1. 在对应的Tester/Controller中创建 `public` 方法
2. 在 `GameTestWindow.cs` 中对应的Section里添加按钮
3. 使用 `GUILayout.Button()` 创建按钮
4. 在点击事件中调用对应方法

**示例代码**：
```csharp
if (GUILayout.Button("新功能", GUILayout.Height(BUTTON_HEIGHT)))
{
    m_MyController.MyTestMethod();
}
```

# 修复棋子状态持久化问题 - 提案

## 问题描述

当出战的棋子在战斗过程中死亡后，下一次战斗时该棋子的状态没有正确同步：
- **现象**：每次战斗棋子血量都恢复到满状态
- **预期**：棋子的状态应该全局继承，只有回到局外才能通过特殊道具恢复状态
- **影响**：破坏了游戏的战斗连贯性和难度平衡

## 核心需求

1. **状态**：棋子在战斗中受到的状态效果（如中毒、冻结等）应该在战斗结束后重置（状态是需要清除的，不会留到下一次战斗）
2. **血量管理**：棋子的当前血量应该在战斗间隙保持，而不是每次战斗都重置
3. **全局继承**：棋子的状态数据应该从全局数据源读取，而不是从战斗场景本地数据读取
4. **恢复机制**：离开局内，回到基地（局外时）自动恢复，其他时候只有使用特殊道具才能恢复。需要注意受伤和死亡：当棋子血量<=0时死亡，棋子可以回复血量（通过道具、技能等），但是没有特殊说明，只有受伤的棋子可以回复血量（已死亡的不能回复）

## 技术影响范围

- **棋子数据管理**：需要区分"全局棋子数据"和"战斗场景棋子数据"
- **战斗初始化**：战斗开始时需要从全局数据同步棋子状态
- **战斗结束处理**：战斗结束时需要将棋子状态回写到全局数据
- **UI 更新**：需要确保 UI 显示的是最新的全局状态

## 预期收益

- 修复游戏逻辑 Bug，提升游戏体验
- 增强战斗的策略性和难度感
- 为后续的状态管理系统奠定基础

## 实现优先级

**高** - 这是一个影响核心游戏逻辑的 Bug，需要尽快修复

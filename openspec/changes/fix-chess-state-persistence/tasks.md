# 修复棋子状态持久化问题 - 实现任务

## 任务清单

### 第一阶段：数据结构和基础设施

#### ✅ Task 1.1: 定义全局棋子状态数据结构
- **目标**：创建 `GlobalChessState` 类，用于存储棋子的持久化状态
- **包含内容**：
  - 棋子 ID、当前血量、最大血量
  - 等级、经验等其他持久化属性（需要加一个新的规则配置表，专门配置不同等级需要的经验，以及多少级可以进阶。注意，棋子等级也是局内效果，不属于全局的局外养成，即每次从基地开始游戏，进入局内时，都会刷新棋子的等级和经验）
  - **不包含**：临时状态效果（Buff）
- **文件**：`Assets/Scripts/Game/Chess/Data/GlobalChessState.cs`
- **验收标准**：
  - 类定义完整，包含所有必要字段
  - 包含日志输出（DebugEx）

#### ✅ Task 1.2: 定义战斗棋子数据结构
- **目标**：创建 `BattleChessData` 类，用于战斗场景中的临时数据
- **包含内容**：
  - 血量相关字段（当前血量、最大血量）
  - 临时状态效果列表（Buff）
  - `IsDead` 属性：判断血量 <= 0
  - `CanRecover` 属性：判断是否可以恢复血量（未死亡且血量 < 最大值）
- **文件**：`Assets/Scripts/Game/Battle/Data/BattleChessData.cs`
- **验收标准**：
  - 类定义完整
  - 提供从 `GlobalChessState` 创建副本的方法
  - 包含日志输出（DebugEx）

#### ✅ Task 1.3: 创建全局棋子管理器
- **目标**：实现 `GlobalChessManager` 单例，管理全局棋子状态
- **职责**：
  - 维护所有棋子的全局状态数据（仅血量和基础属性）
  - 提供状态查询接口：`GetChessState(int chessId)`
  - 提供血量更新接口：`UpdateChessHP(int chessId, int newHP)`
  - 提供基地恢复接口：`RestoreAllChessHP()`
  - 提供道具恢复接口：`TryRecoverChessHP(int chessId, int recoverAmount)`（仅限受伤的棋子）
  - 可以提供复活的接口，有效果可以复活（复活血量可以设置）
- **文件**：`Assets/Scripts/Game/Chess/Manager/GlobalChessManager.cs`
- **验收标准**：
  - 单例模式实现正确
  - 所有接口实现完整
  - 包含日志输出（DebugEx）
  - 道具恢复时检查棋子是否已死亡

#### ✅ Task 1.4: 创建战斗棋子管理器
- **目标**：实现 `BattleChessManager`，管理战斗场景中的棋子数据
- **职责**：
  - 初始化战斗棋子数据（从全局管理器同步血量）
  - 提供战斗数据查询接口：`GetBattleChessData(int chessId)`
  - 提供伤害接口：`DamageChess(int chessId, int damageAmount)`
  - 提供 Buff 管理接口：`AddBuffToChess()`, `RemoveBuffFromChess()`
  - 处理战斗结束时的数据回写（仅血量，清除所有 Buff）
- **文件**：`Assets/Scripts/Game/Battle/Manager/BattleChessManager.cs`
- **验收标准**：
  - 提供 `Initialize(List<int> chessIds)` 方法
  - 提供 `OnBattleEnd()` 方法
  - 包含日志输出（DebugEx）

### 第二阶段：战斗系统集成

#### ✅ Task 2.1: 修改战斗初始化流程
- **目标**：在战斗开始时调用数据同步
- **修改内容**：

  - 确保所有棋子数据正确加载

- **验收标准**：

  - 临时状态效果初始化为空

#### ✅ Task 2.2: 修改战斗结束流程
- **目标**：在战斗结束时回写棋子状态
- **修改内容**：
  - 在战斗结束时调用 `BattleChessManager.OnBattleEnd()`
  - 将战斗数据的血量回写到全局管理器
  - 清除所有临时状态效果

- **文件**：战斗管理器相关文件

  - 日志显示数据回写过程

#### ✅ Task 2.3: 确保所有伤害和 Buff 修改使用战斗数据
- **目标**：审查战斗逻辑，确保所有状态修改都作用于 `BattleChessData`
- **修改内容**：
  - 检查棋子受伤逻辑，使用 `BattleChessManager.DamageChess()`
  - 检查 Buff 应用逻辑，使用 `BattleChessManager.AddBuffToChess()`
  - 检查 Buff 移除逻辑，使用 `BattleChessManager.RemoveBuffFromChess()`

### 第三阶段：基地和道具系统集成

- **修改内容**：

#### ✅ Task 3.2: 实现道具恢复逻辑
- **目标**：使用恢复道具时，恢复受伤棋子的血量
- **修改内容**：
  - 在使用恢复道具时调用 `GlobalChessManager.TryRecoverChessHP()`

- **文件**：道具系统相关文件
- **验收标准**：

  - 已死亡的棋子不能通过道具恢复

## 关键实现要点

### 数据流向
```
全局数据 (血量)
    ↓ 战斗开始时复制
战斗数据 (血量 + 临时 Buff)
    ↓ 战斗进行中修改
战斗数据 (修改后的血量 + Buff)
    ↓ 战斗结束时回写（仅血量）
全局数据 (更新后的血量)
    ↓ 回到基地时恢复
全局数据 (满血量)
```

### 状态效果处理
- **战斗中**：状态效果存储在 `BattleChessData` 中
- **战斗结束**：所有状态效果被清除
- **全局数据**：不存储任何临时状态效果

### 血量恢复规则
- **受伤棋子**：可以通过道具或技能恢复血量
- **已死亡棋子**：不能通过道具或技能恢复，只能回到基地后自动恢复
- **基地恢复**：所有棋子血量恢复到最大值

# 卡牌解锁物品配置表设计文档

## 需求分析

### 功能目标
1. 为每张策略卡创建对应的解锁物品
2. 这些物品是消耗品，使用后可解锁对应卡牌
3. 已解锁的卡牌不能再使用对应解锁物品
4. 按品质对解锁物品进行分类

## 配置表修改方案

### 一、ItemTable 扩展

#### 新增字段说明
无需新增字段，利用现有字段：
- **UseEffectId**: 用于执行"解锁卡牌"的特殊效果（新增效果ID）
- **Quality**: 与对应卡牌 Rarity 保持一致
- **CanUse**: 固定为 1（可使用）
- **CanStack**: 固定为 1（可堆叠）
- **MaxStackCount**: 固定为 1（最多堆叠1个）

## 关键设计决策

### 1. ID 分配策略
- 卡牌解锁物ID从100开始顺序分配
- 保留足够的缓冲空间（当前最大ID 125）
- 便于后续扩展更多卡牌

### 2. 品质对应关系
```
ItemTable.Quality ←→ CardTable.Rarity
    稀有(2)     ←→   稀有(2)
    史诗(3)     ←→   史诗(3)
    传说(4)     ←→   传说(4)
```

### 3. UseEffectId 规范
- 使用 20xx 的 ID 段用于卡牌解锁效果
- 便于与其他效果区分
- 满足未来扩展需求

### 4. 堆叠限制
- MaxStackCount = 1: 每次只能使用一个
- 避免多次消耗同一物品
- 增强稀有卡牌的获取成本感

## 后续开发提示

### 需要实现的业务逻辑

1. **卡牌解锁效果处理** (`UseEffectId: 2001-2026`)
   ```csharp
   // 检查卡牌是否已解锁
   bool isUnlocked = playerData.IsCardUnlocked(cardId);
   if (isUnlocked) {
       // 显示"卡牌已解锁"提示，禁用使用
       return false;
   }
   
   // 解锁卡牌
   playerData.UnlockCard(cardId);
   // 消耗物品
   inventory.RemoveItem(itemId, 1);
   ```

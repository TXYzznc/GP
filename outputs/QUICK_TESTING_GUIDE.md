# 快速测试指南 - 背包快照与结算资源计算

**用途**: 快速验证核心功能是否正确运行  
**耗时**: 每个场景 3-5 分钟  
**工具**: Unity Editor 或真机测试  

---

## 前置检查

在开始测试前，请确认以下项目已就位：

- [ ] ItemTable 包含 Value 字段（用于计算背包价值）
- [ ] PlayerAccountDataManager 已实现 AddOriginStone() 方法
- [ ] SettlementUIForm 已在 UIViews 枚举中定义
- [ ] Unity 项目已编译通过，无语法错误

**如果以上任何项目未完成，请先完成配置后再开始测试。**

---

## 快速验证流程

### 快速验证 1：编译检查（2 分钟）

**目标**: 确保所有代码编译正确

**步骤**:
1. 打开 Unity Editor
2. 等待脚本编译完成
3. 检查 Console 窗口是否有 compile errors

**预期结果**:
```
✅ 无编译错误
✅ 无 missing method 错误
✅ 无类型不匹配错误
```

**如果失败**:
- 检查是否所有的 using 语句都正确
- 验证 PlayerAccountDataManager 中是否有 AddOriginStone() 方法
- 查看错误日志具体指向哪个文件

---

### 快速验证 2：快照创建（3 分钟）

**目标**: 验证进入局内时快照是否正确创建

**步骤**:
1. 启动游戏
2. 进入局内（任意地图）
3. 打开 Console 窗口
4. 搜索日志关键字：`已创建背包快照`

**预期日志**:
```
[InGameState] 进入局内状态
[InGameState] 已创建背包快照
```

**如果失败**:
- 检查 InGameState.OnEnter() 中是否正确调用了 CreateSnapshot()
- 验证 InventoryManager.Instance 是否为 null
- 查看是否有其他初始化错误日志

---

### 快速验证 3：背包价值计算（4 分钟）

**目标**: 验证背包价值计算是否正确

**准备**:
1. 进入局内，背包为空
2. 拾取几个物品（建议拾取价值已知的物品）

**步骤**:
1. 打开 Console 窗口
2. 触发结算（传送回基地或死亡）
3. 搜索日志关键字：`背包价值对比`

**预期日志**:
```
[SettlementManager] 背包价值对比: 进入局内=0, 当前=XX, 收益=XX
```

**验证方法**:
- 如果进入局内背包为空，进入局内价值应该=0
- 如果拾取了价值为 10 的物品 3 个，当前价值应该=30
- 收益应该等于 当前价值 - 进入局内价值

**如果失败**:
- 检查 ItemTable 是否为所有物品配置了 Value 字段
- 验证 CalculateInventoryValue() 是否正确访问 ItemTable
- 查看是否有 ItemTable 未加载的警告日志

---

### 快速验证 4：宝箱虚拟物品处理（5 分钟）

**目标**: 验证 TakeAll() 时虚拟物品是否正确转换

**准备**:
1. 进入局内
2. 打开宝箱（宝箱内包含虚拟物品）
3. 记录操作前的账号资源（金币、起源石）

**步骤**:
1. 点击 "全部拿走" 按钮
2. 打开 Console 窗口
3. 搜索日志关键字：`TreasureBoxContainer`

**预期日志**:
```
[TreasureBoxContainer] 金币 xN → 账号资源
[TreasureBoxContainer] 起源石 xN → 账号资源
[TreasureBoxContainer] 灵石 xN → 删除（局内货币）
[TreasureBoxContainer] 全部拿走: 成功 X 件
```

**验证方法**:
1. 检查账号资源是否增加了相应的金币和起源石
2. 检查背包中是否仍有虚拟物品（应该没有）
3. 检查灵石是否被删除（应该不存在于任何地方）

**如果失败**:
- 检查 TakeAll() 中虚拟物品处理的 switch 语句
- 验证 PlayerAccountDataManager.AddGold() 和 AddOriginStone() 是否正确
- 检查宝箱是否正确生成了虚拟物品

---

### 快速验证 5：完整结算流程（5 分钟）

**目标**: 验证从进入局内到结算的完整流程

**准备**:
1. 记录操作前的账号资源：当前金币 = C1, 起源石 = S1
2. 进入局内（初始背包为空或已知物品）

**步骤**:
1. 拾取物品（建议拾取价值已知的物品，如苹果 x3，假设每个 Value=10）
2. 打开宝箱，全部拿走（假设获得金币 x100，起源石 x10，灵石 x50）
3. 触发结算（传送或死亡）
4. 打开 Console，搜索以下日志：

**预期日志序列**:
```
[SettlementManager] 开始收集结算数据...
[SettlementManager] 背包价值对比: 进入局内=0, 当前=30, 收益=30
[InventoryManager] 虚拟物品清理: 金币 x0 → 账号资源  (x0 because TakeAll 已清理)
[SettlementManager] 数据收集完成: 资源收益=30, 金币=0, 起源石=0, 灵石=0
[SettlementManager] 开始应用结算奖励
[SettlementManager] 获得资源（价值）: 30
[SettlementManager] 灵石（局内货币）已清理: 50
[InventoryManager] 背包快照已清除
[SettlementManager] 奖励应用完成，存档已保存
```

**最终验证**:
- 当前金币 = C1 + 30（背包价值差）+ 100（宝箱虚拟金币）= C1 + 130
- 当前起源石 = S1 + 10（宝箱虚拟起源石）

**如果失败**:
- 检查日志序列中是否有缺失步骤
- 验证账号资源增加值是否正确
- 查看是否有异常错误日志

## 常见问题诊断

### 问题：快照未创建

**症状**: 日志中找不到 `已创建背包快照`

**诊断步骤**:
1. 检查 InGameState.OnEnter() 中是否有 CreateSnapshot() 调用
2. 检查 InventoryManager.Instance 是否为 null
3. 添加 Debug.Log 跟踪执行流程

**解决方案**:
- 确保 InventoryManager 已在场景中初始化
- 确保 InGameState 的 OnEnter 被正确调用
- 检查是否有其他初始化错误阻断了执行

---

### 问题：背包价值计算为 0

**症状**: 日志显示 `收益=0`，即使拾取了物品

**诊断步骤**:
1. 检查 ItemTable 是否正确加载
2. 检查拾取的物品是否配置了 Value 字段
3. 打开 ItemTable，搜索相应物品的 Value 值

**解决方案**:
- 为所有物品配置 Value 字段（可使用 SellPrice 作为基础）
- 确保 ItemTable 已在 DataTable 中加载
- 检查是否有 "ItemTable 未加载" 的警告日志

---

### 问题：虚拟物品未转换

**症状**: TakeAll() 后，虚拟物品仍在背包中

**诊断步骤**:
1. 检查虚拟物品 ID 是否为 999/9999/99999
2. 检查 TakeAll() 中的 switch 语句是否覆盖了这些 ID
3. 检查 PlayerAccountDataManager.AddGold() 是否正确

**解决方案**:
- 验证虚拟物品的 ID 与 InventoryManager 中定义的常量一致
- 检查宝箱生成物品时是否使用了正确的虚拟物品 ID
- 添加 Debug.Log 跟踪 TakeAll() 的执行流程

---

### 问题：账号资源未增加

**症状**: 结算完成后，账号金币/起源石未增加

**诊断步骤**:
1. 检查日志中是否有 "获得金币" 或 "获得起源石" 的日志
2. 检查 PlayerAccountDataManager 中是否正确实现了增加方法
3. 检查存档是否正确保存

**解决方案**:
- 验证 ApplyRewardsAsync() 中的所有 Add 方法调用
- 确保 PlayerAccountDataManager.AddGold() 和 AddOriginStone() 都已实现
- 检查是否有异常错误阻止了奖励应用

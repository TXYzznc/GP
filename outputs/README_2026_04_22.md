## 🎓 关键概念速览

### 背包快照系统

```
进入局内
  ↓
CreateSnapshot()     ← 保存当前背包
  ↓
[游戏进行]
  ↓
结算触发
  ↓
GetSnapshot()        ← 获取之前保存的背包
  ↓
对比价值差           ← 计算收益
  ↓
ClearSnapshot()      ← 清除快照
```

### 虚拟物品双重处理

```
宝箱获得虚拟物品
  ├─ TakeAll() 处理
  │  └─ 虚拟物品 → 账号资源（立即）
  │
  └─ 结算 ConvertVirtualItems() 处理
     └─ 残留虚拟物品 → 账号资源（保底）
```

### 资源收益计算

```
ResourceGain = max(0, 当前背包价值 - 进入前背包价值)

最终金币 = 原有金币 + ResourceGain + VirtualGold
最终起源石 = 原有起源石 + VirtualOriginStone
```

### 必须完成的配置

在进行功能测试前，请确认：

1. **ItemTable.Value 字段**
   - 所有物品必须配置 Value 字段
   - 建议值 = SellPrice 或单独估值

2. **PlayerAccountDataManager 方法**
   - AddOriginStone() 必须实现
   - 建议参考 AddGold() 的实现方式

3. **SettlementUIForm 定义**
   - 必须在 UIViews 枚举中定义
   - 必须有对应的 Prefab 和脚本

### 常见陷阱

- ❌ 不要在 ItemTable 中重复配置虚拟物品的 Value
- ❌ 不要修改虚拟物品的 ID 常量（999/9999/99999）
- ❌ 不要跳过 ClearSnapshot() 的调用
- ❌ 不要在 TakeAll() 中修改虚拟物品处理逻辑

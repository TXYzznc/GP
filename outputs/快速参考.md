## 📝 关键代码路径

### 物品使用流程
```
ConsumableItem.OnUse()
  ↓
ItemEffectExecutor.ExecuteEffect(effectId)
  ↓
ItemEffectFactory.Create(effectType)  // 工厂模式
  ↓
effect.Execute(context)               // 多态调用
```

### 效果执行参数
```csharp
// 在 EffectParams 中配置 JSON
context.GetParam<int>("cardId", 0)    // 获取卡牌 ID
context.GetPlayerData()                // 获取玩家数据
```

## 🎓 架构对比

### 旧架构（switch-case）
```csharp
switch (effectType)
{
    case "AddGold":
        return ExecuteAddGold(effectData);
    case "UnlockCard":
        return ExecuteUnlockCard(effectData);
    // ... 每次新增效果都要修改这里
}
```

### 新架构（工厂模式）
```csharp
var effect = ItemEffectFactory.Create(effectType);
return effect?.Execute(context) ?? false;
// 新增效果只需：1. 创建类，2. 注册，3. 配置表
```

## 📞 常见问题

**Q: 如何自定义效果参数？**  
A: 在 EffectParams JSON 中添加参数，使用 `context.GetParam<T>(key)` 读取

**Q: 旧消耗品还能用吗？**  
A: 可以。有工厂就走工厂，没有就回退到旧 switch-case

**Q: 需要修改现有的消耗品吗？**  
A: 不需要。新旧并存，逐步迁移

**Q: 如何测试新效果？**  
A: 在 ItemTable 中配置 UseEffectId，然后使用物品即可

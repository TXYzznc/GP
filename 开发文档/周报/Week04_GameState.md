# 第四周周报：游戏状态管理与角色状态机

## 一、思维导图状态
- **本周已完成的内容（绿色）**：
  - 核心玩法开发-角色系统（状态切换）
  - 探索与战斗系统（状态机架构）

## 二、本周完成的具体工作
1. **状态机架构**：搭建 `GameStateManager` 核心架构，定义 `OutOfGame`（局外）和 `InGame`（局内）主状态。
2. **子状态实现**：实现局内子状态机：`ExplorationState`（探索）和 `CombatState`（战斗），并实现切换逻辑。
3. **角色状态**：完善 `PlayerController` 的状态管理，实现 Idle -> Walk -> SlowRun -> FastRun 的状态切换逻辑（支持双击W奔跑）。
4. **UI联动**：实现 `StateAwareUIForm` 基类，让 UI 能根据游戏状态自动显示/隐藏。

## 三、遇到的问题及解决方案
- **问题**：状态切换时 UI 没有正确关闭或开启，且代码耦合度高。
- **解决方案**：引入事件系统（EventComponent），在状态进入/退出时派发事件，UI 组件监听事件自动响应，解耦状态与UI。

2. 实现新游戏创建与角色数据初始化。

## 🚀 新项目初始化步骤

### 步骤 1：清理示例代码

**方式一：手动删除（推荐，更安全）**

1. 删除示例脚本：
   - 删除 `Scripts/Demo/` 整个文件夹
   - 删除 `Scripts/DataModel/PlayerDataModel.cs`
   - 删除示例 DataTable（保留 Core 文件夹）
   - 删除示例 Entity（可选保留通用组件）
   - 删除示例 EventArgs（保留 GFEventArgs.cs）
   - 删除示例 Procedures（保留 PreloadProcedure.cs 和 ChangeSceneProcedure.cs）
   - 删除示例 UI（可选保留 ToastTips.cs、SettingDialog.cs、LanguagesDialog.cs）

2. 删除示例资源：
   - 删除对应的 UI Prefabs
   - 删除示例 Entity Prefabs
   - 删除示例模型和材质
   - 删除 `Scene/Test.unity`

3. 删除示例数据表：
   - 删除 `AAAGameData/DataTables/` 下的示例 Excel 文件
   - 保留 `Core/` 文件夹下的所有文件

**方式二：使用脚本批量删除（快速但需谨慎）**

创建一个编辑器脚本来批量删除示例文件（我可以为你生成）

### 步骤 2：清理数据表配置

1. 打开 `AAAGameData/DataTables/Core/UITable.xlsx`
2. 删除所有示例 UI 的配置行（保留表头）
3. 只保留 ToastTips 的配置（如果保留了该 UI）

### 步骤 3：重新生成数据表代码

1. 在 Unity 编辑器中，点击菜单：`Tools/GameData/Generate DataTables`
2. 这会重新生成所有数据表代码，删除不存在的表

### 步骤 4：修改 AppConfigs 配置

1. 在 Unity 中找到 `Assets/AAAGame/ScriptableAssets/Core/AppConfigs`
2. 修改配置：
   - **DataTables**: 删除示例数据表名称，只保留 Core 表
   - **Procedures**: 修改流程列表为你的新流程

示例配置：
```
DataTables:
- EntityGroupTable
- LanguagesTable
- SoundGroupTable
- UIGroupTable
- UITable

Procedures:
- PreloadProcedure
- ChangeSceneProcedure
- YourMenuProcedure      # 你的新菜单流程
- YourGameProcedure      # 你的新游戏流程
```

### 步骤 5：创建新的启动流程

创建你的第一个流程（例如主菜单流程）：

```csharp
// Assets/AAAGame/Scripts/Procedures/MainMenuProcedure.cs
using GameFramework.Fsm;
using GameFramework.Procedure;
using UnityGameFramework.Runtime;

public class MainMenuProcedure : ProcedureBase
{
    protected override void OnEnter(IFsm<IProcedureManager> procedureOwner)
    {
        base.OnEnter(procedureOwner);
        
        // 打开主菜单 UI
        // GF.UI.OpenUIForm(UIViews.MainMenu);
        
        GF.Log("进入主菜单流程");
    }

    protected override void OnLeave(IFsm<IProcedureManager> procedureOwner, bool isShutdown)
    {
        // 清理工作
        base.OnLeave(procedureOwner, isShutdown);
    }
}
```

### 步骤 6：修改预加载流程

修改 `PreloadProcedure.cs`，让它在预加载完成后进入你的新流程：

```csharp
// 在 OnUpdate 方法的最后，修改切换流程的代码
if (loadedProgress >= totalProgress && smoothProgress >= 0.99f)
{
    preloadAllCompleted = true;
    InitGameFrameworkSettings();
    GF.Log("预加载完成, 进入主菜单.");
    
    // 修改这里，切换到你的新流程
    ChangeState<MainMenuProcedure>(procedureOwner);
}
```

## ⚙️ 项目配置

### 1. 修改项目名称和包名

1. **修改项目名称**
   - Unity 菜单：`Edit > Project Settings > Player`
   - 修改 `Product Name`

2. **修改包名**
   - 修改 `Package Name`（例如：com.yourcompany.yourproject）

3. **修改应用图标**
   - 设置 `Default Icon`

### 2. 配置 UI 设计分辨率

1. 找到 `Assets/Resources/AppSettings`
2. 修改 `Design Resolution`（例如：1920x1080）
3. 点击 `Conform` 按钮应用

### 3. 配置资源模式

在 `AppSettings` 中设置：
- **Package**: 单机模式，资源打包在应用内
- **Updatable**: 可更新模式，支持资源热更新
- **UpdatableWhilePlaying**: 边玩边下载模式

### 4. 配置热更新 URL

如果使用热更新，在 `AppSettings` 中设置：
- `Check Version Url`: 版本检测服务器地址

### 5. 配置多语言

1. 打开 `AAAGameData/Languages/` 目录
2. 编辑多语言 Excel 文件
3. 使用工具生成 JSON 文件

## 🎯 开发第一个功能

### 示例：创建一个简单的主菜单

#### 1. 创建数据表

**创建 Excel 文件**：`AAAGameData/DataTables/PlayerTable.xlsx`

```
| ID注释 | 名称注释 | 等级注释 |
| Id     | Name     | Level    |
| int    | string   | int      |
| 1      | 新手     | 1        |
```

**生成代码**：`Tools/GameData/Generate DataTables`

#### 2. 创建 UI 界面

**创建 UI 脚本**：`Assets/AAAGame/Scripts/UI/MainMenuUI.cs`

```csharp
using UnityEngine;
using UnityEngine.UI;

public class MainMenuUI : UIFormBase
{
    // 使用序列化字段工具自动生成引用
    private Button btnStart;
    private Button btnSettings;
    private Button btnQuit;

    protected override void OnInit(object userData)
    {
        base.OnInit(userData);
        
        // 获取组件引用（或使用序列化字段工具）
        btnStart = transform.Find("BtnStart").GetComponent<Button>();
        btnSettings = transform.Find("BtnSettings").GetComponent<Button>();
        btnQuit = transform.Find("BtnQuit").GetComponent<Button>();
        
        // 绑定事件
        btnStart.onClick.AddListener(OnClickStart);
        btnSettings.onClick.AddListener(OnClickSettings);
        btnQuit.onClick.AddListener(OnClickQuit);
    }

    protected override void OnOpen(object userData)
    {
        base.OnOpen(userData);
        GF.Log("主菜单打开");
    }

    private void OnClickStart()
    {
        GF.Sound.PlayEffect("ui/ui_click.wav");
        // 切换到游戏流程
        // ChangeState<GameProcedure>(procedureOwner);
        GF.UI.ShowToast("开始游戏", UIExtension.ToastStyle.Green);
    }

    private void OnClickSettings()
    {
        GF.Sound.PlayEffect("ui/ui_click.wav");
        // 打开设置界面
        GF.UI.OpenUIForm(UIViews.SettingDialog);
    }

    private void OnClickQuit()
    {
        GF.Sound.PlayEffect("ui/ui_click.wav");
        Application.Quit();
    }
}
```

**创建 UI 预制体**：
1. 在 Unity 中创建 Canvas
2. 添加按钮等 UI 元素
3. 添加 `MainMenuUI` 脚本
4. 保存为 Prefab：`Assets/AAAGame/Prefabs/UI/MainMenuUI.prefab`

#### 3. 配置 UI 数据表

在 `UITable.xlsx` 中添加：

```
| Id | UIPrefab      | UIGroupId | SortOrder | PauseCoveredUI | EscapeClose |
| 1  | MainMenuUI    | 1         | 10        | TRUE           | FALSE       |
```

重新生成数据表代码。

#### 4. 创建主菜单流程

```csharp
// Assets/AAAGame/Scripts/Procedures/MainMenuProcedure.cs
using GameFramework.Fsm;
using GameFramework.Procedure;

public class MainMenuProcedure : ProcedureBase
{
    private int m_MainMenuUIId;

    protected override void OnEnter(IFsm<IProcedureManager> procedureOwner)
    {
        base.OnEnter(procedureOwner);
        
        // 打开主菜单 UI
        m_MainMenuUIId = GF.UI.OpenUIForm(UIViews.MainMenuUI);
    }

    protected override void OnLeave(IFsm<IProcedureManager> procedureOwner, bool isShutdown)
    {
        // 关闭主菜单 UI
        if (GF.UI.HasUIForm(m_MainMenuUIId))
        {
            GF.UI.CloseUIForm(m_MainMenuUIId);
        }
        
        base.OnLeave(procedureOwner, isShutdown);
    }
}
```

#### 5. 配置流程

在 `AppConfigs` 中添加流程：
```
Procedures:
- PreloadProcedure
- ChangeSceneProcedure
- MainMenuProcedure
```

#### 6. 修改预加载流程

在 `PreloadProcedure.cs` 中，修改完成后的流程切换：

```csharp
ChangeState<MainMenuProcedure>(procedureOwner);
```

#### 7. 测试运行

1. 打开 `Launch` 场景
2. 点击 Play 运行
3. 应该能看到主菜单界面

## 💡 开发建议

### 1. 项目结构建议

```
Scripts/
├── Common/              # 通用工具类
├── Config/              # 配置类
├── DataModel/           # 数据模型
├── DataTable/           # 数据表（自动生成）
├── Entity/              # 游戏实体
├── EventArgs/           # 事件参数
├── Manager/             # 管理器类
├── Procedures/          # 游戏流程
├── UI/                  # UI 界面
└── Gameplay/            # 游戏玩法逻辑
```

### 2. 命名建议

- **UI 类**: `xxxUI` 或 `xxxUIForm`（例如：MainMenuUI）
- **Entity 类**: `xxxEntity`（例如：PlayerEntity）
- **DataModel 类**: `xxxDataModel`（例如：PlayerDataModel）
- **Procedure 类**: `xxxProcedure`（例如：GameProcedure）
- **EventArgs 类**: `xxxEventArgs`（例如：PlayerDeathEventArgs）

### 3. 开发顺序建议

1. **数据层**：先设计数据表和数据模型
2. **流程层**：设计游戏流程和状态机
3. **UI 层**：开发 UI 界面
4. **逻辑层**：实现游戏逻辑
5. **优化层**：性能优化和 Bug 修复

### 4. 版本控制建议

**.gitignore 建议添加**：
```
# Unity 生成文件
/[Ll]ibrary/
/[Tt]emp/
/[Oo]bj/
/[Bb]uild/
/[Bb]uilds/
/[Ll]ogs/
/[Uu]ser[Ss]ettings/
```

## ❓ 常见问题

### Q1: 删除示例代码后编译报错？
A: 
1. 检查是否有遗漏的引用
2. 重新生成数据表代码
3. 清理并重新编译项目

### Q2: UI 打不开？
A: 
1. 检查 UITable 配置
2. 确认 UI Prefab 路径正确
3. 检查 UIViews 枚举是否生成

### Q3: 数据表加载失败？
A: 
1. 确认数据表文件存在
2. 检查 AppConfigs 中的配置
3. 重新生成数据表

### Q4: 如何保留部分示例代码作为参考？
A: 
1. 可以将示例代码移到 `_Examples/` 文件夹
2. 或者创建新的 Git 分支保存示例代码
3. 在需要时参考，但不要在新项目中使用

## 一、高优先级（应尽快修复，直接影响帧率/GC）

### 1.1 PlayerSaveData JSON 反复序列化/反序列化 ⭐最严重
- **文件**: `DataModel/PlayerAccountData.cs:227-242, 249-254`
- **问题**: `GetInventoryItems()`/`SetInventoryItems()` 每次增删物品都做完整的 JSON 反序列化+序列化。`AddItem`(行756)、`ConsumeItem`(行795)、`GetUnlockedSummonerIds()`、`GetSavedDecks()` 等多处高频调用
- **优化**: 增加运行时缓存字段（`[NonSerialized]`），首次调用时反序列化并缓存，后续直接返回。仅存档时序列化。加脏标记（dirty flag）
- **收益**: 消除每次物品操作的 JSON 往返开销

### 1.2 PlayerSkillUI 每帧反射调用
- **文件**: `UI/PlayerSkillUI.cs:342-368, 384-398`
- **问题**: `CheckCooldownComplete()` 每帧通过反射 `GetField().GetValue()` 获取技能冷却时间，每帧 3 次反射 + 装箱（float→object）
- **优化**: 在 Skill 类上暴露公开的 `CdRemain` 属性，彻底消除反射和装箱
- **收益**: 消除每帧反射开销和 GC 分配

### 1.3 PlayerSkillUI 每帧无条件刷新
- **文件**: `UI/PlayerSkillUI.cs:137-143`
- **问题**: `OnUpdate` 中每帧调用 `UpdateAllSlots()` 遍历所有技能槽位刷新，即使没有变化
- **优化**: 改为事件驱动 + 脏标记，只在技能冷却变化时刷新；将 CD 显示用独立子 Canvas 隔离
- **收益**: 减少每帧 Canvas 重建

### 1.4 CombatEntityTracker 查询每次 new List
- **文件**: `Game/Combat/CombatEntityTracker.cs`（GetEnemies/GetAllies/GetEnemyCache 等方法）
- **问题**: 每次查询都 `new List<>`，这些方法在战斗中被 AI 索敌高频调用
- **优化**: 预分配列表，清空后复用（Clear + 填充），或返回 `IReadOnlyList` 缓存
- **收益**: 消除战斗热路径上的 GC 分配

### 1.5 DissolveTransitionManager FindObjectsOfType
- **文件**: `Game/Combat/DissolveTransitionManager.cs`
- **问题**: 战斗过渡时使用 `FindObjectsOfType<>()` 全场景搜索
- **优化**: 缓存引用或通过注册/注销模式管理
- **收益**: 消除战斗过渡时的卡顿

### 1.6 CameraOcclusionHandler 每帧 RaycastAll + new List
- **文件**: `Game/Camera/CameraOcclusionHandler.cs`
- **问题**: 每帧调用 `RaycastAll`（返回新数组）+ `new List` 临时分配
- **优化**: 使用 `RaycastNonAlloc` + 预分配数组；复用 List
- **收益**: 消除每帧的 GC 分配

### 1.7 CombatUI FindObjectsOfType<Canvas>
- **文件**: `UI/CombatUI.cs:140-149`
- **问题**: `CreateCardPreviewManager()` 中 `FindObjectOfType<Canvas>()` + `FindObjectsOfType<Canvas>()` 遍历全场景

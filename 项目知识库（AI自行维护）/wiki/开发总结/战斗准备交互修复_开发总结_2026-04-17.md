## 修复内容

### 问题1：点击已出战棋子不显示详情
**表现**：战斗准备阶段，点击已出战的棋子对象时，无法显示棋子详情界面

**原因**：
- `ChessItemUI.OnButtonClick()` 中有检查：如果 `instance.IsDeployed`，直接返回，不触发点击回调
- 导致已出战的棋子无法通过点击卡片显示详情

**修复**：
1. **ChessItemUI.cs 第261-268行**：
   - 移除了对 `IsDeployed` 的检查
   - 允许已出战的棋子点击触发回调
   ```csharp
   // ⭐ 已出战的棋子也可以点击显示详情（由回调方决定如何处理）
   m_OnSelectCallback?.Invoke(m_InstanceId);
   ```

2. **CombatPreparationUI.cs 第299-350行**（OnChessItemSelected）：
   - 在回调中检查 `instance.IsDeployed`
   - 已出战的棋子：只显示详情，无动效
   - 未出战的棋子：支持选中/取消选中切换
   ```csharp
   // 已出战的棋子只显示详情，不支持选中/取消选中切换
   if (instance.IsDeployed)
   {
       // 直接显示详情（无动效）
       if (varDetailInfoUI != null)
       {
           var detailUI = varDetailInfoUI.GetComponent<DetailInfoUI>();
           if (detailUI != null)
           {
               // ... 显示详情逻辑
           }
       }
       return;
   }
   ```

### 问题2：拖拽放置还需要额外点击
**表现**：拖拽棋子卡片放置时，松开鼠标后只显示预览，还需要再点击才能确认放置

**原因**：
- `ChessItemUI.OnBeginDrag()` 中有重复的 `StartPlacement()` 调用
- 与 `CombatPreparationUI.OnChessItemDragBegin` 中的调用冲突
- 延迟了拖拽模式的正确设置

**修复**：
1. **ChessItemUI.cs 第279-327行**（OnBeginDrag）：
   - 移除了 `ChessPlacementManager.Instance.StartPlacement()` 的直接调用
   - 只触发 `m_OnDragBeginCallback`，让 `CombatPreparationUI` 统一处理
   ```csharp
   // ⭐ 触发拖拽开始回调（由 CombatPreparationUI 处理放置逻辑）
   m_OnDragBeginCallback?.Invoke(m_InstanceId);
   // 移除了下面这行重复调用：
   // ChessPlacementManager.Instance.StartPlacement(m_InstanceId, true);
   ```

2. **CombatPreparationUI.cs 第463-495行**（OnChessItemDragBegin/End）：
   - 确保拖拽开始时调用 `StartPlacement(isDragMode=true)`
   - 拖拽结束时直接调用 `ConfirmPlacementFromDrag()`，无需额外点击
   ```csharp
   private void OnChessItemDragBegin(string instanceId)
   {
       if (ChessPlacementManager.Instance != null)
       {
           ChessPlacementManager.Instance.StartPlacement(instanceId, isDragMode: true);
       }
   }

   private void OnChessItemDragEnd(string instanceId)
   {
       // 拖拽结束时直接确认放置（如果在有效区域）
       if (ChessPlacementManager.Instance != null)
       {
           ChessPlacementManager.Instance.ConfirmPlacementFromDrag();
       }
   }
   ```

## 交互流程（修复后）

### 未出战棋子
```
① 点击卡片 → 显示详情 + 选中高亮动效 ✓
② 再次点击 → 取消选中 + 隐藏详情 ✓
③ 拖拽卡片 → 显示Ghost预览
   └─ 松开鼠标（有效区域）→ 直接放置，无需再点击 ✓
   └─ 松开鼠标（无效区域）→ 自动取消，回归未拖拽状态 ✓
   └─ 松开鼠标（UI上）→ 自动取消 ✓
```

### 已出战棋子
```
① 点击卡片 → 显示详情（无动效，无选中高亮）✓
② 右键（战斗阶段）→ 召回棋子 ✓
```

## 关键代码逻辑

**拖拽放置的验证流程**（ChessPlacementManager.ConfirmPlacementFromDrag）：
1. 检查 `m_IsPlacing && m_IsDragMode` 状态
2. 检查释放位置：
   - 如果在 UI 上 → 取消放置
   - 如果不在我方区域 → 取消放置
   - 如果在有效的我方区域 → `ConfirmPlacementInternal()` 放置棋子

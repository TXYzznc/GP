## 🏗️ 核心架构

### ISlotContainer 接口系统

```
ISlotContainer（接口）
    ↓
SlotContainerBase（抽象基类）
    ├─ InventorySlotContainerImpl（背包容器）
    ├─ WarehouseSlotContainerImpl（仓库容器）
    ├─ FastBarSlotContainerImpl（快捷栏容器）
    └─ EquipSlotContainerImpl（装备栏容器框架）
```

**核心设计原则**：
1. **单一责任**：MoveToXXX 方法只处理目标相关逻辑，源格子清理由 ExecuteMove 统一处理
2. **对称性**：容器内操作支持交换（SwapSlots），跨容器操作只支持存入/堆叠
3. **数据一致性**：避免重复操作，清空源格子的条件基于目标格子状态

### 清空源格子的统一条件

```csharp
if (success && targetContainer is not [SameContainerType] &&
    (targetIsEmpty || targetSlot.ItemId == itemId))
{
    ClearSource();
}
```

**语义**：
- ✓ 操作成功
- ✓ 非容器内部操作（各自已处理）
- ✓ 目标格子为空 **OR** 同种物品堆叠

### Bug 1: WarehouseSlotContainerImpl 仓库内交换丢失数据 ✅
**文件**: `WarehouseSlotContainerImpl.cs:63-64`
**原因**: SwapSlots 完成交换后，还调用了 RemoveItem

### Bug 2: FastBarSlotContainerImpl 快捷栏交换丢失数据 ✅
**文件**: `FastBarSlotContainerImpl.cs:56-57`
**原因**: SwapSlots 完成交换后，还调用了 ClearSlot
**修复**: 条件判断 → 仅非快捷栏→快捷栏时才清空

### Bug 3: 快捷栏 ↔ 仓库 不合理的直接交互 ✅
**文件**:
- `FastBarSlotContainerImpl.cs:25-29`
- `WarehouseSlotContainerImpl.cs:36-40`

**原因**: 快捷栏应是"背包的快速访问栏"，不应直接与仓库交互
**修复**: CanInteractWith 规则调整，禁止快捷栏↔仓库直接拖拽

```csharp
protected void RefreshAll()
{
    RefreshInventory();

    RefreshEquipSlots();

// 负重刷新方法
private void RefreshWeightState()
{
    if (varWeightStateText == null)
    {
        DebugEx.Warning("InventoryUI", "RefreshWeightState: varWeightStateText 未连接");
        return;
    }

    if (m_InventoryManager == null)
    {
        varWeightStateText.text = "负重: 0";
        return;
    }

    float currentWeight = m_InventoryManager.CalculateCurrentWeight();
    varWeightStateText.text = $"负重: {currentWeight:F0}";
    DebugEx.Log("InventoryUI", $"RefreshWeightState: 负重 = {currentWeight}");
}
```

- ✓ 打开背包界面（OnOpen → RefreshAll）
- ✓ 物品拖拽操作（源格子/目标格子变化触发事件）
- ✓ 整理背包（SortInventory 后）
- ✓ 快捷栏变化（OnItemStored/OnItemRetrieved）

## 💡 核心设计洞察

### 问题根源
原设计用**容器类型**决定是否清空源格子，但实际上应该用**目标格子状态**决定——这是更清晰和灵活的方式。

### 解决方案
统一所有三个容器的 ExecuteMove 逻辑：
```
清空源格子 ⟸ 目标格子为空 OR 同种物品堆叠
```

### 好处
- ✓ 逻辑清晰易理解
- ✓ 消除特殊情况（无需按容器类型判断）
- ✓ 易于扩展（新容器只需实现 MoveToXXX）
- ✓ 数据安全（避免重复操作）

# 棋子卡片交互开发总结

> **时间**: 2026-04-02
> **涉及文件**: `CombatPreparationUI.cs`、`ChessItemUI.cs`、`DetailInfoUI.cs`

---

## 一、功能概述

实现战斗准备阶段棋子卡片的完整交互体系：
- 点击选中/取消选中，显示/隐藏 DetailInfoUI
- 选中时播放向上移动 + 脉冲动效（参考策略卡实现）
- 拖拽放置独立于选中状态
- 部署完成后自动清除选中状态
- DetailInfoUI 支持准备阶段（配置数据）和战斗阶段（实体+Buff）两种显示模式

---

## 二、核心设计

### 单选模式
- `CombatPreparationUI` 中用 `m_SelectedChessInstanceId` 记录当前选中棋子
- 点击新棋子时自动取消之前选中的棋子
- 再次点击已选中棋子时取消选中

### 三个回调
```csharp

```

## 三、实现过程（两次迭代）

### 第一版设计

**交互逻辑**：
- 点击 → OnChessItemSelected → 选中/取消选中 + 显示/隐藏 DetailInfoUI
- 拖拽开始 → OnChessItemDragBegin → **清除选中状态** + 进入放置模式
- 拖拽结束 → OnChessItemDragEnd → ConfirmPlacementFromDrag

**已实现**：
- SelectChess / DeselectChess 方法
- DetailInfoUI 集成（SetChessUnitData → RefreshUI → ShowWithAnimation）
- 棋子被撤回时检查并清除选中状态

---

### 第二版修正（最终方案）

**问题**：拖拽开始就清除选中违反用户体验，用户在放置时应能继续看到棋子信息

**改动**：

| 场景 | 第一版 | 第二版（最终） |
|------|--------|-------------|
| 拖拽期间 | DetailInfoUI 隐藏 | 保持显示 |
| 部署完成 | 手动清除 | OnChessDeployedHandler 自动清除 |
| 撤回时 | 检查选中状态并清除 | 不检查（撤回时卡片无法交互，无需处理） |

**具体代码变化**：

```csharp
// OnChessItemDragBegin — 移除选中清除逻辑
private void OnChessItemDragBegin(string instanceId)
{
    if (ChessPlacementManager.Instance != null)
        ChessPlacementManager.Instance.StartPlacement(instanceId, true);
}

// OnChessDeployedHandler — 新增选中清除逻辑
private void OnChessDeployedHandler(ChessDeploymentTracker.ChessInstanceData instance)
{
    if (m_SelectedChessInstanceId == instance.InstanceId)
        DeselectChess();  // 部署完成后自动隐藏 DetailInfoUI
    RefreshChessPanel();
}

// OnChessRecalledHandler — 简化，无需处理选中状态
private void OnChessRecalledHandler(ChessDeploymentTracker.ChessInstanceData instance)
{
    RefreshChessPanel();
}
```

## 四、DetailInfoUI 扩展：准备阶段支持

### 问题
准备阶段选中棋子时，棋子未部署（Entity = null），无法用 SetChessUnitData 显示信息。

### 解决方案
新增 `SetChessConfig(config, globalState)` 方法，接收 SummonChessConfig 和 GlobalChessState：

```csharp
public void SetChessConfig(SummonChessConfig config, GlobalChessState globalState)
// 设置 m_CurrentMode = 2，调用 RefreshChessConfigUI()
```

内部通过 `DisplayChessInfo(config, currentHp, maxHp)` 统一处理显示逻辑，与战斗阶段复用。

### 显示内容对比

| 阶段 | 标题 | 属性 | Buff | 动效 |
|------|------|------|------|------|
| 准备（配置模式） | 名称+星级 | 血量/法力/攻击/护甲/人口消耗 | 隐藏 | 向上移动+脉冲 |
| 战斗（实体模式） | 名称+星级 | 血量/法力/攻击/护甲/人口消耗 | 显示 | 无 |

### 数据流（准备阶段）
```
点击棋子卡片
  ↓
SelectChess(instanceId)
  ├─ ChessDataManager.TryGetConfig(chessId) → SummonChessConfig
  ├─ GlobalChessManager.GetChessState(chessId) → GlobalChessState
  └─ detailUI.SetChessConfig(config, globalState)
       └─ DisplayChessInfo → 显示完整信息
  ↓
PlayChessSelectAnimation
  ├─ 向上移动 20px（0.3秒，SELECTED_OFFSET）
  └─ 脉冲动画（1.0 → 1.1 → 1.0）
```

## 六、完整交互流程

```
【准备阶段】

用户点击棋子卡片
  ↓
OnChessItemSelected()
  ├─ 已选中 → DeselectChess() → 隐藏 DetailInfoUI + 反向动效
  └─ 未选中 → SelectChess() → 显示 DetailInfoUI + 选中动效

用户拖拽棋子卡片
  ↓
OnChessItemDragBegin()
  ├─ 不清除选中状态（DetailInfoUI 保持显示）
  └─ ChessPlacementManager.StartPlacement(isDragMode=true)

用户释放鼠标（有效位置）
  ↓
ChessPlacementManager.ConfirmPlacementInternal()
  └─ ChessDeploymentTracker.DeployChess()
  ↓
OnChessDeployedHandler()
  ├─ DeselectChess() → 隐藏 DetailInfoUI
  └─ RefreshChessPanel() → 卡片变"已出战"灰显

【战斗阶段】

点击棋子对象（ChessSelectionManager）
  └─ entity.OutlineController.SetSelected(true) → 显示描边
```

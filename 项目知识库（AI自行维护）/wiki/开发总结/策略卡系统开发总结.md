## 三、关键技术决策

1. **范围预览**: Gizmos 临时实现，后续改成 Shader（避免创建 Prefab）
2. **CardEffectExecutor**: 非单例，参考 Buff 的动态创建方式，在战斗流程中对应时机创建
3. **效果执行**: 反射 + 工厂模式，支持动态创建效果实例
4. **事件系统**: CardManager 使用 Action 委托，CombatUI 订阅事件自动刷新
5. **单例初始化**: 动态添加组件模式（不依赖预制体），如 `new GameObject("CardManager").AddComponent<CardManager>()`

### 卡牌效果实现统一模式
```
1. 获取所有棋子 → BattleChessManager.Instance.GetAllChessEntities()
2. 按 Camp 筛选目标（CampType.Player 或 CampType.Enemy）
3. 执行具体效果（伤害 / 治疗 / Buff / 范围预览）
4. 播放特效
```

### 初始化顺序
```
GameProcedure：技能管理器 → 卡牌系统 → 战斗特效系统 → UI 系统 → 角色生成
CombatState：棋子管理器 → 玩家控制器 → 卡牌系统 → 战斗特效系统 → 敌人生成
```

# 项目知识库整理方案

## 📊 当前问题分析

### 问题1：文件夹结构混乱
**现状**：
- ✅ 标准类别（系统设计、技术文档、问题诊断、开发总结、项目规则）
- ❌ 额外非标准文件夹（UI系统、战斗系统、探索系统等）
- ❌ 根目录直接放置 .md 文件（应都归入子文件夹）
- ❌ 临时文件夹存在（论文写作、00_文档模板/修复总结等）

### 问题2：文件命名格式不统一
**现状**：
- ❌ `[标签]名称.md`（当前大量使用）
- ✅ `名称_日期.md`（少数文件）
- ❌ 缺少统一的日期和状态标记

**标准应为**：`[功能名称]_[文档类型]_[日期].md`

示例：
```
❌ [使用]FloatingBoxTip提示框.md
✅ FloatingBoxTip提示框_使用说明_2026-04-14.md

❌ [总结]伤害飘字系统调试.md
✅ 伤害飘字系统_调试总结_2026-04-14.md
```

### 问题3：元数据不统一
- 部分文件有日期，部分没有
- 缺少`状态`字段（有效/过时/草稿）
- 缺少统一的分隔线格式

### 问题4：临时和过时文件需要清理
- `论文写作/temp_docx/` - 临时文件，应删除
- `00_文档模板/修复总结_*` - 应归到开发总结
- `graphify-out/` - 工具输出，应隐藏或移除

### Phase 3：统一文件命名（45分钟）

**转换规则**：
1. `[标签]名称.md` → `名称_[标签类型]_[日期].md`
2. `标签对应关系`：
   - `[使用]` → `_使用说明_`
   - `[设计]` → `_设计方案_`
   - `[总结]` → `_完成总结_`
   - `[规则]` → `_开发规则_`
   - `[配置]` → `_配置指南_`
   - `[分析]` → `_问题分析_`
   - `[修复]` → `_bug修复_`
   - `[实现]` → `_实现记录_`

**示例转换**：
```
UI系统/[使用]FloatingBoxTip提示框.md
→ 技术文档/使用说明/FloatingBoxTip提示框_使用说明_2026-04-14.md

战斗系统/[设计]伤害飘字系统.md
→ 系统设计/UI系统/伤害飘字系统_设计方案_2026-04-14.md

开发工具/[规则]MCP工具使用规范.md
→ 项目规则/MCP工具_开发规则_2026-04-14.md
```

### Phase 4：补充元数据（30分钟）

对所有 .md 文件添加统一的元数据头：
```markdown
> **最后更新**: YYYY-MM-DD  
> **状态**: 有效/过时/草稿  
> **分类**: [系统设计|技术文档|开发注意|问题诊断|开发总结|项目规则]

---
```

**标记指南**：
- `有效` - 文档内容准确，可直接参考
- `草稿` - 文档未完成，需进一步完善
- `过时` - 文档内容不再适用，保留以备查看

### Phase 5：更新索引（15分钟）

- 重新生成 INDEX.md
- 更新 README.md 中的统计数据
- 检查所有交叉引用

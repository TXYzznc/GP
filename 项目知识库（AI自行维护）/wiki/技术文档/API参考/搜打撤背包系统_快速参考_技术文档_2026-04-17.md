## 📋 拖拽规则速查

| 操作 | 是否允许 | 是否支持交换 | 备注 |
|-----|--------|-----------|------|
| 背包←→背包 | ✅ | ✅ | MoveItem 处理 |
| 背包→仓库 | ✅ | ❌ | 存入或堆叠 |
| 背包→快捷栏 | ✅ | ❌ | 存入或堆叠 |
| 仓库→背包 | ✅ | ❌ | 存入或堆叠 |
| 仓库←→仓库 | ✅ | ✅ | SwapSlots |
| 快捷栏→背包 | ✅ | ❌ | 存入或堆叠 |
| 快捷栏←→快捷栏 | ✅ | ✅ | SwapSlots |
| 仓库↔快捷栏 | ❌ | — | 禁止直接交互 |

## 🐛 常见问题排查

### Q: 拖拽物品后消失了
**A**: 检查 ExecuteMove 的清空逻辑
- 确认 `targetIsEmpty || targetSlot.ItemId == itemId` 条件正确
- 检查 Manager 的 RemoveItem/ClearSlot 是否被重复调用

### Q: 仓库物品无法拖拽到快捷栏
**A**: 这是**设计决策**，快捷栏与仓库不支持直接交互
- 快捷栏应通过背包中转获取物品
- 若需要修改，在 CanInteractWith 中添加 FastBar

### Q: 负重不显示
**A**: 检查以下几点
- [ ] Inspector 中 varWeightStateText 是否赋值
- [ ] Console 中是否有 "varWeightStateText 未连接" 的警告
- [ ] Text 组件颜色是否与背景相同

### Q: 交换物品时数据不一致
**A**: 容器内部操作（如 SwapSlots）已处理，不需要额外清空
- 检查是否在 MoveToXXX 和 ExecuteMove 中重复清空

## 🔧 修改建议

### 如果要添加容器（如装备栏）

1. **创建实现类**
```csharp
public class EquipSlotContainerImpl : SlotContainerBase
{
    public override SlotContainerType ContainerType => SlotContainerType.Equipment;

    public override bool CanInteractWith(SlotContainerType otherContainerType)
    {
        return otherContainerType == SlotContainerType.Inventory;
    }

    protected override bool ExecuteMove(int fromSlotIndex, ISlotContainer targetContainer, int targetSlotIndex)
    {
        // 实现装备栏特有逻辑
        // 遵循统一的清空条件
    }
}
```

2. **在 UI 中初始化**
```csharp
m_EquipSlotContainer = GetComponent<EquipSlotContainerImpl>();
if (m_EquipSlotContainer == null)
    m_EquipSlotContainer = gameObject.AddComponent<EquipSlotContainerImpl>();
```

### 如果要改进负重显示

1. **添加最大负重限制**
```csharp
// InventoryManager 中
public float MaxWeight { get; set; } = 50f;
public bool IsOverweight => CalculateCurrentWeight() > MaxWeight;
```

2. **更新显示**
```csharp
private void RefreshWeightState()
{
    float current = m_InventoryManager.CalculateCurrentWeight();
    float max = m_InventoryManager.MaxWeight;
    varWeightStateText.text = $"负重: {current:F0}/{max:F0}";
}
```

## 📊 性能注意

- **RefreshAll()** 每次拖拽都会调用，影响性能
- 建议优化为仅刷新相关的 UI（而非全部）
- 使用对象池减少格子 UI 的创建销毁

---

## 🎓 学习要点

1. **接口驱动设计**：ISlotContainer 让拖拽逻辑与具体容器解耦
2. **单一责任**：MoveToXXX 只处理目标，清空由 ExecuteMove 统一处理
3. **条件判断优化**：用目标状态判断而非容器类型，更灵活

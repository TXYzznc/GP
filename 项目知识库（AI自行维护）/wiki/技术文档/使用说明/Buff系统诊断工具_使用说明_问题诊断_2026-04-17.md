# Buff 系统诊断工具使用指南

## 问题背景

用户反馈给棋子应用多个攻击Buff后，攻击力和伤害增加不明显：
- 应用的6个Buff：神力增益、日落长弓、先手·攻击强化、战意激昂、狂怒之心、战争号角
- 预期攻击力应该大幅增加，但实际增加不多

## 根本原因分析

### Buff 修改机制

每个 Buff 的属性修改遵循以下规则：

1. **百分比修改**：基于**当前属性值**计算
   ```
   修改值 = 当前属性值 × 百分比
   最终值 = 当前属性值 + 修改值
   ```

2. **多个百分比Buff的复利增长**
   ```
   基础攻击力: 100
   +25% Buff: 100 × 1.25 = 125
   +15% Buff: 125 × 1.15 = 143.75
   +15% Buff: 143.75 × 1.15 = 165.3125
   ...
   ```

### 配置验证

根据 BuffTable 配置，6个攻击Buff的修改值：

| Buff名称 | BuffID | 攻击力修改 | 备注 |
|---------|--------|---------|------|
| 神力增益 | 4 | +25% | - |
| 日落长弓 | 5 | +15% | 还有+30% 射程 |
| 先手·攻击强化 | 2002 | +15% | - |
| 战意激昂 | 4001 | +15% | 还有+20% 攻速 |
| 狂怒之心 | 4002 | +15% | - |
| 战争号角 | 5003 | +30% | 还有+20% 移速 |

### 理论计算

6个Buff的复利增长应该达到：
```
100 × 1.25 × 1.15 × 1.15 × 1.15 × 1.15 × 1.30 ≈ 400+%
```

如果实际增幅远小于此，说明存在以下可能的问题

## 诊断步骤

### 方法1：使用编辑器诊断工具

1. 打开菜单：**工具 → Clash of Gods → Test Manager**
2. 在 **Buff 测试系统** 部分：
   - 选择棋子作为目标
   - 应用6个攻击Buff
   - 在 **Buff 效果诊断** 部分点击 **生成诊断报告**

### 方法2：查看详细修改信息

在编辑器窗口中，**Buff 修改详情** 部分会显示：
- 每个Buff应用了哪些属性修改
- 修改的百分比和实际值

### 方法3：Console 日志分析

查看 Console 中的日志，搜索关键词：
- `[StatModBuff]` - Buff修改日志
- `属性修改已应用` - 确认Buff是否被正确应用

## 常见问题排查

### 问题1：Buff 显示有，但属性值没变

**可能原因：**
- Buff被应用到了错误的目标
- Buff的类型不是 StatModBuff（某些Buff可能有特殊逻辑）

**检查方法：**
1. 确认 Console 日志中有 `[StatModBuff] Buff(ID=...) 属性修改已应用`
2. 确认目标棋子有 BuffManager 和 ChessAttribute 组件

### 问题2：百分比修改计算错误

**可能原因：**
- JSON 配置解析错误（例如"25%"被解析错误）
- IsPercent 标志未正确设置

**验证方式：**
诊断报告中查看 `修改详情` 部分，确认：
- 显示的是 `+25%` 格式（而非 `+0.25`）
- `IsPercent` 标志为 True

### 问题3：多个Buff之间有冲突

**可能原因：**
- 某个Buff覆盖了其他Buff的修改
- Buff 栈的处理逻辑有问题

**排查步骤：**
1. 逐个应用Buff，观察属性变化
2. 对比理论值和实际值的差异

## 修复建议

### 如果发现 Buff 没有应用

检查 StatModBuff.cs：
1. `ApplyMods()` 方法是否在 `OnEnter()` 中被调用
2. `m_Mods` 是否被正确初始化
3. `Ctx.OwnerAttribute` 是否为 null

### 如果发现百分比计算错误

检查 `TryParseModValue()` 方法：
```csharp
value = p / 100.0;  // "25%" 应该被解析为 0.25
isPercent = true;
```

### 如果需要调整Buff效果

编辑 `AAAGameData/DataTables/BuffTable.xlsx`：
1. 修改对应Buff的 `StatMods` 字段
2. 运行 **GameFramework → DataTable → Generate**
3. 重新测试

## 性能影响考虑

如果应用大量Buff后性能下降，可能是：
1. 每个Buff都在 Update 中做计算（应该只在应用/移除时计算）
2. 事件监听器泄漏
3. BuffManager 的列表没有正确清理

# 卡牌扇形排列系统 - Prefab 配置指南

## 概述

新增 `CardSlotContainer` 脚本用于管理卡牌的扇形排列和动效。需要移除原有的 LayoutGroup 组件，改由脚本统一控制。

---

## 手动配置步骤

### 1. 找到卡牌容器 Prefab

路径：`Assets/AAAGame/Prefabs/UI/CombatUI.prefab` 或 `varCardSlots` 对象

在 Hierarchy 中找到 `varCardSlots`（卡牌容器的 RectTransform 对象）

### 2. 移除 LayoutGroup 组件

选中 `varCardSlots` 对象，在 Inspector 中找到 LayoutGroup 组件（可能是 HorizontalLayoutGroup、VerticalLayoutGroup 或 GridLayoutGroup），点击右上角的 ⋮ 菜单选择 **Remove Component**

### 3. 添加 CardSlotContainer 脚本

仍在 `varCardSlots` 对象上：
- 点击 **Add Component**
- 搜索 `CardSlotContainer`
- 添加组件

### 4. 配置 CardSlotContainer 参数

在 Inspector 中配置以下参数（可选，已有默认值）：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| Fan Radius | 800f | 圆弧半径（单位：像素） |
| Max Fan Angle | 50f | 最大展开角度（度数，卡牌展开范围） |
| Circle Center | (0, 0) | 圆心 X 偏移 |
| Circle Center Offset Y | -100f | 圆心相对于容器底部的 Y 偏移距离 |
| Enter Duration | 0.35f | 卡牌进场动画时长（秒） |
| Rearrange Duration | 0.25f | 补位动画时长（秒） |
| Rearrange Ease | OutCubic | 补位动画缓动函数 |

**建议配置**：
- 如果卡牌数量多（5+ 张），调大 `Fan Radius` 至 1000-1200，减小 `Max Fan Angle` 至 30-40°
- 如果卡牌数量少（2-3 张），保持默认或稍微增大 `Max Fan Angle` 至 60-80°

### 5. 验证配置

1. 运行游戏进入战斗模式
2. 观察卡牌是否以扇形排列显示在容器底部
3. 新卡牌进场时应有滑入动画
4. 使用/移除卡牌后，其余卡牌平滑补位
5. 鼠标悬停时卡牌上移放大，松开时恢复

## 扇形排列原理

```
圆心在容器底部偏下（Y = containerHeight + offsetY）
卡牌沿半径为 R 的圆弧分布
总展开角度 = maxFanAngle（例如 50°）
每张卡位置 = 圆心 + (sin(angle) * R, cos(angle) * R - R)
每张卡旋转 = -angle（与圆弧切线对齐）

视觉效果：
  [Card] [Card] [Card]     ← 卡牌以扇形展开
 /                   \
[C]                   [C]   ← 边缘卡稍微旋转
```

## 动效说明

### 进场动画 (0.35s)
- 卡牌从容器底部中央 Y-200 处开始
- 透明度从 0 淡入到 1
- 同时移动和旋转到目标扇形位置

### 补位动画 (0.25s)
- 移除卡牌后，周围卡牌平滑滑动到新位置
- 缓动函数：OutCubic（快速减速）
- 实时更新每张卡的基准位置和旋转

### 悬停动画 (0.2s)
- 鼠标进入时：卡牌向上移动 30 像素，放大到 1.05 倍
- 鼠标离开时：恢复到基准位置，缩小回 1.0 倍
- 不影响其他卡牌

### 拖拽让位
- 开始拖拽时其他卡牌临时偏移给出空间
- 左边卡向左偏移，右边卡向右偏移
- 松手后立即恢复正常排列

## 常见问题

**Q: 卡牌排列不对称？**
A: 检查圆心偏移参数。默认圆心在容器中央水平线下 100 像素。

**Q: 动画过快/过慢？**
A: 调整 `Enter Duration` 和 `Rearrange Duration` 参数。

**Q: 卡牌重叠？**
A: 增大 `Fan Radius` 或减小 `Max Fan Angle`。

**Q: 悬停时卡牌位置错误？**
A: 确保 `CardSlotItem` 上的 `SetBaseFanTransform` 被正确调用（由 Container 自动调用）。

## 技术细节

### 核心类

- **CardSlotContainer.cs** - 扇形排列和动效调度中枢
  - `AddCardAsync()` - 新增卡牌并播放进场动画
  - `RemoveCard()` - 移除卡牌并触发补位动画
  - `OnCardBeginDrag/OnDrag/OnEndDrag()` - 拖拽事件处理
  - `CalculateFanPositions()` - 计算所有卡的扇形位置

- **CardSlotItem.cs** - 单张卡牌脚本（修改部分）
  - `SetBaseFanTransform()` - 由 Container 设置基准位置
  - `OnPointerEnter/Exit()` - 悬停动画（基于基准位置叠加偏移）
  - `OnBeginDrag/OnDrag/OnEndDrag()` - 拖拽事件通知 Container

- **CombatUI.cs** - 战斗 UI 主脚本（修改部分）
  - `RefreshCardSlots()` - 改为调用 Container 的 `AddCardAsync()`
  - `OnCardRemoved()` - 改为调用 Container 的 `RemoveCard()`
  - `GetCardSlotContainer()` - 辅助方法获取 Container 实例

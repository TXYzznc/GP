## 系统概述

本项目使用两套描边系统实现不同类型对象的描边效果：

1. **OutlineDisplayManager** - 层级自动管理器（环境物体、道具）
2. **ChessOutlineController** - 棋子描边控制器（基于阵营关系的自动管理）

---

## 系统架构

### 方案一：Layer 自动管理（OutlineDisplayManager）

**工作原理：**
- 通过 GameObject 的 Layer 自动判断是否需要描边
- 适用于场景中的环境物体、道具等
- 性能优化：LOD 系统、距离检测、队列处理

**优点：**
- 无需手动控制，自动检测
- 性能优化完善
- 适合大量对象

**缺点：**
- 需要正确配置 Layer
- 不支持动态阵营关系

---

### 方案二：阵营关系管理（ChessOutlineController）

**工作原理：**
- 根据棋子的阵营关系自动选择描边配置
- 支持选中状态的特殊描边
- 与 CampRelationService 集成

**优点：**
- 自动根据阵营关系显示不同颜色
- 支持选中状态
- 适合战斗单位

**缺点：**
- 仅适用于棋子实体
- 需要手动初始化

## 配置步骤

### 第一步：创建 OutlineConfig 资源

1. 在 Project 窗口右键 → `Create` → `Rendering` → `Outline Config`
2. 创建以下配置文件：
   - `Enemy.asset` - 敌人描边（红色）
   - `Ally.asset` - 友军描边（绿色）
   - `Selected.asset` - 选中描边（黄色/白色）
   - `Interactive.asset` - 可交互对象描边（黄色）

3. 配置参数（每个配置文件）：
   - **Enable**: 启用/禁用描边
   - **Mode**: 描边模式
   - **Color**: 描边颜色
   - **Width**: 描边宽度
   - **Smoothness**: 平滑度

### 第二步：Layer 配置（OutlineDisplayManager）

1. 打开 Unity Editor → Tags and Layers
2. 创建以下 Layer（如果不存在）：
   - `Interactive` - 可交互对象
   - `WorldProp` - 世界道具
   - `CollectableItem` - 可收集物品

3. 将需要描边的物体分配到相应的 Layer

### 第三步：场景设置

1. 创建空物体作为描边管理器
2. 根据情况选择：
   - 使用 OutlineDisplayManager（处理环境物体）
   - 使用 ChessOutlineController（处理棋子）
3. 配置管理器的引用参数

## 工作流程

### OutlineDisplayManager 工作流程

```
初始化
  ↓
定时扫描场景（可配置时间间隔）
  ↓
按 Layer 识别对象
  ↓
检查距离和 LOD 级别
  ↓
更新描边状态（启用/禁用）
  ↓
性能监控（如果超过限制，降级处理）
```

### ChessOutlineController 工作流程

```
初始化
  ↓
注册到棋子实体
  ↓
监听阵营变化事件
  ↓
获取当前阵营关系
  ↓
选择对应的 OutlineConfig
  ↓
应用描边效果
```

## 阵营区分系统

### 关系定义

| 关系类型 | 描边颜色 | 示例 |
|---------|--------|------|
| **敌人** | 红色 | 对方棋子 |
| **友军** | 绿色 | 自己的棋子 |
| **中立** | 灰色 | 环境物体 |
| **选中** | 黄色/白色 | 当前操作目标 |

### 实现方式

```csharp
// 查询阵营关系
var relation = campRelationService.GetRelation(caster, target);
switch (relation)
{
    case CampRelation.Enemy:
        outlineController.SetOutlineConfig(enemyConfig);
        break;
    case CampRelation.Ally:
        outlineController.SetOutlineConfig(allyConfig);
        break;
    case CampRelation.Neutral:
        outlineController.SetOutlineConfig(neutralConfig);
        break;
}
```

## 常见问题

### Q: 描边显示不出来

**原因：**
1. OutlineConfig 未正确配置
2. 材质不支持描边
3. Layer 配置错误

**解决：**
1. 检查 OutlineConfig 的 Enable 选项
2. 查看是否使用了正确的 Shader
3. 验证 GameObject 的 Layer 设置

### Q: 描边颜色不对

**原因：**
1. OutlineConfig 颜色配置错误
2. 光照影响了表现

**解决：**
1. 调整 OutlineConfig 中的 Color 参数
2. 在不同光照条件下测试

### Q: 性能下降明显

**原因：**
1. 描边对象过多
2. LOD 系统未生效
3. 距离检测范围过大

**解决：**
1. 减少描边对象数量
2. 启用 LOD 系统
3. 调整距离检测范围

## 代码示例

### 使用 OutlineDisplayManager

```csharp
var manager = GetComponent<OutlineDisplayManager>();
manager.Initialize();
// 自动处理，无需手动管理
```

### 使用 ChessOutlineController

```csharp
var controller = gameObject.AddComponent<ChessOutlineController>();
controller.SetOutlineConfigs(enemyConfig, allyConfig, selectedConfig);
controller.Initialize(campRelationService);

// 手动更新描边
controller.UpdateOutlineByRelation(targetEntity);
```

### 自定义描边逻辑

```csharp
public void SetCustomOutline(GameObject target, OutlineConfig config)
{
    var renderer = target.GetComponent<Renderer>();
    if (renderer != null)
    {
        renderer.material.SetColor("_OutlineColor", config.Color);
        renderer.material.SetFloat("_OutlineWidth", config.Width);
    }
}
```

## 调试技巧

### 实时查看描边状态

1. 使用 `DebugEX` 打印描边信息
2. 在 Inspector 中启用"Gizmos"显示
3. 使用 Profiler 监控性能

### 常用调试命令

```csharp
// 启用/禁用描边
outlineController.EnableOutline(true/false);

// 查询当前描边状态
var isEnabled = outlineController.IsOutlineEnabled;

// 强制刷新
outlineController.RefreshOutline();
```

# 描边系统规则文档

> **最后更新**: 2026-04-17
> **状态**: 已验证有效

## 一、系统架构概览

```
OutlineConfig (ScriptableObject)        ← 配置层：颜色/宽度/距离衰减
    ├── ChessOutlineController          ← 棋子专用：按阵营关系选配置
    └── OutlineDisplayManager           ← 通用管理器：按 Layer 自动扫描
            ↓
        OutlineTest (MonoBehaviour)     ← 轻量组件：缓存 Renderer，转发调用
            ↓
        OutlineRenderFeature (URP)      ← 渲染层：RT + 模糊 + 合成
            ↓
        Custom_Outline.shader           ← 着色器：绘制/模糊/合成三个 Pass
        Custom_Outline_Blur.shader
```

**存在两套独立的描边管理系统**，各自负责不同场景：

| 系统 | 职责 | 触发方式 |
|------|------|----------|
| `ChessOutlineController` | 棋子实体专用，按阵营关系显示不同颜色 | 棋子初始化时自动挂载 |
| `OutlineDisplayManager` | 通用场景物体，按 Layer 自动管理 | 场景中的单例，协程扫描范围内物体 |

两套系统共享底层的 `OutlineRenderFeature` 渲染管线。

## 二、配置层 — OutlineConfig

**文件**: `Assets/AAAGame/Scripts/TA/OuterGlow/OutlineConfig.cs`
**资产位置**: `Assets/TA/OuterGlow/`

| 资产文件 | 用途 | 对应阵营关系 |
|----------|------|-------------|
| `Enemy.asset` | 敌人描边（红色） | CampRelation.Enemy |
| `Ally.asset` | 友军描边（绿色） | CampRelation.Ally |
| `Interactive.asset` | 中立/可交互（黄色） | CampRelation.Neutral |
| `Selected.asset` | 选中状态（特殊颜色） | 被 ChessSelectionManager 选中时 |

### 配置字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `OutlineColor` | Color | 描边颜色 |
| `OutlineSize` | float (1-100) | 描边宽度（像素） |
| `EnableDistanceScaling` | bool | 是否启用距离衰减 |
| `MinDistance` | float | 最小距离（此内保持最大宽度） |
| `MaxDistance` | float | 最大距离（超过则不渲染） |
| `DistanceFalloffCurve` | AnimationCurve | 距离衰减曲线 |
| `AutoRefresh` | bool | 是否自动刷新 |
| `RefreshRate` | int | 刷新频率（Hz） |
| `AutoApplyOnStart` | bool | 启动时自动应用 |

### 距离计算公式

```
if (distance <= MinDistance) → OutlineSize（全宽度）
if (distance >= MaxDistance) → 0（不渲染）
else → OutlineSize × DistanceFalloffCurve.Evaluate(归一化距离)
```

## 三、棋子描边 — ChessOutlineController

**文件**: `Assets/AAAGame/Scripts/Game/Combat/Camp/ChessOutlineController.cs`

### 生命周期

1. `ChessEntity.Initialize()` 时自动获取/添加 `ChessOutlineController` 组件
2. 调用 `OutlineController.Initialize(this)` 传入实体引用
3. 初始化时缓存所有子级 Renderer、加载 OutlineConfig 资产
4. 调用 `RefreshOutline()` 根据阵营关系设置初始描边

### 描边优先级规则

```
优先级从高到低：
1. 选中状态 (m_IsSelected = true) → Selected.asset
2. 阵营关系判断：
   - CampRelation.Enemy  → Enemy.asset（红色）
   - CampRelation.Ally   → Ally.asset（绿色）
   - CampRelation.Neutral → Interactive.asset（黄色）
   - CampRelation.Self   → null（不描边）
```

### 阵营关系判断

通过 `CampRelationService.GetRelationToLocalPlayer(entity.Camp)` 判断：

```
CampType 枚举：
  Player = 0   （玩家阵营）
  Enemy = 1    （敌人阵营）
  Neutral = 2  （中立阵营）
  Team1~4 = 10~13（PVP 预留）

CampRelation 枚举：
  Self    — 自己
  Ally    — 友军（同阵营）
  Enemy   — 敌人（敌对阵营）
  Neutral — 中立
```

**默认敌对关系**：Player ↔ Enemy 互为敌对，Neutral 不与任何阵营敌对。

### 选中交互

由 `ChessSelectionManager` 驱动：
- 左键点击玩家棋子 → `entity.OutlineController.SetSelected(true)` → 描边切换为 Selected 配置
- 再次点击 / 点击空地 / 右键 → `SetSelected(false)` → 恢复阵营描边

### ⚠️ 需要修改的问题

1. **配置加载仅限 Editor** — `LoadOutlineConfigs()` 使用 `#if UNITY_EDITOR` + `AssetDatabase.LoadAssetAtPath`，**打包后无法加载配置**。需要改为 Resources.Load 或通过 Prefab 序列化引用。

2. **CampRelation.Self 判断缺失** — `CampRelationService.GetRelationToLocalPlayer()` 在 `entityCamp == s_LocalPlayerCamp` 时返回 `Ally` 而不是 `Self`，导致**玩家自己的棋子也会显示友军描边**（绿色），而代码中 `Self` 分支返回 null（不描边）。实际效果：玩家棋子有绿色描边。需要确认这是否是预期行为。

3. **距离缩放依赖 CameraRegistry** — `ApplyOutline()` 中使用 `CameraRegistry.PlayerCamera`，如果摄像机未就绪会跳过距离缩放，使用默认宽度。

4. **没有刷新机制** — `RefreshOutline()` 只在初始化和选中状态变化时调用。如果棋子在运行时改变阵营（如被控制），描边不会自动更新，需要外部手动调用 `RefreshOutline()`。

5. **Renderer 缓存不更新** — `CacheRenderers()` 只在初始化时调用一次。如果棋子模型运行时变化（换装/特效），描边会遗漏新的 Renderer。

## 四、通用描边 — OutlineDisplayManager

**文件**: `Assets/AAAGame/Scripts/TA/OuterGlow/OutlineDisplayManager.cs`

### 工作模式

- 单例模式（`SingletonBase<OutlineDisplayManager>`），场景中放置
- 通过 `LayerMappings` 配置哪些 Layer 的物体需要描边
- 两种检测方式：物理检测（OverlapSphere）+ Renderer 查找（FindObjectsOfType 兜底）
- 协程定时扫描，自动追踪/移除范围内物体

### 核心流程

```
ScanCoroutine (10Hz)
  → PerformScan() 检测范围内物体
    → ProcessDetectedObject() 按 Layer 匹配配置
      → CreateTracked() / UpdateTracked()
        → 入队优先级队列

ProcessQueues (每帧)
  → High/Mid/Low 优先级处理
    → ApplyOutline() → OutlineTest.ApplyOutline()

LayerCheckCoroutine (30Hz)
  → CheckLayerChanges() 检测 Layer 变化
    → HandleLayerChange() 自动切换描边配置
```

### LOD 系统

| 距离范围 | 更新间隔（帧） |
|----------|---------------|
| < NearDistance (30m) | 每帧 |
| < MidDistance (80m) | 每 3 帧 |
| < DetectionRadius×0.8 | 每 5 帧 |
| 更远 | 每 10 帧 |

### ⚠️ 需要修改的问题

1. **使用协程而非 UniTask** — `InitializeAsync()`、`ScanCoroutine()`、`LayerCheckCoroutine()` 都使用 `IEnumerator` + `StartCoroutine`，**违反项目异步规范**（应统一使用 UniTask）。

2. **FindObjectsOfType 性能问题** — `UseRendererFallback` 开启时每次扫描调用 `FindObjectsOfType<Renderer>()`，在大量物体的场景中**严重影响性能**。

3. **与 ChessOutlineController 职责重叠** — 如果 Chess Layer 同时被 `OutlineDisplayManager` 的 `LayerMappings` 包含，会导致两套系统同时操作同一个物体的描边，产生**冲突和闪烁**。

4. **摄像机获取方式** — `InitializeAsync()` 中用协程轮询等待 `CameraRegistry.PlayerCamera`，应改为事件监听或 UniTask.WaitUntil。

5. **HashSet 每帧分配** — `PerformScan()` 中每次 `new HashSet<int>(500)`，会产生 GC 压力。应改为复用。

## 五、渲染组件 — OutlineTest

**文件**: `Assets/AAAGame/Scripts/TA/OuterGlow/OutlineTest.cs`

轻量桥接组件，挂在需要描边的 GameObject 上：
- 缓存子级所有 Renderer
- `ApplyOutline(config, size)` → 调用 `OutlineRenderFeature.DrawOrUpdateOutlines()`
- `RemoveOutline()` → 调用 `OutlineRenderFeature.RemoveDrawOutlines()`
- `OnDestroy` 时自动移除描边

**注意**：`OutlineDisplayManager` 会自动给追踪到的物体添加 `OutlineTest` 组件；`ChessOutlineController` 不使用 `OutlineTest`，而是直接调用 `OutlineRenderFeature`。

## 六、渲染管线 — OutlineRenderFeature

**文件**: `Assets/AAAGame/Scripts/TA/OuterGlow/OutlineRenderFeature.cs`

### 渲染流程（每个 OutlineData 独立渲染，不合批）

```
Step 1: 绘制轮廓蒙版
  → 用 OutlineShader Pass 0 将目标 Renderer 画到 _srcRt（R8G8 格式）

Step 2: 3 次高斯模糊
  → BlurShader 逐步模糊，步长 = OutlineSize / 3
  → 模糊 RT 使用降采样（RTDownsample = 2 时为半分辨率）

Step 3: 生成描边
  → OutlineShader Pass 1：用原始蒙版减去模糊结果 → _outlineRt

Step 4: 合成到屏幕
  → OutlineShader Pass 2：带颜色和强度合成到 cameraColorTarget
```

### 性能优化参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `RTDownsample` | 2 | RT 降采样倍数（1=全分辨率，4=1/4） |
| `EnableFrustumCulling` | true | 视锥剔除 |
| `EnableDistanceLOD` | true | 距离裁剪 |
| `MaxOutlineDistance` | 100m | 超过此距离不渲染 |
| `ClipSize` | 0.5 | 描边边缘锐利程度 |
| `Intensity` | 1.0 | 描边强度 |
| `ExpPower` | 1.0 | 衰减曲线指数 |

### 数据管理 API

| 方法 | 说明 |
|------|------|
| `DrawOrUpdateOutlines(renderers, color, size)` | 添加或更新描边 |
| `RemoveDrawOutlines(renderers)` | 移除描边 |
| `UpdateOutlines(renderers, color, size)` | 仅更新已有描边 |
| `StopDrawOutline()` | 清除所有描边 |

### ⚠️ 需要修改的问题

1. **Material 泄漏** — `OutlineDrawer.Clear()` 使用 `DestroyImmediate`，在运行时应使用 `Destroy`。

2. **RenderTexture 管理** — 每个 `OutlineDrawer` 持有独立的 RT 组（src + 3 blur + outline = 5 张）。大量描边对象时 RT 数量激增。

3. **未合批渲染** — 注释中说明"无合批版本"，每个 OutlineData 独立走一遍完整的 渲染→模糊→合成 流程，DrawCall 较高。同色同宽的描边应该可以合批处理。

4. **CheckId 使用线性搜索** — `_outlineDatas` 是 List，每次 CheckId 遍历整个列表。应改为 Dictionary<int, OutlineData>。

## 七、交互流程汇总

### 棋子从创建到显示描边

```
ChessEntity.Initialize()
  → AddComponent<ChessOutlineController>()
  → OutlineController.Initialize(this)
    → CacheRenderers()（获取所有子级 Renderer）
    → LoadOutlineConfigs()（加载 4 个 .asset 配置）
    → RefreshOutline()
      → GetTargetOutlineConfig()（查阵营关系）
      → ApplyOutline(config)
        → OutlineRenderFeature.Instance.DrawOrUpdateOutlines(renderers, color, size)
```

### 选中棋子时

```
ChessSelectionManager.Tick()
  → HandleMouseClick()
    → 射线检测 Chess Layer
    → SelectChess(entity)
      → entity.OutlineController.SetSelected(true)
        → RefreshOutline()
          → 返回 m_SelectedOutlineConfig（选中优先级最高）
          → ApplyOutline(Selected.asset)
```

### 取消选中

```
DeselectChess()
  → entity.OutlineController.SetSelected(false)
    → RefreshOutline()
      → 恢复阵营描边（Enemy/Ally/Neutral/null）
```

### 通用物体描边（OutlineDisplayManager）

```
ScanCoroutine → OverlapSphere + FindObjectsOfType
  → 物体 Layer 在 LayerMappings 中 → CreateTracked
    → 入队 → ProcessQueues → AddComponent<OutlineTest>
      → OutlineTest.ApplyOutline() → OutlineRenderFeature.DrawOrUpdateOutlines()

Layer 变化时：
  → LayerCheckCoroutine → HandleLayerChange
    → 移除旧描边 → 按新 Layer 查配置 → 重新应用
```

## 八、需要修改的问题汇总

| # | 所在类 | 问题 | 严重程度 | 建议 |
|---|--------|------|----------|------|
| 1 | ChessOutlineController | 配置加载使用 `#if UNITY_EDITOR`，打包后失效 | **严重** | 改为 Prefab 序列化引用或 Resources.Load |
| 2 | ChessOutlineController | 无运行时刷新机制，阵营变化时描边不更新 | 中等 | 添加事件监听或定时刷新 |
| 3 | ChessOutlineController | Renderer 缓存不更新，换装后描边遗漏 | 低 | 换装时调用 RefreshRenderers |
| 4 | OutlineDisplayManager | 使用协程而非 UniTask，违反项目规范 | 中等 | 改为 UniTask |
| 5 | OutlineDisplayManager | FindObjectsOfType 性能差 | **严重** | 改为注册制或 Tag 查找 |
| 6 | OutlineDisplayManager | 与 ChessOutlineController 职责重叠 | 中等 | 明确各自管理范围，Chess Layer 排除 |
| 7 | OutlineDisplayManager | HashSet 每帧 new | 低 | 复用缓存 |
| 8 | OutlineRenderFeature | Material 用 DestroyImmediate | 低 | 改为 Destroy |
| 9 | OutlineRenderFeature | 不合批，每个对象独立渲染 | 中等 | 同色同宽对象合批 |
| 10 | OutlineRenderFeature | CheckId 线性搜索 | 低 | 改为 Dictionary |
| 11 | CampRelationService | 同阵营返回 Ally 而非 Self | 需确认 | 确认设计意图 |

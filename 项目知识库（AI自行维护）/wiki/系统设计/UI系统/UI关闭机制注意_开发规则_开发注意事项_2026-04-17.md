### 1. UI 关闭方式选择
- **带动画关闭**：`GF.UI.Close(UIForm)` - 适用于重要 UI，用户体验优先
- **直接关闭**：`GF.UI.CloseUIForm(uiFormId)` - 适用于简单提示 UI，性能优先

### 2. 等待机制选择
- **事件驱动**：订阅 `CloseUIFormCompleteEventArgs` 事件，立即响应
- **轮询检测**：`while(HasUIForm())` 每帧检查，存在延迟风险

### 3. 事件订阅注意事项
```csharp
// 正确的事件类型转换
System.EventHandler<GameFramework.Event.GameEventArgs> handler = (sender, e) =>
{
    if (e is UnityGameFramework.Runtime.CloseUIFormCompleteEventArgs closeArgs)
    {
        // 处理关闭事件
    }
};
```

## 最佳实践
1. **统一关闭检测**：关闭方式和检测方式保持一致
2. **优先事件驱动**：避免轮询等待，提高响应性能
3. **简单 UI 直接关闭**：跳过不必要的动画延迟
4. **添加超时保护**：防止事件监听失效导致的死锁

## 常见陷阱
- 混用 `Close()` 和 `CloseUIForm()` 导致检测不一致
- 忘记取消事件订阅导致内存泄漏
- 事件类型转换错误导致编译失败

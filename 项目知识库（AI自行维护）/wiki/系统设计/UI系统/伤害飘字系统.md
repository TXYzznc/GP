## 系统架构

### 核心组件

| 组件 | 职责 |
|------|------|
| `DamageFloatingTextManager` | 主管理器，定义伤害类型枚举，管理配置和对象池，提供统一显示接口 |
| `DamageFloatingTextAnimationLibrary` | 动画库，含19种飘字动画（基础上升/弹跳缩放/震动/螺旋等） |
| `DamagePopupItem` | 飘字显示组件，支持描边，对象池管理 |
| `CombatVFXManager` | 战斗特效管理器，调用 ShowDamagePopup 触发飘字 |

### 文件清单

- `Assets/AAAGame/Scripts/Game/Combat/Effects/DamageFloatingTextManager.cs`
- `Assets/AAAGame/Scripts/Game/Combat/Effects/DamageFloatingTextAnimationLibrary.cs`
- `Assets/AAAGame/Scripts/Game/Combat/Effects/DamagePopupItem.cs`
- `Assets/AAAGame/Scripts/Game/Combat/Effects/CombatVFXManager.cs`
- `Assets/AAAGame/Scripts/DataTable/DamageFloatingTextTable.cs`（工具自动生成，勿手改）

### 技术特性

- **20种伤害类型**：普通伤害、暴击伤害、法术伤害、真实伤害、各种元素伤害（毒/火/冰/雷）、治疗效果、护盾吸收、经验/金币获得等
- **19种动画效果**：对应不同伤害类型，通过配置表的 AnimationType 字段选择
- **对象池优化**：减少 GC 压力，自动回收复用
- **配置化**：所有视觉参数均从 DamageFloatingTextTable 读取，不硬编码

## 配置表

### 文件位置

- **TXT 源文件**：`MCP工作区/配置表/DamageFloatingTextTable.txt`
- **XLSX 文件**：`MCP工作区/配置表/DamageFloatingTextTable.xlsx`

### 字段说明

| 字段名 | 类型 | 说明 |
|--------|------|------|
| ID | int | 伤害类型唯一标识 |
| Name | string | 伤害类型名称 |
| FontSize | float | 字体大小 |
| TextColor | string | 文字颜色（HTML格式，如 `#FF6B35`） |
| OutlineColor | string | 描边颜色（HTML格式） |
| IsGradient | bool | 是否使用渐变效果 |
| GradientStartColor | string | 渐变起始色 |
| GradientEndColor | string | 渐变结束色 |
| AnimationType | int | 动画类型ID（1-19有效） |
| ScaleMultiplier | float | 缩放倍数 |
| Duration | float | 持续时间（秒） |
| MoveDistance | float | 移动距离（像素） |
| FadeInTime | float | 淡入时间（秒） |
| FadeOutTime | float | 淡出时间（秒） |
| EffectId | int | 特效资源ID（预留） |
| SoundId | int | 音效资源ID（预留） |
| Priority | int | 显示优先级 |

### 颜色格式规范

- 必须使用 HTML 格式（`#RRGGBB` 或 `#RRGGBBAA`）
- 不支持颜色名称（如 "red"）
- 解析失败时使用默认白色并输出警告日志

## 使用方式

### 显示飘字

```csharp
// 推荐：使用枚举方式
DamageFloatingTextManager.Instance.ShowDamageText(
    DamageFloatingTextManager.DamageType.暴击伤害,
    1250f,
    targetPosition
);

// 也支持 ID 方式
DamageFloatingTextManager.Instance.ShowDamageText(
    2, // 暴击伤害 ID
    "1250",
    targetPosition
);

// 治疗效果
DamageFloatingTextManager.Instance.ShowDamageText(
    DamageFloatingTextManager.DamageType.治疗效果,
    500f,
    playerPosition
);
```

### 获取配置信息

```csharp
var config = DamageFloatingTextManager.Instance.GetConfig(
    DamageFloatingTextManager.DamageType.暴击伤害

if (config != null)
{
    DebugEx.Log($"伤害类型: {config.Name}, 字体大小: {config.FontSize}");
}
```

## 扩展指南

1. 在配置表中添加新行数据
2. 在 `DamageType` 枚举中添加对应项
3. 可选：在动画库中添加专属动画效果

### 新增动画效果

2. 在 `DamageFloatingTextAnimationLibrary` 中实现动画方法

## 注意事项

1. **配置表加载**：确保配置表已正确加载，配置表类由工具自动生成，勿手改
2. **动画类型 ID**：必须在 1-19 范围内，超出范围会使用默认的基础上升动画
3. **对象池**：使用对象池管理飘字对象，避免频繁创建销毁
4. **异步动画**：动画使用异步方式，不会阻塞主线程
5. **WorldCanvas**：飘字依赖场景中的 WorldCanvas 对象，场景中不能出现多个同名 Canvas

### 后续可扩展

- 特效集成：根据配置表的 effectId 加载粒子特效
- 音效支持：根据 soundId 播放对应音效
- 优先级系统：基于 priority 的显示优先级管理

## 问题记录：重复显示

**发现时间**：2026-03-11

### 诊断结论

通过 Unity Skills API 诊断发现场景中存在 **12 个 Canvas 对象**，多个 Canvas 可能同时渲染伤害飘字。

**可能的根本原因**：

1. **多个 WorldCanvas 实例**：场景中存在多个同名 WorldCanvas，每个都在显示飘字
2. **Canvas 渲染模式问题**：WorldCanvas 的渲染模式设置不正确导致重复渲染
3. **对象池状态异常**：飘字对象未被正确重置，导致显示异常

### 解决方案

#### 1. CombatVFXManager 初始化时添加 Canvas 严格检查

在初始化逻辑（约第 185-195 行）中，查找所有名为 `WORLD_CANVAS_NAME` 的 Canvas，若存在多个，保留第一个并禁用其余。

```csharp
var worldCanvasObjects = GameObject.FindObjectsOfType<Canvas>()
    .Where(c => c.name == WORLD_CANVAS_NAME)
    .ToArray();

if (worldCanvasObjects.Length > 1)
{
    DebugEx.WarningModule("CombatVFXManager", $"发现多个 {WORLD_CANVAS_NAME}，禁用多余实例");
    for (int i = 1; i < worldCanvasObjects.Length; i++)
    {
        worldCanvasObjects[i].gameObject.SetActive(false);
    }
}
```

#### 2. 增强对象池重置逻辑

`GetPopupFromPool()` 返回对象时，必须重置 `localScale`、`localPosition` 和 `CanvasGroup.alpha`，防止状态残留。

#### 3. 飘字位置去重（可选）

在 `ShowDamagePopup` 中检查是否在相同位置已有飘字（阈值 0.5 单位），避免同帧重复显示：

```csharp
const float MIN_DISTANCE = 0.5f;
bool hasDuplicatePosition = s_ActivePopups.Any(popup =>
    popup != null &&
    Vector3.Distance(popup.transform.position, worldPosition) < MIN_DISTANCE);

if (hasDuplicatePosition)
{
    DebugEx.WarningModule("CombatVFXManager", $"位置 {worldPosition} 附近已有飘字，跳过");
    return;
}
```

### 验证步骤

1. 在 Unity 中搜索所有名为 "WorldCanvas" 的对象，确保只有一个
2. 验证 WorldCanvas 的 RenderMode 为 "World Space"
3. 检查主摄像机的 Culling Mask 是否正确排除 WorldCanvas 所在 Layer
4. 实际触发伤害，观察飘字是否还有重复

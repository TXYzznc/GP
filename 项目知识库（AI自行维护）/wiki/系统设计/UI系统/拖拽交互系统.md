## 架构设计

### 核心理念

**旧架构问题**：6 个不同的 Handle* 方法，逻辑重复；新增容器需要 N-1 个新方法；事件驱动导致时序问题。

**新设计原则**：
- **格子是一等公民**：每个 `InventorySlotUI` 知道自己属于哪个容器
- **容器自主交互**：容器定义自己能与哪些容器交互，如何交互
- **统一拖拽接口**：所有拖拽通过 `ISlotContainer.TryMoveToContainer()` 执行

### 核心效果

**旧架构**：HandleDrop 有 6+ 个分支，每个 Handle* 方法 20-50 行，逻辑重复。

**新架构**：
```csharp
private void HandleDrop(InventorySlotUI src, InventorySlotUI dst)
{
    bool success = src.SlotContainer.TryMoveToContainer(
        src.SlotIndex,
        dst.SlotContainer,
        dst.SlotIndex
    );
}
```

## 核心组件

### ISlotContainer 接口

```csharp
public interface ISlotContainer
{
    SlotContainerType ContainerType { get; }
    InventorySlot GetSlot(int slotIndex);
    bool TryMoveToContainer(int fromSlotIndex, ISlotContainer targetContainer, int targetSlotIndex);
    bool CanInteractWith(SlotContainerType otherContainerType);
}
```

### SlotContainerBase 基类

```csharp
public abstract class SlotContainerBase : MonoBehaviour, ISlotContainer
{
    public abstract SlotContainerType ContainerType { get; }
    public abstract InventorySlot GetSlot(int slotIndex);
    public abstract bool CanInteractWith(SlotContainerType otherContainerType);
    public abstract bool ExecuteMove(int fromSlotIndex, ISlotContainer targetContainer, int targetSlotIndex);

    public bool TryMoveToContainer(int fromSlotIndex, ISlotContainer targetContainer, int targetSlotIndex)
    {
        // 通用检查逻辑，委托给 ExecuteMove() 执行具体操作
    }
}
```

### SlotContainerType 枚举

```csharp
public enum SlotContainerType
{
    Inventory,  // 背包
    Warehouse,  // 仓库
    Equip,      // 装备栏
    FastBar,    // 快捷栏
}
```

### 具体容器实现

| 容器类 | CanInteractWith |
|--------|-----------------|
| `InventorySlotContainer` | Inventory、Warehouse、FastBar |
| `WarehouseSlotContainer` | Inventory、Warehouse |
| `FastBarSlotContainer` | Inventory、FastBar |
| `EquipSlotContainer` | Equip（暂仅自身） |

每个容器的 `ExecuteMove()` 通过 switch 表达式分发到对应的移动方法（MoveToInventory / MoveToWarehouse / MoveToFastBar 等）。

### FastBarManager

管理快捷栏数据的独立容器管理器（与 FastBarSlotContainer 配合使用）。

**关键方法**：`Initialize()`、`StoreItemToSlot()`、`SwapSlots()`、`ClearSlot()`、`GetSlot()`、`GetAllSlots()`

**注意**：FastBarManager 管理数据，FastBarSlotContainer 提供容器接口，两者配合，不要删除 FastBarManager。

## 交互规则

### 支持的拖拽组合

| 源 | 目标 | 操作 |
|----|------|------|
| Inventory | Inventory | 移动/交换 |
| Inventory | Warehouse | 存入 |
| Inventory | FastBar | 存入 |
| Warehouse | Inventory | 取出（空/堆叠/交换三种情况） |
| Warehouse | Warehouse | 交换 |
| FastBar | Inventory | 移出（空/堆叠/交换三种情况） |
| FastBar | FastBar | 交换 |

### 不支持的组合

- Warehouse → FastBar（仓库不能直接到快捷栏）
- FastBar → Warehouse（快捷栏不能直接到仓库）
- Equip → * / * → Equip（装备栏暂不支持外部拖拽）

## UI 初始化流程

初始化顺序必须保证：**OnInit 创建容器组件 → CollectSlotUIs 绑定格子**。

```csharp
// InventoryUI 示例
protected override void OnInit(object userData)
{
    m_InventorySlotContainer = GetComponent<InventorySlotContainer>()
        ?? gameObject.AddComponent<InventorySlotContainer>();
}

private void CollectSlotUIs()
{
    for (int i = 0; i < totalSlots; i++)
    {
        var slot = CreateSlot(varInventorySlotUI);
        slot.SetSlotIndex(i);
        slot.SetContainerType(SlotContainerType.Inventory);
        slot.SetSlotContainer(m_InventorySlotContainer);  // 必须在 SetContainerType 之后
        m_InventorySlots.Add(slot);
    }
}
```

WarehouseUI、FastBar 集成方式类似，各自创建并绑定对应容器类型。

## 注意事项

1. **SetContainerType() 必须先于 SetSlotContainer() 调用**
2. **每个 UI 只需一个容器组件**（所有同类格子共享同一容器实例）
3. **交换策略**：
   - 相同容器 → `SwapSlots()`（交换 SlotIndex/ItemStack）
   - 不同容器 → 清空目标后 SetItem/SetItemStack，返回物品到源
4. **CreateItem() 返回 ItemBase**，与 StoreItemToSlot 参数类型对齐

### 扩展新容器

1. 创建 `NewSlotContainer : SlotContainerBase`
2. 实现 `CanInteractWith()` 定义交互规则
3. 实现 `ExecuteMove()` 定义移动逻辑
4. 在对应 UI 的初始化中调用 `SetSlotContainer(newContainer)`

拖拽系统自动支持，无需修改 InventoryDragHandler。

## InventoryItemUI 组件问题

**问题**：InventoryDragHandler 无法正确接收拖拽事件。

### 组件层级

```
InventorySlotUI (父)
  └─ InventoryItemUI (Image, raycastTarget=1 ⚠️)
      ├─ ItemImg (Image, raycastTarget=0 ✓)
      └─ ItemBtn (Button, raycastTarget=1)
```

### 根本原因

InventoryItemUI 自身的 Image 组件 `raycastTarget=1` 且完全透明（alpha=0），会优先拦截射线检测，影响 InventoryDragHandler 接收拖拽事件。

### 修复方案（推荐）

在 Prefab 中直接添加 InventoryDragHandler 组件（不动态添加），并将 InventoryItemUI 的 Image 设为 `raycastTarget=0`。

```
1. 打开 Assets/AAAGame/Prefabs/UI/Items/InventoryItemUI.prefab
2. 移除代码中动态 AddComponent<InventoryDragHandler>() 的逻辑
3. Inspector 中直接添加 InventoryDragHandler 组件
4. 设置 InventoryItemUI > Image > raycastTarget = 0
5. 保存 Prefab
```

### 关键配置清单

- [x] Button (ItemBtn) - raycastTarget = 1
- [x] Image (ItemImg) - raycastTarget = 0
- [ ] **InventoryItemUI.Image - 需改为 raycastTarget = 0**
- [ ] **InventoryDragHandler - 确认正确挂载**
- [ ] EventSystem 存在

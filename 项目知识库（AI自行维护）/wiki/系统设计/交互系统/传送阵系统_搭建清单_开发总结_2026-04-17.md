## ⚠️ 必须手动完成的步骤

### 第 1 步：将 OverworldUI 添加到 UITable

**文件**：`AAAGameData/DataTables/Core/UITable.xlsx`

1. 打开 Excel 文件
2. 在最后一行后新增一行，设置：
   - **Id**：33（或下一个递增 ID，当前最大是 32）
   - **UIFormName**：OverworldUI
   - **UIGroupId**：1（主菜单层级）
   - **IsSingleton**：true
   - **CacheType**：0

3. 保存文件
4. 在 Unity 编辑器运行：**GameFramework → DataTable → Generate**
5. 生成后检查 `UIViews.cs` 中是否添加了 `OverworldUI = 33` 的枚举项

**结果**：OverworldUI 会自动生成到 UIViews 枚举中

---

### 第 2 步：搭建 OverworldUI Prefab

**位置**：`Assets/AAAGame/Prefabs/UI/OverworldUI.prefab`

#### 2.1 创建 Canvas 节点

```
Prefab: OverworldUI
├─ Canvas
│  ├─ RenderMode: Screen Space - Camera
│  ├─ Canvas Scaler: 1920x1080
│  └─ GraphicRaycaster
```

#### 2.2 添加脚本和 Variables

1. 在 Canvas 节点上添加脚本：`OverworldUI.cs`
2. 右键 Canvas → **DataTable → Generate UI Variables**
3. 生成 `OverworldUI.Variables.cs`

#### 2.3 添加 UI 子对象

```
Canvas (OverworldUI)
├─ Background (Image)
│  └─ Color: 半透明黑色（0,0,0,0.5）
│
├─ Title (Text)
│  ├─ Text: "选择目的地"
│  └─ Font Size: 40
│
├─ MapContainer (GridLayoutGroup)
│  ├─ Grid Constraint: Flexible
│  ├─ Cell Size: (200, 200)
│  ├─ Spacing: (20, 20)
│  └─ Padding: (20,20,20,20)
│
└─ CloseBtn (Button)
   ├─ Text: "关闭"
   └─ Anchors: Bottom Right
```

#### 2.4 配置 Variables 字段

在 OverworldUI.Variables.cs 中（或通过 Hierarchy 右键），配置：

- **varCloseBtn** ← 关闭按钮 (Button)
- **varMapItemUI** ← MapContainer (GameObject)

---

### 第 3 步：创建 MapItemUI 预制体

**位置**：`Assets/AAAGame/Prefabs/UI/MapItemUI.prefab`

#### 3.1 创建按钮节点

```
Prefab: MapItemUI (继承自 Button)
├─ MapItemUI (Button)
│  ├─ MapItemUI.cs (脚本)
│  ├─ MapItemSceneIdHolder.cs (脚本)
│  ├─ LayoutElement (preferred width: 200, height: 200)
│  │
│  └─ Content (Panel)
│     ├─ Btn (Button) → varBtn
│     ├─ MapName (Text) → varMapName
│     ├─ Mask (Image) → varMask
│     └─ InteractUI (Panel) → varInteractUI
```

#### 3.2 子对象详细配置

**MapName (Text)**
- 显示场景名称
- Font Size: 18
- Color: 白色

**Mask (Image)**
- 遮罩图片（未解锁时显示）
- Color: 半透明黑色（0,0,0,0.7）
- 默认状态：Active（运行时根据解锁状态切换）

**InteractUI (Panel)**
- 鼠标悬浮时显示的信息面板
- 包含文本：显示解锁条件或"点击传送"
- 默认状态：Inactive（运行时在 OnPointerEnter/Exit 时切换）

#### 3.3 配置 MapItemSceneIdHolder

在 MapItemUI 上添加 `MapItemSceneIdHolder` 组件，在 Inspector 中设置：

- **SceneId**：这个地图项对应的场景 ID（从 SceneTable）

#### 3.4 配置 Variables 字段

在 MapItemUI.Variables.cs 中，配置：

- **varBtn** ← Btn (Button)
- **varMapName** ← MapName (Text)
- **varMask** ← Mask (Image)
- **varInteractUI** ← InteractUI (RectTransform)

---

### 第 4 步：将 MapItemUI 实例化到 OverworldUI

1. 打开 OverworldUI 预制体
2. 在 MapContainer 下创建 **MapItemUI 的实例**
3. 为每个实例配置 **MapItemSceneIdHolder.SceneId**
   - 第一个地图项：SceneId = 1
   - 第二个地图项：SceneId = 2
   - 以此类推...

示例：

```
MapContainer
├─ MapItemUI_Home (SceneId=1)
├─ MapItemUI_World1 (SceneId=2)
└─ MapItemUI_Dungeon1 (SceneId=3)
```

### 第 5 步：配置 SceneTable（如尚未配置）

**文件**：`AAAGameData/DataTables/SceneTable.xlsx`

确保包含所有可传送的场景，例如：

| Id | SceneName | SceneType | DisplayName | ConditionType | ConditionParam |
|----|-----------|-----------|-------------|---------------|----------------|
| 1 | MainBase | 1 | 主基地 | 0 | |
| 2 | World1 | 2 | 大世界 第一章 | 1 | |
| 3 | Dungeon1 | 4 | 副本 第一关 | 3 | 5 |

**说明**：
- **ConditionType 0**：无条件（任何人都可以进入）
- **ConditionType 1**：完成教程
- **ConditionType 3**：达到等级（参数为等级值）
- **ConditionType 2**：完成任务（参数为任务 ID）
- 其他参考 [场景条件系统文档](传送阵系统设计_系统设计_2026-04-17.md)

运行 **GameFramework → DataTable → Generate** 生成代码。

### 第 6 步：在场景中放置传送阵

在你的游戏场景中：

1. 创建一个空 GameObject，命名为 `TeleportGate`
2. 添加脚本：`TeleportGateInteractable.cs`
3. 添加 **SphereCollider**：
   - ✅ Is Trigger: true

4. 在 Inspector 中配置：
   - **interactionTip**: "进入传送阵"
   - **priority**: 1
   - **interactionRadius**: 2

放在适当位置（如主基地出口）。

## 🚨 常见问题排查

| 问题 | 原因 | 解决 |
|------|------|------|
| 编译错误：`UIViews.OverworldUI 不存在` | UITable 中没有添加 OverworldUI | 完成第 1 步，运行 DataTableGenerator |
| 场景名称显示为空 | SceneTable 没有配置 DisplayName | 检查 SceneTable，运行 DataTableGenerator |
| Mask 不显示 | PlayerSaveData 没有正确初始化 | 确保游戏启动时正确加载了存档数据 |
| 点击不响应 | 按钮没有正确连接 OnClick 事件 | 检查 MapItemUI.Awake() 中的事件注册 |
| InteractUI 始终显示 | 鼠标事件没有触发 | 确保 Canvas 上有 GraphicRaycaster，MapItemUI 是 Button 组件 |

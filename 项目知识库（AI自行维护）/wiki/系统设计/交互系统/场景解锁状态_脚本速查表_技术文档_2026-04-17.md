## 📋 检查流程速查

### 完整检查流程

```csharp
// Step 1: 获取存档数据
var saveData = PlayerAccountDataManager.Instance.CurrentSaveData;
if (saveData == null)
{
    Log.Error("没有加载存档数据");
    return;
}

// Step 2: 获取场景数据
var sceneTable = GF.DataTable.GetDataTable<SceneTable>();
var sceneData = sceneTable.GetDataRow(sceneId);
if (sceneData == null)
{
    Log.Error($"场景 {sceneId} 不存在");
    return;
}

// Step 3: 检查条件
bool isUnlocked = sceneData.CheckCondition(saveData);

// Step 4: 显示 UI
if (isUnlocked)
{
    // 场景已解锁，可交互
    ShowUnlockedState();
}
else
{
    // 场景未解锁，显示条件
    string conditionMsg = sceneData.GetConditionNotMetMessage();
    ShowLockedState(conditionMsg);
}
```

## 📝 SceneTable 配置快速参考

### 格式说明

| 列名 | 类型 | 说明 | 示例 |
|------|------|------|------|
| Id | int | 场景 ID | 1 |
| SceneName | string | Unity 场景名 | MainBase |
| DisplayName | string | 显示名（本地化 Key） | Home |
| ConditionType | int | 条件类型 (0-5) | 3 |
| ConditionParam | string | 条件参数 | "5" 或 "1,2,3" |

### 配置示例

```
场景 1 (无条件):
Id=1, ConditionType=0, ConditionParam=""
→ 所有人都可进入

场景 2 (需要等级 5):
Id=2, ConditionType=3, ConditionParam="5"
→ GlobalLevel >= 5 时可进入

场景 3 (需要完成任务 1 和 2):
Id=3, ConditionType=2, ConditionParam="1,2"
→ CompletedQuestIds 包含 1 和 2 时可进入

场景 4 (需要解锁科技 10):
Id=4, ConditionType=5, ConditionParam="10"
→ UnlockedTechIds 包含 10 时可进入
```

## ⚠️ 常见陷阱

| 问题 | 原因 | 解决 |
|------|------|------|
| `CheckCondition` 始终返回 false | PlayerSaveData 为 null | 确保存档已加载 |
| 修改后场景仍显示未解锁 | 没有调用 SaveCurrentSave() | 记得保存存档 |
| 参数解析失败 | ConditionParam 格式错误 | 检查是否正确使用逗号分隔 |
| 条件提示显示重复 | 多次调用 GetConditionNotMetMessage() | 缓存结果，避免重复调用 |

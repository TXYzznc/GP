## 📊 系统架构

场景解锁状态的管理涉及三层体系：

```
PlayerAccountDataManager (单例管理器)
    ↓
PlayerSaveData (当前存档数据)
    ├─ HasCompletedTutorial: bool          ← 新手教程完成状态
    ├─ GlobalLevel: int                    ← 玩家等级
    ├─ CompletedQuestIds: List<int>        ← 已完成任务列表
    ├─ UnlockedTechIds: List<int>          ← 已解锁科技列表
    └─ InventoryItems: string (JSON)       ← 背包物品（包含ID）
         ↓
    SceneTableExtensions.CheckCondition() (条件检查)
         ↓
    SceneConditionType 枚举 (6 种条件)
```

### 3. SceneTableExtensions (条件检查逻辑)

**位置**：`Assets/AAAGame/Scripts/DataTablePartial/SceneTableExtensions.cs`

**核心方法**：

#### CheckCondition()
```csharp
public static bool CheckCondition(this SceneTable table, PlayerSaveData saveData)
{
    var conditionType = table.GetConditionTypeEnum();
    
    switch (conditionType)
    {
        case SceneConditionType.None:
            return true;  // 无条件，始终可进入
            
        case SceneConditionType.CompleteTutorial:
            return saveData.HasCompletedTutorial;
            
        case SceneConditionType.CompleteQuest:
            // 检查是否完成所有指定任务
            var requiredQuestIds = table.GetConditionParamAsIntList();
            foreach (var questId in requiredQuestIds)
            {
                if (!saveData.CompletedQuestIds.Contains(questId))
                    return false;
            }
            return true;
            
        case SceneConditionType.ReachLevel:
            // 检查等级是否满足
            int requiredLevel = table.GetConditionParamAsInt();
            return saveData.GlobalLevel >= requiredLevel;
            
        case SceneConditionType.HasItem:
            // 检查是否拥有指定物品
            var requiredItemIds = table.GetConditionParamAsIntList();
            var inventoryItems = saveData.GetInventoryItems();
            foreach (var itemId in requiredItemIds)
            {
                if (!inventoryItems.Exists(item => item.ItemId == itemId))
                    return false;
            }
            return true;
            
        case SceneConditionType.UnlockTech:
            // 检查是否解锁指定科技
            var requiredTechIds = table.GetConditionParamAsIntList();
            foreach (var techId in requiredTechIds)
            {
                if (!saveData.UnlockedTechIds.Contains(techId))
                    return false;
            }
            return true;
            
        default:
            return false;
    }
}
```

#### GetConditionNotMetMessage()
```csharp
public static string GetConditionNotMetMessage(this SceneTable table)
{
    // 返回玩家友好的提示文本
    var conditionType = table.GetConditionTypeEnum();
    
    return conditionType switch
    {
        SceneConditionType.CompleteTutorial => "需要完成新手引导才能进入此区域",
        SceneConditionType.CompleteQuest => "需要完成指定任务才能进入此区域",
        SceneConditionType.ReachLevel => $"需要达到 {table.GetConditionParamAsInt()} 级才能进入此区域",
        SceneConditionType.HasItem => "需要特定物品才能进入此区域",
        SceneConditionType.UnlockTech => "需要解锁特定科技才能进入此区域",
        _ => "不满足进入条件"
    };
}
```

## 📥 检查场景是否解锁（传送阵使用）

### 标准流程

```csharp
// 1. 获取玩家存档数据

// 2. 获取场景数据
var sceneTable = GF.DataTable.GetDataTable<SceneTable>();
var sceneData = sceneTable.GetDataRow(sceneId);

// 3. 检查是否满足条件

// 4. 根据结果显示 UI
if (isUnlocked)
{
    // 显示已解锁状态，可点击

else
{
    // 显示 Mask 并提示解锁条件

}
```

```csharp
private bool CheckSceneUnlocked()
```

## 🐛 常见问题

### Q: 如何在运行时动态改变场景解锁状态？

A: 直接修改 PlayerSaveData 的相应字段，然后调用 SaveCurrentSave()：

```csharp
var saveData = PlayerAccountDataManager.Instance.CurrentSaveData;
saveData.GlobalLevel = 10;
PlayerAccountDataManager.Instance.SaveCurrentSave();
```

### Q: 多个条件如何同时满足？

A: 在 ConditionParam 中用逗号分隔多个 ID，系统会检查是否全部满足：

```
ConditionType=2 (完成任务)
ConditionParam="1,2,3"
→ 需要同时完成任务 1, 2, 3
```

### Q: 条件检查性能如何？

A: 条件检查非常高效：
- 教程和等级：O(1) 比较
- 任务/科技/物品：O(n) 列表查找（n 通常 < 100）
- 可以实时检查，无需缓存

### Q: 如何添加自定义条件？

A: 在 SceneTableExtensions.CheckCondition() 中添加新的 case：

```csharp
case SceneConditionType.Custom:
    // 自定义逻辑
    return CheckCustomCondition(table.ConditionParam, saveData);
```

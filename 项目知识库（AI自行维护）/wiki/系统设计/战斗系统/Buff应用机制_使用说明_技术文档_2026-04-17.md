## 概述

战斗系统提供了两种 Buff 应用方式，分别适用于不同的使用场景：
1. **配置表方式**：通过配置表自动应用（无条件）
2. **回调方式**：通过技能代码自定义逻辑（有条件）

## 一、配置表方式（系统自动应用）

### 适用场景
- 每次命中都会触发的 Buff
- 不需要任何条件判断
- 简单、通用的 Buff 效果

### 配置方法

在 `SummonChessSkillTable.txt` 中配置：

| 字段 | 说明 | 示例 |
|:---|:---|:---|
| `BuffTriggerType` | 设置为 `1`（命中时应用） | `1` |
| `BuffIds` | 给目标添加的 Buff ID 数组 | `2`（寒霜）或 `1,3`（多个） |
| `SelfBuffIds` | 给自己添加的 Buff ID 数组 | `4`（神力增益） |

### 执行流程

```
投射物命中 → ProjectileHitDetector.OnProjectileHit()
           → HitDetectorBase.ApplyDamage()
           → EffectExecutor.ApplyBuffsOnHit()  ⭐ 在这里自动应用
           → 遍历 BuffIds，逐个添加到目标
```

### 代码示例：嫦娥普攻（无条件应用寒霜）

**配置表：**
```
22  嫦娥普攻  2  2  0.8  2  2503  1  0  0  10  0  0  0  0  1  1  2  0  15  ...
                                      ↑              ↑
                                BuffTriggerType=1  BuffIds=2（寒霜Buff）
```

**技能代码：**
```csharp
public class ChangENormalAttack : ChessNormalAttackBase
{
    public override void ExecuteAttack(ChessEntity caster, ChessEntity target)
    {
        // 1. 计算伤害
        double damage = CalculateDamage(caster, out bool isCritical);

        // 2. 构建命中检测上下文
        HitContext context = new HitContext
        {
            Attacker = caster,
            LockedTarget = target,
            BaseDamage = damage,
            IsCritical = isCritical,
            SkillConfig = m_Config,  // ⭐ 配置表会自动应用 BuffIds
            OnHitCallback = null     // ⭐ 不需要回调
        };

        // 3. 执行命中检测
        IHitDetector detector = HitDetectorFactory.GetDetector(AttackHitType.Projectile);
        detector.Execute(context);
    }
}
```

**结果：**
- 每次命中敌人，自动给目标添加寒霜 Buff（ID=2）
- 不需要写任何额外代码

## 二、回调方式（技能自定义逻辑）

### 适用场景
- 需要条件判断才触发的 Buff
- 需要检查某些状态（如其他 Buff 是否存在）
- 复杂的、可扩展的 Buff 逻辑

### 配置方法

在 `SummonChessSkillTable.txt` 中：

| 字段 | 说明 | 示例 |
|:---|:---|:---|
| `BuffTriggerType` | 设置为 `1`（命中时应用） | `1` |
| `BuffIds` | 留空或设置为 `0` | `0` |
| `SelfBuffIds` | 留空或设置为 `0` | `0` |

在技能代码中实现 `OnHitCallback`。

### 执行流程

```
投射物命中 → ProjectileHitDetector.OnProjectileHit()
           → HitDetectorBase.ApplyDamage()
           → EffectExecutor.ApplyBuffsOnHit()  （跳过，因为 BuffIds=0）
           → context.OnHitCallback?.Invoke()   ⭐ 在这里执行自定义逻辑
```

### 代码示例：后羿普攻（条件应用灼烧）

**配置表：**
```
12  后羿普攻  2  1  1  2  2501  1  0  0  10  0  0  0  0  1  1  0  0  20  ...
                                    ↑              ↑
                              BuffTriggerType=1  BuffIds=0（不配置）
```

**技能代码：**
```csharp
public class HouyiNormalAttack : ChessNormalAttackBase
{
    public override void ExecuteAttack(ChessEntity caster, ChessEntity target)
    {
        // 1. 计算伤害
        double damage = CalculateDamage(caster, out bool isCritical);

        // 2. 构建命中检测上下文
        HitContext context = new HitContext
        {
            Attacker = caster,
            LockedTarget = target,
            BaseDamage = damage,
            IsCritical = isCritical,
            SkillConfig = m_Config,
            OnHitCallback = OnAttackHit  // ⭐ 设置命中回调
        };

        // 3. 执行命中检测
        IHitDetector detector = HitDetectorFactory.GetDetector(AttackHitType.Projectile);
        detector.Execute(context);
    }

    /// <summary>
    /// 普攻命中回调（投射物命中目标时调用）
    /// </summary>
    private void OnAttackHit(ChessEntity target, double damage, bool isCritical)
    {
        if (target == null || m_Ctx == null) return;

        // ⭐ 只有在"烈焰箭激活"时才应用灼烧 Buff
        if (m_Ctx.BuffManager != null && m_Ctx.BuffManager.HasBuff(4)) // 烈焰箭 Buff ID=4
        {
            if (target.BuffManager != null)
            {
                target.BuffManager.AddBuff(1, m_Ctx.Owner, m_Ctx.Attribute); // 灼烧 ID=1
                DebugEx.LogModule("HouyiNormalAttack",
                    $"烈焰箭激活，对 {target.Config?.Name} 附带灼烧效果");
            }
        }
    }
}
```

**结果：**
- 只有当后羿有"烈焰箭"Buff（ID=4）时，普攻才会给目标添加灼烧 Buff（ID=1）
- 没有烈焰箭时，普攻不会应用灼烧

## 三、完整的伤害应用流程

当投射物命中目标时，`HitDetectorBase.ApplyDamage()` 会按以下顺序执行：

```csharp
protected void ApplyDamage(ChessEntity target, HitContext context)
{
    // 1. 播放受击特效
    if (context.HitEffectPath > 0)
    {
        CombatVFXManager.PlayEffect(context.HitEffectPath, target.transform.position);
    }

    // 2. 造成伤害
    target.Attribute.TakeDamage(
        context.BaseDamage,
        context.IsMagicDamage,
        context.IsTrueDamage,
        context.IsCritical
    );

    // ⭐ 3. 应用"命中时"的 Buff（配置表方式）
    if (context.SkillConfig != null)
    {
        EffectExecutor.ApplyBuffsOnHit(
            context.SkillConfig,
            context.Attacker,
            target
        );
    }

    // ⭐ 4. 触发命中回调（回调方式）
    context.OnHitCallback?.Invoke(target, context.BaseDamage, context.IsCritical);
}
```

## 四、两种方式对比

| 对比项 | 配置表方式 | 回调方式 |
|:---|:---|:---|
| **触发时机** | 第 3 步（自动） | 第 4 步（手动） |
| **适用场景** | 无条件应用 | 有条件应用 |
| **配置复杂度** | 简单（只需填表） | 复杂（需要写代码） |
| **灵活性** | 低 | 高 |
| **典型示例** | 嫦娥普攻（每次都附带寒霜） | 后羿普攻（烈焰箭激活时才附带灼烧） |
| **是否会重复触发** | 不会（BuffIds=0 时跳过） | 不会（只在回调中应用） |

## 五、常见问题

### Q1: 两种方式会不会触发两次 Buff？

**A:** 不会。因为：
- 配置表方式：只有 `BuffIds` 不为空时才应用
- 回调方式：只在回调中手动应用
- 后羿普攻配置表中 `BuffIds=0`，所以第 3 步会跳过

### Q2: 什么时候用配置表方式？

**A:** 当 Buff 应用是无条件的，每次命中都触发时使用。例如：
- 嫦娥普攻：每次都附带寒霜
- 某个技能：每次命中都减速敌人

### Q3: 什么时候用回调方式？

**A:** 当 Buff 应用需要条件判断时使用。例如：
- 后羿普攻：只有烈焰箭激活时才附带灼烧
- 某个技能：只有暴击时才附带额外 Buff
- 某个技能：根据目标血量决定是否应用 Buff

### Q4: 可以同时使用两种方式吗？

**A:** 可以！例如：
- 配置表中设置 `BuffIds=2`（无条件应用寒霜）
- 回调中检查条件，额外应用灼烧
- 结果：每次命中都有寒霜，满足条件时还有灼烧

### Q5: 如何避免混淆？

**A:** 建议：
- 简单技能：只用配置表方式
- 复杂技能：只用回调方式
- 在配置表注释中说明使用方式

## 六、最佳实践

### 1. 配置表注释规范

在配置表中添加清晰的注释：

```
# BuffIds: 无条件应用的Buff（每次命中都触发）
# 如果需要条件判断，请在技能代码中使用 OnHitCallback，并将 BuffIds 设置为 0
```

### 2. 技能代码注释规范

在技能类中添加说明：

```csharp
/// <summary>
/// 后羿：远程投射物普通攻击 (ID=12)
/// 造成伤害：100%攻击系数
/// 烈焰箭激活时，攻击附带灼烧效果（通过回调实现条件判断）
/// </summary>
public class HouyiNormalAttack : ChessNormalAttackBase
{
    // ...
}
```

### 3. 日志输出规范

在回调中添加详细的日志：

```csharp
private void OnAttackHit(ChessEntity target, double damage, bool isCritical)
{
    if (m_Ctx.BuffManager != null && m_Ctx.BuffManager.HasBuff(4))
    {
        target.BuffManager?.AddBuff(1, m_Ctx.Owner, m_Ctx.Attribute);
        DebugEx.LogModule("HouyiNormalAttack",
            $"烈焰箭激活，对 {target.Config?.Name} 附带灼烧效果");
    }
    else
    {
        DebugEx.LogModule("HouyiNormalAttack",
            $"烈焰箭未激活，普攻不附带灼烧");
    }
}
```

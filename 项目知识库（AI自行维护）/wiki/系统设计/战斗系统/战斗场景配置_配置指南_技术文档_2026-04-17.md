## 概述

战斗场景由 `BattleArenaManager` 管理，负责在战斗开始时动态生成战斗场地，并提供区域划分功能（我方区域、敌方区域）。

## 一、战斗场地预制体结构

### 预制体位置
```
Assets/AAAGame/Prefabs/Combat/BattleArena.prefab
```

### 必需的子对象结构

战斗场地预制体必须包含以下子对象：

```
BattleArena (根对象)
├── PlayerAnchor (Transform)      - 玩家锚点，用于对齐玩家位置
├── PlayerZone (GameObject)        - 我方区域（必须有 Renderer 组件）
└── EnemyZone (GameObject)         - 敌方区域（必须有 Renderer 组件）
```

### 各子对象说明

#### 1. PlayerAnchor（玩家锚点）
- **作用**：标记玩家在战斗场地中的位置
- **要求**：
  - 必须是场地根对象的直接子对象
  - 名称必须为 `PlayerAnchor`
  - 只需要 Transform 组件
- **工作原理**：
  - 生成场地时，系统会让 `PlayerAnchor` 的世界坐标对齐玩家当前位置
  - 计算公式：`场地位置 = 玩家位置 - PlayerAnchor本地坐标`

**示例配置**：
```
PlayerAnchor
  Position: (0, 0, -5)    // 玩家站在场地前方 5 米处
  Rotation: (0, 0, 0)
  Scale: (1, 1, 1)
```

#### 2. PlayerZone（我方区域）
- **作用**：定义玩家可以放置棋子的区域
- **要求**：
  - 必须是场地根对象的直接子对象
  - 名称必须为 `PlayerZone`
  - 必须有 `Renderer` 组件（用于计算边界）
  - 建议使用 Plane 或 Quad
- **推荐设置**：
  - Layer: `PlacementPlane`（用于射线检测）
  - Material: 半透明绿色材质（视觉提示）

**示例配置**：
```
PlayerZone (Plane)
  Position: (0, 0, -5)     // 在玩家脚下
  Rotation: (0, 0, 0)
  Scale: (2, 1, 2)         // 20x20 米区域
  Layer: PlacementPlane
  Material: PlayerZoneMat (绿色半透明)
```

#### 3. EnemyZone（敌方区域）
- **作用**：定义敌人生成和站位的区域
- **要求**：
  - 必须是场地根对象的直接子对象
  - 名称必须为 `EnemyZone`
  - 必须有 `Renderer` 组件（用于计算边界）
  - 建议使用 Plane 或 Quad
- **推荐设置**：
  - Material: 半透明红色材质（视觉提示）

**示例配置**：
```
EnemyZone (Plane)
  Position: (0, 0, 5)      // 在玩家对面
  Rotation: (0, 0, 0)
  Scale: (2, 1, 2)         // 20x20 米区域
  Material: EnemyZoneMat (红色半透明)
```

## 二、场地生成流程

### 1. 初始化管理器

在战斗准备阶段初始化：

```csharp
// 在 CombatPreparationState.OnEnter() 中
BattleArenaManager.Instance.Initialize(arenaResourceId);
```

- `arenaResourceId`：战斗场地预制体的 ResourceId（从配置表读取）
- 如果传入 0 或无效 ID，会创建默认测试场地

### 2. 生成战斗场地

```csharp
// 获取玩家当前位置
Vector3 playerPosition = PlayerManager.Instance.GetPlayerPosition();

// 异步生成场地（会自动对齐玩家位置）
GameObject arena = await BattleArenaManager.Instance.SpawnArenaAsync(playerPosition);
```

**生成逻辑**：
1. 加载预制体
2. 查找预制体中的 `PlayerAnchor`
3. 计算场地生成位置，使 `PlayerAnchor` 对齐玩家位置
4. 实例化场地
5. 缓存 `PlayerZone` 和 `EnemyZone` 的 Renderer 引用

### 3. 销毁战斗场地

```csharp
// 在战斗结束时调用
BattleArenaManager.Instance.DestroyArena();
```

## 三、区域查询 API

### 1. 检查坐标是否在区域内

```csharp
// 检查是否在我方区域
bool isInPlayerZone = BattleArenaManager.Instance.IsInPlayerZone(worldPosition);

// 检查是否在敌方区域
bool isInEnemyZone = BattleArenaManager.Instance.IsInEnemyZone(worldPosition);
```

**使用场景**：
- 棋子放置时检查是否在合法区域
- 技能释放时检查目标位置
- AI 寻路时限制移动范围

### 2. 获取区域边界

```csharp
// 获取我方区域边界（世界坐标）
Bounds playerBounds = BattleArenaManager.Instance.GetPlayerZoneBounds();

// 获取敌方区域边界（世界坐标）
Bounds enemyBounds = BattleArenaManager.Instance.GetEnemyZoneBounds();
```

**Bounds 常用属性**：
- `center`：区域中心点
- `min`：最小坐标（左下角）
- `max`：最大坐标（右上角）
- `size`：区域尺寸
- `Contains(point)`：检查点是否在区域内

### 3. 获取敌方区域中心

```csharp
// 获取敌方区域中心点（用于计算敌人站位）
Vector3 enemyCenter = BattleArenaManager.Instance.GetEnemyZoneCenter();
```

**使用场景**：
- 敌人站位计算的基准点
- 技能目标点选择

## 四、配置表集成

### 方案 1：通过 SceneTable 配置（推荐）

在 `SceneTable` 中为每个场景配置战斗场地：

```
场景ID | 场景名称 | 战斗场地ResourceId | ...
1      | 新手村   | 1001              | ...
2      | 森林     | 1002              | ...
```

**使用方式**：
```csharp
// 从场景配置读取
var sceneData = GF.DataTable.GetDataTable<SceneTable>().GetDataRow(sceneId);
int arenaResourceId = sceneData.BattleArenaResourceId;

BattleArenaManager.Instance.Initialize(arenaResourceId);
```

### 方案 2：通过 EnemyTable 配置

在 `EnemyTable` 中为每个敌人配置专属场地：

```
敌人ID | 敌人名称 | 战斗场地ResourceId | ...
1      | 哥布林   | 1001              | ...
2      | 巨龙     | 1003              | ...
```

**使用方式**：
```csharp
// 从敌人配置读取
var enemyData = GF.DataTable.GetDataTable<EnemyTable>().GetDataRow(enemyId);
int arenaResourceId = enemyData.BattleArenaResourceId;

BattleArenaManager.Instance.Initialize(arenaResourceId);
```

### 方案 3：使用默认场地

```csharp
// 不传参数或传入 0，使用默认测试场地
BattleArenaManager.Instance.Initialize(0);
```

## 五、实际使用示例

### 完整的战斗流程

```csharp
// ========== 战斗准备阶段 ==========
public class CombatPreparationState : GameStateBase
{
    protected override async void OnEnter(IFsm<GameStateManager> fsm)
    {
        // 1. 初始化战斗场地管理器
        int arenaResourceId = GetArenaResourceId(); // 从配置表读取
        BattleArenaManager.Instance.Initialize(arenaResourceId);

        // 2. 生成战斗场地
        Vector3 playerPos = PlayerManager.Instance.GetPlayerPosition();
        await BattleArenaManager.Instance.SpawnArenaAsync(playerPos);

        // 3. 加载敌人配置
        EnemySpawnManager.Instance.LoadFromEnemyTable(enemyId);

        // 4. 生成敌人（会自动使用 EnemyZone 的中心点）
        await EnemySpawnManager.Instance.SpawnWaveAsync();

        // 5. 进入战斗状态
        ChangeState<CombatState>(fsm);
    }
}

// ========== 战斗结束阶段 ==========
public class CombatState : GameStateBase
{
    protected override void OnLeave(IFsm<GameStateManager> fsm, bool isShutdown)
    {
        // 清理战斗场地
        BattleArenaManager.Instance.DestroyArena();
        
        // 清理敌人
        EnemySpawnManager.Instance.Cleanup();
    }
}
```

### 棋子放置时的区域检查

```csharp
public class ChessPlacementManager
{
    private bool CanPlaceAtPosition(Vector3 worldPosition, int camp)
    {
        // 我方棋子只能放在 PlayerZone
        if (camp == 0)
        {
            return BattleArenaManager.Instance.IsInPlayerZone(worldPosition);
        }
        // 敌方棋子只能放在 EnemyZone
        else if (camp == 1)
        {
            return BattleArenaManager.Instance.IsInEnemyZone(worldPosition);
        }
        
        return false;
    }
}
```

## 六、常见问题

### Q1: 场地生成后位置不对？
**A**: 检查预制体中的 `PlayerAnchor` 本地坐标是否正确设置。`PlayerAnchor` 的本地坐标决定了玩家相对于场地的位置。

### Q2: 区域检测不准确？
**A**: 确保 `PlayerZone` 和 `EnemyZone` 都有 `Renderer` 组件。系统通过 `Renderer.bounds` 来计算区域边界。

### Q3: 找不到 PlayerAnchor 或 Zone？
**A**: 检查子对象的名称是否完全匹配（区分大小写）：
- `PlayerAnchor`
- `PlayerZone`
- `EnemyZone`

### Q4: 如何调整区域大小？
**A**: 修改 `PlayerZone` 和 `EnemyZone` 的 Scale 即可。例如 Scale=(2,1,2) 表示 20x20 米的区域。

### Q5: 可以有多个战斗场地预制体吗？
**A**: 可以。为不同场景或敌人创建不同的场地预制体，通过配置表的 ResourceId 来切换。

## 七、调试技巧

### 1. 可视化区域边界

在 Scene 视图中选中场地对象，可以看到 `PlayerZone` 和 `EnemyZone` 的 Gizmos 边界。

### 2. 日志输出

`BattleArenaManager` 会输出详细的日志：
- 场地生成位置计算
- 区域边界信息
- 锚点对齐信息

查看 Console 中带 `[BattleArenaManager]` 标签的日志。

### 3. 测试默认场地

如果预制体有问题，可以先使用默认场地测试：

```csharp
BattleArenaManager.Instance.Initialize(0); // 使用默认场地
```

默认场地会自动创建标准的 PlayerZone 和 EnemyZone。

## 八、最佳实践

1. **统一坐标系**：所有场地预制体使用相同的坐标系和朝向
2. **合理的区域大小**：根据棋子数量设计区域大小，避免过于拥挤
3. **视觉反馈**：使用半透明材质区分我方/敌方区域
4. **性能优化**：区域检测使用的是 Bounds，性能开销很小
5. **配置灵活性**：通过配置表管理场地 ResourceId，方便后期调整

## 九、扩展建议

### 1. 多波次战斗
```csharp
// 清理上一波敌人，但保留场地
EnemySpawnManager.Instance.DestroyAllEnemies();

EnemySpawnManager.Instance.LoadFromEnemyTable(nextWaveId);
```

## 总结

战斗场景配置的核心要点：
1. 预制体必须包含 `PlayerAnchor`、`PlayerZone`、`EnemyZone` 三个子对象
2. Zone 必须有 Renderer 组件用于边界计算
3. 通过配置表管理场地 ResourceId
4. 使用 `BattleArenaManager` 的 API 进行区域查询
5. 战斗结束时记得清理场地

按照本指南配置后，战斗场地系统就可以正常工作了！

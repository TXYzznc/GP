## 探索战斗融合设计方案

### 问题背景

当前设计中探索和战斗状态严格分离，存在以下问题：
1. 玩家缺少主动权（敌人发现后只能逃跑）
2. 缺少传统"搜打撤"的核心体验
3. 策略深度不足

### 推荐方案：遭遇战机制 + 召唤师形态系统

结合方案一（遭遇战机制）和方案四（召唤师形态），打造完整的探索战斗混合系统。技术架构保持探索/战斗状态分离，不需要大规模重构。

#### 1. 探索形态能力

- 召唤1个护卫棋子跟随（战斗力70%，血量全局继承）
- 可使用召唤师技能攻击敌人
- 攻击敌人后触发"遭遇战"，进入战斗状态

#### 2. 遭遇战触发效果

| 触发类型 | 效果 |
|---------|------|
| 偷袭休息中的敌人 | 敌人血量-30%，眩晕1回合，玩家获得先手优势Buff(攻击+10%) |
| 主动攻击警戒中的敌人 | 玩家获得先手优势Buff |
| 被敌人发现 | 正常进入战斗，无额外优势 |

#### 3. 快速脱战机制

```
触发条件：战斗开始后3秒内，玩家选择"撤退"
成功时：立即返回探索，敌人恢复到战斗前状态，玩家无惩罚
失败时：继续战斗，召唤师受到50点固定伤害，敌人获得"警觉"Buff(攻击+5%)
```

#### 4. 方案四各变体对比

| 方案 | 推荐度 | 特点 |
|-----|--------|------|
| 方案一：遭遇战机制 | ⭐⭐⭐ | 不需要大改架构，增加主动权 |
| 方案二：混合战斗模式 | ⭐⭐ | 需要两套战斗系统，复杂度高 |
| 方案三：警戒阶段互动 | ⭐ | 实现简单但互动深度有限 |
| 方案四：召唤师形态 | ⭐⭐⭐⭐ | 符合世界观，护卫棋子机制有陪伴感 |
| **推荐：1+4混合** | ⭐⭐⭐⭐ | 兼顾架构稳定和玩法深度 |

### 先手优势Buff参考实现

```csharp
public class FirstStrikeBuff : BuffBase
{
    private float m_AttackBonus = 0.1f;

    public override void OnApply(ChessEntity target)
    {
        base.OnApply(target);
        target.Attribute.ModifyAttack(m_AttackBonus, true);
        DebugEx.LogModule("FirstStrikeBuff", $"先手优势生效: 攻击力+{m_AttackBonus * 100}%");
    }

    public override void OnRemove(ChessEntity target)
    {
        target.Attribute.ModifyAttack(-m_AttackBonus, true);
        base.OnRemove(target);
    }
}
```

### 实现优先级

1. **第一阶段**（1-2周）：探索状态下基础攻击能力 + 先手优势Buff系统

3. **第三阶段**（1-2周）：偷袭机制 + 战斗优势数值效果
4. **第四阶段**（1周）：快速脱战机制 + 结果UI

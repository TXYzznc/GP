## 系统概述

战斗提示（`EnterCombatTip`）是进入战斗时显示的提示UI，负责在战斗准备阶段前显示战斗开始动画，等待完成后再继续初始化战斗。

**相关文件**：
- `Assets/AAAGame/Scripts/UI/EnterCombatTip.cs` - 提示UI
- `Assets/AAAGame/Scripts/GameState/States/CombatPreparationState.cs` - 战斗准备状态

---

## 重构后的流程设计

### 问题背景

原始设计存在以下架构问题：
1. **重复逻辑** - `GameTestManager`、`GameProcedure`、`CombatPreparationState` 都在处理战斗提示
2. **事件协调复杂** - 通过 `GameplayEventArgs.EnterCombatTipClosed` 协调UI和状态切换
3. **职责不清** - 战斗提示应该是战斗准备状态的内置流程

### 新流程
```
任何入口 → GameStateManager.SwitchToCombatPreparation()
         → CombatPreparationState.OnEnter()
         → ShowEnterCombatTipAndContinue()（异步）
         → 显示战斗提示UI → 等待UI动画完成 → ContinueCombatPreparationInit()
```

### 修改文件清单

**1. GameTestManager.cs**
- 简化O键处理，直接调用 `GameStateManager.Instance.SwitchToCombatPreparation()`
- 删除整个 `#region 战斗进入提示` 区域的事件协调代码

**2. CombatPreparationState.cs**
- 新增方法：`ShowEnterCombatTipAndContinue()` - 显示提示UI并等待完成
- 新增方法：`WaitForUIFormClosedByEvent()` - 事件驱动等待（主用）
- 新增方法：`WaitForUIFormClosed()` - 轮询等待（备用）
- 新增方法：`ContinueCombatPreparationInit()` - 继续战斗准备初始化

**3. EnterCombatTip.cs**
- OnClose 方法中移除事件触发逻辑（不再需要通过事件通知）
- 使用 `GF.UI.CloseUIForm()` 替代 `GF.UI.Close()`（跳过关闭动画，避免延迟）
- 添加 CancellationTokenSource 管理异步任务生命周期

**4. GameProcedure.cs**
- 删除：`OnGameplayEvent`、`OnEnterCombatTipClosed`、`StartCombatPreparation` 方法
- 消除重复逻辑，统一架构

**5. GameplayEventArgs.cs（可选）**
- 可移除 `EnterCombatTipClosed` 枚举值（不再使用）

## 技术要点

### 事件驱动等待（主方案）

```csharp
// CombatPreparationState.cs
private async UniTask WaitForUIFormClosedByEvent(int uiFormId)
{
    System.EventHandler<GameFramework.Event.GameEventArgs> onUIFormClosed = (sender, e) =>
    {
        if (e is UnityGameFramework.Runtime.CloseUIFormCompleteEventArgs closeArgs)
        {
            if (closeArgs.SerialId == uiFormId)
            {
                // 处理关闭事件
            }
        }
    };
    // 使用事件驱动而非轮询，提高UI响应性能
}
```

### 轮询等待（备用方案）

```csharp
private async UniTask WaitForUIFormClosed(int uiFormId)
{
    while (GF.UI.HasUIForm(uiFormId))
    {
        await UniTask.Yield();
    }
    await UniTask.Yield(); // 额外等待确保清理完成
}
```

### 异常处理规范

- UI打开失败时继续初始化流程，确保游戏不中断
- 使用 try-catch 包装异步操作
- 统一使用 `DebugEx.LogModule` 替代 `Log.Info`

## Bug记录：延迟问题（已修复）

### 问题描述

`EnterCombatTip` 存在稳定延迟问题，第二次及以后触发战斗时出现明显卡顿。

### 根本原因

1. **UI关闭方式不一致**：
   - `EnterCombatTip.cs` 使用 `GF.UI.Close(UIForm)` → 触发关闭动画
   - `CombatPreparationState.cs` 使用轮询 `GF.UI.HasUIForm(uiFormId)` 检测

2. **关闭动画延迟链**：
   ```
   GF.UI.Close(UIForm) → CloseWithAnimation() → Internal_PlayCloseUIAnimation(callback)
   // 动画完成后才真正销毁，而轮询每帧都在等待
   ```

### 修复方案

- **方案一（已实施）**：事件驱动机制，使用UI关闭完成事件替代轮询
- **方案二（已实施）**：`EnterCombatTip` 跳过关闭动画，使用 `GF.UI.CloseUIForm()` 直接关闭

## Bug记录：状态机异常（已修复）

### 问题描述

```
[InGameState] 切换子状态失败 - Exception has been thrown by the target of an invocation.
```

多次进入战斗后出现状态机切换异常，第三次战斗开始时触发。

### 根本原因

`EnterCombatTip` 的异步生命周期管理问题：
1. `UniTask.Delay` 没有取消机制，UI关闭后异步任务仍在运行
2. `m_IsAutoClosing` 状态标志在重新打开时没有重置
3. 多个异步任务并发导致竞态条件

### 修复方案（EnterCombatTip.cs）

```csharp
private CancellationTokenSource m_CancellationTokenSource;

protected override void OnOpen(object userData)
{
    m_IsAutoClosing = false;

    // 取消之前的异步任务，创建新令牌
    m_CancellationTokenSource?.Cancel();
    m_CancellationTokenSource?.Dispose();
    m_CancellationTokenSource = new CancellationTokenSource();

    StartAutoCloseTimer(m_CancellationTokenSource.Token).Forget();
}

protected override void OnClose(bool isShutdown, object userData)
{
    // 取消并释放异步任务
    m_CancellationTokenSource?.Cancel();
    m_CancellationTokenSource?.Dispose();
    m_CancellationTokenSource = null;
    m_IsAutoClosing = false;
}

private async UniTaskVoid StartAutoCloseTimer(CancellationToken cancellationToken)
{
    try
    {
        await UniTask.Delay((int)(m_DisplayDuration * 1000), cancellationToken: cancellationToken);
        if (cancellationToken.IsCancellationRequested) return;
        if (this != null && UIForm != null)
            GF.UI.CloseUIForm(UIForm);  // 跳过动画直接关闭
    }
    catch (System.OperationCanceledException)
    {
        // 正常的取消操作，不需要处理
    }
    finally
    {
        m_IsAutoClosing = false;
    }
}
```

**修复效果**：
- 正确的生命周期管理：UI重新打开时会取消之前的异步任务
- 避免竞态条件：使用取消令牌确保异步任务能被正确中断
- 状态一致性：OnOpen和OnClose都会正确重置状态标志
- 资源清理：CancellationTokenSource会被正确释放

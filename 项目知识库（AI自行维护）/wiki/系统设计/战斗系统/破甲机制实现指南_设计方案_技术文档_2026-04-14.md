## 概述

**破甲（Armor Penetration）** 是一种战斗机制，通过**负数护甲**来削弱敌方防御，从而增加伤害输出。

### 设计理念

- **非对称性**：防御强度 > 破甲强度（防御生态更健康）
- **平缓性**：破甲增伤 < 同等护甲减伤（防止伤害爆炸）
- **策略性**：鼓励作为战斗策略，但不是绝对压制

### 核心数据

```
示例场景：
- 攻击方基础伤害: 100
- 目标护甲: 0

无破甲时: 伤害 = 100
施加-100护甲破甲后: 伤害 = 115 (+15%)

对比：
- +100护甲: 减伤30% → 伤害=70
- -100破甲: 增伤15% → 伤害=115
（相差45点伤害，防御优于破甲）
```

## 破甲工作原理

### 执行流程

```
1. 获取目标基础护甲值 (BaseArmor)
2. 计算所有护甲修改（如减防Debuff）
3. 应用破甲效果 (负数护甲)
4. 代入护甲减伤公式
5. 最终伤害输出

示例时序：
目标基础护甲: 100
敌方应用-30护甲破甲 → 临时护甲: 70
再应用-50护甲破甲 → 临时护甲: 20
最终护甲: max(20, -200) = 20 (不能低于-200)
```

### 护甲修改来源

```
1. 基础护甲（属性系统）
   └─ ChessAttribute.Armor

2. 护甲加成Buff（正数）
   └─ 如: 铁皮术 +50护甲

3. 破甲Buff（负数护甲修改）
   └─ 如: 破甲术 -30护甲

4. 减防Debuff（负数护甲加成）
   └─ 如: 脆皮诅咒 -20护甲
```

## 数值对标

### 物理破甲 vs 护甲增伤

```
破甲强度公式（负数护甲）：
  增伤% = -护甲 × lowRate × negativeArmorRate
  = -护甲 × 3% × 0.5
  = -护甲 × 1.5%

护甲减伤公式（正数护甲）：
  减伤% = 护甲 × lowRate
  = 护甲 × 3%

对比：
┌─────────┬──────────┬──────────────┐
│ 护甲值  │ 护甲减伤 │ 对应破甲增伤 │
├─────────┼──────────┼──────────────┤
│ ±50     │ ±1.5%    │ ±0.75%       │
│ ±100    │ ±3%      │ ±1.5%        │
│ ±150    │ ±4.5%    │ ±2.25%       │
│ ±200    │ ±6%      │ ±3%          │
└─────────┴──────────┴──────────────┘

结论：破甲强度 = 护甲强度的 50%（设计目标）
```

### 极限情况（破甲上限）

```
目标初始护甲: 100
应用多个破甲：

情况1（单体破甲）：
  -200护甲破甲 → 目标护甲 = 100 - 200 = -100
  伤害系数 = 1 + 100 × 1.5% = 1.15 (+15%)
  达到增伤上限

情况2（多个破甲叠加）：
  -50破甲 × 4 = -200总破甲
  目标护甲 = 100 - 200 = -100
  伤害系数 = 1.15 (+15%)
  同样达到上限

护甲下限：-200（硬上限，防止过度）
```

### 推荐：混合方案（Buff + 护甲修改器）

```csharp
public class ChessAttribute : MonoBehaviour
{
    private float m_BaseArmor;        // 基础护甲
    private float m_ArmorModifier;    // 护甲修改（Buff、破甲等）
    
    /// <summary>
    /// 获取最终护甲值（包括所有修改）
    /// </summary>
    public float GetFinalArmor()
    {
        float finalArmor = m_BaseArmor + m_ArmorModifier;
        
        // 硬上限：护甲不能低于-200（防止伤害爆炸）
        return Mathf.Max(finalArmor, -200f);
    }
    
    /// <summary>
    /// 应用破甲效果
    /// </summary>
    public void ApplyArmorPenetration(float penetration)
    {
        // penetration为负数：如 -50, -100
        m_ArmorModifier += penetration;
        DebugEx.LogModule("ChessAttribute", 
            $"应用破甲: {penetration:F0}, 最终护甲: {GetFinalArmor():F0}");
    }
    
    /// <summary>
    /// 恢复护甲修改
    /// </summary>
    public void RestoreArmorModifier(float restoration)
    {
        m_ArmorModifier -= restoration;
        DebugEx.LogModule("ChessAttribute", 
            $"恢复护甲: {restoration:F0}, 最终护甲: {GetFinalArmor():F0}");
    }
}
```

```
# 说明：

# EffectType=5: 防御修改（新增）
# EffectValue: 负数表示破甲
# Duration: 持续回合数
# MaxStack: 最多叠加层数
```

### Buff生效流程

```csharp
// 在 BuffManager 中处理防御修改类 Buff

public class BuffManager
{
    public void ApplyBuff(int buffId, ChessEntity target)
    {
        var buffConfig = GetBuffConfig(buffId);
        
        if (buffConfig.EffectType == BuffEffectType.ArmorModification)
        {
            // 应用破甲效果
            target.Attribute.ApplyArmorPenetration(buffConfig.EffectValue);
            
            // 记录Buff时长，用于移除时恢复
            ScheduleBuffRemoval(target, buffId, buffConfig.Duration);
        }
    }
    
    public void RemoveBuff(int buffId, ChessEntity target)
    {
        var buffConfig = GetBuffConfig(buffId);
        
        if (buffConfig.EffectType == BuffEffectType.ArmorModification)
        {
            // 移除时恢复护甲
            target.Attribute.RestoreArmorModifier(
                -buffConfig.EffectValue  // 反向操作
            );
        }
    }
}
```

## 技能集成示例

### 示例1：破甲技能（造成伤害 + 应用破甲）

```csharp
public class BreakArmorAttack : ChessSkillBase
{
    /// <summary>
    /// 破甲术：造成100攻击力伤害，并破甲-50（3回合）
    /// </summary>
    public override void ExecuteSkill(ChessEntity caster, ChessEntity target)
    {
        // 1. 计算伤害
        double damage = caster.Attribute.Attack * 1.0;  // 1.0系数
        
        // 2. 造成伤害
        target.TakeDamage(damage);
        
        // 3. 应用破甲Buff（通过BuffManager）
        if (target.BuffManager != null)
        {
            target.BuffManager.AddBuff(4001, caster);  // 破甲术 Buff
            DebugEx.LogModule("BreakArmorAttack",
                $"对{target.Name}施加破甲术，-50护甲");
        }
    }
}
```

### 示例2：组合技（先减防再破甲）

```csharp
public class CombinedDebuff : ChessSkillBase
{
    /// <summary>
    /// 虚弱诅咒：先降低30防御，再破甲100
    /// </summary>
    public override void ExecuteSkill(ChessEntity caster, ChessEntity target)
    {
        // 1. 降低防御（也是减护甲）
        target.BuffManager?.AddBuff(3001);  // 脆皮诅咒 Buff
        
        // 2. 破甲
        target.BuffManager?.AddBuff(4002);  // 致命破甲 Buff
        
        // 结果：累计 -130 护甲
        double totalPenetration = -30 + (-100);
    }
}
```

### 示例3：攻击附带破甲（条件触发）

```csharp
public class ConditionalBreakArmor : ChessNormalAttackBase
{
    /// <summary>
    /// 普攻：暴击时附带-30破甲（回调方式）
    /// </summary>
    public override void ExecuteAttack(ChessEntity caster, ChessEntity target)
    {
        double damage = CalculateDamage(caster, out bool isCritical);
        
        HitContext context = new HitContext
        {
            Attacker = caster,
            LockedTarget = target,
            BaseDamage = damage,
            IsCritical = isCritical,
            SkillConfig = m_Config,
            OnHitCallback = OnCriticalHit  // 暴击时触发
        };
        
        IHitDetector detector = HitDetectorFactory.GetDetector(AttackHitType.Projectile);
        detector.Execute(context);
    }
    
    private void OnCriticalHit(ChessEntity target, double damage, bool isCritical)
    {
        if (isCritical && target.BuffManager != null)
        {
            target.BuffManager.AddBuff(4001);  // 破甲术
            DebugEx.LogModule("ConditionalBreakArmor",
                $"暴击触发，对{target.Name}施加破甲术");
        }
    }
}
```

## 平衡建议

### 1. 破甲强度等级划分

```
轻度破甲（日常应用）：
  -30 ~ -50 护甲 → +0.45% ~ +0.75% 增伤
  适用：普通技能附加效果

中度破甲（核心技能）：
  -80 ~ -100 护甲 → +1.2% ~ +1.5% 增伤
  适用：技能主要效果

重度破甲（大招）：
  -150 ~ -200 护甲 → +2.25% ~ +3% 增伤
  适用：终极技能，但受上限限制
```

### 2. 破甲Buff的持续时间

```
推荐配置：

轻度破甲（-30）: 5回合    → 长效，但效果弱
中度破甲（-100）: 3回合   → 标准时长
重度破甲（-150）: 2回合   → 短时强效

逻辑：时间越短，强度越高（平衡机制）
```

### 3. 破甲Buff的叠加规则

```
建议使用最强覆盖机制（而非叠加）：

示例：
- 应用第一层破甲: -50 → 目标护甲 = 100 - 50 = 50
- 应用第二层破甲: -80 → 目标护甲 = 100 - 80 = 20（覆盖，不叠加）
- 应用第三层破甲: -30 → 目标护甲 = 100 - 30 = 70（覆盖回增）

这样设计的好处：
1. 逻辑清晰：最强的破甲生效
2. 防止滥用：避免无脑堆破甲Buff
3. 战术深度：需要选择最优破甲时机
```

### 4. 防护措施

```
1. 护甲下限：-200（防止伤害爆炸）
   → 最高增伤30%（远小于85%减伤）

2. 破甲持续时间不宜过长
   → 一般3~5回合，大招2回合

3. 破甲Buff的获取难度应该相对较高
   → 通过技能/Buff获取，不是基础属性

4. 监控日志
   → 记录所有护甲修改，便于平衡调试
```

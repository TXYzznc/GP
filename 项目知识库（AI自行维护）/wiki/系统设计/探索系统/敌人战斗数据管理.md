# 敌人战斗数据管理

合并自：敌人战斗数据传递_问题分析 + 敌人战斗数据保存优化_设计方案

---

## 一、战斗数据传递流程

### 数据流向概览

```
EnemyEntity（探索场景）
    ↓ 保存引用/数据
EnemyEntityManager.CurrentCombatEnemy / m_CurrentCombatData
    ↓ 读取配置
CombatPreparationState.LoadEnemyWaveConfig()
    ↓ 加载
EnemySpawnManager.LoadFromEnemyTable(battleConfigId, enemyGuid)
    ↓ 生成
敌人棋子实例（战斗场景）
```

### 配置表关联关系

```
EnemyEntityTable（探索场景的敌人）
    └── BattleConfigId: 1  →  EnemyTable（战斗配置）
                                    └── ChessIds: [1001, 1002]
```

**EnemyEntityTable 字段**：EnemyID、Name、BattleConfigId（关联 EnemyTable.ID）

**EnemyTable 字段**：ID、EnemyName、ChessIds（出战棋子ID数组）、FormationType（阵型）、Spacing（间距）

### 详细步骤

1. `EnemyChaseState.TriggerCombat()` → 调用 `EnemyEntityManager.TriggerCombat(entity)`
2. `EnemyEntityManager` 保存敌人引用，调用 `entity.EnterCombat()`，切换战斗准备状态
3. `CombatPreparationState.LoadEnemyWaveConfig()` 从 `CurrentCombatEnemy.Config.BattleConfigId` 读取配置
4. `EnemySpawnManager.LoadWaveFromConfig(battleConfigId)` 读取 EnemyTable，构造 EnemyWaveConfig
5. 战斗开始后 `SpawnWaveAsync()` 按 ChessIds 生成棋子

## 二、战斗数据保存优化方案

### 当前方案问题

保存 `EnemyEntity` 引用（GameObject），若实体在战斗准备阶段被销毁，引用变为 null，无法加载战斗配置。

### 优化方案：保存战斗配置数据而非实体引用

新增 `EnemyCombatData` 数据类，在触发战斗时立即从实体提取所有必要信息：

```csharp
public class EnemyCombatData
{
    public int EntityConfigId { get; set; }
    public string EnemyName { get; set; }
    public int BattleConfigId { get; set; }
    public EnemyType EnemyType { get; set; }
    public int RewardTier { get; set; }
    public int PurificationCardId { get; set; }
    public Vector3 TriggerPosition { get; set; }

    public static EnemyCombatData FromEntity(EnemyEntity entity)
    {
        return new EnemyCombatData
        {
            EntityConfigId = entity.EntityConfigId,
            EnemyName = entity.Config.Name,
            BattleConfigId = entity.Config.BattleConfigId,
            // ...
        };
    }
}
```

### 修改 EnemyEntityManager

```csharp
public void TriggerCombat(EnemyEntity entity)
{
    // 立即保存战斗数据（不依赖实体生命周期）
    m_CurrentCombatData = EnemyCombatData.FromEntity(entity);
    m_CurrentCombatEnemy = entity;  // 临时引用，用于销毁
    m_IsInCombat = true;
    // ...
}
```

### 修改 LoadEnemyWaveConfig

```csharp
private void LoadEnemyWaveConfig()
{
    var combatData = EnemyEntityManager.Instance?.CurrentCombatData;
    if (combatData != null)
    {
        EnemySpawnManager.Instance.LoadWaveFromConfig(combatData.BattleConfigId);
    }
    else
    {
        // 默认配置
        EnemySpawnManager.Instance.LoadWaveFromConfig(1);
    }
}
```

## 潜在问题与注意事项

### 敌人实体不会被提前销毁

敌人触发战斗后不立即 Destroy，只是 `SetActive(false)`（详见敌人棋子数据重构文档）。`m_CurrentCombatEnemy` 引用在战斗准备阶段仍然有效。

### 配置表关联错误

若 `EnemyEntityTable.BattleConfigId` 与 `EnemyTable.ID` 不匹配，会生成错误的敌人。建议添加配置验证逻辑：

```csharp
var dataTable = GF.DataTable.GetDataTable<EnemyTable>();
if (dataTable == null || dataTable.GetDataRow(battleConfigId) == null)
{
    DebugEx.ErrorModule("CombatPreparationState", $"战斗配置不存在: BattleConfigId={battleConfigId}");
    EnemySpawnManager.Instance.LoadWaveFromConfig(1);  // 降级默认
}
```

#### 字段说明

**基础信息**：
- `EnemyID`：敌人实体ID（唯一标识）
- `Name`：敌人名称（显示用）
- `Description`：描述信息
- `EnemyType`：敌人类型（0=普通，1=精英，2=Boss，3=特殊）

**战斗相关**：
- `BattleConfigId`：关联的战斗配置ID（EnemyTable）
- `PurificationCardId`：净化后获得的卡牌ID（Boss/特殊敌人）
- `Difficulty`：难度系数（1-5）

**广播机制**：
- `CanBroadcast`：是否会广播（精英/Boss）
- `BroadcastDistance`：广播范围（米）

**奖励系统**：
- `RewardTier`：掉落奖励等级（1-3）
- `PrefabId`：预制体ID

**AI参数**：
- `PatrolRadius`：巡逻半径（米）
- `PatrolSpeed`：巡逻速度
- `RestProbability`：进入深度休息的概率（0-1）
- `RestDuration`：休息时长（秒）

**检测参数**：
- `AlertDistance`：警戒距离（米，远距离慢速发现）
- `AlertTime`：警戒时长（秒，需要停留多久才发现）
- `DetectDistance`：发现距离（米，近距离快速发现）
- `DetectTime`：近距离发现时长（秒）

**追击参数**：
- `ChaseSpeed`：追击速度
- `ChaseDistance`：追击距离（超过此距离放弃追击）
- `CombatDistance`：进入战斗距离（米）

#### 配置建议

**普通敌人**：
```
EnemyType = 0
CanBroadcast = False
PatrolRadius = 10
AlertDistance = 10
ChaseDistance = 20
CombatDistance = 3
Difficulty = 1
```

**精英敌人**：
```

CanBroadcast = True
BroadcastDistance = 15-20
PatrolRadius = 15
AlertDistance = 12

```

**Boss敌人**：

## 二、场景配置

### 2.1 准备工作

#### 1. 烘焙 NavMesh

1. 打开探索场景
2. 选择 `Window > AI > Navigation`
3. 切换到 `Bake` 标签
4. 设置参数：
   - Agent Radius: 0.5
   - Agent Height: 2.0
   - Max Slope: 45
   - Step Height: 0.4
5. 点击 `Bake` 按钮

#### 2. 设置 Layer

确保项目中有 `Enemy` Layer：
1. `Edit > Project Settings > Tags and Layers`
2. 在 `Layers` 中添加 `Enemy`

#### 3. 设置玩家 Tag

确保玩家 GameObject 有 `Player` Tag：
1. 选择玩家对象
2. 在 Inspector 顶部设置 Tag 为 `Player`

---

### 2.2 创建敌人实体

#### 方法一：手动创建

1. **创建空对象**
   ```
   Hierarchy > 右键 > Create Empty
   命名：Enemy_Goblin_1001
   ```

2. **添加组件**
   - `NavMeshAgent`
   - `EnemyEntity` 脚本
   - `Capsule Collider`（或其他碰撞体）

3. **配置 EnemyEntity**
   ```
   Entity Config Id: 1001（对应配置表ID）
   Show Debug: True（显示调试信息）
   ```

4. **配置 Collider**
   ```
   Radius: 0.5
   Height: 2.0
   Center: (0, 1, 0)
   ```

5. **设置 Layer**
   ```
   Layer: Enemy
   ```

6. **添加视觉模型**（可选）
   - 在敌人对象下创建子对象
   - 添加 Mesh Renderer 或 Sprite Renderer
   - 添加模型/精灵

#### 方法二：使用预制体（推荐）

1. **创建预制体**
   - 按照方法一创建好敌人实体
   - 拖拽到 `Assets/AAAGame/Prefabs/Enemy/` 文件夹
   - 命名：`Enemy_Goblin_1001.prefab`

2. **在场景中实例化**
   - 从 Project 窗口拖拽预制体到场景
   - 调整位置

### 4.3 常见问题排查

#### 问题 1：敌人不移动

**可能原因**：
- NavMesh 未烘焙
- NavMeshAgent 组件缺失
- 敌人位置不在 NavMesh 上

**解决方法**：
1. 检查 NavMesh 是否烘焙
2. 检查敌人是否有 NavMeshAgent 组件
3. 在 Scene 视图查看 NavMesh（蓝色区域）

---

#### 问题 2：敌人不检测玩家

**可能原因**：
- 玩家没有 `Player` Tag
- 玩家距离超出警戒范围

**解决方法**：
1. 检查玩家 Tag
2. 调整 AlertDistance 参数
3. 查看 Console 日志

---

#### 问题 3：广播不生效

**可能原因**：
- 敌人 Layer 不是 `Enemy`
- BroadcastDistance 太小
- 接收敌人不满足条件

**解决方法**：
1. 检查所有敌人的 Layer
2. 增大 BroadcastDistance
3. 查看广播响应日志

## 五、性能优化建议

### 5.1 敌人数量控制

**推荐配置**：
- 同屏敌人数量：≤ 20
- 激活的敌人数量：≤ 50
- 总敌人数量：≤ 200

### 5.2 检测优化

**建议**：
- 使用分层检测（近、中、远）
- 降低远距离敌人的更新频率
- 使用对象池管理敌人实体

### 5.3 NavMesh 优化

**建议**：
- 合理设置 NavMesh 精度
- 避免过于复杂的地形
- 使用 NavMesh Obstacles

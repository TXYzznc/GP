## 一、PlayerController 组件配置

### 1. 在角色 GameObject 上添加组件

确保你的角色 GameObject 上有以下组件：
- `PlayerController` 脚本
- `CharacterController` 组件（如果没有会自动添加）
- `Animator` 组件（通常在角色模型的子对象上）

### 2. PlayerController Inspector 参数设置

#### 移动设置 (Movement Settings)
```
Walk Speed: 5          // 行走速度
Slow Run Speed: 8      // 慢跑速度
Fast Run Speed: 12     // 快跑速度
Acceleration: 10       // 加速度
Deceleration: 15       // 减速度
Gravity: -9.81         // 重力
```

#### 状态设置 (State Settings)
```
Double Click Window: 0.4   // 双击检测时间窗口（秒）
```

#### 动画设置 (Animation Settings)
```
Animator: [拖入角色的 Animator 组件]
```

**推荐值说明：**
- **Walk Speed (5)**: 正常行走速度，适合探索
- **Slow Run Speed (8)**: 慢跑速度，比行走快但不消耗太多体力
- **Fast Run Speed (12)**: 快跑速度，用于快速移动
- **Double Click Window (0.4)**: 双击检测窗口，0.3-0.5秒比较合适

## 二、Animator Controller 配置

### 1. 创建 Animator Controller

1. 在 Project 窗口右键 → Create → Animator Controller
2. 命名为 `PlayerAnimatorController`
3. 将其拖到角色的 Animator 组件的 Controller 字段

### 2. 添加动画参数 (Parameters)

在 Animator 窗口的 Parameters 面板中添加以下参数：

| 参数名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `State` | Int | 0 | 当前状态 (0=Idle, 1=Walk, 2=SlowRun, 3=FastRun, 4=Interact) |
| `Speed` | Float | 0 | 归一化速度 (0-1) |
| `InteractIndex` | Int | 0 | 交互动画索引 |

### 3. 创建动画状态 (Animation States)

在 Animator 窗口中创建以下状态：

#### 基础状态
- **Idle** - 待机动画
- **Walk** - 行走动画
- **SlowRun** - 慢跑动画
- **FastRun** - 快跑动画

#### 交互状态（可选，根据需要添加）
- **Interact_0** - 交互动画 1（例如：开门）
- **Interact_1** - 交互动画 2（例如：拾取物品）
- **Interact_2** - 交互动画 3（例如：使用机关）

### 4. 配置状态转换 (Transitions)

#### 方案 A：使用 State 参数直接切换（推荐）

使用 Blend Tree 或者直接通过 State 参数控制：

```
Any State → Idle
  Condition: State Equals 0
  
Any State → Walk
  Condition: State Equals 1
  
Any State → SlowRun
  Condition: State Equals 2
  
Any State → FastRun
  Condition: State Equals 3
  
Any State → Interact
  Condition: State Equals 4
```

**转换设置：**
- Has Exit Time: ❌ 取消勾选（除了 Interact 状态）
- Transition Duration: 0.1-0.2 秒
- Interruption Source: Current State

#### 方案 B：使用 Blend Tree（更平滑）

1. 创建一个 Blend Tree 状态命名为 "Movement"
2. 设置 Blend Type 为 1D
3. 设置 Parameter 为 "Speed"
4. 添加动画：
   - Threshold 0.0 → Idle
   - Threshold 0.3 → Walk
   - Threshold 0.6 → SlowRun
   - Threshold 1.0 → FastRun

### 5. Interact 状态特殊配置

如果有多个交互动画，可以使用 Sub-State Machine：

```
Interact (Sub-State Machine)
  ├─ Interact_0 (Condition: InteractIndex Equals 0)
  ├─ Interact_1 (Condition: InteractIndex Equals 1)
  └─ Interact_2 (Condition: InteractIndex Equals 2)
```

**Interact 状态转换设置：**
- 进入：Any State → Interact (State Equals 4)
- 退出：Interact → Idle (Has Exit Time ✓, Exit Time = 1.0)

## 三、动画资源准备

### 需要的动画片段

确保你有以下动画资源：

1. **Idle.anim** - 待机动画（循环）
2. **Walk.anim** - 行走动画（循环）
3. **SlowRun.anim** - 慢跑动画（循环）
4. **FastRun.anim** - 快跑动画（循环）
5. **Interact_X.anim** - 交互动画（不循环，根据需要添加多个）

### 动画导入设置

对于循环动画（Idle, Walk, SlowRun, FastRun）：
- Loop Time: ✓ 勾选
- Loop Pose: ✓ 勾选

对于交互动画（Interact）：
- Loop Time: ❌ 取消勾选

## 四、测试配置

### 1. 基础移动测试

1. 运行场景
2. 按 `Tab` 键锁定鼠标
3. 按 `WASD` 移动，观察状态切换：
   - 开始移动 → Walk 状态
   - 双击 `W` → SlowRun 状态
   - 慢跑时按 `Shift` → FastRun 状态
   - 停止移动 → Idle 状态

### 2. 双击检测测试

测试以下情况：
- ✅ `W` → 松开 → `W`（间隔<0.4秒）→ 应该触发慢跑
- ❌ `W` → `A` → `W` → 不应该触发慢跑
- ❌ `W` → 松开 → `W`（间隔>0.4秒）→ 不应该触发慢跑

### 3. 交互测试

在其他脚本中调用：
```csharp
PlayerController controller = GetComponent<PlayerController>();
controller.TriggerInteract(0);  // 触发交互动画 0

// 动画结束后调用
controller.EndInteract();  // 结束交互，返回 Idle
```

## 五、常见问题排查

### 问题 1：动画不切换
**检查项：**
- Animator 组件是否正确引用
- Animator Controller 是否配置了参数
- 动画状态转换条件是否正确

### 问题 2：双击检测不灵敏
**解决方案：**
- 调整 `Double Click Window` 参数（增大到 0.5）
- 检查输入系统是否正常工作

### 问题 3：速度切换不平滑
**解决方案：**
- 调整 `Acceleration` 和 `Deceleration` 参数
- 使用 Blend Tree 实现更平滑的动画过渡

### 问题 4：交互动画无法触发
**检查项：**
- 确保调用了 `TriggerInteract(index)`
- 检查 Animator 中是否有对应的 Interact 状态
- 检查 `InteractIndex` 参数是否正确设置

## 六、进阶配置

### 1. 添加更多交互动画

```csharp
// 在需要触发交互的地方调用
playerController.TriggerInteract(0);  // 开门动画
playerController.TriggerInteract(1);  // 拾取物品动画
playerController.TriggerInteract(2);  // 使用机关动画
```

### 2. 自定义速度配置

根据游戏需求调整速度：
- **慢节奏游戏**: Walk=3, SlowRun=5, FastRun=8
- **快节奏游戏**: Walk=6, SlowRun=10, FastRun=15

### 3. 添加状态事件监听

可以扩展 PlayerController 添加状态改变事件：
```csharp
public event Action<PlayerState> OnStateChanged;

// 在状态改变时触发
OnStateChanged?.Invoke(m_CurrentState);
```

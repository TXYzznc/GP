# 嫦娥棋子问题诊断

## 一、大招特效不显示（2026-04-13）

### 问题描述

嫦娥释放大招"月华天倾"时，法阵没有生成特效。

**根本原因**：`CombatVFXManager.PlayEffect()` 内部调用 `PlayEffectAsync().Forget()`，在法阵创建之前特效还未完成异步加载，时序不确定。

### 修复方案

将特效播放移到 `ChangeMagicCircle.Initialize()` 内部，法阵位置设置后立即触发异步加载。

**修改文件**：`Assets/AAAGame/Scripts/Game/SummonChess/Chess/嫦娥/ChangeMagicCircle.cs`

```csharp
public void Initialize(SummonChessSkillTable config, ChessEntity caster, Vector3 targetPosition)
{
    transform.position = targetPosition;

    if (m_Config.EffectId > 0)
    {
        CombatVFXManager.PlayEffect(m_Config.EffectId, targetPosition);
        DebugEx.LogModule("ChangeMagicCircle", $"法阵特效播放: ID={m_Config.EffectId}, 位置={targetPosition}");
    }

    m_IsInitialized = true;
}
```

**修改文件**：`Assets/AAAGame/Scripts/Game/SummonChess/Chess/嫦娥/ChangeUltimate.cs`

删除 `CreateMagicCircleSync()` 中重复的特效播放代码，由 `Initialize()` 统一管理。

### 相关配置

| 配置项 | 值 |
|--------|-----|
| 技能ID | 24（月华天倾） |
| 特效ID | 3006 |
| 持续时间 | 5s |
| 命中次数 | 10 |

**验证日志**：`法阵特效播放: ID=3006, 位置=...`

## 二、嫦娥战斗平衡问题（2026-04-17）

### 问题描述

嫦娥无法攻击到后羿，表面上像是阵营识别 Bug，实际是数值不平衡。

### 根本原因

后羿被动技能"日月长弓"（ID=5）在白天激活，攻击范围 +30，实际攻击范围从 5 变为 35。嫦娥基础攻击范围仅为 4，两者距离约 11.74 米，嫦娥无法找到目标。

**系统工作正常**：阵营关系、AI 寻敌逻辑均无误，这是游戏平衡问题，非 Bug。

### 排查过程

1. 最初以为是阵营关系问题 → 验证正确
2. 以为是配置表数值错误 → 数值本身正常
3. 最终定位：被动技能导致运行时数值大幅偏移

### 解决方案

**方案1（推荐）**：将嫦娥 `AtkRange` 从 4 调整为 15-20

**方案2**：为嫦娥添加夜晚被动（攻击范围+20），与后羿昼夜对称

**方案3**：将后羿"日月长弓"攻击范围加成从 +30 降低到 +15

> ⚠️ **矛盾标注**：两份文档记录的时间一致，内容互补（特效 Bug + 平衡问题），无实质矛盾。

用户反馈：技能系统初始化时，并没有正确的对技能进行初始化（比如 PlayerSkillContext 就没有被正确初始化，里面的值都是空的）。

## 问题分析

### 原始代码（错误的顺序）

在 `GameProcedure.cs` 的 `OnCharacterSpawned()` 方法中：

```csharp
// 第187行：先加载技能
m_SkillManager.UpdateSkillsFromPlayerData(playerSkillIds);

// 第188行：后设置角色
m_SkillManager.SetPlayerCharacter(character);
```

### 问题原因

1. **调用顺序错误**：
   - `UpdateSkillsFromPlayerData()` 在 `SetPlayerCharacter()` **之前**调用
   - 此时 `PlayerSkillContext` 还是空的（在 Awake 中初始化为空）
   - 技能初始化时使用的是空的上下文

2. **导致的结果**：
   - `ctx.Owner` = null
   - `ctx.Transform` = null
   - `ctx.Controller` = null
   - 技能无法正常工作（传送技能找不到目标对象）

### 正确的初始化流程

技能系统的初始化必须按照以下顺序：

1. **创建 PlayerSkillManager**
2. **设置技能参数注册表** - `SetParamRegistry(registry)`
3. **设置玩家角色** - `SetPlayerCharacter(character)` ← **必须在加载技能之前**
4. **加载技能列表** - `UpdateSkillsFromPlayerData(skillIds)`

## 解决方案

### 修改文件：GameProcedure.cs

**文件路径**：`Assets/AAAGame/Scripts/Procedures/GameProcedure.cs`

**修改位置**：`OnCharacterSpawned()` 方法，第 182-189 行

**修改前**：
```csharp
// 从配置表读取所有技能ID
List<int> playerSkillIds = DataTableExtension.GetAllIds<PlayerSkillTable>();

if (playerSkillIds.Count > 0)
{
    m_SkillManager.UpdateSkillsFromPlayerData(playerSkillIds);  // ❌ 先加载技能
    m_SkillManager.SetPlayerCharacter(character);                // ❌ 后设置角色
    Log.Info($"GameProcedure: 已加载 {playerSkillIds.Count} 个技能");

    // UI会自动检测并绑定技能，无需手动操作
    Log.Info("GameProcedure: 技能加载完成，UI将自动更新");
}
```

**修改后**：
```csharp
// 从配置表读取所有技能ID
List<int> playerSkillIds = DataTableExtension.GetAllIds<PlayerSkillTable>();

if (playerSkillIds.Count > 0)
{
    // 先设置玩家角色，再加载技能（顺序很重要！）
    m_SkillManager.SetPlayerCharacter(character);                // ✅ 先设置角色
    m_SkillManager.UpdateSkillsFromPlayerData(playerSkillIds);  // ✅ 后加载技能
    Log.Info($"GameProcedure: 已加载 {playerSkillIds.Count} 个技能");

    // UI会自动检测并绑定技能，无需手动操作
    Log.Info("GameProcedure: 技能加载完成，UI将自动更新");
}
```

**修改原因**：
- 必须先设置玩家角色，建立正确的技能上下文
- 然后再加载技能，这样技能初始化时就能使用正确的上下文

## 完整的初始化流程

### 1. InitializeSkillManager()
```csharp
// 创建技能管理器
GameObject skillManagerObj = new GameObject("PlayerSkillManager");
m_SkillManager = skillManagerObj.AddComponent<PlayerSkillManager>();

// 加载技能参数注册表
var paramRegistry = await ResourceExtension.LoadScriptableObjectAsync<SkillParamRegistrySO>(...);
m_SkillManager.SetParamRegistry(paramRegistry);
```

### 2. OnCharacterSpawned(character)
```csharp
// 1. 先设置玩家角色
m_SkillManager.SetPlayerCharacter(character);
// → 此时 ctx.Owner = character
// → 此时 ctx.Transform = character.transform
// → 此时 ctx.Controller = character.GetComponent<PlayerController>()

// 2. 再加载技能
m_SkillManager.UpdateSkillsFromPlayerData(playerSkillIds);
// → 技能初始化时使用正确的上下文
// → 所有技能都能正确访问玩家角色
```

## 关键要点

### ⚠️ 初始化顺序至关重要

技能系统的初始化顺序**不能颠倒**：

1. ✅ **正确顺序**：
   ```
   SetPlayerCharacter() → UpdateSkillsFromPlayerData()
   ```

2. ❌ **错误顺序**：
   ```
   UpdateSkillsFromPlayerData() → SetPlayerCharacter()
   ```

### 为什么顺序重要？

- `UpdateSkillsFromPlayerData()` 会调用 `skill.Init(ctx, common, param)`
- 如果此时 `ctx` 是空的，技能就会使用空的上下文初始化
- 即使之后调用 `SetPlayerCharacter()` 更新了 `ctx`，已经初始化的技能仍然持有旧的空上下文
- 技能对象不会自动更新上下文，必须重新初始化

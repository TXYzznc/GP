## 什么时候需要更新索引?

✅ 以下情况需要更新技能索引:

1. **添加了新技能**
   - 在 `.kiro/skills/` 下创建了新的技能文件夹
   - 添加了新的 SKILL.md 文件

2. **修改了技能信息**
   - 更新了 SKILL.md 的 frontmatter (name, description, keywords)
   - 修改了技能的描述或关键词
   - 调整了技能的适用场景

3. **删除了技能**
   - 移除了某个技能文件夹
   - 删除了 SKILL.md 文件

## 如何更新索引?

### 方法 1: 使用 Kiro UI (推荐)

1. 打开 Kiro 侧边栏的 **Agent Hooks** 面板
2. 找到 **"更新 SKILL 索引"** 钩子
3. 点击右侧的 **"触发"** 按钮 (▶️)
4. 等待 Agent 完成索引更新
5. 查看更新报告

### 方法 2: 使用命令面板

1. 按 `Ctrl+Shift+P` (Windows/Linux) 或 `Cmd+Shift+P` (Mac)
2. 输入 `Trigger Hook`
3. 选择 **"更新 SKILL 索引"**
4. 等待执行完成

### 方法 3: 在对话中请求

直接在 Kiro 对话框中输入:

```
请更新技能索引
```

或

```
触发"更新 SKILL 索引"钩子
```

## 更新流程

钩子会自动执行以下步骤:

```
1. 扫描技能目录
   └─ 读取 .kiro/skills/**/SKILL.md

2. 提取技能信息
   ├─ 技能名称 (name)
   ├─ 技能描述 (description)
   └─ 关键词列表 (keywords)

3. 分析关键词
   ├─ 从 description 提取关键词
   ├─ 从文档内容提取关键词
   └─ 生成触发词映射

4. 更新索引钩子
   └─ 修改 .kiro/hooks/skill-auto-indexer.kiro.hook

5. 生成报告
   ├─ 新增的技能
   ├─ 更新的关键词
   └─ 索引统计信息
```

## 验证更新结果

更新完成后,可以通过以下方式验证:

### 1. 查看更新报告
Agent 会输出详细的更新报告,包括:
- 扫描到的技能数量
- 新增的技能列表
- 更新的关键词映射
- 索引文件的变更

### 2. 测试智能推荐
在对话中输入相关关键词,测试是否能正确推荐技能:

```
# 测试示例
"我需要创建一个配置表"  → 应该推荐 txt-to-xlsx-converter
"帮我分析代码架构"      → 应该推荐 gitnexus-exploring
"生成一个 Excel 文件"   → 应该推荐 xlsx 技能
```

### 3. 检查索引文件
打开 `.kiro/hooks/skill-auto-indexer.kiro.hook`,确认:
- 新技能已添加到触发词列表
- 关键词映射正确
- JSON 格式正确

## 技能 SKILL.md 格式要求

为了确保索引正确生成,SKILL.md 文件应该包含 frontmatter:

```markdown
---
name: 技能名称
description: 技能的简短描述,说明主要功能和适用场景
keywords: 关键词1, 关键词2, 关键词3
license: MIT
---

# 技能标题

技能的详细文档内容...
```

### Frontmatter 字段说明

- **name** (必需): 技能的唯一标识符,使用小写字母和连字符
- **description** (必需): 简短描述,会用于生成关键词
- **keywords** (可选): 逗号分隔的关键词列表
- **license** (可选): 许可证信息

### 关键词提取规则

如果没有明确的 keywords 字段,系统会从 description 中智能提取:

1. **功能关键词**: 转换、生成、创建、分析等动词
2. **技术关键词**: TXT、XLSX、Excel、配置表等名词
3. **场景关键词**: 游戏开发、数据处理、文档生成等

## 常见问题

### Q: 更新索引需要多长时间?
A: 通常在 10-30 秒内完成,取决于技能数量。

### Q: 更新会影响现有的 GitNexus 技能吗?
A: 不会。更新只会修改"其他技能触发词"部分,GitNexus 技能触发词保持不变。

### Q: 如果更新失败怎么办?
A: 检查以下几点:
1. SKILL.md 文件的 frontmatter 格式是否正确
2. 是否有写入 `.kiro/hooks/` 目录的权限
3. 查看 Agent 的错误日志

### Q: 可以手动编辑索引文件吗?
A: 可以,但不推荐。手动编辑可能导致格式错误,建议使用钩子自动更新。

### Q: 更新后需要重启 Kiro 吗?
A: 不需要。钩子配置会自动重新加载。

## 最佳实践

1. **及时更新**: 每次修改技能后立即更新索引
2. **验证结果**: 更新后测试智能推荐是否正常工作
3. **保持格式**: 确保 SKILL.md 的 frontmatter 格式正确
4. **清晰描述**: 在 description 中使用清晰的关键词
5. **定期检查**: 定期检查索引文件,确保所有技能都被正确索引

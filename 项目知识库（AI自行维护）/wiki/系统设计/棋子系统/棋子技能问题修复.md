# 棋子技能问题修复

合并自：技能初始化顺序（2026-04-17）+ 技能输入问题（2026-04-17）+ 棋子技能卡死问题（2026-04-17）

---

## 一、技能初始化顺序错误

### 问题

`PlayerSkillContext` 未正确初始化（Owner/Transform/Controller 全部为 null），导致技能无法正常工作（如传送技能找不到目标对象）。

### 根本原因

`GameProcedure.OnCharacterSpawned()` 中调用顺序错误：

```csharp
// ❌ 错误顺序
m_SkillManager.UpdateSkillsFromPlayerData(playerSkillIds);  // 先加载技能（此时 ctx 是空的）
m_SkillManager.SetPlayerCharacter(character);               // 后设置角色
```

`UpdateSkillsFromPlayerData()` 调用 `skill.Init(ctx, common, param)`，如果 ctx 为空则技能用空上下文初始化。之后设置角色也不会自动更新已初始化的技能。

### 修复

**文件**：`Assets/AAAGame/Scripts/Procedures/GameProcedure.cs`

```csharp
// ✅ 正确顺序：先设置角色，再加载技能
m_SkillManager.SetPlayerCharacter(character);
m_SkillManager.UpdateSkillsFromPlayerData(playerSkillIds);
```

### 完整初始化流程

1. `InitializeSkillManager()` → 创建 PlayerSkillManager，加载 SkillParamRegistrySO
2. `OnCharacterSpawned()` → 先 `SetPlayerCharacter(character)`，再 `UpdateSkillsFromPlayerData(skillIds)`

## 二、技能输入无响应（J/K/L 按键）

### 问题

按下技能对应按键（J/K/L）后，技能没有反应。

### 根本原因

`PlayerSkillManager.Update()` 只处理槽位1（只释放 `Skills[0]`），没有遍历全部槽位，也没有建立槽位索引到技能的映射。

### 修复

**文件**：`Assets/AAAGame/Scripts/Game/Player/PlayerSkill/PlayerSkillManager.cs`

```csharp
private void Update()
{
    float dt = Time.deltaTime;
    for (int i = 0; i < Skills.Count; i++)
        Skills[i].Tick(dt);

    if (PlayerInputManager.Instance != null)
    {
        for (int slot = 1; slot <= 3; slot++)
        {
            if (PlayerInputManager.Instance.SkillDown(slot))
            {
                IPlayerSkill skill = FindSkillBySlot(slot);
                if (skill != null) skill.TryCast();
            }
        }
    }
}

private IPlayerSkill FindSkillBySlot(int slotIndex)
{
    for (int i = 0; i < Skills.Count; i++)
    {
        var tb = GF.DataTable.GetDataTable<PlayerSkillTable>();
        var row = tb?.GetDataRow(Skills[i].SkillId);
        if (row != null && row.SlotIndex == slotIndex)
            return Skills[i];
    }
    return null;
}
```

**按键映射**：

| 槽位 | 按键 | 技能示例 |
|------|------|---------|
| 1 | J | Dash（飞雷神，ID=1001） |
| 2 | K | Heal（治疗术，ID=1002） |
| 3 | L | Bomb（爆裂弹，ID=1003） |

**性能优化建议（可选）**：在 `UpdateSkillsFromPlayerData()` 时建立 `Dictionary<int, IPlayerSkill>` 槽位缓存，避免每帧查表。

## 三、棋子技能决策循环卡死

### 问题

日志中出现大量重复的 `[DefaultSkillReleaseStrategy] 后羿 决策: 使用技能1`，棋子既不攻击也不释放技能，卡死。

**触发条件**：棋子处于攻击状态时技能冷却恰好完成。

### 根本原因

1. `TickAttacking` 每帧调用 `ShouldUseSkill()`，条件满足时设 `m_ShouldUseSkillAfterAttack = true`，但棋子仍在攻击中，状态不切换，下帧继续进入，形成循环
2. 没有防抖机制，每帧都输出决策日志

### 修复

**文件**：`Assets/AAAGame/Scripts/Game/SummonChess/AI/FSM/ChessAIBase.cs`

```csharp
// 新增防抖字段
protected float m_SkillDecisionCooldown;
protected const float SKILL_DECISION_COOLDOWN = 0.2f;

protected virtual bool CanMakeSkillDecision()
{
    if (m_SkillDecisionCooldown > 0) return false;
    if (m_ShouldUseSkillAfterAttack) return false;
    return true;
}

// TickAttacking 中
if (m_CurrentState == ChessAIState.Attacking && CanMakeSkillDecision())
{
    if (ShouldUseSkill())
    {
        m_SkillDecisionCooldown = SKILL_DECISION_COOLDOWN;
        // ... 后续逻辑
    }
}

// OnAttackComplete 中
m_SkillDecisionCooldown = 0f;  // 攻击完成后重置防抖
```

### 修复效果

- 消除无限循环，每 0.2 秒最多一次技能决策
- 减少日志刷屏，提高性能
- 向后兼容，不影响正常技能释放

## 问题分析

### 1. 系统流程梳理

**正常流程：**
1. 玩家点击棋子 → ChessSelectionManager.SelectChess()
2. 玩家点击地面 → ChessSelectionManager.MoveChessTo()
3. 调用 ChessCombatController.SetPlayerMoveCommand()
4. 设置 AI 状态为 PlayerMoving，开始移动
5. 移动完成 → OnMovementArrived() 触发
6. 调用 AI.ResetTargetAfterPlayerMove() 重置目标
7. AI 重新搜索目标并进入战斗状态

### 2. 代码流程验证

**ChessCombatController.OnMovementArrived()：**
```csharp
private void OnMovementArrived()
{
    if (m_AIState == CombatAIState.PlayerMoving)
    {
        m_PlayerMoveTarget = null;  // ✅ 正确清空移动指令
        m_AIState = CombatAIState.Idle;  // ✅ 正确设置状态
        
        // ✅ 正确调用AI重置方法
        if (m_Entity.AI != null)
        {
            var ai = m_Entity.AI as ChessAIBase;
            if (ai != null)
            {
                ai.ResetTargetAfterPlayerMove();
            }
        }
    }
}
```

**ChessAIBase.ResetTargetAfterPlayerMove()：**
```csharp
public virtual void ResetTargetAfterPlayerMove()
{
    // ✅ 正确清除当前目标
    m_CurrentTarget = null;
    
    // ✅ 正确重置搜索计时器，触发立即搜索
    m_TargetSearchTimer = 0f;
}
```

**ChessAIBase.TickIdle()：**
```csharp
protected virtual void TickIdle(float dt)
{
    // ✅ 搜索逻辑正常

    }
}
```

### 3. 潜在问题点分析

**问题1：AI执行条件检查**
```csharp
protected virtual bool CanExecuteAI()
{
    // 检查 CombatController 是否启用
    if (m_Context.Entity.CombatController == null || !m_Context.Entity.CombatController.IsEnabled)
    {
        return false;
    }
    
    // ⚠️ 关键检查：如果有玩家移动指令，AI暂停工作
    if (m_Context.Entity.CombatController.HasPlayerMoveCommand)
    {
        return false;
    }
    
    return true;
}
```

**HasPlayerMoveCommand 属性：**
```csharp
public bool HasPlayerMoveCommand => m_PlayerMoveTarget.HasValue;
```

**分析：** 在 OnMovementArrived() 中，m_PlayerMoveTarget 被正确设置为 null，所以 HasPlayerMoveCommand 应该返回 false，这个逻辑看起来是正确的。

**问题2：敌人缓存机制**
```csharp
public List<EnemyInfoCache> GetEnemyCache(int myCamp)

        BuildEnemyCache();  // 自动构建缓存
```

**EnemyInfoCache.IsAlive：**
```csharp

```

**分析：** 敌人缓存机制看起来也是正常的。

## 可能的根本原因

### 推测1：时序问题
可能存在以下时序问题：
1. OnMovementArrived() 被调用
2. ResetTargetAfterPlayerMove() 被调用，设置 m_TargetSearchTimer = 0f
3. 但是在下一帧的 Tick() 中，CanExecuteAI() 仍然返回 false

### 推测2：状态同步问题
可能 m_PlayerMoveTarget 的清空和 AI 的 Tick() 执行之间存在时序差异。

### 推测3：敌人缓存未更新
可能敌人缓存在战斗过程中没有正确更新，导致 FindTarget() 返回 null。

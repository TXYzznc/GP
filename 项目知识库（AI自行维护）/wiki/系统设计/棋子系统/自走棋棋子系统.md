## 架构设计要点

### 接口驱动
- `IChessAI`：定义 AI 行为（Tick、FindTarget、Attack）
- `IChessSkill`：定义技能行为（ExecuteSkill、CalculateDamage、GetTargets）
- 便于不同 AI 和技能类型的扩展

### 工厂模式
- `ChessFactory` 使用 `Dictionary<string, Func<T>>` 存储创建器函数
- 避免反射，提升运行时性能
- 支持运行时动态注册

### 数据驱动
- 所有参数通过 `SummonChessTable` 配置，代码不写死数值
- `ChessDataManager` 单例管理配置缓存，支持按品质/星级/种族/职业查询

### 事件系统
- `ChessAttribute` 提供 `OnHpChanged`、`OnMpChanged`、`OnDead` 等事件
- 自动清理事件订阅，避免内存泄漏

## 场景配置步骤

1. **配置表**：在 `SummonChessTable.xlsx` 填写棋子数据，运行 DataTableGenerator
2. **场景对象**：
   - 创建空物体 `ChessSystemManager`，挂载 `ChessSystemInitializer`，勾选 `Auto Initialize`
   - 创建空物体 `SummonChessManager`，挂载 `SummonChessManager`（单例，只能一个）
3. **预制体**：创建棋子预制体 `Prefabs/SummonChess/Chess_xxx.prefab`
   - 不需要手动挂 ChessEntity 和 ChessAttribute，系统自动添加
   - 将预制体加入 Addressables，设置地址与资源 ID 对应

## 核心 API

```csharp
// 生成棋子
ChessEntity entity = await SummonChessManager.Instance.SpawnChessAsync(chessId, position, camp);

// 查询棋子
var allChess = SummonChessManager.Instance.GetAllChess();
var campChess = SummonChessManager.Instance.GetChessByCamp(0);
var nearbyChess = SummonChessManager.Instance.GetChessInRange(Vector3.zero, 10f);

// 属性操作
entity.Attribute.ModifyHp(-100);
entity.Attribute.ModifyMp(50);
double physDmg = entity.Attribute.CalculatePhysicalDamage(100);

// 解锁
ChessUnlockManager.Instance.UnlockChess(chessId);
ChessUnlockManager.Instance.GetUnlockedCount();
```

## 常见问题排查

| 症状 | 原因 | 解决方案 |
|------|------|---------|
| SummonChessTable 未加载 | 配置表未导入或名称错误 | 确认表名、运行 DataTableGenerator |
| 预制体加载失败 | 资源ID或 Addressables 地址错误 | 检查 ResourceConfigTable 和 Addressables 配置 |
| 棋子生成后立即消失 | 初始生命值为 0 或被外部销毁 | 检查 MaxHp 配置和事件订阅 |
| 存档数据未保存 | SaveCurrentSave() 未调用 | 确认存档时序 |

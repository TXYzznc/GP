### 1. 初始化（游戏启动时）
```csharp
// 设置账号ID
PlayerAccountDataManager.Instance.SetCurrentAccountId("000001");

// 自动加载最近存档
bool loaded = PlayerAccountDataManager.Instance.AutoLoadLastSave();
```

### 2. 创建新存档
```csharp
var saveData = PlayerAccountDataManager.Instance.CreateNewSave(
    "我的冒险",  // 存档名称
    1001         // 召唤师ID
);
```

### 3. 获取存档列表
```csharp
var saveInfos = PlayerAccountDataManager.Instance.GetAllSaveBriefInfos();

foreach (var info in saveInfos)
{
    Debug.Log($"{info.SaveName} - 等级{info.GlobalLevel}");
}
```

### 4. 加载存档
```csharp
var saveData = PlayerAccountDataManager.Instance.LoadSave(saveId);
```

### 5. 保存当前存档
```csharp
PlayerAccountDataManager.Instance.QuickSave();
```

### 6. 删除存档
```csharp
PlayerAccountDataManager.Instance.DeleteSave(saveId);
```

### 7. 访问当前存档
```csharp
var current = PlayerAccountDataManager.Instance.CurrentSaveData;
if (current != null)
{
    Debug.Log($"等级: {current.GlobalLevel}");
    Debug.Log($"金币: {current.Gold}");
}
```

## 📊 数据结构

### PlayerSaveData（存档数据）
```csharp
saveData.SaveId           // 存档ID（GUID）
saveData.SaveName         // 存档名称
saveData.GlobalLevel      // 等级
saveData.CurrentExp       // 经验
saveData.Gold             // 金币
saveData.Diamond          // 钻石
saveData.HolyWater        // 圣水
saveData.CreateTime       // 创建时间
saveData.LastPlayTime     // 最后游玩时间
```

### SaveBriefInfo（存档简要信息）
```csharp
info.SaveId               // 存档ID
info.SaveName             // 存档名称
info.GlobalLevel          // 等级
info.LastPlayTime         // 最后游玩时间
info.CreateTime           // 创建时间
```

## 🔄 API 对照表

| 操作 | 旧API | 新API |
|------|-------|-------|
| 获取当前数据 | `CurrentAccountData` | `CurrentSaveData` |
| 创建存档 | `CreateNewAccount(name, id, index)` | `CreateNewSave(name, id)` |
| 加载存档 | `LoadAccountData(index)` | `LoadSave(saveId)` |
| 保存存档 | `SaveAccountData(data)` | `SaveCurrentSave()` |
| 删除存档 | `DeleteSaveData(index)` | `DeleteSave(saveId)` |
| 获取列表 | `GetAllSaveInfos()` | `GetAllSaveBriefInfos()` |

## ⚠️ 重要变化

1. **不再使用存档索引（0-2）**，改用唯一的 SaveId（GUID）
2. **存档数量无限制**（旧系统固定3个）
3. **存档自动排序**（最近使用的在最前面）
4. **支持多账号**（每个账号有独立的存档目录）

## 📁 文件位置

```
Application.persistentDataPath/PlayerSaves/
└── 000001/                    # 账号ID
    ├── AccountInfo.json       # 账号信息
    ├── Save_[GUID1].json     # 存档1
    ├── Save_[GUID2].json     # 存档2
    └── Save_[GUID3].json     # 存档3
```

Windows 示例路径：
```
C:\Users\[用户名]\AppData\LocalLow\[公司名]\[产品名]\PlayerSaves\000001\
```

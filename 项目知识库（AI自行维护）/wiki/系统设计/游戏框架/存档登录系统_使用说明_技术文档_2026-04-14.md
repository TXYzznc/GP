### 1. **GameItem（存档信息UI）**
- ✅ 显示存档名称（玩家名称）
- ✅ 显示最后修改时间
- ✅ 显示章节名（暂时固定为"【序章】"）

- ✅ **新游戏按钮**：打开 NewGameUI 创建新角色
- ✅ **继续按钮**：自动加载最近的存档（栈顶）

### 3. **NewGameUI（新游戏）**
- ✅ 创建角色后自动添加到存档列表
- ✅ 自动设置为当前存档
- ✅ 创建后自动进入游戏

### 4. **LoadGameUI（存档列表）**
- ✅ 显示所有存档（按最近使用排序）
- ✅ 点击存档项选择
- ✅ 点击确认加载选中的存档
- ✅ 加载后自动将存档移到栈顶
- ✅ 点击关闭隐藏界面

## 🎮 完整流程说明

### 流程1：新游戏
```
1. 玩家在 StartMenuUI 点击"新游戏"按钮
   ↓
2. 打开 NewGameUI 界面
   ↓
3. 玩家输入名字、选择职业
   ↓
4. 创建新存档（调用 CreateNewSave）
   ↓
5. 存档ID自动添加到 PlayerAccountInfo.SaveIdStack 栈顶
   ↓
6. 设置为当前存档
   ↓
7. 进入游戏（调用 EnterGame）
```

### 流程2：继续游戏
```
1. 玩家在 StartMenuUI 点击"继续"按钮
   ↓
2. 自动加载 SaveIdStack[0]（栈顶存档）
   ↓
3. 设置为当前存档
   ↓
4. 进入游戏（调用 EnterGame）
```

### 流程3：读取存档
```
1. 玩家在 StartMenuUI 点击"读取存档"按钮
   ↓
2. 打开 LoadGameUI 界面
   ↓
3. 显示所有存档列表（按 SaveIdStack 顺序）
   ↓
4. 玩家点击某个 GameItem 的"选择"按钮
   ↓
5. 记录选中的 SaveId
   ↓
6. 玩家点击"确认"按钮
   ↓
7. 加载选中的存档（调用 LoadSave）
   ↓
8. 该存档自动移到 SaveIdStack 栈顶
   ↓
9. 设置为当前存档
   ↓
10. 关闭 LoadGameUI
   ↓
11. 进入游戏（调用 EnterGame）
```

### 修改的文件
1. **GameItem.cs** - 存档信息UI项
2. **LoadGameUI.cs** - 存档列表界面
3. **StartMenuUI.cs** - 主菜单界面

### 数据流
```
PlayerAccountInfo (账号信息)
├── AccountId: "000001"
├── SaveIdStack: [最近, 次近, 最旧]  # 栈结构
└── CreateTime, LastLoginTime

PlayerSaveData (存档数据)
├── SaveId: GUID
├── SaveName: 玩家名称
├── GlobalLevel: 等级
├── CurrentSummonerId: 召唤师ID
└── ... 其他游戏数据
```

## 🎯 存档栈机制

### 工作原理
```
初始状态：
SaveIdStack: []

创建存档A：
SaveIdStack: [A]

创建存档B：
SaveIdStack: [B, A]

加载存档A：
SaveIdStack: [A, B]  # A被移到栈顶

创建存档C：
SaveIdStack: [C, A, B]
```

### 优势
- 最近使用的存档总是在最前面
- "继续游戏"总是加载最近的存档
- 存档列表自动按使用时间排序

## ⚠️ 注意事项

### 1. 进入游戏功能
当前 `EnterGame()` 方法只是打印日志，需要你实现实际的场景切换：

```csharp
private void EnterGame()
{
    // 方式1：使用 Procedure 系统
    GF.Procedure.ChangeState<GameProcedure>();
    
    // 方式2：直接加载场景
    GF.Scene.LoadScene("GameScene");
    
    // 方式3：打开游戏主界面
    GF.UI.OpenUIForm(UIViews.GameMainUI);
}
```

### 2. 账号ID
当前账号ID固定为 "000001"，未来实现登录系统后需要修改：

```csharp
// 登录成功后
PlayerAccountDataManager.Instance.SetCurrentAccountId(loginAccountId);
```

### 3. 章节和天数
当前章节和天数是固定值，如果需要动态显示：

**方式1：扩展 SaveBriefInfo**
```csharp
public class SaveBriefInfo
{
    public string SaveId;
    public string SaveName;
    public int GlobalLevel;
    public double LastPlayTime;
    public double CreateTime;
    public string CurrentChapter;  // 新增
    public int GameDays;           // 新增
}
```

**方式2：在 GameItem 中读取完整存档**
```csharp
private void UpdateDisplay()
{
    var fullSaveData = LoadFullSaveData(m_SaveInfo.SaveId);
    if (fullSaveData != null)
    {
        varCharacter.text = fullSaveData.CurrentChapter;
        varDay.text = $"Day {fullSaveData.GameDays}";
    }
}
```

### 4. 存档删除功能
如果需要删除存档，可以在 GameItem 中添加删除按钮：

```csharp
// 在 GameItem 中添加
private Button varDeleteButton;

private void OnDeleteClick()
{
    // 显示确认对话框
    // 确认后调用
    PlayerAccountDataManager.Instance.DeleteSave(m_SaveInfo.SaveId);
    
    // 刷新列表
    // ...
}
```

## 📞 常见问题

### Q: GameItem 预制体加载失败？
**A:** 检查预制体路径是否正确，确保在 `Resources/UI/Items/` 目录下。

### Q: 存档列表为空？
**A:** 检查是否调用了 `SetCurrentAccountId("000001")`，确保账号信息已加载。

### Q: 职业名称显示"未知职业"？
**A:** 检查 SummonerTable 是否已加载，确保召唤师ID正确。

### Q: 时间显示异常？
**A:** 检查时间戳格式，确保使用 `DateTime.Ticks`。

### Q: 点击"继续"没反应？
**A:** 检查是否有存档，按钮是否被禁用。

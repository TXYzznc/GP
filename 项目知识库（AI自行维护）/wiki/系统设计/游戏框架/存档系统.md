## 一、系统架构

### 存档目录结构

```
Application.persistentDataPath/PlayerSaves/
├── 000001/                          # 账号ID目录
│   ├── AccountInfo.json             # 账号信息（存档栈）
│   ├── Save_[GUID1].json           # 存档1
│   ├── Save_[GUID2].json           # 存档2
│   └── Save_[GUID3].json           # 存档3
└── 000002/                          # 另一个账号
    └── ...
```

### 数据结构

```
PlayerAccountInfo (账号信息)
├── AccountId: "000001"
├── SaveIdStack: [最近, 次近, 最旧]  # 栈结构，按最近使用排序
└── CreateTime, LastLoginTime

PlayerSaveData (存档数据)
├── SaveId: GUID
├── SaveName: 玩家名称
├── GlobalLevel: 等级
├── CurrentSummonerId: 召唤师ID
└── ... 其他游戏数据
```

## 二、核心 API

### 旧 API → 新 API 对照

| 旧API | 新API | 说明 |
|-------|-------|------|
| `CurrentAccountData` | `CurrentSaveData` | 获取当前存档数据 |
| `CreateNewAccount(name, summonerId, saveIndex)` | `CreateNewSave(saveName, summonerId)` | 创建新存档 |
| `LoadAccountData(saveIndex)` | `LoadSave(saveId)` | 加载存档 |
| `SaveAccountData(accountData)` | `SaveCurrentSave()` | 保存当前存档 |
| `HasSaveData(saveIndex)` | - | 不再需要（通过列表判断） |
| `GetSaveInfo(saveIndex)` | `GetSaveBriefInfo(saveId)` | 获取存档信息 |
| `GetAllSaveInfos()` | `GetAllSaveBriefInfos()` | 获取所有存档 |
| `DeleteSaveData(saveIndex)` | `DeleteSave(saveId)` | 删除存档 |

### 常用 API 示例

```csharp
// 游戏启动时设置账号ID
PlayerAccountDataManager.Instance.SetCurrentAccountId("000001");

// 自动加载最近存档
bool success = PlayerAccountDataManager.Instance.AutoLoadLastSave();

// 创建新存档
var saveData = PlayerAccountDataManager.Instance.CreateNewSave(playerName, summonerId);

// 加载指定存档
var saveData = PlayerAccountDataManager.Instance.LoadSave(saveId);

// 获取所有存档列表（按最近使用排序）
var saveInfos = PlayerAccountDataManager.Instance.GetAllSaveBriefInfos();

// 获取当前存档
var currentSave = PlayerAccountDataManager.Instance.CurrentSaveData;

// 快速保存
PlayerAccountDataManager.Instance.QuickSave();

// 删除存档
PlayerAccountDataManager.Instance.DeleteSave(saveId);
```

## 三、存档栈机制

### 工作原理

新建存档自动添加到栈顶（索引0），加载存档自动移到栈顶，显示列表时按栈顺序显示（最近使用在前）。

```
初始：     SaveIdStack: []
创建A：    SaveIdStack: [A]
创建B：    SaveIdStack: [B, A]
加载A：    SaveIdStack: [A, B]   # A被移到栈顶
创建C：    SaveIdStack: [C, A, B]
```

### 优势
- 最近使用的存档总是在最前面
- "继续游戏"总是加载最近的存档
- 存档列表自动按使用时间排序
- 无限存档数量（旧系统固定3个槽位）

## 四、UI 集成

### 相关 UI 文件

| 文件 | 功能 |
|------|------|
| `GameItem.cs` | 存档信息UI项 |
| `LoadGameUI.cs` | 存档列表界面 |
| `StartMenuUI.cs` | 主菜单界面 |
| `NewGameUI.cs` | 新游戏界面 |

### 完整流程

**新游戏流程**
```
StartMenuUI 点击"新游戏" → NewGameUI 输入名字/职业 → CreateNewSave() →
存档ID加入 SaveIdStack 栈顶 → 设置当前存档 → EnterGame()
```

**继续游戏流程**
```
StartMenuUI 点击"继续" → AutoLoadLastSave()（加载 SaveIdStack[0]）→ EnterGame()
```

**读取存档流程**
```
StartMenuUI 点击"读取存档" → LoadGameUI 显示列表 → 选择存档 → LoadSave() →
存档移到 SaveIdStack 栈顶 → 关闭 LoadGameUI → EnterGame()
```

### GameItem 显示内容

| 字段 | 数据来源 | 备注 |
|------|---------|------|
| varNameText | SaveBriefInfo.SaveName | 存档名称 |
| varTimeText | SaveBriefInfo.LastPlayTime | 格式化时间 |
| varCharacter | - | 当前固定"【序章】" |
| varDay | - | 当前固定"Day 1" |
| varOccupationText | PlayerSaveData.CurrentSummonerId | 召唤师名称 |
| varGradeText | SaveBriefInfo.GlobalLevel | Lv.X |

## 五、注意事项

### 账号 ID
当前账号ID固定为 "000001"，未来实现登录系统后需要修改：
```csharp
PlayerAccountDataManager.Instance.SetCurrentAccountId(loginAccountId);
```

### 进入游戏
`EnterGame()` 需要实现实际的场景切换，例如：
```csharp
GF.Procedure.ChangeState<GameProcedure>();
// 或
GF.UI.OpenUIForm(UIViews.GameMainUI);
```

### 迁移搜索关键词
旧代码中搜索以下关键词进行替换：
`CurrentAccountData`、`CreateNewAccount`、`LoadAccountData`、`SaveAccountData`、`HasSaveData(`、`GetSaveInfo(`、`DeleteSaveData`

### 数据兼容性
旧存档数据（PlayerAccount_0.json 等）无法直接迁移到新系统，需要手动实现迁移逻辑。

### 存档备份路径
- Windows: `C:\Users\[用户名]\AppData\LocalLow\[公司名]\[产品名]\PlayerSaves\000001\`
- Android: `/storage/emulated/0/Android/data/[包名]/files/PlayerSaves/000001/`

## 六、常见问题

| 问题 | 排查 |
|------|------|
| GameItem 预制体加载失败 | 检查预制体路径 `Resources/UI/Items/GameItem.prefab` |
| 存档列表为空 | 检查是否调用了 `SetCurrentAccountId("000001")` |
| 职业名称显示"未知职业" | 检查 SummonerTable 是否已加载，召唤师ID是否正确 |
| 时间显示异常 | 检查时间戳格式，确保使用 `DateTime.Ticks` |
| 点击"继续"没反应 | 检查是否有存档，按钮是否被禁用 |
| 如何限制存档数量 | 调用 `GetAllSaveBriefInfos()` 判断数量后再开放创建 |

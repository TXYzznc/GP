## 一、功能需求

### 物品分类

| 类型 | ID 段 | 特性 |
|------|-------|------|
| 消耗品 | 10000-19999 | 可使用，部分可堆叠 |
| 任务道具 | 20000-29999 | 不可使用，不可堆叠，详情显示地图等 |
| 宝物 | 30000-39999 | 不可堆叠，可装备，固定特殊效果 + 1-3个随机词条 + 羁绊效果 |
| 装备 | 40000-49999 | 不可堆叠，可装备，固定属性加成 + 可选特殊效果 |

### 核心功能

- 背包管理（增删改查、堆叠、排序）
- 物品使用（消耗品效果执行）
- 装备系统（装备/卸下、属性计算）
- 宝物系统（词条生成、羁绊检测）
- 配置表加载与解析

```
│   ├── TreasureManager.cs       # 宝物管理器（待实现）
│   └── SynergyDetector.cs       # 羁绊检测器（待实现）
├── Effect/

    ├── SpecialEffectData.cs     # 特殊效果数据
    ├── AffixData.cs             # 词条数据
    └── SynergyData.cs           # 羁绊数据
```

### 数据流

```
InventoryManager（单例）
  - m_Slots[100]: List<Slot>
  - OnInventoryChanged: Event
      │
      ├─ InventorySlot（格子数据）
      │   ├─ ItemId, Count, IsEmpty
      │   └─ ItemStack
      │
      └─ ItemStack（堆叠数据）
          ├─ Item (InventoryItem)
          └─ Count
```

## 三、配置表结构

### ItemTable

| 字段名 | 类型 | 说明 |
|--------|------|------|
| Id | int | 物品 ID（分段规则见上） |
| Name | string | 物品名称 |
| Type | int | 类型（1=消耗品，2=任务道具，3=宝物，4=装备） |
| Quality | int | 品质（1=普通，2=优秀，3=稀有，4=史诗，5=传说） |
| Description | string | 物品描述 |
| IconPath | string | 缩略图路径 |
| DetailIconPath | string | 详细图路径（为空则用 IconPath） |
| CanStack | int | 是否可堆叠（0/1） |
| MaxStackCount | int | 最大堆叠数 |
| CanUse | int | 是否可使用 |
| UseEffectId | int | 使用效果 ID |
| CanEquip | int | 是否可装备 |
| EquipType | int | 装备类型（0=无，1=武器，2=防具，3=饰品，4=宝物） |
| SpecialEffectId | int | 特殊效果 ID |
| AffixPoolIds | string | 词条池 ID 列表（逗号分隔） |
| AffixCount | string | 词条数量范围（格式：最小-最大，如 "1-3"） |
| SynergyIds | string | 羁绊 ID 列表（逗号分隔） |
| BaseAttributes | string | 基础属性（JSON 格式） |
| SellPrice | int | 售价 |

## 四、关键技术要点

### 堆叠系统

- 使用 `ItemStack` 封装物品实例和数量
- 相同 ID 且可堆叠的物品自动合并到现有堆叠
- 超出最大堆叠数时创建新堆叠

### 宝物词条系统

- 宝物获得时从词条池随机抽取 1-3 个词条
- 使用权重算法控制稀有度（Weight 字段）
- 词条一旦生成固定不变

### 物品移动流程

```
MoveItem(fromSlot, toSlot)
  ├─ 目标为空 → 直接转移，清空源
  └─ 目标非空：
      ├─ 同 ID 且可堆叠 → 合并
      └─ 不同 → 互换两格内容
  └─ 触发 OnInventoryChanged 事件
```

### 整理算法

```
Step 1: 收集所有非空堆叠
Step 2: 合并同 ID 可堆叠物品
Step 3: 多级排序
  1. ItemType 升序（消耗品 < 装备 < 宝物 < 任务道具）
  2. Quality 降序（高品质在前）
  3. ItemId 升序
Step 4: 重新填回格子
```

### 日志规范

所有关键操作使用 `DebugEx` 输出，包含模块名称，区分普通/成功/警告/错误。

## 五、使用示例

```csharp
// 添加物品
InventoryManager.Instance.AddItem(10001, 10);  // 添加10个钱包
InventoryManager.Instance.AddItem(30001, 1);   // 添加1个宝物

// 使用物品（第0格）
InventoryManager.Instance.UseItem(0);

// 创建物品实例
var treasure = ItemManager.Instance.CreateItem(30001) as TreasureItem;
foreach (var affix in treasure.Affixes)
    Debug.Log(affix.GetFormattedDescription());

// 查询数量
int count = InventoryManager.Instance.GetItemCount(10001);
```

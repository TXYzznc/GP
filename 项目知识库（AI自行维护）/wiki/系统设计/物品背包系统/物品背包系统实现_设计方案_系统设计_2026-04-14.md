## 功能需求

### 物品分类

1. **消耗品**
   - 可使用
   - 部分可堆叠，部分不可堆叠
   - 示例：钱包（使用后随机获得金币）

2. **任务道具**
   - 不可使用
   - 不可堆叠
   - 示例：神秘地图（详情中显示地图图片）

3. **宝物**
   - 不可堆叠
   - 可装备（装备逻辑与装备不同）
   - 固定特殊效果 + 1-3个随机词条 + 羁绊效果
   - 示例：八尺琼勾玉

4. **装备**
   - 不可堆叠
   - 可装备
   - 固定属性加成 + 可选特殊效果
   - 示例：石中剑

### 核心功能

- 背包管理（增删改查、堆叠、排序）
- 物品使用（消耗品效果执行）
- 装备系统（装备/卸下、属性计算）
- 宝物系统（词条生成、羁绊检测）
- 配置表加载与解析

## 技术方案

### 架构设计

```

├── Core/
│   ├── ItemBase.cs              # 物品基类
│   ├── ConsumableItem.cs        # 消耗品

│   ├── EquipmentManager.cs      # 装备管理器
```

## 技术要点

1. **物品ID分段规则**
   - 10000-19999：消耗品
   - 20000-29999：任务道具
   - 30000-39999：宝物
   - 40000-49999：装备

2. **堆叠逻辑**
   - 使用 ItemStack 记录物品ID和数量
   - 相同ID且可堆叠的物品自动合并
   - 超过最大堆叠数量时创建新堆叠

3. **词条生成**
   - 宝物获得时从词条池随机抽取
   - 使用权重系统控制稀有度
   - 生成后词条固定，不再变化

4. **羁绊检测**
   - 装备/卸下宝物时触发检测
   - 检查是否满足羁绊激活条件
   - 激活/失效时应用/移除羁绊效果

5. **效果参数**
   - 使用 JSON 格式存储灵活参数
   - 支持多种效果类型扩展
   - 效果执行器根据类型分发处理

## 注意事项

1. 所有物品操作需要通过 InventoryManager 统一管理
2. 装备和宝物的装备逻辑需要分开处理
3. 羁绊效果需要在装备管理器中统一计算
4. 配置表加载失败需要有明确的错误提示
5. 物品使用需要检查使用条件和冷却时间
6. 背包满时需要提示玩家

## 技术要点

### 1. 物品ID分段设计
- 10000-19999：消耗品
- 20000-29999：任务道具
- 30000-39999：宝物
- 40000-49999：装备
- 便于快速识别物品类型

### 2. 堆叠系统设计
- 使用 ItemStack 封装物品和数量
- 支持可堆叠和不可堆叠物品
- 自动合并相同物品到现有堆叠

### 3. 宝物词条系统
- 配置词条池和权重
- 获得时随机生成1-3个词条
- 使用权重算法保证稀有度

### 4. 配置表灵活性
- 使用JSON格式存储复杂数据（属性、效果参数）
- 使用逗号分隔存储ID列表
- 使用范围格式存储数值范围（如"1-3"）

### 5. 日志系统集成
- 所有关键操作都使用 DebugEx 输出日志
- 日志包含模块名称，便于定位
- 区分普通、成功、警告、错误日志

### 6. 单例模式应用
- InventoryManager、ItemManager、ItemEffectExecutor 都使用单例
- 便于全局访问和管理
- 避免重复初始化

## 使用示例

### 添加物品到背包
```csharp
// 添加10个钱包
InventoryManager.Instance.AddItem(10001, 10);

// 添加1个宝物
InventoryManager.Instance.AddItem(30001, 1);
```

### 使用物品
```csharp
// 使用第0个格子的物品
InventoryManager.Instance.UseItem(0);
```

### 创建物品实例
```csharp
// 创建宝物
var treasure = ItemManager.Instance.CreateItem(30001) as TreasureItem;

// 查看词条
foreach (var affix in treasure.Affixes)
{
    Debug.Log(affix.GetFormattedDescription());
}
```

### 查询物品数量
```csharp
// 查询钱包数量
int count = InventoryManager.Instance.GetItemCount(10001);
Debug.Log($"拥有钱包数量: {count}");
```

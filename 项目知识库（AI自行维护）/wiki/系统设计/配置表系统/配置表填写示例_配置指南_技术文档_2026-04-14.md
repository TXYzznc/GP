### 1.2 数组字段填写格式

**重要：** `BuffIds` 和 `SelfBuffIds` 是数组类型，填写格式如下：

```
空数组：     []
单个元素：   [1]
多个元素：   [1,2,3]
```

**注意事项：**
- 使用英文方括号 `[]`
- 多个元素用英文逗号 `,` 分隔
- 不要有空格
- 不要有中文符号

## 五、填表注意事项

### 5.1 数组字段格式

**正确示例：**
```
[]           ← 空数组
[1]          ← 单个元素
[1,2,3]      ← 多个元素
[1,1,1]      ← 重复元素（用于叠层）
```

**错误示例：**
```
❌ [ 1, 2, 3 ]    ← 有空格
❌ [1，2，3]      ← 中文逗号
❌ 【1,2,3】      ← 中文方括号
❌ 1,2,3          ← 缺少方括号
```

### 5.2 Buff叠层技巧

如果想让技能一次性施加多层 Buff，可以在 `BuffIds` 中重复写同一个 ID：

```
BuffIds = [1,1,1]  ← 施加 3 层灼烧
```

### 5.3 配置驱动 vs 代码实现

**配置驱动（推荐）：**
- 简单的 Buff 应用（如嫦娥普攻附带冰霜）
- 固定的伤害系数
- 不需要条件判断的效果

**代码实现（必要时）：**
- 复杂的条件判断（如后羿烈焰箭激活时才附带灼烧）
- 特殊的元素反应（如融化）
- 需要动态计算的效果

### 5.4 命中检测类型选择

| 类型 | 值 | 适用场景 | 性能 |
|:---|:---|:---|:---|
| **瞬发** | 0 | 自动战斗、单体技能 | ⭐⭐⭐⭐⭐ 最优 |
| **近战** | 1 | 近战武器挥砍 | ⭐⭐⭐⭐ 良好 |
| **投射物** | 2 | 弓箭、飞弹 | ⭐⭐⭐ 中等 |
| **AOE** | 3 | 范围技能 | ⭐⭐ 较低 |
| **射线** | 4 | 激光、穿透 | ⭐⭐⭐⭐ 良好 |

**建议：**
- 自动战斗的远程棋子优先使用 `瞬发（0）`，性能最优
- 需要物理表现的攻击使用 `投射物（2）`
- 近战棋子使用 `近战（1）`

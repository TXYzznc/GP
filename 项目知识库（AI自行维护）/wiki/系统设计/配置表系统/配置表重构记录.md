## 一、资源路径 → 资源ID 重构

**重构日期**：2026-03-08

### 背景

原来配置表中的资源引用使用字符串路径（如 `PrefabPath`、`IconPath`），改为使用整数资源ID（如 `PrefabId`、`IconId`），通过 `ResourceConfigTable` 集中管理 ID 到路径的映射。

### 字段变更清单

| 配置表 | 旧字段 | 新字段 |
|--------|--------|--------|
| SummonerTable | PrefabPath | PrefabId |
| PlayerSkillTable | IconPath | IconId |
| SummonChessTable | PrefabPath, IconPath | PrefabId, IconId |
| SummonChessSkillTable | EffectPath, HitEffectPath, IconPath | EffectId, HitEffectId, IconId |
| ItemTable | IconPath, DetailIconPath | IconId, DetailIconId |
| BuffTable | SpritePath, EffectPath | SpriteId, EffectId |
| CardTable | SpritePath | SpriteId |
| EquipmentTable | SpritePath | SpriteId |

### 代码修改模式

```csharp
// 旧代码：直接使用路径字符串
string path = config.PrefabPath;
await GameEntry.Resource.LoadAssetAsync<GameObject>(config.PrefabPath);

// 新代码：通过ResourceExtension
int prefabId = config.PrefabId;
string path = ResourceExtension.GetResourcePath(prefabId);
await ResourceExtension.LoadPrefabAsync(config.PrefabId);
```

**字段直接替换**：

```csharp
// 旧
EffectPath = m_Config.EffectPath,
HitEffectPath = m_Config.HitEffectPath,

// 新
EffectId = m_Config.EffectId,
HitEffectId = m_Config.HitEffectId,
```

### 资源类型规范

`ResourceConfigTable` 中的资源类型枚举：
- 1 = Sprite
- 2 = Prefab
- 3 = Effect
- 4 = Material
- 5 = Texture
- 6 = ScriptableObject

ID 范围：10001-40003（物品图标等资源）

## 三、技术要点

1. 项目使用 `ResourceExtension` 类统一管理所有资源加载
2. 配置表存储资源 ID（int 类型），运行时通过 `ResourceConfigTable` 查路径
3. DataTable 生成的 C# 类已自动更新字段名（字段迁移后重新生成）

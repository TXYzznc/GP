## 3个主要执行流

### 1️⃣ 战斗流程（最核心）

```mermaid
sequenceDiagram
    participant Input as PlayerInputManager
    participant Event as EventManager
    participant Combat as CombatManager
    participant Chess as ChessEntity
    participant Skill as SkillExecutor
    participant Hit as HitDetector
    participant Buff as BuffManager
    
    Input->>Event: 发布输入事件
    Event->>Combat: 战斗事件
    Combat->>Chess: 执行行动
    Chess->>Skill: 执行技能
    Skill->>Hit: 命中检测
    Hit->>Buff: 应用Buff
    Buff->>Combat: 更新状态
```

### 2️⃣ UI交互流程

```mermaid
sequenceDiagram
    participant Input as PlayerInputManager
    participant Event as EventManager
    participant UI as UIForm
    participant Card as CardManager
    participant Inventory as InventoryManager
    
    Input->>Event: 用户点击
    Event->>UI: UI事件
    UI->>Card: 卡牌操作
    UI->>Inventory: 物品操作
    Card->>Event: 发布效果事件
    Inventory->>Event: 发布更新事件
```

### 3️⃣ 探索与敌人AI流程

```mermaid
sequenceDiagram
    participant Explore as ExploreManager
    participant Chess as ChessEntity/EnemyAI
    participant Input as PlayerInputManager
    participant Event as EventManager
    
    Input->>Explore: 玩家移动
    Explore->>Chess: 更新敌人状态
    Chess->>Chess: AI决策（FSM）
    Chess->>Event: 发布行动事件
    Event->>Explore: 更新场景
```

## 如何查询图结构？

使用 GitNexus exploring 技能进行高级查询：

```bash
# 查询特定类的所有依赖
gitnexus explore-dependencies CombatManager

# 查询特定函数的调用链
gitnexus explore-callers ExecuteSkill

# 查询某个类的所有继承者
gitnexus explore-implementations IChessSkill

# 查询模块间的通信
gitnexus explore-communication Combat Buff Skill
```

> **注**：GitNexus 数据存储在 KuzuDB（图数据库）中，目前没有内置的可视化导出功能，但可以通过技能查询获取详细信息。

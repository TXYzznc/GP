# 第 6-7 章 策略卡系统与技术美术系统

## 第 6 章 策略卡系统的设计与实现

### 6.1 卡牌系统架构

策略卡系统是游戏的创新系统之一，提供了丰富的战斗策略选择。系统架构如下：

```
卡牌系统
├── 卡牌管理（CardManager）
│   ├── 卡牌数据（CardData）
│   ├── 卡牌配置表（CardTable）
│   └── 卡牌列表管理
├── 卡牌效果（CardEffect）
│   ├── 效果执行器（CardEffectExecutor）
│   ├── 12 个卡牌效果脚本
│   └── 效果参数配置
├── 卡牌 UI（CardUI）
│   ├── 卡槽显示（CardSlotItem）
│   ├── 卡牌选中
│   ├── 拖拽交互
│   └── 范围预览（CardRangePreview）
└── 卡牌平衡（CardBalance）
    ├── 灵力消耗
    ├── 效果强度
    └── 冷却时间
```

### 6.2 卡牌数据结构

**6.2.1 卡牌属性**

每张卡牌包含以下属性：
- **ID**：卡牌唯一标识
- **名称**：卡牌名称
- **描述**：卡牌效果描述
- **灵力消耗**：使用卡牌消耗的灵力
- **目标类型**：单体、范围、全体等
- **效果类型**：伤害、治疗、Buff 等
- **范围**：效果作用范围

### 6.3 卡牌效果执行引擎

**6.3.1 效果执行流程**

```
选择卡牌
    ↓
选择目标
    ↓
消耗灵力
    ↓
执行效果
    ↓
播放特效
    ↓
更新 UI
    ↓
卡牌销毁
```

**6.3.2 效果执行器**

CardEffectExecutor 负责：
- 创建效果实例
- 执行效果逻辑
- 处理目标选择
- 发送效果事件

**6.3.3 效果参数配置**

每个效果支持参数化配置：
- 伤害值
- 治疗值
- Buff 类型
- 特效 ID
- 范围大小

### 6.4 卡牌 UI 交互

**6.4.1 卡槽显示**

- 显示 8 张卡牌
- 显示卡牌名称、描述、灵力消耗
- 支持卡牌选中高亮

**6.4.2 卡牌选中**

- 点击卡牌进行选中
- 选中卡牌向上移动
- 显示卡牌详细信息
- 支持取消选中

**6.4.3 拖拽交互**

- 拖拽卡牌到战场
- 创建半透明拖拽预览
- 检测有效放置位置
- 支持吸附到快捷栏

### 6.5 范围预览系统

**6.5.1 预览类型**

- **单体目标**：显示小型指示器
- **范围目标**：显示圆形范围
- **直线目标**：显示直线范围
- **特殊范围**：显示自定义范围

**6.5.2 预览实现**

- 使用 Gizmos 绘制预览
- 支持动态调整大小
- 支持颜色自定义
- 支持透明度调整

**6.5.3 预览优化**

- 使用对象池缓存预览对象
- 减少创建销毁开销
- 支持快速切换预览

### 6.6 卡牌平衡性设计

**6.6.1 平衡原则**

- 不同效果的卡牌灵力消耗不同
- 强力效果消耗更多灵力
- 支持多种战斗策略
- 避免某张卡牌过于强势

**6.6.2 卡牌参数设计**

- 伤害卡牌：高伤害，高灵力消耗
- 治疗卡牌：中等治疗，中等灵力消耗
- 控制卡牌：特殊效果，中等灵力消耗
- 增益卡牌：属性提升，低灵力消耗

**6.6.3 平衡性测试**

- 测试不同卡牌组合
- 测试不同敌人配置
- 收集玩家反馈
- 定期调整参数

## 第 7 章 技术美术（TA）系统的设计与实现

### 7.1 TA 系统架构

技术美术系统负责游戏的视觉效果，包括渲染、特效、动画等。系统架构如下：

```
TA 系统
├── Shader 层
│   ├── 卡通渲染 Shader
│   ├── 溶解 Shader
│   ├── 描边 Shader
│   └── 其他特效 Shader
├── 脚本层
│   ├── 效果控制脚本
│   ├── 参数管理脚本
│   └── 动画控制脚本
└── 配置层
    ├── 材质配置
    ├── 参数配置
    └── 效果配置
```

### 7.2 卡通渲染系统（Cartoon Shader）

**7.2.1 卡通渲染原理**

卡通渲染是一种非真实感渲染（NPR）技术，通过以下方式实现：

1. **分层着色**：将光照分为几个离散的颜色层
2. **边缘检测**：检测模型边缘并添加描边
3. **简化光照**：使用简化的光照计算

**7.2.2 实现方式**

- 使用 PlayerCartoon.shader 渲染玩家角色
- 使用 OilPaintingToon.shader 渲染油画风格
- 支持溶解效果集成
- 支持参数化配置

**7.2.3 应用场景**

- 玩家角色渲染
- 敌人角色渲染
- 召唤物渲染

### 7.3 溶解效果系统（Dissolve Shader）

**7.3.1 溶解原理**

溶解效果通过以下步骤实现：

1. 采样噪声纹理
2. 比较噪声值与溶解阈值
3. 丢弃低于阈值的像素
4. 在边缘添加发光效果

**7.3.2 实现方式**

- 使用 Dissolve.shader 实现溶解效果
- 支持自定义溶解方向
- 支持边缘发光颜色
- 支持参数动画化

**7.3.3 应用场景**

- 敌人死亡动画
- 物体消失效果
- 传送特效

### 7.4 描边效果系统（Outline/OuterGlow）

**7.4.1 描边原理**

描边效果通过两个 Pass 实现：

1. **第一 Pass**：正常渲染模型
2. **第二 Pass**：放大模型顶点并使用单色渲染

**7.4.2 实现方式**

- 使用 Custom_Outline.shader 实现基础描边
- 使用 Custom_Outline_Blur.shader 实现模糊描边
- 支持多种配置（友方、敌方、可交互、选中）
- 支持颜色和宽度自定义

**7.4.3 配置方案**

| 配置 | 颜色 | 用途 |
|------|------|------|
| Ally.asset | 绿色 | 友方单位 |
| Enemy.asset | 红色 | 敌方单位 |
| Interactive.asset | 黄色 | 可交互物体 |
| Selected.asset | 白色 | 选中状态 |

### 7.5 天空盒系统（Sky）

**7.5.1 天空盒类型**

- 立方体贴图天空盒
- 程序化天空盒
- 动态天空盒

**7.5.2 应用场景**

- 探索场景背景
- 战斗场景背景
- 基地场景背景

### 7.6 UI 特效系统（UI）

**7.6.1 UI 特效类型**

- 按钮特效：点击反馈
- 面板特效：打开/关闭动画
- 过渡特效：场景切换动画
- 粒子效果：特殊效果展示

**7.6.2 实现方式**

- 使用 DOTween 实现 UI 动画
- 使用粒子系统实现粒子效果
- 支持特效叠加
- 支持性能优化

### 7.7 性能优化与渲染管线

**7.7.1 渲染优化**

- **批处理**：合并 Draw Call，使用 GPU Instancing
- **纹理优化**：压缩纹理，使用 Mipmap，纹理图集
- **剔除优化**：剔除不可见物体，使用 LOD 系统

**7.7.2 TA 系统优化**

- 使用对象池缓存特效
- 减少 Shader 计算量
- 使用预计算优化性能
- 定期检查性能指标

**7.7.3 性能指标**

- 帧率：60 FPS
- 内存占用：< 500 MB
- Draw Call：< 100
- 三角形数：< 1M

## 📝 初稿内容

### 摘要

本论文以 Clash of Gods（众神之战）游戏项目为研究对象，深入研究了回合制 RPG 游戏的系统架构设计与实现。该项目采用 Unity 引擎和 GameFramework 框架，实现了一个完整的游戏系统，包括战斗系统、物品背包系统、策略卡系统、技术美术系统等核心模块。

论文首先分析了游戏开发中的关键技术和设计模式，包括事件驱动架构、配置表驱动设计、对象池优化等。其次，详细阐述了游戏的总体架构设计，包括模块划分、数据流管理、资源加载策略等。再次，重点介绍了四个核心系统的设计与实现：（1）战斗系统，包括棋子管理、Buff 系统、AI 决策等；（2）物品背包系统，包括容器设计、拖拽交互、数据持久化等；（3）策略卡系统，包括卡牌效果执行、范围预览、UI 交互等；（4）技术美术系统，包括卡通渲染、溶解效果、描边效果等。最后，论文总结了系统集成、性能优化和热修复实践的经验。

通过本研究，我们验证了配置表驱动、事件驱动、对象池等设计模式在游戏开发中的有效性，为游戏架构设计、性能优化和工程化实践提供了有价值的参考。项目已完成 14 周的开发周期，实现了 100+ 个功能模块，代码规模达到 50000+ 行，具有较高的工程化水平和可维护性。

**关键词**：游戏开发；系统架构；GameFramework；配置表驱动；事件驱动；性能优化；技术美术；热修复

### ABSTRACT

This thesis investigates the system architecture design and implementation of turn-based RPG games, taking the Clash of Gods project as the research subject. The project employs the Unity engine and GameFramework framework to implement a complete game system, including core modules such as combat systems, inventory systems, card systems, and technical art systems.

The thesis first analyzes key technologies and design patterns in game development, including event-driven architecture, configuration-driven design, and object pool optimization. Second, it elaborates on the overall architecture design of the game, including module division, data flow management, and resource loading strategies. Third, it focuses on the design and implementation of four core systems: (1) Combat System, including chess piece management, Buff system, and AI decision-making; (2) Inventory System, including container design, drag-and-drop interaction, and data persistence; (3) Card System, including card effect execution, range preview, and UI interaction; (4) Technical Art System, including cartoon rendering, dissolve effects, and outline effects. Finally, the thesis summarizes the experience in system integration, performance optimization, and hot-fix practices.

Through this research, we have verified the effectiveness of design patterns such as configuration-driven, event-driven, and object pool optimization in game development, providing valuable references for game architecture design, performance optimization, and engineering practices. The project has completed a 14-week development cycle, implementing 100+ functional modules with a code scale of 50,000+ lines, demonstrating high engineering standards and maintainability.

**Keywords**: Game Development; System Architecture; GameFramework; Configuration-Driven; Event-Driven; Performance Optimization; Technical Art; Hot-Fix

### 第 1 章 绪论

#### 1.1 研究背景

随着游戏产业的快速发展，游戏开发的复杂度不断增加。特别是在移动游戏和网络游戏领域，如何设计高效、可扩展、易维护的游戏系统成为了一个重要课题。回合制 RPG 游戏因其策略性强、可玩性高、易于平衡等特点，在游戏市场中占据重要地位。

当前游戏开发面临的主要挑战包括：
1. **系统复杂度高**：现代游戏包含众多相互关联的系统（战斗、物品、技能、UI 等），系统间的耦合度高，难以维护和扩展。
2. **性能要求严格**：移动设备的性能限制要求游戏开发者必须进行精细的性能优化，包括内存管理、渲染优化、逻辑优化等。
3. **快速迭代需求**：游戏需要频繁更新和调整，传统的硬编码方式难以满足快速迭代的需求。
4. **团队协作困难**：大型游戏项目通常由多个团队协作开发，需要统一的架构设计和开发规范。

为了应对这些挑战，游戏开发社区提出了多种解决方案，包括：
- **事件驱动架构**：通过事件系统解耦各个模块，提高系统的灵活性和可维护性。
- **配置表驱动设计**：将游戏数据从代码中分离出来，通过配置表管理，提高数据的可维护性和快速迭代能力。
- **对象池优化**：通过对象池技术减少内存分配和垃圾回收的开销，提高游戏性能。
- **热修复支持**：支持在不重启游戏的情况下更新代码和数据，提高游戏的可用性。

Unity 引擎作为当前最流行的游戏开发引擎之一，提供了强大的开发工具和丰富的资源库。GameFramework 是一个基于 Unity 的开源游戏框架，提供了完整的游戏开发基础设施，包括流程管理、状态机、事件系统、资源管理等。

#### 1.2 研究意义

本研究以 Clash of Gods 游戏项目为实践平台，深入研究回合制 RPG 游戏的系统架构设计与实现，具有以下意义：

**学术意义**：
1. 验证现有设计模式在游戏开发中的有效性，特别是事件驱动、配置表驱动等模式的实际应用效果。
2. 提出针对回合制 RPG 游戏的系统架构设计方案，为游戏架构研究提供新的思路。
3. 总结游戏开发中的最佳实践，为游戏工程化提供理论支持。

**实践意义**：
1. 为游戏开发团队提供可参考的架构设计方案，降低游戏开发的复杂度。
2. 通过配置表驱动、事件驱动等技术手段，提高游戏的快速迭代能力。
3. 通过性能优化和热修复支持，提高游戏的稳定性和可用性。
4. 建立完整的开发规范和文档体系，便于团队协作和知识传递。

**创新点**：
1. 提出了一套完整的配置表驱动设计方案，包括配置表的设计、生成、加载、使用等全流程。
2. 实现了高效的事件驱动架构，支持模块间的松耦合通信。
3. 设计了灵活的卡牌效果执行引擎，支持复杂的卡牌效果组合。
4. 实现了完整的热修复支持，包括代码热修复和数据热更新。

# 毕业论文规范格式指南

> **最后更新**: 2026-04-06
> **适用范围**: Clash of Gods 项目毕业论文
> **参考标准**: 高校毕业设计（论文）规范

---

## 一、论文基本要求

### 1.1 论文结构

标准毕业论文应包含以下部分（按顺序）：

| 序号 | 部分 | 说明 |
|------|------|------|
| 1 | 封面 | 论文标题、作者、指导教师、完成日期等 |
| 2 | 摘要（中文） | 150-300 字，概括论文核心内容 |
| 3 | ABSTRACT（英文） | 英文摘要，与中文摘要内容对应 |
| 4 | 目录 | 自动生成，列出所有章节和页码 |
| 5 | 绪论 | 研究背景、意义、目标、主要内容 |
| 6 | 正文 | 核心章节（通常 5-8 章） |
| 7 | 结论 | 总结研究成果、存在问题、后续工作 |
| 8 | 参考文献 | 引用的所有文献 |
| 9 | 附录 | 可选，包含代码、配置表等 |
| 10 | 致谢 | 感谢指导教师、团队成员等 |

### 1.2 页面设置

| 项目 | 要求 |
|------|------|
| **纸张** | A4 (210mm × 297mm) |
| **页边距** | 上 2.5cm、下 2.5cm、左 3cm、右 2cm |
| **行距** | 1.5 倍行距 |
| **字体** | 正文：宋体 12pt；标题：黑体 14-18pt |
| **对齐** | 正文两端对齐，标题居中 |
| **页码** | 从绪论开始编号，位于页脚中央 |

### 1.3 标题层级

```
# 一级标题（黑体 18pt，居中）
## 二级标题（黑体 14pt，左对齐）
### 三级标题（黑体 12pt，左对齐）
#### 四级标题（宋体 12pt 加粗，左对齐）
```

## 二、各部分详细要求

### 2.1 摘要（中文）

**要求**：
- 字数：150-300 字
- 内容：简明扼要地说明研究内容、方法、结果和意义
- 格式：单独成页，标题"摘要"居中，黑体 14pt
- 关键词：3-5 个，用分号分隔

**结构**：
1. 研究背景和意义（1-2 句）
2. 研究内容和方法（2-3 句）
3. 主要成果（2-3 句）
4. 应用价值（1-2 句）

**示例**：
```
摘要

本论文以 Clash of Gods 游戏项目为研究对象，研究了回合制 RPG 游戏的系统架构设计与实现。
针对游戏中的战斗系统、物品系统、卡牌系统等核心模块，采用 GameFramework 框架和 Unity 引擎，
设计并实现了高效的数据管理、事件驱动和 UI 交互系统。通过配置表驱动、热修复支持等技术手段，
实现了游戏的快速迭代和灵活扩展。本研究对游戏开发中的架构设计、性能优化和工程化实践具有参考价值。

关键词：游戏开发；系统架构；GameFramework；配置表驱动；热修复
```

### 2.2 ABSTRACT（英文）

**要求**：
- 字数：150-300 词
- 内容：与中文摘要对应，但不是逐字翻译
- 格式：单独成页，标题"ABSTRACT"居中，黑体 14pt
- Keywords：3-5 个，用semicolons 分隔

**示例**：
```
ABSTRACT

This thesis investigates the system architecture design and implementation of turn-based RPG games,
taking the Clash of Gods project as the research subject. Focusing on core modules including combat systems,
inventory systems, and card systems, we designed and implemented efficient data management, event-driven,
and UI interaction systems using the GameFramework and Unity engine. Through configuration-driven approaches
and hot-fix support, we achieved rapid iteration and flexible extension of the game. This research provides
valuable insights for game development in architecture design, performance optimization, and engineering practices.

Keywords: Game Development; System Architecture; GameFramework; Configuration-Driven; Hot-Fix
```

### 2.3 绪论

**要求**：
- 字数：1500-2000 字
- 内容：研究背景、意义、目标、主要内容、论文结构
- 格式：第一章，标题"绪论"或"第一章 绪论"

**结构**：
1. **研究背景**（300-400 字）
   - 游戏开发行业现状
   - 回合制 RPG 游戏的特点
   - 现有技术和框架

2. **研究意义**（300-400 字）
   - 学术意义：对游戏架构设计的贡献
   - 实践意义：对游戏开发的指导作用
   - 创新点：新的设计思路或优化方案

3. **研究目标**（200-300 字）
   - 主要目标：实现完整的游戏系统
   - 具体目标：各个子系统的设计和实现

4. **主要内容**（300-400 字）
   - 论文涵盖的主要内容
   - 各章节的简要说明

5. **论文结构**（200-300 字）
   - 各章节的组织方式
   - 逻辑关系说明

### 2.4 正文章节

**通常包含 5-8 章**，建议结构：

| 章节 | 内容 | 字数 |
|------|------|------|
| 第 2 章 | 相关技术与理论基础 | 2000-3000 |
| 第 3 章 | 系统总体设计 | 2000-3000 |
| 第 4 章 | 核心模块实现（如战斗系统） | 3000-4000 |
| 第 5 章 | 核心模块实现（如物品系统） | 3000-4000 |
| 第 6 章 | 核心模块实现（如卡牌系统） | 3000-4000 |
| 第 7 章 | 测试与优化 | 2000-3000 |
| 第 8 章 | 总结与展望 | 1500-2000 |

**每章要求**：
- 标题清晰，层级分明
- 内容逻辑严密，论证充分
- 包含图表、代码示例等辅助说明
- 字数 2000-4000 字

### 2.5 结论

**要求**：
- 字数：1000-1500 字
- 内容：总结研究成果、存在问题、后续工作
- 格式：最后一章或单独章节

**结构**：
1. **研究成果总结**（400-500 字）
   - 完成的主要工作
   - 取得的成果

2. **存在的问题**（300-400 字）
   - 技术层面的不足
   - 设计层面的改进空间

3. **后续工作**（300-400 字）
   - 功能扩展方向
   - 性能优化方向
   - 新技术应用

### 2.6 参考文献

**要求**：
- 格式：按引用顺序编号 [1]、[2] 等
- 数量：通常 15-30 篇
- 类型：学术论文、技术文档、书籍等

**格式示例**：
```
[1] 张三, 李四. 游戏引擎设计与实现[M]. 北京: 清华大学出版社, 2020.
[2] Smith, J., Johnson, K. Game Architecture Patterns[J]. Game Developer Magazine, 2021, 28(3): 45-52.
[3] Unity Technologies. Unity Manual[EB/OL]. https://docs.unity3d.com, 2026.
```

## 三、格式规范

### 3.1 字体和字号

| 内容 | 字体 | 字号 | 加粗 |
|------|------|------|------|
| 一级标题 | 黑体 | 18pt | 是 |
| 二级标题 | 黑体 | 14pt | 是 |
| 三级标题 | 黑体 | 12pt | 是 |
| 正文 | 宋体 | 12pt | 否 |
| 代码 | Courier New | 10pt | 否 |
| 图表标题 | 宋体 | 11pt | 是 |

### 3.2 段落格式

- **行距**：1.5 倍
- **段前间距**：0pt
- **段后间距**：6pt
- **首行缩进**：2 字符（0.74cm）
- **对齐方式**：两端对齐

### 3.3 图表规范

**图片**：
- 格式：PNG、JPG（分辨率 ≥ 300dpi）
- 大小：宽度不超过 12cm
- 标题：图 X.X 标题（宋体 11pt，居中）
- 位置：图片下方

**表格**：
- 格式：三线表（顶线、表头线、底线）
- 标题：表 X.X 标题（宋体 11pt，居中）
- 位置：表格上方
- 字体：宋体 10-11pt

**代码**：
- 格式：代码块，使用等宽字体
- 标题：代码 X.X 标题（宋体 11pt）
- 位置：代码上方
- 字体：Courier New 10pt

### 3.4 引用规范

**文献引用**：
```
根据文献 [1] 的研究，...
如 Smith 等人所述 [2]，...
```

**脚注**：
```
使用脚注进行补充说明¹
¹ 脚注内容
```

## 四、常见错误

| 错误 | 正确做法 |
|------|---------|
| 标题不统一 | 使用统一的字体、字号、加粗 |
| 行距不一致 | 全文使用 1.5 倍行距 |
| 图表无标题 | 所有图表都要有编号和标题 |
| 代码格式混乱 | 使用代码块，统一字体和字号 |
| 参考文献格式不统一 | 按照规范格式编写所有文献 |
| 页码不连续 | 从绪论开始编号，连续编号 |
| 目录不自动更新 | 使用 Word 的自动目录功能 |

## 五、Word 文档生成建议

### 5.1 使用样式

在 Word 中定义样式，确保格式一致：
- 定义"标题 1"、"标题 2"、"标题 3"样式
- 定义"正文"样式
- 定义"代码"样式
- 定义"图表标题"样式

### 5.2 自动生成目录

1. 在目录位置插入"目录"
2. 选择"自动目录"
3. 设置为"从大纲级别 1-3"

### 5.3 页码设置

1. 在绪论前插入"分页符"
2. 在绪论页脚插入页码
3. 设置为"从当前位置开始"

### 5.4 页眉页脚

- 页眉：论文标题（宋体 10pt）
- 页脚：页码（宋体 10pt，居中）

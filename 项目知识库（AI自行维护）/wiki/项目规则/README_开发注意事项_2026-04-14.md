## 重要原则

### 1. 统一使用 GF 框架

所有资源加载、UI 管理、Entity 管理等必须通过 GF 框架的组件进行，禁止直接使用 Unity 原生 API。

### 2. 统一使用 DebugEx 输出日志

禁止使用原生 `Debug.Log`，统一使用 `DebugEx` 工具类，并且必须传入脚本名称作为模块标签。

### 3. 遵循命名规范

- 类名、方法名、属性名：PascalCase
- 私有字段：m_camelCase
- 局部变量、参数：camelCase
- 常量：UPPER_CASE

### 4. 代码组织规范

使用 `#region` 组织代码块，按照推荐的顺序排列：
1. 常量
2. 字段
3. 属性
4. Unity 生命周期方法
5. 公共方法
6. 私有方法

### 5. 性能优化

- 避免在 Update 中进行不必要的计算
- 使用对象池复用对象
- 缓存组件引用
- 减少 GC 分配

### 6. 架构设计

- 单一职责原则
- 低耦合高内聚
- 使用事件系统解耦
- 使用管理器模式

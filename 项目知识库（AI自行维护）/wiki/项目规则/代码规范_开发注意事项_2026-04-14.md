# 代码规范

> **最后更新**: 2026-03-23
> **状态**: 有效
> **适用范围**: 所有 C# 代码

---

## 概述

本文档定义了项目的代码规范，包括命名规范、代码组织、日志规范、架构设计等。所有开发人员必须遵循这些规范，以确保代码的一致性、可读性和可维护性。

**核心原则**：
- 一致性 - 所有代码风格统一
- 可读性 - 代码易于理解和维护
- 可维护性 - 易于查找和修改

## 命名规范

### 1. 类名

- **规则**：PascalCase（帕斯卡命名法）
- **示例**：`PlayerController`、`EnemyAI`、`ItemManager`

```csharp
// ✅ 正确
public class PlayerController : MonoBehaviour { }
public class EnemyAI : MonoBehaviour { }

// ❌ 错误
public class playerController : MonoBehaviour { }
public class enemy_ai : MonoBehaviour { }
```

### 2. 方法名

- **规则**：PascalCase
- **示例**：`Initialize()`、`OnPlayerDeath()`、`GetItemById()`

```csharp
// ✅ 正确
public void Initialize() { }
private void OnPlayerDeath() { }
public Item GetItemById(int id) { }

// ❌ 错误
public void initialize() { }
private void on_player_death() { }
```

### 3. 私有字段

- **规则**：m_camelCase（m_ 前缀 + 驼峰命名）
- **示例**：`m_playerLevel`、`m_moveSpeed`、`m_isAlive`

```csharp
// ✅ 正确
private int m_playerLevel;
private float m_moveSpeed;
private bool m_isAlive;

// ❌ 错误
private int playerLevel;
private float _moveSpeed;
private bool IsAlive;
```

### 4. 公共属性

- **规则**：PascalCase
- **示例**：`PlayerLevel`、`MoveSpeed`、`IsAlive`

```csharp
// ✅ 正确
public int PlayerLevel { get; private set; }
public float MoveSpeed { get; set; }
public bool IsAlive { get; private set; }
```


### 5. 局部变量和参数

- **规则**：camelCase（驼峰命名法）
- **示例**：`playerLevel`、`moveSpeed`、`isAlive`

```csharp
// ✅ 正确
public void SetLevel(int playerLevel)
{
    int newLevel = playerLevel + 1;
    bool isMaxLevel = newLevel >= 100;
}

// ❌ 错误
public void SetLevel(int PlayerLevel)
{
    int NewLevel = PlayerLevel + 1;
}
```

### 6. 常量

- **规则**：UPPER_CASE（全大写 + 下划线）
- **示例**：`MAX_LEVEL`、`DEFAULT_SPEED`、`PLAYER_TAG`

```csharp
// ✅ 正确
public const int MAX_LEVEL = 100;
public const float DEFAULT_SPEED = 5.0f;
public const string PLAYER_TAG = "Player";

// ❌ 错误
public const int maxLevel = 100;
public const float defaultSpeed = 5.0f;
```

### 7. 枚举

- **枚举名**：PascalCase
- **枚举值**：PascalCase

```csharp
// ✅ 正确
public enum ItemType
{
    Weapon = 1,
    Armor = 2,
    Consumable = 3
}

// ❌ 错误
public enum item_type
{
    weapon = 1,
    armor = 2
}
```

### 8. 接口

- **规则**：I + PascalCase
- **示例**：`IPlayerSkill`、`IDamageable`、`IInteractable`

```csharp
// ✅ 正确
public interface IPlayerSkill { }
public interface IDamageable { }

// ❌ 错误
public interface PlayerSkill { }
public interface Damageable { }
```

## 代码组织

### 1. 使用 #region 组织代码

```csharp
public class PlayerController : MonoBehaviour
{
    #region 常量
    
    private const float DEFAULT_SPEED = 5.0f;
    
    #endregion
    
    #region 字段
    
    [SerializeField] private float m_MoveSpeed = DEFAULT_SPEED;
    private Rigidbody2D m_Rigidbody;
    
    #endregion
    
    #region 属性
    
    public float MoveSpeed => m_MoveSpeed;
    public bool IsMoving { get; private set; }
    
    #endregion
    
    #region Unity 生命周期
    
    private void Awake()
    {
        DebugEx.Log("PlayerController", "Awake 初始化开始");
        InitializeComponents();
    }
    
    private void Start()
    {
        DebugEx.Log("PlayerController", "Start 初始化开始");
    }
    
    private void Update()
    {
        HandleMovement();
    }
    
    #endregion
    
    #region 公共方法
    
    public void SetSpeed(float speed)
    {
        m_MoveSpeed = speed;
        DebugEx.Log("PlayerController", $"移动速度设置为: {speed}");
    }
    
    #endregion
    
    #region 私有方法
    
    private void InitializeComponents()
    {
        m_Rigidbody = GetComponent<Rigidbody2D>();
        DebugEx.Success("PlayerController", "组件初始化完成");
    }
    
    private void HandleMovement()
    {
        // 移动逻辑
    }
    
    #endregion
}
```

### 2. 代码块顺序

推荐的代码块顺序：
1. 常量
2. 字段（序列化字段在前，私有字段在后）
3. 属性
4. Unity 生命周期方法
5. 公共方法
6. 私有方法
7. 事件处理方法

## 日志输出规范

### 1. 使用 DebugEx 工具类

**禁止使用原生 Debug.Log**，统一使用 `DebugEx` 工具类。

```csharp
// ❌ 禁止使用
Debug.Log("初始化完成");
Debug.LogWarning("目标丢失");
Debug.LogError("连接超时");

// ✅ 正确使用
DebugEx.Log("PlayerController", "初始化完成");
DebugEx.Warning("EnemyAI", "目标丢失");
DebugEx.Error("NetworkManager", "连接超时");
```

### 2. 日志方法

| 场景 | 方法 | 示例 |
|:---|:---|:---|
| 普通信息 | `DebugEx.Log(module, message)` | `DebugEx.Log("PlayerController", "初始化完成");` |
| 成功状态 | `DebugEx.Success(module, message)` | `DebugEx.Success("GameManager", "游戏启动成功");` |
| 警告信息 | `DebugEx.Warning(module, message)` | `DebugEx.Warning("EnemyAI", "目标丢失");` |
| 错误信息 | `DebugEx.Error(module, message)` | `DebugEx.Error("NetworkManager", "连接超时");` |
| 高亮信息 | `DebugEx.Log(module, message, color)` | `DebugEx.Log("UIManager", "打开面板", DebugEx.Color.Cyan);` |

### 3. 模块标签规范

- **必须传入脚本名称**作为模块标签
- 方便在 Console 中快速定位日志来源
- 使用脚本类名，不要使用缩写

```csharp
// ✅ 正确
DebugEx.Log("PlayerController", "玩家移动");
DebugEx.Log("EnemyAI", "敌人巡逻");

// ❌ 错误
DebugEx.Log("PC", "玩家移动");  // 缩写不清晰
DebugEx.Log("Enemy", "敌人巡逻");  // 不是完整类名
```

### 4. 日志输出时机

```csharp
public class PlayerController : MonoBehaviour
{
    private void Awake()
    {
        // ✅ 关键初始化步骤输出日志
        DebugEx.Log("PlayerController", "Awake 初始化开始");
        InitializeComponents();
        DebugEx.Success("PlayerController", "Awake 初始化完成");
    }
    
    private void Update()
    {
        // ❌ 避免在 Update 中频繁输出日志
        // DebugEx.Log("PlayerController", "Update");
        
        // ✅ 仅在关键事件时输出
        #if UNITY_EDITOR
        if (Input.GetKeyDown(KeyCode.Space))
        {
            DebugEx.Log("PlayerController", "跳跃键被按下");
        }
        #endif
    }
    
    public void TakeDamage(float damage)
    {
        // ✅ 重要逻辑输出日志
        DebugEx.Log("PlayerController", $"受到伤害: {damage}");
        
        if (m_Health <= 0)
        {
            DebugEx.Warning("PlayerController", "玩家死亡");
            OnDeath();
        }
    }
}
```

## 注释规范

### 1. 类注释

```csharp
/// <summary>
/// 玩家控制器 - 负责玩家的移动、跳跃和交互
/// </summary>
public class PlayerController : MonoBehaviour
{
}
```

### 2. 方法注释

```csharp
/// <summary>
/// 初始化玩家组件
/// </summary>
private void InitializeComponents()
{
    m_Rigidbody = GetComponent<Rigidbody2D>();
    DebugEx.Success("PlayerController", "组件初始化完成");
}

/// <summary>
/// 设置玩家移动速度
/// </summary>
/// <param name="speed">新的移动速度</param>
public void SetSpeed(float speed)
{
    m_MoveSpeed = speed;
    DebugEx.Log("PlayerController", $"移动速度设置为: {speed}");
}

/// <summary>
/// 玩家受到伤害
/// </summary>
/// <param name="damage">伤害值</param>
/// <returns>是否死亡</returns>
public bool TakeDamage(float damage)
{
    m_Health -= damage;
    DebugEx.Log("PlayerController", $"受到伤害: {damage}, 剩余生命: {m_Health}");
    return m_Health <= 0;
}
```

### 3. 复杂逻辑注释

```csharp
private void HandleCombat()
{
    // 检查是否有敌人在攻击范围内
    Collider2D[] enemies = Physics2D.OverlapCircleAll(transform.position, m_AttackRange);
    
    foreach (var enemy in enemies)
    {
        // 只攻击敌人标签的对象
        if (enemy.CompareTag("Enemy"))
        {
            // 计算伤害（基础伤害 * 攻击力加成）
            float damage = m_BaseDamage * m_AttackMultiplier;
            
            // 应用伤害
            enemy.GetComponent<EnemyHealth>()?.TakeDamage(damage);
            
            DebugEx.Log("PlayerController", $"攻击敌人，造成伤害: {damage}");
        }
    }
}
```

## 性能优化规范

### 1. 避免在 Update 中进行不必要的计算

```csharp
// ❌ 错误：每帧查找组件
private void Update()
{
    var rigidbody = GetComponent<Rigidbody2D>();
    rigidbody.velocity = Vector2.right * m_MoveSpeed;
}

// ✅ 正确：缓存组件引用
private Rigidbody2D m_Rigidbody;

private void Awake()
{
    m_Rigidbody = GetComponent<Rigidbody2D>();
}

private void Update()
{
    m_Rigidbody.velocity = Vector2.right * m_MoveSpeed;
}
```

### 2. 使用对象池

```csharp
// ❌ 错误：频繁创建销毁对象
private void SpawnBullet()
{
    GameObject bullet = Instantiate(m_BulletPrefab);
    Destroy(bullet, 3f);
}

// ✅ 正确：使用对象池
private void SpawnBullet()
{
    var bullet = m_BulletPool.Spawn();
    // 使用完毕后回收
    m_BulletPool.Unspawn(bullet);
}
```

### 3. 减少 GC 分配

```csharp
// ❌ 错误：频繁创建临时对象
private void Update()
{
    string message = "Player position: " + transform.position.ToString();
    DebugEx.Log("PlayerController", message);
}

// ✅ 正确：避免频繁字符串拼接
private void OnPositionChanged()
{
    DebugEx.Log("PlayerController", $"Player position: {transform.position}");
}
```

## 架构设计规范

### 1. 单一职责原则

每个类只负责一个功能：

```csharp
// ✅ 正确：职责分离
public class PlayerMovement : MonoBehaviour
{
    // 只负责移动
}

public class PlayerCombat : MonoBehaviour
{
    // 只负责战斗
}

public class PlayerHealth : MonoBehaviour
{
    // 只负责生命值管理
}
```

### 2. 低耦合高内聚

使用事件系统解耦：

```csharp
// ✅ 正确：使用事件通信
public class PlayerHealth : MonoBehaviour
{
    public event Action OnDeath;
    
    public void TakeDamage(float damage)
    {
        m_Health -= damage;
        if (m_Health <= 0)
        {
            OnDeath?.Invoke();
        }
    }
}

public class PlayerController : MonoBehaviour
{
    private void Start()
    {
        m_PlayerHealth.OnDeath += OnPlayerDeath;
    }
    
    private void OnPlayerDeath()
    {
        DebugEx.Warning("PlayerController", "玩家死亡");
        // 处理死亡逻辑
    }
}
```

### 3. 使用管理器模式

```csharp
/// <summary>
/// 玩家管理器 - 单例模式
/// </summary>
public class PlayerManager : MonoBehaviour
{
    #region 单例
    
    private static PlayerManager s_Instance;
    
    public static PlayerManager Instance
    {
        get
        {
            if (s_Instance == null)
            {
                s_Instance = FindObjectOfType<PlayerManager>();
                if (s_Instance == null)
                {
                    DebugEx.Error("PlayerManager", "场景中未找到 PlayerManager");
                }
            }
            return s_Instance;
        }
    }
    
    #endregion
    
    private void Awake()
    {
        if (s_Instance != null && s_Instance != this)
        {
            Destroy(gameObject);
            return;
        }
        
        s_Instance = this;
        DontDestroyOnLoad(gameObject);
        
        DebugEx.Success("PlayerManager", "PlayerManager 初始化完成");
    }
}
```

# 资源加载规则

> **最后更新**: 2026-04-17
> **状态**: 有效
> **资源系统**: GameFramework AssetBundle

---

## 概述

本项目基于 GameFramework (GF) 框架，使用 AssetBundle 进行资源管理和加载。项目已经封装了统一的资源加载方式，所有资源加载必须通过 `ResourceExtension` 扩展类进行，禁止直接使用 Unity 原生的 Resources 或 AssetDatabase API。

**核心原则**：
- 所有资源路径统一在 `ResourceConfigTable` 配置表中管理
- 其他配置表中不应填入资源路径，而是填入资源 ID
- 使用 `ResourceIds` 静态类中定义的常量引用资源
- 通过 `ResourceExtension` 扩展类加载资源

---

## 资源管理架构

### 三层架构

```
ResourceIds (资源ID常量)
    ↓
ResourceConfigTable (资源配置表)
    ↓
ResourceExtension (资源加载扩展)
```

### 1. ResourceIds - 资源ID常量类

定义所有资源的 ID 常量，方便代码中引用。

**文件位置**：`Assets/AAAGame/Scripts/Config/ResourceIds.cs`

**ID 分段规则**：
- 1000-1099：UI 背景图片
- 1100-1199：物品图标
- 1200-1299：卡牌图标
- 1300-1399：棋子图标
- 1400-1499：Buff 图标
- 1500-1599：技能图标
- 1600-1699：装备图标
- 1700-1799：召唤师技能图标
- 1800-1899：棋子技能图标
- 2000-2099：UI 预制体
- 2100-2199：角色/NPC 预制体
- 2200-2299：道具预制体
- 2300-2399：神话角色预制体
- 2400-2499：特效预制体
- 2500-2599：子弹预制体
- 2900-2999：战斗场景预制体
- 6000-6999：ScriptableObject 资源

**使用示例**：
```csharp
// ✅ 正确：使用 ResourceIds 常量
int iconId = ResourceIds.ICON_GOLD;
int prefabId = ResourceIds.PREFAB_CHAR_HOUYI;

// ❌ 错误：硬编码资源 ID
int iconId = 1101;
```

### 2. ResourceConfigTable - 资源配置表

管理所有资源的路径和类型信息。

**文件位置**：
- Excel：`AAAGameData/DataTables/ResourceConfigTable.xlsx`
- TXT：`Assets/AAAGame/DataTable/ResourceConfigTable.txt`
- C#：`Assets/AAAGame/Scripts/DataTable/ResourceConfigTable.cs`

**配置表字段**：
- `Id`：资源 ID（对应 ResourceIds 中的常量）
- `Type`：资源类型（1=Sprite, 2=Prefab, 3=Effect, 4=Material, 5=Texture, 6=ScriptableObject）
- `Path`：资源相对路径（相对于资源类型目录）
- `Name`：资源名称（中文说明）

**配置示例**：
```
| ID   | Type | Path                              | Name           |
|------|------|-----------------------------------|----------------|
| 1101 | 1    | UI/Items/Gold.png                 | 金币图标       |
| 2301 | 2    | 角色/友方神话角色/后羿            | 棋子-后羿预制体 |
| 6001 | 6    | Skills/SkillParamRegistry         | 技能参数注册表  |
```

### 3. ResourceExtension - 资源加载扩展类

封装了所有资源加载方法，基于配置表动态加载资源。

**文件位置**：`Assets/AAAGame/Scripts/Extension/ResourceExtension.cs`

**资源类型枚举**：
```csharp
public enum ResourceType
{
    Sprite = 1,              // 图片（需要扩展名：.png, .jpg）
    Prefab = 2,              // 预制体
    Effect = 3,              // 特效
    Material = 4,            // 材质
    Texture = 5,             // 纹理（需要扩展名：.png, .jpg, .tga）
    ScriptableObject = 6,    // ScriptableObject 配置
}
```

**资源类型对应的目录**：
- Sprite (1)：`Assets/AAAGame/Sprites/`
- Prefab (2)：`Assets/AAAGame/Prefabs/`
- Effect (3)：`Assets/AAAGame/Entities/`
- Material (4)：`Assets/AAAGame/Materials/`
- Texture (5)：`Assets/AAAGame/Textures/`
- ScriptableObject (6)：`Assets/AAAGame/Config/`

## 资源加载方式

### 方式一：基于配置表 ID 加载（推荐）

这是项目的标准资源加载方式，所有资源都应该通过配置表 ID 加载。

#### 1. UniTask 异步加载（推荐）

```csharp
using GameExtension;
using Cysharp.Threading.Tasks;

public class ItemManager : MonoBehaviour
{
    private async void LoadItemIcon()
    {
        DebugEx.Log("ItemManager", "开始加载金币图标");
        
        // 加载 Sprite
        Sprite goldIcon = await ResourceExtension.LoadSpriteAsync(ResourceIds.ICON_GOLD);
        if (goldIcon != null)
        {
            DebugEx.Success("ItemManager", "金币图标加载成功");
            // 使用图标
            m_IconImage.sprite = goldIcon;
        }
        else
        {
            DebugEx.Error("ItemManager", "金币图标加载失败");
        }
    }
    
    private async void LoadCharacterPrefab()
    {
        DebugEx.Log("ItemManager", "开始加载后羿预制体");
        
        // 加载 GameObject
        GameObject houyiPrefab = await ResourceExtension.LoadPrefabAsync(ResourceIds.PREFAB_CHAR_HOUYI);
        if (houyiPrefab != null)
        {
            DebugEx.Success("ItemManager", "后羿预制体加载成功");
            // 实例化
            GameObject instance = Instantiate(houyiPrefab, transform);
        }
    }
    
    private async void LoadMaterial()
    {
        // 加载 Material
        Material mat = await ResourceExtension.LoadMaterialAsync(ResourceIds.SOME_MATERIAL_ID);
    }
    
    private async void LoadTexture()
    {
        // 加载 Texture
        Texture tex = await ResourceExtension.LoadTextureAsync(ResourceIds.SOME_TEXTURE_ID);
    }
    
    private async void LoadScriptableObject()
    {
        // 加载 ScriptableObject
        var registry = await ResourceExtension.LoadScriptableObjectAsync<SkillParamRegistry>(
            ResourceIds.SO_SKILL_PARAM_REGISTRY
        );
    }
}
```

## 配置表中使用资源 ID

### 正确做法

在其他配置表中，不应该填入资源路径，而是填入资源 ID。

**示例：ItemTable 配置表**

```
| ID | Name   | IconId | PrefabId |
|----|--------|--------|----------|
| 1  | 金币   | 1101   | 2201     |
| 2  | 灵石   | 1102   | 2202     |
```

**对应的代码**：

```csharp
public class ItemManager : MonoBehaviour
{
    private async void LoadItemData(int itemId)
    {
        // 从 ItemTable 获取物品数据
        var itemTable = GF.DataTable.GetDataTable<ItemTable>();
        var itemData = itemTable.GetDataRow(itemId);
        
        if (itemData == null)
        {
            DebugEx.Error("ItemManager", $"物品数据不存在: ID={itemId}");
            return;
        }
        
        DebugEx.Log("ItemManager", $"开始加载物品: {itemData.Name}");
        
        // ✅ 正确：使用 IconId 字段（资源 ID）加载图标
        Sprite icon = await ResourceExtension.LoadSpriteAsync(itemData.IconId);
        if (icon != null)
        {
            DebugEx.Success("ItemManager", $"物品图标加载成功: {itemData.Name}");
            m_IconImage.sprite = icon;
        }
        
        // ✅ 正确：使用 PrefabId 字段（资源 ID）加载预制体
        GameObject prefab = await ResourceExtension.LoadPrefabAsync(itemData.PrefabId);
        if (prefab != null)
        {
            DebugEx.Success("ItemManager", $"物品预制体加载成功: {itemData.Name}");
        }
    }
}
```

### 错误做法

```csharp
// ❌ 错误：在配置表中直接填入资源路径
| ID | Name   | IconPath                  |
|----|--------|---------------------------|
| 1  | 金币   | UI/Items/Gold.png         |

// ❌ 错误：直接使用路径加载
string iconPath = itemData.IconPath;
GF.Resource.LoadAsset(iconPath, typeof(Sprite), callbacks);
```

## 性能优化建议

### 1. 使用对象池

对于频繁创建销毁的预制体，使用对象池复用：

```csharp
public class BulletPool : MonoBehaviour
{
    private Queue<GameObject> m_Pool = new Queue<GameObject>();
    private GameObject m_BulletPrefab;
    
    private async void Initialize()
    {
        // 加载子弹预制体
        m_BulletPrefab = await ResourceExtension.LoadPrefabAsync(ResourceIds.PREFAB_BULLET);
        
        // 预创建对象池
        for (int i = 0; i < 20; i++)
        {
            GameObject bullet = Instantiate(m_BulletPrefab);
            bullet.SetActive(false);
            m_Pool.Enqueue(bullet);
        }
        
        DebugEx.Success("BulletPool", "对象池初始化完成");
    }
    
    public GameObject Spawn()
    {
        if (m_Pool.Count > 0)
        {
            GameObject bullet = m_Pool.Dequeue();
            bullet.SetActive(true);
            return bullet;
        }
        
        return Instantiate(m_BulletPrefab);
    }
    
    public void Recycle(GameObject bullet)
    {
        bullet.SetActive(false);
        m_Pool.Enqueue(bullet);
    }
}
```

### 2. 批量加载

使用批量加载方法减少异步操作次数：

```csharp
// ✅ 推荐：批量加载
int[] iconIds = new int[] { 1101, 1102, 1103 };
Sprite[] icons = await ResourceExtension.LoadSpritesAsync(iconIds);

// ❌ 不推荐：逐个加载
Sprite icon1 = await ResourceExtension.LoadSpriteAsync(1101);
Sprite icon2 = await ResourceExtension.LoadSpriteAsync(1102);
Sprite icon3 = await ResourceExtension.LoadSpriteAsync(1103);
```

### 3. 避免重复加载

```csharp
// ❌ 错误：每次都加载
private async void Update()
{
    var sprite = await ResourceExtension.LoadSpriteAsync(ResourceIds.ICON_GOLD);

}

{
    // 使用缓存的资源
    if (m_CachedSprite != null)
    {
        // ...
    }
}
```

## 常见问题

### Q1: 资源加载失败怎么办？

**排查步骤**：
1. 检查 ResourceIds 中的常量是否正确
2. 确认 ResourceConfigTable 中是否有对应的配置
3. 检查资源文件是否存在于指定路径
4. 查看 Console 错误日志

**调试方法**：
```csharp
// 检查资源配置是否存在
bool exists = ResourceExtension.HasResourceConfig(ResourceIds.ICON_GOLD);
DebugEx.Log("Debug", $"资源配置存在: {exists}");

// 获取资源路径
string path = ResourceExtension.GetResourcePath(ResourceIds.ICON_GOLD);
DebugEx.Log("Debug", $"资源路径: {path}");

// 获取资源类型
ResourceType? type = ResourceExtension.GetResourceType(ResourceIds.ICON_GOLD);
DebugEx.Log("Debug", $"资源类型: {type}");
```

### Q2: ResourceConfigTable 数据表未加载？

**解决方案**：

确保在 `AppConfigs` ScriptableObject 中配置了 ResourceConfigTable：

```csharp
[SerializeField]
private string[] m_DataTableNames = new string[]
{
    "ResourceConfigTable",  // ← 必须包含
    "ItemTable",
    "EnemyTable"
};
```

框架会在 `PreloadProcedure` 中自动加载这些配置表。

### Q3: 如何在配置表中引用资源？

**正确做法**：

在配置表中使用资源 ID 字段，而不是资源路径字段。

```
// ✅ 正确：ItemTable 配置
| ID | Name   | IconId | PrefabId |
|----|--------|--------|----------|
| 1  | 金币   | 1101   | 2201     |

// ❌ 错误：不要使用路径字段
| ID | Name   | IconPath              |
|----|--------|-----------------------|
| 1  | 金币   | UI/Items/Gold.png     |
```

### Q4: 资源 ID 冲突怎么办？

**解决方案**：

遵循 ResourceIds 中的 ID 分段规则，不同类型的资源使用不同的 ID 范围。

如果需要新增资源类型，在 ResourceIds 中添加新的分段：

```csharp
#region 新资源类型 (4000-4099)
public const int NEW_RESOURCE_TYPE_1 = 4001;
#endregion
```

### ❌ 禁止直接使用 GF.Resource

```csharp
// ❌ 禁止：直接使用 GF.Resource
GF.Resource.LoadAsset("UI/Items/Gold.png", typeof(Sprite), callbacks);

// ✅ 正确：使用 ResourceExtension
var sprite = await ResourceExtension.LoadSpriteAsync(ResourceIds.ICON_GOLD);
```

### ❌ 禁止硬编码资源 ID

```csharp
// ❌ 禁止：硬编码 ID
var sprite = await ResourceExtension.LoadSpriteAsync(1101);

// ✅ 正确：使用 ResourceIds 常量
var sprite = await ResourceExtension.LoadSpriteAsync(ResourceIds.ICON_GOLD);
```

### ❌ 禁止在配置表中填入资源路径

```csharp
// ❌ 禁止：在配置表中填入路径
```

## 总结

1. **统一使用 ResourceExtension**：所有资源加载必须通过 ResourceExtension 扩展类
2. **使用 ResourceIds 常量**：通过常量引用资源 ID，避免硬编码
3. **配置表管理路径**：所有资源路径在 ResourceConfigTable 中统一管理
4. **其他配置表使用 ID**：其他配置表中不填路径，只填资源 ID
5. **优先异步加载**：使用 UniTask 异步加载，避免阻塞主线程
6. **添加日志输出**：使用 DebugEx 输出关键日志，方便调试
7. **缓存频繁使用的资源**：避免重复加载，提升性能
8. **使用对象池**：频繁创建销毁的对象使用对象池复用

遵循这些规则，可以确保项目的资源管理规范、高效、易维护。

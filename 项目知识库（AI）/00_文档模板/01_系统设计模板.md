## 📌 说明

这个模板用于文档化系统设计和架构决策。使用本模板时：

1. **复制完整内容**到新文件
2. **替换所有 `<!-- 说明 -->` 部分**为实际内容
3. **删除可选部分**（标记为「可选」）
4. **在顶部添加元数据**（见下方"完整示例"）

## 📋 完整示例

以下是一个使用此模板的完整示例（简化版）：

```markdown
# Buff 应用系统设计

> **最后更新**: 2026-03-23
> **状态**: 有效
> **版本**: Phase 16.1

---

## 问题陈述

目前 Buff 应用逻辑分散在多个地方，缺乏统一的应用流程，导致维护困难。

## 解决方案概述

创建 BuffApplyHelper 统一处理 Buff 应用，支持单体和全体应用。

## 系统架构

### 核心组件

| 组件 | 职责 |
|-----|------|
| BuffApplyHelper | 统一处理 Buff 应用逻辑 |
| BuffTable | Buff 配置数据 |
| CombatSystem | 战斗系统调用 Buff 应用 |

## 设计细节

### 1. Buff 应用分离

**设计目标**: 支持单体和全体应用

**实现方式**:
```csharp
public class BuffApplyHelper
{
    public static void ApplyBuff(int buffId, Entity target, bool isGroupTarget)
    {
        if (isGroupTarget)
            ApplyToGroup(buffId, target);
        else
            ApplyToSingle(buffId, target);
    }
}
```

## 实施步骤

1. 创建 BuffApplyHelper 类
2. 迁移现有 Buff 应用逻辑
3. 更新各系统调用接口
4. 测试全体应用场景
```

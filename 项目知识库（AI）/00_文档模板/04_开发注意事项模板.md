## 概述

UI 关闭机制涉及多个步骤，包括隐藏、销毁、资源释放等。正确处理关闭机制对防止内存泄漏很重要。

## 常见问题

### 问题 1: UI 关闭后仍然占用内存

**描述**: UI 关闭后，其资源没有被释放

**症状**: 内存占用不断增加，关闭多个 UI 后明显

**根本原因**:
- 没有调用 OnDispose() 方法
- 监听的事件没有取消订阅

**解决方案**:
在 UIFormBase.Close() 中必须调用 OnDispose()，并在其中取消所有事件订阅。

**代码示例**:
```csharp
// ❌ 错误做法
public override void OnClose()
{
    // 只隐藏了 UI，但没有释放资源
    gameObject.SetActive(false);
}

// ✅ 正确做法
public override void OnClose()
{
    // 1. 取消事件订阅
    EventManager.Instance.Unsubscribe(...);

    // 2. 释放资源
    OnDispose();

    // 3. 隐藏 UI
    gameObject.SetActive(false);
}
```

## 最佳实践

### 实践 1: 完整的 UI 生命周期管理

**说明**: 确保 UI 的每个生命周期方法都实现正确

**代码示例**:
```csharp
public class MyUIForm : UIFormBase
{
    public override void OnInitialize()
    {
        // 初始化 UI 结构
    }

    public override void OnOpen(UIParams params)
    {
        // 订阅事件，更新显示
        EventManager.Instance.Subscribe(...);
    }

    public override void OnClose()
    {
        // 取消订阅，释放资源
        OnDispose();
    }
}
```

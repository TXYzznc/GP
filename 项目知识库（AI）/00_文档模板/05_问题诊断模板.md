## 📌 说明

这个模板用于记录 bug 分析、问题修复和性能优化工作。使用本模板时：

1. **复制完整内容**到新文件
2. **替换所有 `<!-- 说明 -->` 部分**为实际内容
3. **删除不适用的部分**（标记为「可选」）
4. **在顶部添加元数据**（见下方"完整示例"）

本模板强调**问题诊断的全过程**：现象 → 根因 → 修复 → 验证 → 预防。

## 📋 完整示例

以下是一个使用此模板的完整示例：

```markdown
# EnterCombatTip 延迟问题修复

> **最后更新**: 2026-03-16
> **状态**: 有效
> **严重程度**: 中等
> **修复状态**: 已修复

---

## 问题概述

进入战斗提示界面时，显示出现延迟，用户需要等待 1-2 秒才能看到界面。

## 症状描述

### 表现形式

- 进入战斗提示界面时出现黑屏
- 需要等待 1-2 秒才能看到界面
- 界面显示不稳定，有时快有时慢

### 复现步骤

1. 启动游戏
2. 移动角色靠近敌人，触发战斗
3. 观察战斗提示界面显示速度

**预期行为**: 战斗提示界面应该立即显示

**实际行为**: 战斗提示界面延迟 1-2 秒才显示

### 影响范围

- 每次进入战斗都会发生
- 影响用户体验

---

## 根本原因分析

### 初步诊断

可能是因为：
- UI 预制体加载缓慢
- 数据初始化缓慢
- 异步操作阻塞

### 深入调查

添加 Profiler 分析，发现：
- ShowCombatInteractUI() 中有 await UniTask.Yield()
- 该 yield 没有正确等待，导致主流程继续执行
- ContinueCombatPreparationInitAsync() 立即返回，没有等待前面的异步操作

### 根本原因

ShowEnemyInitiativeBuffIfNeeded() 被声明为 `async void`，导致：
1. 方法立即返回
2. 后续的 await 在后台执行
3. 主流程不等待，导致时序混乱

---

## 修复方案

### 修复思路

将 `async void` 改为 `async UniTask`，让主流程能正确等待异步操作完成。

### 代码修改

**修改文件**: CombatPreparationState.cs

```csharp
// ❌ 修改前
private async void ShowEnemyInitiativeBuffIfNeeded()
{
    // ...
    await UniTask.Yield();
    // ...
}

// ✅ 修改后
private async UniTask ShowEnemyInitiativeBuffIfNeededAsync()
{
    // ...
    await UniTask.Yield();
    // ...
}

// 调用方式也要改
await ShowEnemyInitiativeBuffIfNeededAsync();
```

### 代码审查重点

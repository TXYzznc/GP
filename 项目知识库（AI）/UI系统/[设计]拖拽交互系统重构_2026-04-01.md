# 拖拽交互系统重构 - 新架构

> **最后更新**: 2026-04-01
> **状态**: 有效
> **版本**: 1.0

---

**重构日期**: 2026-04-01

## 核心设计理念

### 问题诊断
旧架构的问题：
- ❌ 6 个不同的 `Handle*` 方法，逻辑重复，代码冗余
- ❌ 新增容器需要 N-1 个新方法（容器爆炸）
- ❌ 容器语义混乱（FastBarManager 独立存储 vs m_HotbarBoundSlots 绑定）
- ❌ 事件驱动导致时序问题和重复刷新
- ❌ 拖拽逻辑散落在各处，难以维护

### 新设计原则
- **格子是一等公民**：每个 `InventorySlotUI` 都知道自己属于哪个容器
- **容器自主交互**：容器定义自己能与哪些容器交互，如何交互
- **统一拖拽接口**：所有拖拽都通过 `ISlotContainer.TryMoveToContainer()` 执行
- **规则即代码**：交互规则清晰地写在各容器的实现中

## 架构组件

### 1. ISlotContainer 接口

```csharp
public interface ISlotContainer
{
    SlotContainerType ContainerType { get; }
    InventorySlot GetSlot(int slotIndex);
    bool TryMoveToContainer(int fromSlotIndex, ISlotContainer targetContainer, int targetSlotIndex);
    bool CanInteractWith(SlotContainerType otherContainerType);
}
```

**含义**：容器必须实现的合约
- `ContainerType` - 容器身份标识
- `GetSlot()` - 获取格子数据
- `TryMoveToContainer()` - 对目标容器执行一次移动操作
- `CanInteractWith()` - 定义交互规则

### 2. SlotContainerBase 基类

    public abstract bool ExecuteMove(int fromSlotIndex, ISlotContainer targetContainer, int targetSlotIndex);

    public bool TryMoveToContainer(int fromSlotIndex, ISlotContainer targetContainer, int targetSlotIndex)

    }

### 3. 具体容器实现

#### InventorySlotContainer - 背包容器
```csharp
public class InventorySlotContainer : SlotContainerBase
{
    public override bool CanInteractWith(SlotContainerType otherContainerType)
    {
        // 背包可以与背包、仓库、快捷栏交互
        return otherContainerType == SlotContainerType.Inventory ||
               otherContainerType == SlotContainerType.Warehouse ||
               otherContainerType == SlotContainerType.FastBar;
    }

    protected override bool ExecuteMove(int fromSlotIndex, ISlotContainer targetContainer, int targetSlotIndex)
    {
        return targetContainer switch
        {
            InventorySlotContainer inv => MoveToInventory(inv, fromSlotIndex, targetSlotIndex),
            WarehouseSlotContainer wh => MoveToWarehouse(wh, fromSlotIndex, targetSlotIndex),
            FastBarSlotContainer fb => MoveToFastBar(fb, fromSlotIndex, targetSlotIndex),
            _ => false
        };
    }
}
```

## 交互规则（清晰定义）

### 支持的交互组合

| 源 | 目标 | 操作 | 规则 |
|-------|-------|------|------|
| **Inventory** | Inventory | 移动/交换 | 背包内自由移动 |
| **Inventory** | Warehouse | 存入 | 存到仓库指定格子 |
| **Inventory** | FastBar | 存入 | 存到快捷栏指定格子 |
| **Warehouse** | Inventory | 取出 | 3-case：空/堆叠/交换 |
| **Warehouse** | Warehouse | 交换 | 仓库内交换两个格子 |
| **FastBar** | Inventory | 移出 | 3-case：空/堆叠/交换 |
| **FastBar** | FastBar | 交换 | 快捷栏内交换 |
| **Equip** | Equip | （暂不支持） | 后续实现 |

### 不支持的组合
- Warehouse → FastBar（仓库不能直接到快捷栏）
- FastBar → Warehouse（快捷栏不能直接到仓库）
- Equip → * （装备栏暂不支持拖拽）
- * → Equip （暂不支持拖到装备栏）

## 代码对比：新 vs 旧

### 旧架构（问题代码）
```csharp
private void HandleDrop(InventorySlotUI src, InventorySlotUI dst)
{
    var srcType = src.ContainerType;
    var dstType = dst.ContainerType;

    if (srcType == Inventory && dstType == Inventory)
        HandleInventoryToInventory(src, dst);
    else if (srcType == Inventory && dstType == Warehouse)
        HandleInventoryToWarehouse(src, dst);
    else if (srcType == Inventory && dstType == FastBar)
        HandleInventoryToFastBar(src, dst);
    // ... 还有6个分支
    else
        Debug.Warning("不支持...");
}

// 每个 Handle* 方法 20-50 行代码，逻辑重复
private void HandleInventoryToWarehouse(...) { /* 25 行代码 */ }
private void HandleInventoryToFastBar(...) { /* 25 行代码 */ }
private void HandleWarehouseToInventory(...) { /* 50 行代码 */ }
// ... 等等
```

### 新架构（优化代码）
```csharp
private void HandleDrop(InventorySlotUI src, InventorySlotUI dst)
{
    bool success = src.SlotContainer.TryMoveToContainer(
        src.SlotIndex,
        dst.SlotContainer,
        dst.SlotIndex
    );
}
```

全部逻辑内聚在各容器的 `ExecuteMove()` 中，清晰且易于扩展。

## 扩展性

### 添加新容器（e.g., 装备栏）

只需：
1. 创建 `EquipSlotContainer : SlotContainerBase`
2. 实现 `CanInteractWith()` 定义交互规则
3. 实现 `ExecuteMove()` 定义移动逻辑
4. InventoryUI 在初始化装备槽时调用 `SetSlotContainer(equipContainer)`

拖拽系统自动支持 ✅

## 总结

**新架构的关键特性**：
- ✅ **一个方法统治所有**：HandleDrop 只有 3 行
- ✅ **容器自主**：每个容器定义自己的规则
- ✅ **易于扩展**：新容器无需改动拖拽系统
- ✅ **逻辑清晰**：规则写在代码里，一目了然
- ✅ **类型安全**：通过 switch 表达式和接口

### 2. 配置表字段映射
配置表字段直接对应类属性，无需额外转换：
- `config.FontSize` - 字体大小
- `config.TextColor` - 文字颜色（字符串格式，需解析）
- `config.OutlineColor` - 描边颜色（字符串格式，需解析）
- `config.IsGradient` - 是否使用渐变
- `config.GradientStartColor` / `config.GradientEndColor` - 渐变颜色
- `config.AnimationType` - 动画类型ID
- `config.ScaleMultiplier` - 缩放倍数
- `config.Duration` - 持续时间
- `config.MoveDistance` - 移动距离

### 3. 颜色解析处理
- 配置表中的颜色字段为字符串格式（如 "#FF6B35"）
- 添加了 `ParseColor()` 方法处理颜色字符串解析
- 使用 `ColorUtility.TryParseHtmlString()` 进行安全解析
- 解析失败时使用默认白色并输出警告日志

### 4. 伤害类型枚举
保留了伤害类型枚举，避免硬编码：
```csharp
public enum DamageType
{
    普通伤害 = 1,
    暴击伤害 = 2,
    法术伤害 = 3,
    // ... 其他类型
}
```

### 5. 动画库集成
- 保持了 `DamageFloatingTextAnimationLibrary` 动画库
- 支持19种不同的动画效果
- 通过配置表的 `AnimationType` 字段选择对应动画

## 使用方式

### 显示飘字
```csharp
// 使用枚举方式（推荐）
DamageFloatingTextManager.Instance.ShowDamageText(
    DamageFloatingTextManager.DamageType.暴击伤害, 
    1250f, 
    targetPosition
);

// 使用ID方式
DamageFloatingTextManager.Instance.ShowDamageText(
    2, // 暴击伤害ID
    "1250", 
    targetPosition
);
```

### 获取配置信息
```csharp
// 获取配置表数据
var config = DamageFloatingTextManager.Instance.GetConfig(
    DamageFloatingTextManager.DamageType.暴击伤害
);

if (config != null)
{
    Debug.Log($"伤害类型: {config.Name}, 字体大小: {config.FontSize}");
}
```

## 注意事项

### 1. 配置表加载
- 确保配置表已正确加载到项目中
- 配置表类由工具自动生成，请勿手动修改
- 配置表数据通过项目的配置表系统统一管理

### 2. 颜色格式
- 配置表中的颜色必须使用HTML格式（如 "#FF6B35"）
- 不支持颜色名称（如 "red"）
- 解析失败时会使用默认白色并输出警告

### 3. 动画类型
- 动画类型ID必须在1-19范围内
- 超出范围的动画类型会使用默认的基础上升动画
- 新增动画类型需要在动画库中实现对应方法

### 4. 性能考虑
- 使用对象池管理飘字对象，避免频繁创建销毁
- 配置表数据在运行时缓存，避免重复查询
- 动画使用异步方式，不会阻塞主线程

## 扩展指南

### 添加新的伤害类型
1. 在配置表中添加新行数据
2. 在 `DamageType` 枚举中添加对应项
3. 可选：在动画库中添加专属动画效果

### 添加新的动画效果
1. 在 `AnimationType` 枚举中定义新类型
2. 在 `DamageFloatingTextAnimationLibrary` 中实现动画方法
3. 在配置表中指定使用新动画类型

## 常见问题

### Q: 配置表数据没有生效？
A: 检查配置表是否正确加载，确认配置表文件格式正确

### Q: 颜色显示不正确？
A: 检查配置表中的颜色格式是否为HTML格式（#RRGGBB）

### Q: 动画效果异常？
A: 确认动画类型ID在有效范围内，检查动画库中是否有对应实现

### Q: 飘字不显示？
A: 检查预制体和父节点是否正确设置，确认配置表中对应ID的数据存在

# 敌人AI状态机转换图

## Graphviz DOT 格式（优化正交线条版）

**适用于**: https://edotor.net/

```dot
digraph AI_StateMachine {
    rankdir=TB;              // 上到下布局（自然的状态流）
    splines=orthogonal;      // 强制正交线条（90度转向）
    nodesep=1.0;             // 节点间距（优化可读性）
    ranksep=1.2;             // 层级间距（充分的垂直空间）
    layout=dot;
    
    // 全局节点样式
    node [shape=ellipse, style="filled", fontname="微软雅黑", fontsize=10];
    edge [arrowsize=1.2, penwidth=1.5];
    
    // 开始和结束节点
    START [shape=point, label="", fillcolor="#000"];
    END [shape=point, label="", fillcolor="#000"];
    
    // ===== 状态节点 =====
    // 基础状态（蓝色）
    Idle [label="⏸️ Idle\n(休息)", fillcolor="#bbdefb", fontcolor="#000"];
    Patrol [label="🚶 Patrol\n(巡逻)", fillcolor="#bbdefb", fontcolor="#000"];
    
    // 警戒状态（黄色）
    Alert [label="⚠️ Alert\n(警戒)", fillcolor="#fff9c4", fontcolor="#000"];
    
    // 追击状态（橙色）
    Chase [label="🏃 Chase\n(追击)", fillcolor="#ffe0b2", fontcolor="#000"];
    
    // 战斗状态（红色）
    Combat [label="⚔️ Combat\n(战斗)", fillcolor="#ffccbc", fontcolor="#000"];
    
    // ===== 状态转移边 =====
    START -> Idle [label="初始化", fontsize=9];
    
    // Idle 转移
    Idle -> Patrol [label="休息结束", fontsize=9];
    Idle -> Alert [label="检测玩家\n进入范围", fontsize=9];
    
    // Patrol 转移
    Patrol -> Idle [label="随机休息\n(概率)", fontsize=9];
    Patrol -> Alert [label="检测玩家\n进入范围", fontsize=9];
    
    // Alert 转移
    Alert -> Idle [label="玩家离开\n(立即)", fontsize=9];
    Alert -> Patrol [label="警戒结束\n(超时)", fontsize=9];
    Alert -> Chase [label="玩家仍在\n范围内", fontsize=9];
    
    // Chase 转移
    Chase -> Alert [label="玩家逃离\n(距离增加)", fontsize=9];
    Chase -> Patrol [label="失败/离开", fontsize=9];
    Chase -> Combat [label="进入战斗距离\n(触发管理器)", fontsize=9];
    
    // Combat 转移
    Combat -> Patrol [label="战斗结束\n(返回巡逻)", fontsize=9];
    Combat -> END [label="敌人净化\n或Boss胜利", fontsize=9];
    
    // 自环（改进的显示）
    Idle -> Idle [label="继续等待", fontsize=8, constraint=false];
    Patrol -> Patrol [label="随机移动", fontsize=8, constraint=false];
}
```

### 📐 状态节点说明

| 状态 | 颜色 | 含义 | 备注 |
|------|------|------|------|
| **Idle** | 蓝色 | 休息待机 | 原地待机，可被偷袭 |
| **Patrol** | 蓝色 | 巡逻移动 | NavMesh导航，配置范围内随机移动 |
| **Alert** | 黄色 | 警戒发现 | 显示警觉度UI，触发广播协作 |
| **Chase** | 橙色 | 追击追逐 | 追击玩家，接近触发战斗 |
| **Combat** | 红色 | 战斗中 | 战斗系统接管，敌人数据隔离 |

### 📐 优化参数说明

| 参数 | 值 | 作用 |
|------|-----|------|
| `rankdir` | TB | 上到下布局（自然的状态流）|
| `splines` | orthogonal | **强制正交线条**（90°转向） |
| `nodesep` | 1.0 | 节点间距（可读性优先） |
| `ranksep` | 1.2 | 层级间距（充分垂直空间） |

### ✨ 优化特性

1. ✅ **强制正交线条** - 所有箭头完全直角
2. ✅ **清晰的状态流** - 从上到下的自然流向
3. ✅ **颜色编码** - 不同状态类型使用不同颜色
4. ✅ **详细标签** - 边标签解释转移条件
5. ✅ **空间优化** - 合理的间距最大化可读性

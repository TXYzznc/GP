## 检测方式详细说明

### 1. InstantDetector（瞬发检测）
**场景**: 直接攻击锁定目标，如后羿普攻
```
特点：
- 无需物理碰撞
- 直接命中LockedTarget
- 最快速度，用于即时反应技能
- 不需要额外GameObject

流程：
SkillExecutor → InstantDetector.Execute()
    → 直接返回LockedTarget
    → ApplyDamage()
```

### 2. MeleeDetector（近战检测）
**场景**: 武器挥砍，检测范围内敌人，如近战棋子普攻
```
特点：
- 通过武器Collider的OnTriggerEnter/Stay触发
- 检测碰撞框范围内的敌人
- 支持多目标检测
- 需要武器Collider配置

流程：
OnTriggerEnter(enemy)
    → MeleeDetector.DetectHit()
    → 检查敌人层级和状态
    → ApplyDamage()
```

### 3. ProjectileDetector（投射物检测）
**场景**: 弓箭射击，投射物到达目标时检测，如弓箭手普攻
```
特点：
- 创建Projectile实例飞向目标
- 通过Projectile.OnTriggerEnter检测碰撞
- 支持穿透或单体命中
- 需要配置投射物预制体

流程：
CreateProjectile(target)
    → Projectile.Initialize()
    → Projectile.OnTriggerEnter(enemy)
    → ProjectileDetector.DetectHit()
    → ApplyDamage()
    → DestroyProjectile()
```

### 4. AOEDetector（范围检测）
**场景**: 范围技能，如爆炸、范围Buff，使用OverlapSphere检测
```
特点：
- 使用Physics.OverlapSphere检测范围内所有敌人
- 支持圆形范围检测
- 高效的范围伤害计算
- 可配置忽略层级

流程：
Skill.Execute()
    → AOEDetector.DetectHit()
    → Physics.OverlapSphere(center, radius)
    → FilterTargets()
    → ApplyDamageToAll()
```

### 5. RaycastDetector（射线检测）
**场景**: 激光、穿透攻击，检测射线路径上的敌人
```
特点：
- 使用Physics.Raycast检测射线路径
- 支持穿透多个敌人
- 支持排除某些层级
- 高效的线性检测

流程：
Skill.Execute()
    → RaycastDetector.DetectHit()
    → Physics.Raycast(origin, direction)
    → 迭代检测所有hit
    → ApplyDamageToAll()
```

## 命中检测配置

**SummonChessTable中的配置字段**:
```
AttackHitType: 攻击命中检测类型（0-4）
- 0: Instant（瞬发）
- 1: Melee（近战）
- 2: Projectile（投射物）
- 3: AOE（范围）
- 4: Raycast（射线）

ProjectilePrefabId: 投射物预制体ID（仅在类型=2时使用）
MeleeWeaponId: 近战武器ID（仅在类型=1时使用）
```

## 执行流程总体

```
技能执行 → HitDetectorFactory.GetDetector()
    ↓
根据AttackHitType返回对应Detector实例
    ↓
Detector.Execute(HitContext)
    ├─ 执行检测（DetectHit）
    ├─ 获取击中目标（GetHitTargets）
    ├─ 遍历目标应用效果
    │   ├─ EffectExecutor.ApplyDamage()
    │   ├─ DamageApplier计算伤害
    │   └─ EffectExecutor.ApplyBuffsOnHit()
    ├─ 播放特效和音效
    └─ 返回HitResult
```

## 关键设计特点

1. **策略模式**: IHitDetector接口支持不同检测方式
2. **工厂模式**: HitDetectorFactory根据配置创建检测器
3. **配置驱动**: 所有检测参数来自Excel配置表
4. **高效复用**: 检测器实例可复用，避免重复创建
5. **事件驱动**: 命中后发布事件，与其他系统解耦
6. **多目标支持**: 支持单体和范围检测
7. **灵活扩展**: 新增检测方式只需实现IHitDetector

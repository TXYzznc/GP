# 图3：系统间通信关系

```dot
digraph {
    rankdir=TB;
    nodesep=0.9;
    ranksep=1.2;
    splines=orthogonal;
    
    CombatSys [label="⚔️ 战斗系统", style="filled", fillcolor="#ffccbc", shape="box"];
    ItemSys [label="🎒 物品系统", style="filled", fillcolor="#c8e6c9", shape="box"];
    CardSys [label="🃏 卡牌系统", style="filled", fillcolor="#b3e5fc", shape="box"];
    TASys [label="🎨 TA系统", style="filled", fillcolor="#f8bbd0", shape="box"];
    UISys [label="🖥️ UI系统", style="filled", fillcolor="#ede7f6", shape="box"];
    EventSys [label="📢 事件系统", style="filled", fillcolor="#fff9c4", shape="box"];
    
    CombatSys -> EventSys [label="伤害事件"];
    ItemSys -> EventSys [label="物品变化事件"];
    CardSys -> EventSys [label="卡牌使用事件"];
    TASys -> EventSys [label="特效事件"];
    
    EventSys -> UISys [label="伤害事件"];
    EventSys -> UISys [label="物品事件"];
    EventSys -> UISys [label="卡牌事件"];
    EventSys -> UISys [label="特效事件"];
    
    UISys -> CombatSys [label="用户操作"];
    UISys -> ItemSys [label="用户操作"];
    UISys -> CardSys [label="用户操作"];
}
```

## 说明

各个系统通过事件系统进行通信，实现了低耦合的架构：

- **战斗系统**：发送伤害事件
- **物品系统**：发送物品变化事件
- **卡牌系统**：发送卡牌使用事件
- **TA系统**：发送特效事件
- **UI系统**：订阅所有事件并更新界面
- **事件系统**：中央事件总线，负责事件的发布和订阅

这种设计使得各个系统可以独立开发和测试。

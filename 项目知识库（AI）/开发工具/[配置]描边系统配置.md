## 🎯 系统架构

### 方案一：Layer 自动管理（OutlineDisplayManager）

**工作原理：**
- 通过 GameObject 的 Layer 自动判断是否需要描边
- 适用于场景中的环境物体、道具等
- 性能优化：LOD 系统、距离检测、队列处理

**优点：**
- 无需手动控制，自动检测
- 性能优化完善
- 适合大量对象

**缺点：**
- 需要正确配置 Layer
- 不支持动态阵营关系

---

### 方案二：阵营关系管理（ChessOutlineController）

**工作原理：**
- 根据棋子的阵营关系自动选择描边配置
- 支持选中状态的特殊描边
- 与 CampRelationService 集成

**优点：**
- 自动根据阵营关系显示不同颜色
- 支持选中状态
- 适合战斗单位

**缺点：**
- 仅适用于棋子实体
- 需要手动初始化

## 🔧 配置步骤

### 第一步：创建 OutlineConfig 资源

1. 在 Project 窗口右键 → `Create` → `Rendering` → `Outline Config`
2. 创建以下配置文件：
   - `Enemy.asset` - 敌人描边（红色）
   - `Ally.asset` - 友军描边（绿色）
   - `Selected.asset` - 选中描边（黄色/白色）
   - `Interactive.asset` - 可交互对象描边（黄色）

3. 配置参数：
   ```
   OutlineColor: 描边颜色
   OutlineSize: 描边宽度（1-100）
   EnableDistanceScaling: 是否启用距离缩放
   MinDistance: 最小距离（保持最大宽度）
   MaxDistance: 最大距离（超过不渲染）
   DistanceFalloffCurve: 距离衰减曲线
   ```

**推荐配置：**

| 配置类型 | 颜色 | 宽度 | 最小距离 | 最大距离 |
|:---|:---|:---:|:---:|:---:|
| Enemy | 红色 (1, 0, 0, 1) | 20 | 5 | 50 |
| Ally | 绿色 (0, 1, 0, 1) | 15 | 5 | 50 |
| Selected | 黄色 (1, 1, 0, 1) | 25 | 5 | 100 |
| Interactive | 橙色 (1, 0.5, 0, 1) | 18 | 5 | 30 |

---

### 第二步：配置 OutlineDisplayManager（场景全局管理）

#### 2.1 创建管理器对象

1. 在场景中创建空 GameObject，命名为 `OutlineDisplayManager`
2. 添加 `OutlineDisplayManager` 组件

#### 2.2 配置 Layer 映射

在 Inspector 中配置 `Layer Mappings`：

```
Layer Mappings:
  - Layer Name: "Enemy"
    Outline Config: Enemy.asset
    Enabled: ✓
    
  - Layer Name: "Ally"
    Outline Config: Ally.asset
    Enabled: ✓
    
  - Layer Name: "Interactive"
    Outline Config: Interactive.asset
    Enabled: ✓
```

**重要：** Layer Name 必须与 Unity 的 Layer 设置完全一致！

#### 2.3 配置检测参数

```
Detection Radius: 100 （检测半径）
Target Camera: 留空（自动从 CameraRegistry 获取）
Use Physics Detection: ✓ （使用物理检测，需要 Collider）
Use Renderer Fallback: ✓ （使用 Renderer 查找，无需 Collider）
```

#### 2.4 配置性能参数

```
Scan Frequency: 10 Hz （扫描频率）
Layer Check Frequency: 30 Hz （Layer 检测频率）
Max Process Per Frame: 100 （每帧最大处理数）
Enable LOD: ✓ （启用 LOD 系统）
Near Distance: 30 （近距离阈值）
Mid Distance: 80 （中距离阈值）
```

---

### 第三步：配置 ChessOutlineController（棋子阵营管理）

#### 3.1 在棋子预制体上配置

1. 打开棋子预制体（如 `Chess_后羿`）
2. 确保已添加 `ChessOutlineController` 组件
3. 在 Inspector 中配置：

```
Enemy Outline Config: Enemy.asset
Ally Outline Config: Ally.asset
Selected Outline Config: Selected.asset
Neutral Outline Config: Interactive.asset
```

#### 3.2 配置资源路径（可选）

如果未直接赋值，可以配置资源路径：

```
Enemy Config Path: "Assets/TA/OuterGlow/Enemy.asset"
Ally Config Path: "Assets/TA/OuterGlow/Ally.asset"
Selected Config Path: "Assets/TA/OuterGlow/Selected.asset"
Neutral Config Path: "Assets/TA/OuterGlow/Interactive.asset"
```

**注意：** 这些路径仅在 Unity Editor 中有效，运行时需要直接赋值！

---

### 第四步：配置 Unity Layer

1. 打开 `Edit` → `Project Settings` → `Tags and Layers`
2. 添加以下 Layer：
   - `Enemy` - 敌人单位
   - `Ally` - 友军单位
   - `Interactive` - 可交互对象
   - `Chess` - 棋子（用于选择检测）

3. 将对应的 GameObject 设置到正确的 Layer

## 🔄 工作流程

### OutlineDisplayManager 工作流程

```
1. 初始化阶段
   ├─ 等待摄像机加载（从 CameraRegistry 获取）
   ├─ 构建 Layer → OutlineConfig 映射表
   └─ 启动扫描协程和 Layer 检测协程

2. 扫描阶段（每秒 10 次）
   ├─ 物理检测：OverlapSphere 查找 Collider
   ├─ Renderer 查找：FindObjectsOfType<Renderer>
   ├─ 距离过滤：只处理检测半径内的对象
   └─ 检查 Layer 是否有配置

3. Layer 检测阶段（每秒 30 次）
   ├─ 检测 GameObject.layer 是否变化
   ├─ 检测 GameObject.activeInHierarchy 是否变化
   └─ 触发描边更新

4. 队列处理阶段（每帧）
   ├─ 高优先级队列（近距离对象）
   ├─ 中优先级队列（中距离对象）
   └─ 低优先级队列（远距离对象）

5. 描边应用阶段
   ├─ 添加 OutlineTest 组件
   ├─ 计算距离衰减后的宽度
   └─ 调用 OutlineRenderFeature 渲染
```

---

### ChessOutlineController 工作流程

```
1. 初始化阶段（ChessEntity.Initialize 时调用）
   ├─ 缓存所有 Renderer 组件
   ├─ 加载描边配置资源
   └─ 根据阵营关系设置初始描边

2. 阵营判断阶段
   ├─ 调用 CampRelationService.GetRelationToLocalPlayer()
   ├─ 获取阵营关系：Self / Ally / Enemy / Neutral
   └─ 选择对应的 OutlineConfig

3. 选中状态处理
   ├─ 如果被选中，使用 SelectedOutlineConfig
   └─ 否则使用阵营对应的配置

4. 描边应用阶段
   ├─ 移除旧描边（如果有）
   ├─ 计算距离衰减后的宽度
   └─ 调用 OutlineRenderFeature.DrawOrUpdateOutlines()

5. 状态变化监听
   ├─ SetSelected(bool) - 选中状态变化
   ├─ RefreshOutline() - 手动刷新
   └─ OnDestroy() - 自动清理
```

## 🎨 阵营区分系统

### CampRelationService 工作原理

```csharp
public enum CampRelation
{
    Self,      // 自己
    Ally,      // 友军
    Enemy,     // 敌人
    Neutral    // 中立
}

// 获取阵营关系
CampRelation relation = CampRelationService.GetRelationToLocalPlayer(chessCamp);
```

### 阵营到描边的映射

| 阵营关系 | 描边配置 | 颜色 | 用途 |
|:---|:---|:---|:---|
| Self | 无描边 | - | 玩家自己的单位 |
| Ally | AllyOutlineConfig | 绿色 | 友军单位 |
| Enemy | EnemyOutlineConfig | 红色 | 敌人单位 |
| Neutral | NeutralOutlineConfig | 黄色 | 中立/可交互对象 |
| Selected | SelectedOutlineConfig | 白色/黄色 | 被选中的单位 |

## ⚠️ 常见问题

### 问题 1：描边不显示

**可能原因：**
1. OutlineConfig 资源未赋值
2. Layer 名称配置错误
3. OutlineRenderFeature 未添加到 URP Renderer
4. 摄像机未正确注册到 CameraRegistry

**解决方案：**
```
1. 检查 Inspector 中的配置是否完整
2. 确认 Layer 名称与 Unity 设置一致
3. 检查 URP Renderer 中是否添加了 OutlineRenderFeature
4. 确认 CameraRegistry.PlayerCamera 不为空
```

---

### 问题 2：描边颜色不正确

**可能原因：**
1. 阵营关系判断错误
2. OutlineConfig 资源配置错误
3. 选中状态未正确设置

**解决方案：**
```
1. 检查 CampRelationService 的阵营判断逻辑
2. 确认 OutlineConfig 的颜色配置
3. 使用 SetSelected(true/false) 正确设置选中状态
```

---

### 问题 3：性能问题

**优化建议：**
```
1. 降低扫描频率（Scan Frequency: 5-10 Hz）
2. 减小检测半径（Detection Radius: 50-80）
3. 启用 LOD 系统（Enable LOD: ✓）
4. 减少每帧处理数（Max Process Per Frame: 50-100）
5. 仅使用物理检测（Use Renderer Fallback: ✗）
```

## 📝 代码示例

### 手动控制描边

```csharp
// 获取棋子的描边控制器
ChessOutlineController outlineController = chessEntity.OutlineController;

// 设置选中状态
outlineController.SetSelected(true);

// 刷新描边（阵营关系变化时）
outlineController.RefreshOutline();

// 移除描边
outlineController.RemoveOutline();
```

### 通知 OutlineDisplayManager 刷新

```csharp
// 通知 Layer 变化
OutlineDisplayManager.Instance.NotifyLayerChanged(gameObject, newLayer);

// 刷新单个对象
OutlineDisplayManager.Instance.RefreshObject(gameObject);

// 刷新所有 Layer 映射
OutlineDisplayManager.Instance.RefreshLayerMappings();

// 重新初始化系统（场景切换后）
OutlineDisplayManager.Instance.Reinitialize();
```

# 棋子卡片点击交互逻辑优化 — 修正方案

**完成时间**: 2026-04-02
**修正内容**: 拖拽和选中完全独立，部署完成时自动清除选中

---

## 核心改动

### 问题分析

之前的设计中，在拖拽开始时就清除选中状态，但这违反了以下原则：
- **拖拽** 是放置操作，不应与选中混在一起
- **选中** 应该保持到棋子部署完成，方便用户继续编辑或查看信息

### 解决方案

1. **拖拽和选中完全独立**
   - OnChessItemDragBegin 不再清除选中状态
   - 拖拽期间 DetailInfoUI 保持显示

2. **部署完成时清除选中**
   - OnChessDeployedHandler 检测到部署完成
   - 自动清除对应棋子的选中状态
   - 隐藏 DetailInfoUI

3. **撤回时保持独立**
   - OnChessRecalledHandler 不需要检查选中状态
   - 只负责刷新棋子面板

## 修改详情

### CombatPreparationUI.cs

#### 1. OnChessItemDragBegin - 删除选中清除逻辑

**之前**：
```csharp
private void OnChessItemDragBegin(string instanceId)
{
    if (m_SelectedChessInstanceId == instanceId)
    {
        DeselectChess();  // ❌ 删除
    }
    ChessPlacementManager.Instance.StartPlacement(instanceId, true);
}
```

**现在**：
```csharp
private void OnChessItemDragBegin(string instanceId)
{
    Log.Info($"CombatPreparationUI: 拖拽开始 instanceId={instanceId}");

    // 开始放置，显示预览（进入放置状态）
    if (ChessPlacementManager.Instance != null)
    {
        ChessPlacementManager.Instance.StartPlacement(instanceId, true);
    }
}
```

**影响**：拖拽时保持卡片选中，DetailInfoUI 继续显示

---

#### 2. OnChessDeployedHandler - 增加选中清除逻辑

**之前**：
```csharp
private void OnChessDeployedHandler(ChessDeploymentTracker.ChessInstanceData instance)
{
    Log.Info($"CombatPreparationUI: 棋子已出战 instanceId={instance.InstanceId}");
    RefreshChessPanel();
}
```

**现在**：
```csharp
private void OnChessDeployedHandler(ChessDeploymentTracker.ChessInstanceData instance)
{
    Log.Info($"CombatPreparationUI: 棋子已部署 instanceId={instance.InstanceId}");

    // ✅ 如果部署的是当前选中的棋子，清除选中状态并隐藏 DetailInfoUI
    if (m_SelectedChessInstanceId == instance.InstanceId)
    {
        DeselectChess();
    }

    RefreshChessPanel();
}
```

**影响**：棋子部署完成后自动隐藏 DetailInfoUI，卡片进入"已出战"灰显状态

---

#### 3. OnChessRecalledHandler - 删除多余检查

**之前**：
```csharp
private void OnChessRecalledHandler(ChessDeploymentTracker.ChessInstanceData instance)
{
    Log.Info($"CombatPreparationUI: 棋子已撤回 instanceId={instance.InstanceId}");

    if (m_SelectedChessInstanceId == instance.InstanceId)
    {
        DeselectChess();  // ❌ 删除：撤回时卡片已无法交互
    }

    RefreshChessPanel();
}
```

**现在**：
```csharp
private void OnChessRecalledHandler(ChessDeploymentTracker.ChessInstanceData instance)
{
    Log.Info($"CombatPreparationUI: 棋子已撤回 instanceId={instance.InstanceId}");
    RefreshChessPanel();
}
```

**原因**：棋子卡片在"已出战"状态时无法交互，无需主动清除选中

## 完整交互流程

```
【准备阶段】

用户点击棋子卡片
  ↓
OnChessItemSelected()
  ├─ 已选中 → DeselectChess() → 隐藏 DetailInfoUI ✓
  └─ 未选中 → SelectChess() → 显示 DetailInfoUI ✓

用户拖拽棋子卡片
  ↓
OnChessItemDragBegin()
  ├─ 不清除选中状态 ✓（改动）
  └─ ChessPlacementManager.StartPlacement(isDragMode=true) ✓

用户释放鼠标（放置有效）
  ↓
ChessPlacementManager.ConfirmPlacementInternal()
  └─ ChessDeploymentTracker.DeployChess() ✓

棋子部署完成
  ↓
OnChessDeployedHandler()
  ├─ DeselectChess() → 隐藏 DetailInfoUI ✓（改动）
  └─ RefreshChessPanel() → 卡片变"已出战" ✓

【战斗阶段】

用户点击棋子对象
  ↓
ChessSelectionManager.HandleMouseClick()
  └─ entity.OutlineController.SetSelected(true) → 显示描边 ✓
```

| 场景 | 之前 | 现在 | 优势 |
|------|------|------|------|
| 拖拽期间 | DetailInfoUI 隐藏 | 保持显示 | 用户可持续查看棋子信息 |
| 部署完成 | 手动清除 | 自动清除 | 逻辑清晰，职责分离 |
| 撤回时 | 检查选中 | 不检查 | 删除多余代码 |

## 后续交互体验

**用户场景**：先选中查看信息，再放置
```
1. 点击棋子卡片 → 看到 DetailInfoUI 显示棋子属性
2. 拖拽该棋子 → DetailInfoUI 继续显示，用户一边放置一边参考信息
3. 释放鼠标 → 棋子部署，DetailInfoUI 自动隐藏
4. 点击棋子对象 → ChessSelectionManager 选中，显示描边，可移动
```

这样的设计流程更加自然和符合用户预期。

## 修复内容

### 1. 编译错误修复

#### 问题 1: ChessAttribute.Heal() 方法不存在
- **原因**: ChessAttribute 中没有 `Heal()` 方法
- **解决方案**: 改用 `ModifyHp(healAmount)` 方法
- **文件**: `Assets/AAAGame/Scripts/Game/Card/CardEffectHelper.cs`
- **修改**: 
  ```csharp
  // 修改前
  target.Attribute.Heal(healAmount);
  
  // 修改后
  target.Attribute.ModifyHp(healAmount);
  ```

#### 问题 2: CombatVFXManager 是静态类
- **原因**: 不能用 `FindObjectOfType<>()` 获取静态类
- **解决方案**: 直接调用静态方法 `CombatVFXManager.PlayEffect()`
- **文件**: `Assets/AAAGame/Scripts/Game/Card/CardEffectHelper.cs`
- **修改**:
  ```csharp
  // 修改前
  var vfxManager = Object.FindObjectOfType<CombatVFXManager>();
  if (vfxManager == null) { ... }
  
  // 修改后
  CombatVFXManager.PlayEffect(effectId, position);
  ```

## 架构改进

### 单例管理模式

采用**动态添加**模式确保单例存在：

```csharp
// 如果组件不存在，动态添加
if (CardManager.Instance == null)
{
    GameObject cardManagerObj = new GameObject("CardManager");
    cardManagerObj.AddComponent<CardManager>();
}
```

**优点**:
- ✅ 不依赖预制体引用
- ✅ 自动处理缺失情况
- ✅ 代码更健壮

### 初始化顺序

**GameProcedure** (游戏流程):
1. 技能管理器
2. 卡牌系统 (GameProcedure 中)
3. 战斗特效系统 (GameProcedure 中)
4. UI 系统

**CombatState** (战斗状态):
1. 棋子管理器

3. 卡牌系统 (CombatState 中) ← 战斗特定初始化
4. 战斗特效系统
5. 敌人生成

## 技术决策

### 为什么在 CombatState 中初始化卡牌？
- 卡牌是战斗特定的系统
- 每场战斗需要独立的卡牌实例
- 战斗结束时需要清理卡牌数据
- 符合单一职责原则

### 为什么使用动态添加而不是预制体？
- 避免预制体引用丢失问题
- 代码更灵活，不依赖编辑器配置
- 自动处理缺失情况
- 与项目现有模式一致（如 CombatOpportunityDetector）

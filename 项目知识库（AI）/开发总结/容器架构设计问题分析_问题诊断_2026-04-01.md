## 概览

当前的 ISlotContainer 容器架构在**多容器间拖拽操作**中存在多个严重的逻辑问题，导致数据不一致和物品丢失。

---

## 问题1️⃣：WarehouseSlotContainerImpl 仓库内部交换bug

### 问题位置
[WarehouseSlotContainerImpl.cs:43-68](Assets/AAAGame/Scripts/UI/Components/WarehouseSlotContainerImpl.cs#L43-L68)

### 问题描述
```csharp
bool success = targetContainer switch
{
    InventorySlotContainerImpl inv => MoveToInventory(...),
    WarehouseSlotContainerImpl wh => MoveToWarehouse(fromSlotIndex, targetSlotIndex),
    FastBarSlotContainerImpl fb => MoveToFastBar(...),
    _ => false
};

if (success)
{
    m_WarehouseManager.RemoveItem(fromSlotIndex, count);  // ❌ BUG
}
```

- **MoveToWarehouse()** 调用 `m_WarehouseManager.SwapSlots()`，这个方法已经交换了两个格子的 SlotIndex
- **SwapSlots()** 的实现（WarehouseManager.cs:419-463）已经处理了源目标更新，不需要额外操作
- 之后 `RemoveItem()` 会**再次调用事件并减少数量**，导致：
  - 如果是堆叠物品，数量被错误减少
  - UI 刷新两次，显示不一致

### 修复方案
```csharp
if (success && targetContainer is not WarehouseSlotContainerImpl)
{
    m_WarehouseManager.RemoveItem(fromSlotIndex, count);
}
```

## 问题2️⃣：FastBarSlotContainerImpl 快捷栏内部交换bug

### 问题位置
[FastBarSlotContainerImpl.cs:32-58](Assets/AAAGame/Scripts/UI/Components/FastBarSlotContainerImpl.cs#L32-L58)

### 问题描述
```csharp
bool success = targetContainer switch

    WarehouseSlotContainerImpl wh => MoveToWarehouse(wh, fromSlot, targetSlotIndex),
    FastBarSlotContainerImpl fb => MoveToFastBar(fromSlotIndex, targetSlotIndex),
```

### 修复方案
```csharp
if (success && targetContainer is not FastBarSlotContainerImpl)
{
    m_FastBarManager.ClearSlot(fromSlotIndex);
}
```

## 问题3️⃣：WarehouseSlotContainerImpl 的 MoveToInventory 不完整

### 问题位置
[WarehouseSlotContainerImpl.cs:71-95](Assets/AAAGame/Scripts/UI/Components/WarehouseSlotContainerImpl.cs#L71-L95)

### 问题描述
```csharp
private bool MoveToInventory(InventorySlotContainerImpl targetInv, int itemId, int count, int targetSlotIndex)
{
    var inv = InventoryManager.Instance;
    var targetSlot = inv?.GetSlot(targetSlotIndex);

    if (targetSlot == null)
        return false;

    if (targetSlot.IsEmpty)
    {
        var item = ItemManager.Instance?.CreateItem(itemId);
        if (item == null)
            return false;

        targetSlot.SetItem(item, count);
        return true;
    }
    else if (targetSlot.ItemId == itemId && targetSlot.ItemStack?.Item?.MaxStackCount > 1)
    {
        targetSlot.AddItem(count);
        return true;
    }

    return false;  // ❌ 不支持交换！
}
```

- **不支持交换**：如果目标背包格子有不同物品，操作失败
- InventoryUI 允许拖拽到任何背包格子，但仓库到背包的逻辑**不支持交换**

### 修复方案
增加交换逻辑，类似 FastBarSlotContainerImpl 的实现：
```csharp
else
{
    // 交换逻辑：目标物品交换回仓库
    var dstItem = ItemManager.Instance?.CreateItem(targetSlot.ItemId);
    var srcItem = ItemManager.Instance?.CreateItem(itemId);

    if (dstItem == null || srcItem == null)
        return false;

    int dstItemId = targetSlot.ItemId;
    int dstCount = targetSlot.Count;

    targetSlot.Clear();
    targetSlot.SetItem(srcItem, count);

    return m_WarehouseManager.StoreItemToSlot(dstItemId, dstCount, fromSlotIndex, 0);
}
```

## 问题4️⃣：快捷栏 → 快捷栏 与 CanInteractWith 规则不一致

### 问题位置
[FastBarSlotContainerImpl.cs:25-30](Assets/AAAGame/Scripts/UI/Components/FastBarSlotContainerImpl.cs#L25-L30)

### 问题描述
```csharp
public override bool CanInteractWith(SlotContainerType otherContainerType)
{
    return otherContainerType == SlotContainerType.Inventory ||
           otherContainerType == SlotContainerType.Warehouse ||  // ❌ 不应该直接与仓库交互
           otherContainerType == SlotContainerType.FastBar;
}
```

快捷栏应该定位为**背包的快速访问栏**，**不应该直接与仓库交换物品**。物品流应该是：
- 背包 ↔ 仓库
- 背包 ↔ 快捷栏
- 快捷栏与仓库的交互**应该通过背包中转**

### 修复方案
```csharp
public override bool CanInteractWith(SlotContainerType otherContainerType)
{
    return otherContainerType == SlotContainerType.Inventory ||
           otherContainerType == SlotContainerType.FastBar;
}
```

## 快速补充方法

### 方法 1：使用文本编辑器

1. 打开文件
2. 在文件顶部的标题下方插入光标
3. 按 Enter 创建新行
4. 复制粘贴元数据头（从上面的示例中）
5. 保存文件

### 方法 2：批量补充脚本（PowerShell）

```powershell
# 保存为 补充元数据.ps1

$basePath = "D:\unity\UnityProject\GP\Clash_Of_Gods\项目知识库（AI）\开发总结"
$files = @(
    "容器架构设计问题分析_2026-04-01.md",
    "容器架构修复完成_2026-04-01.md",
    "搜打撤背包系统_完整实现总结_2026-04-01.md",
    "搜打撤背包系统_实现清单_2026-04-01.md"
)

$metadata = @"
> **最后更新**: 2026-04-01  
> **状态**: 有效  
> **分类**: 开发总结

---
"@

foreach ($file in $files) {
    $filePath = Join-Path $basePath $file
    if (Test-Path $filePath) {
        $content = Get-Content $filePath -Raw
        
        # 检查是否已有元数据头
        if ($content -notmatch "^>.*最后更新") {
            # 找到第一个 --- 的位置
            $lines = $content -split "`n"
            $headerEndIndex = -1
            
            for ($i = 0; $i -lt $lines.Length; $i++) {
                if ($lines[$i] -match "^#") {
                    $headerEndIndex = $i
                    break
                }
            }
            
            if ($headerEndIndex -ge 0) {
                # 在标题后插入元数据
                $newLines = @()
                $newLines += $lines[0..($headerEndIndex)]
                $newLines += ""
                $newLines += $metadata
                $newLines += $lines[($headerEndIndex + 1)..($lines.Length - 1)]
                
                $newContent = $newLines -join "`n"
                Set-Content $filePath -Value $newContent -Encoding UTF8
                Write-Host "✅ 已更新: $file"
            }
        } else {
            Write-Host "⏭️  跳过（已有元数据）: $file"
        }
    } else {
        Write-Host "❌ 文件不存在: $file"
    }
}

Write-Host "✅ 元数据补充完成！"
```

### 方法 3：批量补充脚本（Bash）

```bash
#!/bin/bash
# 保存为 补充元数据.sh

BASE_PATH="/d/unity/UnityProject/GP/Clash_Of_Gods/项目知识库（AI）/开发总结"

files=(
    "容器架构设计问题分析_2026-04-01.md"
    "容器架构修复完成_2026-04-01.md"
    "搜打撤背包系统_完整实现总结_2026-04-01.md"
    "搜打撤背包系统_实现清单_2026-04-01.md"
)

metadata="> **最后更新**: 2026-04-01
> **状态**: 有效
> **分类**: 开发总结

---"

for file in "${files[@]}"; do
    filepath="$BASE_PATH/$file"
    
    if [ -f "$filepath" ]; then
        # 检查是否已有元数据头
        if ! grep -q "^> \*\*最后更新\*\*" "$filepath"; then
            # 创建临时文件
            temp_file="${filepath}.tmp"
            
            # 读取文件，找到标题后插入元数据
            head -1 "$filepath" > "$temp_file"
            echo "" >> "$temp_file"
            echo "$metadata" >> "$temp_file"
            tail -n +2 "$filepath" >> "$temp_file"
            
            # 替换原文件
            mv "$temp_file" "$filepath"
            echo "✅ 已更新: $file"
        else
            echo "⏭️  跳过（已有元数据）: $file"
        fi
    else
        echo "❌ 文件不存在: $file"
    fi
done

echo "✅ 元数据补充完成！"
```

## 验证标准

补充完成后，每个文件应该满足以下条件：

✅ **文件结构**:
```
# 标题

> **最后更新**: 日期  
> **状态**: 有效  
> **分类**: 开发总结

---

## 第一个小节
...
```

✅ **元数据内容**:
- 有 `最后更新` 字段
- 有 `状态` 字段（值应为"有效"）
- 有 `分类` 字段（值应为"开发总结"）

✅ **格式规范**:
- 元数据使用 `> ` 前缀（Markdown 引用格式）
- 元数据后面有一行 `---` 分隔符
- 元数据和标题之间有空行

## 常见问题

### Q: 补充元数据后文件无法打开？
**A**: 检查文件编码。使用 UTF-8 编码保存文件。

### Q: 标题位置错了怎么办？
**A**: 检查 Markdown 标题的位置。标题应该是 `# 标题名称` 格式。

### Q: 已经有元数据了还要补充吗？
**A**: 不需要。检查文件顶部是否已有 `> **最后更新**` 的元数据头，如果有则跳过。

### Q: 脚本执行失败？
**A**: 确保：
1. 工作目录正确
2. 文件名完全匹配
3. 有足够的文件权限

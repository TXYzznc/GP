### 1. UI 关闭方式不一致
- **EnterCombatTip.cs**: 使用 `GF.UI.Close(UIForm)` → 触发关闭动画
- **CombatPreparationState.cs**: 使用 `GF.UI.HasUIForm(uiFormId)` 检测 → 只检测 UI 是否存在

### 2. 关闭动画延迟机制
```csharp
// EnterCombatTip.cs - 自动关闭
GF.UI.Close(UIForm); // 触发动画关闭

// UIExtension.Close() 方法
logic.CloseWithAnimation(); // 播放关闭动画

// UIFormBase.CloseWithAnimation()
Internal_PlayCloseUIAnimation(OnCloseAnimationComplete); // 动画完成后才真正销毁
```

### 3. 轮询等待机制效率问题
```csharp
// CombatPreparationState.cs - 轮询等待
while (GF.UI.HasUIForm(uiFormId) && elapsedTime < timeoutSeconds)
{
    await UniTask.Yield(); // 每帧检查，直到动画完成
    elapsedTime += Time.unscaledDeltaTime;
}
```

## 修复方案

### 方案一：事件驱动机制（已实施）
使用 UI 关闭完成事件，避免轮询等待。

### 方案二：直接关闭优化（已实施）
针对 EnterCombatTip 这种简单提示 UI，跳过关闭动画。

## 实施详情

### 1. CombatPreparationState.cs 修改
- **新增方法**: `WaitForUIFormClosedByEvent()` - 事件驱动等待机制
- **修改调用**: `ShowEnterCombatTipAndContinue()` 使用新的等待方法
- **保留备用**: `WaitForUIFormClosed()` 作为备用轮询方案
- **事件处理**: 正确处理 `CloseUIFormCompleteEventArgs` 事件类型转换

### 2. EnterCombatTip.cs 修改
- **跳过动画**: 使用 `GF.UI.CloseUIForm()` 替代 `GF.UI.Close()`
- **直接关闭**: 避免动画播放导致的延迟
- **增强日志**: 记录关闭时机和方式

### 3. 技术细节

        }
    }

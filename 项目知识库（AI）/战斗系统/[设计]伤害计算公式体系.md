## 设计目标

1. **完整性**：覆盖物伤、法伤、真伤、护盾、治疗等所有元素
2. **平衡性**：确保各类伤害类型有合理的互制关系
3. **可配置性**：通过配置表支持多种计算规则
4. **易理解性**：公式逻辑清晰，便于数值调整

---

## 核心计算流程

```
基础伤害计算
  ↓
应用增伤/减伤Buff
  ↓
防御减伤计算（护甲/法抗）
  ↓
暴击伤害计算
  ↓
护盾抵消
  ↓
最终伤害结果
```

## 公式集合

### 一、基础伤害计算

#### 公式1：物理伤害（标准规则）

```
BasePhysicalDamage = AttackerAttack × AttackCoefficient
AppliedDamage = BasePhysicalDamage × (1 + DamageIncrease) × (1 + CriticalMultiplier)
FinalDamage = AppliedDamage × DamageReduction_ByArmor
```

**变量说明**：
- `AttackerAttack`: 攻击方的攻击力属性
- `AttackCoefficient`: 技能的攻击系数（0.5~2.0 范围）
- `DamageIncrease`: 增伤Buff加成（如神力增益 +25%）
- `CriticalMultiplier`: 暴击倍数（1.5× ~ 2.5×）
- `DamageReduction_ByArmor`: 护甲减伤系数（见公式4）

**示例**：
```
攻击方: 100攻击力，目标25%增伤Buff
目标: 100护甲
技能: 1.0系数，20%暴击率，2.0倍爆伤

基础伤害 = 100 × 1.0 = 100
应用增伤 = 100 × (1 + 0.25) = 125
暴击伤害 = 125 × 2.0 = 250
护甲减伤 = 250 × 减伤系数(100护甲) ≈ 250 × 0.57 ≈ 142.5
```

---

#### 公式2：魔法伤害（法抗机制相同）

```
BaseMagicDamage = AttackerMagicPower × MagicCoefficient
AppliedDamage = BaseMagicDamage × (1 + DamageIncrease) × (1 + CriticalMultiplier)
FinalDamage = AppliedDamage × DamageReduction_ByMagicResist
```

**特点**：
- 与物理伤害结构相同，但基于魔力属性
- 护甲和法抗独立计算，不互相影响
- 法抗通常低于护甲（魔法抵抗更弱）

---

#### 公式3：真实伤害

```
BaseTrueDamage = AttackerTruePower × TrueCoefficient
FinalDamage = BaseTrueDamage × (1 + DamageIncrease) × (1 + CriticalMultiplier)
             × (1 + TrueDamageBonus)
```

**特点**：
- **不受护甲/法抗减伤**（MinDamagePassthrough = 100%）
- 只能通过护盾和生命值直接抵消
- 暴击对真伤仍有效
- 真伤强化Buff可以增加真伤系数

### 二、防御减伤计算

#### 公式4：护甲减伤/增伤（两段式递减 + 负数护甲）

```csharp
float CalculateArmorReduction(float armor, float lowThreshold, 
                              float lowRate, float highBase, float highRate,
                              float negativeArmorRate = 0.5f)
{
    // ===== 正数护甲：减伤 =====
    if (armor > 0)
    {
        // 低护甲段（线性）
        if (armor <= lowThreshold)
        {
            return 1.0f - (armor * lowRate);
        }
        
        // 高护甲段（递减）
        float excessArmor = armor - lowThreshold;
        float reduction = highBase + (excessArmor * highRate);
        
        // 限制最大减伤
        return 1.0f - Mathf.Min(reduction, 0.85f);
    }
    
    // ===== 负数护甲：增伤（破甲）=====
    // armor < 0 时，增伤效果比减伤更平缓
    // 示例：-100护甲增伤 < +100护甲减伤
    else if (armor < 0)
    {
        // 负数护甲的增伤效果 = 正数护甲的 50% 
        // (negativeArmorRate 参数可调，建议 0.4~0.6)
        float damageIncrease = -armor * lowRate * negativeArmorRate;
        
        // 限制最大增伤（防止伤害爆炸）
        // 建议上限：30% 增伤（远小于85%减伤）
        float cappedIncrease = Mathf.Min(damageIncrease, 0.30f);
        
        return 1.0f + cappedIncrease;
    }
    
    // armor == 0
    return 1.0f;
}
```

**配置示例**：
```
物理伤害: lowThreshold=100, lowRate=3%, highBase=3%, highRate=1%, negativeArmorRate=0.5

正数护甲（减伤）：
- 0护甲: 伤害系数 100%
- 50护甲: 伤害系数 100% - 50×3% = 85%
- 100护甲: 伤害系数 100% - 100×3% = 70%
- 200护甲: 伤害系数 100% - (3% + 100×1%) = 最高85%减伤 → 15%伤害系数

负数护甲（增伤，破甲）：
- 0护甲: 伤害系数 100%
- -50护甲: 伤害系数 100% + 50×3%×0.5 = 107.5% (+7.5%)
- -100护甲: 伤害系数 100% + 100×3%×0.5 = 115% (+15%)
- -200护甲: 伤害系数 100% + 200×3%×0.5 = 130% (+30%, 达到上限)
```

**数值对比**（非对称设计）：
```
+100护甲 → 减伤30% → 伤害系数70%
-100护甲 → 增伤15% → 伤害系数115%

这样设计的好处：
1. 防御强度 > 破甲强度（防御更有价值）
2. 破甲不会导致伤害过高（增伤最多30%）
3. 鼓励"破甲"作为战斗策略，但不是绝对压制
```

**为什么使用两段式 + 负数处理**：
1. 低护甲段线性递增：初期护甲有明显效果，鼓励前期堆防
2. 高护甲段递减：后期收益递减，防止超级坦克出现
3. 最大减伤上限（85%）：确保有基础伤害穿透
4. **负数护甲增伤平缓**：破甲机制不会过度强势，防御生态健康

#### 公式5：法抗减伤/增伤

```
同护甲公式，但参数不同：

正数法抗（减伤）：
- lowThreshold = 80（法抗阈值更低）
- lowRate = 3.5%（初期减伤更强）
- highBase = 2.5%
- highRate = 0.8%
- MaxReductionRatio = 80%（法抗上限更低）

负数法抗（增伤，破法）：
- negativeArmorRate = 0.6（比破甲稍强，因为法抗本身就弱）
- MaxIncrease = 35%（比物理破甲稍高）
```

**设计意图**：
- 法抗相对弱于护甲，鼓励魔法核心职业输出
- 负数法抗的破法稍强于破甲（平衡法抗本身较弱的问题）
- 但仍有合理的防御空间

### 三、暴击和爆伤

#### 公式6：暴击伤害

```csharp
double CalculateCriticalDamage(double baseDamage, double critRate, 
                               double critPower, bool forceCrit = false)
{
    // 触发暴击检定
    bool isCrit = forceCrit || Random.Range(0, 1) < critRate;
    
    if (isCrit)
    {
        // 爆伤 = 基础倍数 + 额外倍数
        double critMultiplier = 1.5 + critPower;  // 最少1.5倍
        return baseDamage * critMultiplier;
    }
    
    return baseDamage;
}
```

**参数范围**：
```
暴击率(CritRate):    0% ~ 75%（超过则按75%计算）
基础爆伤(CritPower): 0.5× ~ 1.5×（总倍数2.0× ~ 3.0×）

示例：
- 20%暴击率，0.5爆伤 → 平均伤害 = 100 × (0.8×1.0 + 0.2×2.0) = 120
- 50%暴击率，1.0爆伤 → 平均伤害 = 100 × (0.5×1.0 + 0.5×2.5) = 175
```

**平衡建议**：
- 暴击率超过70%时收益明显降低
- 每3点暴击率 ≈ 1%平均伤害提升
- 鼓励暴击和爆伤的组合搭配

### 四、护盾机制

#### 公式7：护盾吸收伤害

```csharp
void ApplyDamageWithShield(ChessEntity target, double damage)
{
    double shieldValue = target.GetTotalShield();
    
    if (shieldValue > 0)
    {
        // 护盾先抵消伤害
        if (damage <= shieldValue)
        {
            target.ReduceShield(damage);
            return;  // 伤害完全抵消
        }
        
        // 护盾不足，扣血
        target.ReduceShield(shieldValue);
        double remainDamage = damage - shieldValue;
        target.TakeDamage(remainDamage);
    }
    else
    {
        // 无护盾，直接扣血
        target.TakeDamage(damage);
    }
}
```

**护盾特性**：
- 优先级最高，最先抵消伤害
- 护盾不计算护甲/法抗减伤（已在生成时计算）
- 护盾破裂时可触发回调（如触发反弹Buff）

---

#### 公式8：护盾生成（示例）

```
ShieldValue = BaseHealth × ShieldRatio × (1 + ShieldBonus)
```

**示例**：
```
生成护盾Buff，参数: ShieldRatio=0.3
目标血量: 1000
生成护盾: 1000 × 0.3 = 300

如果有护盾加强(+20%), 则: 1000 × 0.3 × 1.2 = 360
```

### 五、增伤效果

#### 公式9：叠加式增伤Buff

```csharp
double CalculateTotalDamageIncrease(BuffManager buffManager)
{
    double totalIncrease = 0;
    
    // 遍历所有增伤Buff
    var increaseBuffs = buffManager.GetBuffsByType(BuffType.DamageIncrease);
    foreach (var buff in increaseBuffs)
    {
        totalIncrease += buff.EffectValue;
    }
    
    // 限制最大增伤（防止数值爆炸）
    return Mathf.Min(totalIncrease, 2.0f);  // 最多200%增伤
}
```

**增伤分类**：
```
1. 通用增伤（所有伤害）：神力增益 +25%
2. 物理增伤（仅物伤）：破甲术 +30%
3. 魔法增伤（仅法伤）：法力增强 +25%
4. 真伤增伤（仅真伤）：破裂 +40%
```

**平衡建议**：
- 通用增伤：最多 +100%（单Buff上限）
- 专属增伤：最多 +150%（单Buff上限）
- 总叠加上限：+200%（防止输出爆表）

### 六、治疗效果

#### 公式10：治疗量计算

```csharp
double CalculateHeal(double healPower, double healCoefficient, 
                     double targetMaxHealth, double healBonus = 0)
{
    // 治疗 = 治疗力 × 系数 × (1 + 治疗加成)
    double baseHeal = healPower * healCoefficient;
    double finalHeal = baseHeal * (1 + healBonus);
    
    // 治疗上限：不超过目标最大血量的30%
    return Mathf.Min(finalHeal, targetMaxHealth * 0.3);
}
```

**治疗加成来源**：
```
1. 治疗加成Buff（如恢复术 +20%）
2. 目标已损血量比例加成
3. 治疗者攻击力加成
```

**特殊机制**：
```
过量治疗(Overhealing):
- 如果治疗量超过缺少血量，超出部分生成护盾
- 转换比例: 过量治疗 × 0.5 = 护盾值
- 最多可以转50%的护盾

示例：
目标血量300/1000, 治疗400
- 实际治疗: min(400, 700) = 400（血满到700）
- 过量治疗: 400 - 400 = 0（无过量）

如果治疗500：
- 实际治疗: min(500, 700) = 500（血满到1000）
- 过量治疗: 500 - 700 = -200? 不对

应改为：
- 目标需要血量: 700
- 治疗所有: 500
- 实际治疗: 500, 血量变为 800
- 剩余目标血量: 200
```

重新设计过量治疗：
```
如果 healAmount > targetMissingHealth:
    overHeal = healAmount - targetMissingHealth
    shieldConverted = overHeal × 0.5
```

### 七、完整伤害计算公式（综合）

#### 公式11：完整物理伤害公式

```
Step 1: 计算基础伤害
  BaseDamage = Attack × SkillCoefficient

Step 2: 应用增伤Buff
  DamageAfterBuff = BaseDamage × (1 + DamageIncreaseRatio)
                              × (1 + PhysicalDamageIncrease)

Step 3: 应用暴击
  if (CriticalHit):
      DamageAfterCrit = DamageAfterBuff × (1.5 + CritPower)
  else:
      DamageAfterCrit = DamageAfterBuff

Step 4: 防御减伤
  ArmorReduction = ArmorReductionFunction(TargetArmor)
  DamageAfterDefense = DamageAfterCrit × ArmorReduction

Step 5: 护盾抵消
  if (TargetShield > 0):
      if (DamageAfterDefense <= TargetShield):
          FinalDamage = 0, ShieldRemain = TargetShield - DamageAfterDefense
      else:
          FinalDamage = DamageAfterDefense - TargetShield
          ShieldRemain = 0
  else:
      FinalDamage = DamageAfterDefense

Step 6: 应用最小伤害穿透
  if (FinalDamage < MaxHealth × MinDamagePassthrough):
      FinalDamage = MaxHealth × MinDamagePassthrough  // 15%
```

**实现代码框架**：
```csharp
public double CalculatePhysicalDamage(ChessEntity attacker, ChessEntity target, 
                                      double skillCoefficient)
{
    // 1. 基础伤害
    double baseDamage = attacker.Attribute.Attack * skillCoefficient;
    
    // 2. 增伤Buff
    double damageIncrease = CalculateDamageIncrease(attacker);
    double damageWithBuff = baseDamage * (1 + damageIncrease);
    
    // 3. 暴击
    double damageAfterCrit = CalculateCriticalDamage(
        damageWithBuff, 
        attacker.Attribute.CritRate,
        attacker.Attribute.CritPower
    );
    
    // 4. 护甲减伤
    double armorReduction = CalculateArmorReduction(
        target.Attribute.Armor,
        100,    // lowThreshold
        0.03f,  // lowRate
        0.03f,  // highBase
        0.01f   // highRate
    );
    double damageAfterArmor = damageAfterCrit * armorReduction;
    
    // 5. 护盾抵消
    double finalDamage = ApplyShield(target, damageAfterArmor);
    
    // 6. 最小伤害穿透（15%）
    double minDamage = target.Attribute.MaxHealth * 0.15;
    return Mathf.Max(finalDamage, minDamage);
}
```

## 平衡性建议

### 一、属性系统建议

```
推荐属性：
- 攻击力 (Attack): 基础属性，影响物理伤害
- 魔力 (MagicPower): 基础属性，影响魔法伤害
- 护甲 (Armor): 防御属性，减少物理伤害
- 法抗 (MagicResist): 防御属性，减少魔法伤害
- 暴击率 (CritRate): 0% ~ 75%
- 爆伤 (CritPower): 0.5× ~ 1.5× 范围
- 生命值上限 (MaxHealth)
- 护盾值 (Shield): 临时属性，战斗结束清零
```

### 二、职业伤害对标

```
单次基础伤害（无Buff/防御）：

物理核心职业:
- 攻击力: 120, 技能系数: 1.0, 无暴击
- 基础伤害: 120

魔法核心职业:
- 魔力: 100, 技能系数: 1.2, 无暴击
- 基础伤害: 120

真伤职业:
- 真伤力: 80, 技能系数: 1.5, 无暴击
- 基础伤害: 120

结论: 基础伤害应该持平，但输出稳定性不同
```

### 三、防御压制建议

```
关键指标：
1. 100护甲的敌人应该能减伤30%以上（感受到防御的存在）
2. 300护甲的敌人能减伤约70%（高端防御值）
3. 物理和魔法的防御差异体现在：
   - 物理防御相对强（更容易堆高）
   - 魔法防御相对弱（鼓励魔法输出）
```

### 四、增伤上限建议

```
单一Buff:
- 通用增伤: +50% ~ +100%
- 专属增伤: +75% ~ +150%

总叠加:
- 最多允许 +200% 增伤
- 超过则按200%计算
- 目的：防止输出职业无脑堆增伤Buff导致数值爆炸
```

### 五、治疗平衡建议

```
治疗和伤害的比例：
- 单次治疗应该是伤害的 60% ~ 80%
- 持续治疗应该是伤害的 40% ~ 60%

示例：
- 伤害职业单次伤害: 200
- 治疗职业单次治疗: 150 (75%)
- 治疗职业持续治疗: 100/秒 (50%)
```

## 实现要点

### 1. 暴击判定的时序

```csharp
// ❌ 错误：重复判定
double damage = CalculateDamage();
if (CriticalHit())  // 第一次判定
    damage *= critPower;
OnHit(damage);
if (CriticalHit())  // 第二次判定，可能不同！
    ShowCriticalEffect();

// ✅ 正确：单次判定
bool isCrit = CriticalHit();
double damage = CalculateDamage();
if (isCrit)
    damage *= critPower;
OnHit(damage);
if (isCrit)
    ShowCriticalEffect();
```

### 2. 护甲减伤的浮点精度

```csharp
// 护甲值很高时，需要注意精度
float CalculateArmorReduction(float armor, ...)
{
    // 用 double 计算，避免浮点误差
    double reduction = (double)armor * highRate;
    
    // 最后再转回 float
    return (float)Mathf.Min(reduction, maxReduction);
}
```

### 3. 护盾和伤害的顺序

```csharp
// 护盾 → 伤害 → 护甲减伤是错误的
// 正确顺序应该是：

伤害 → 护甲减伤 → 护盾抵消

原因：护甲是对伤害的减弱，护盾是对伤害的抵消
      护盾应该在完整伤害计算完毕后才发挥作用
```

### 4. 日志输出规范

private void LogDamageCalculation(ChessEntity attacker, ChessEntity target,

        $"防御减伤={((1 - finalDamage/baseDamage)*100):F1}%");
}

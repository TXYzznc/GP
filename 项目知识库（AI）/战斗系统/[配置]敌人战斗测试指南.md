## 前置准备

### 1. 确认配置表已加载

在 Unity Editor 中检查：
- `Assets/AAAGame/DataTable/EnemyTable.txt` 文件存在
- `Assets/AAAGame/DataTable/EnemyEntityTable.txt` 文件存在
- 数据表已通过工具生成二进制文件

### 2. 确认战斗场地预制体已配置

战斗场地预制体需要包含以下子对象：
```
BattleArena
├── PlayerAnchor (空对象)
├── PlayerZone (Plane，带 Renderer)
├── EnemyZone (Plane，带 Renderer)
└── PlacementPlane (放置平面)
```

如果还没有配置，可以先使用默认场地测试（代码会自动创建）。

### 3. 确认测试配置

当前代码使用的测试配置：
- **战斗配置ID**: 2001
- **位置**: `CombatPreparationState.cs` 第 242-249 行

### 步骤 3：检查场景中的敌人棋子

**视觉检查**：
- [ ] 敌方区域生成了 3 个棋子
- [ ] 棋子排列成横排（阵型类型 1）
- [ ] 棋子之间间距约为 2.5 米
- [ ] 棋子位于战斗场地的敌方区域（远离玩家）

**Hierarchy 检查**：
- 在 Hierarchy 中找到生成的棋子对象
- 确认棋子的阵营（Camp）为 1（敌方）
- 确认棋子的 AI 已启用

- [ ] 我方棋子能否正常攻击敌方棋子
- [ ] 敌方棋子能否正常攻击我方棋子
- [ ] 棋子死亡后是否正确移除

## 常见问题排查

### 问题 1：配置加载失败

**日志提示**：
```
[EnemySpawnManager] EnemyTable 数据表未加载
```

**解决方案**：
1. 检查 `EnemyTable.txt` 是否在正确的路径
2. 确认数据表已通过工具生成
3. 检查数据表是否在游戏启动时加载

### 问题 2：敌人没有生成

**日志提示**：
```
[EnemySpawnManager] 未找到敌人配置 ID=2001
```

**解决方案**：
1. 检查 `EnemyTable.txt` 中是否有 ID=2001 的配置
2. 确认配置表格式正确
3. 重新生成数据表文件

### 问题 3：敌人生成位置错误

**可能原因**：
- 战斗场地的 `EnemyZone` 未正确配置
- `BattleArenaManager` 无法获取敌方区域中心

**解决方案**：
1. 检查战斗场地预制体结构
2. 确认 `EnemyZone` 有 Renderer 组件
3. 查看日志中的敌方区域中心坐标

### 问题 4：棋子阵型不正确

**检查项**：
1. 配置表中的 `FormationType` 是否为 1（横排）
2. `Spacing` 是否为 2.5
3. 查看 `EnemyFormationManager` 的日志输出

如果需要测试不同的配置，修改 `CombatPreparationState.cs`：

**文件路径**：`Assets/AAAGame/Scripts/GameState/States/CombatPreparationState.cs`

**修改位置**：第 245 行

```csharp
/// <summary>
/// 加载敌人波次配置
/// </summary>
private void LoadEnemyWaveConfig()
{
    // 修改这里的 ID 来测试不同配置
    int battleConfigId = 2001;  // 改为 2002 或 2003 测试其他配置
    
    EnemySpawnManager.Instance.LoadFromEnemyTable(battleConfigId);
    
    DebugEx.LogModule("CombatPreparationState", $"已加载敌人战斗配置 ID={battleConfigId}");
}
```

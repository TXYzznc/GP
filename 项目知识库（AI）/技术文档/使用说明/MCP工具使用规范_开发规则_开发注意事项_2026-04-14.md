## 工具冲突规则

以下工具**不能在同一个操作流程中混用**:

```
executePwsh  ⚠️  mcp_*
```

**原因**: executePwsh的输出会干扰MCP服务器的JSON-RPC通信

### 正确的使用模式

#### 模式1: 纯MCP工具链

```python
# ✓ 正确
fsWrite("config.txt", content)  # 使用内置工具创建文件
mcp_txt_to_xlsx_converter_convert_txt_to_xlsx(
    txt_file_path="绝对路径",
    output_dir="绝对路径"
)
```

#### 模式2: 分阶段执行

```python
# ✓ 正确 - 阶段1: 准备工作
executePwsh("New-Item -ItemType Directory -Path 'output'")

# 等待阶段1完成后...

# 阶段2: MCP工具调用
mcp_tool(...)
```

#### 模式3: 使用替代工具

```python
# ❌ 错误
executePwsh("Get-Location")  # 获取当前路径
mcp_tool(...)  # 冲突!

# ✓ 正确
# 使用已知的项目路径,不需要动态获取
project_path = "d:/unity/UnityProject/GP/Clash_Of_Gods"
mcp_tool(txt_file_path=f"{project_path}/file.txt")
```

## MCP工具调用最佳实践

### 1. 路径处理

```python
# MCP工具需要绝对路径
# ✓ 正确
mcp_tool(txt_file_path="d:/unity/UnityProject/GP/Clash_Of_Gods/MCP工作区/file.txt")

# ❌ 错误
mcp_tool(txt_file_path="MCP工作区/file.txt")  # 相对路径可能失败
```

### 2. 文件准备

```python
# 使用fsWrite创建文件,而不是executePwsh
# ✓ 正确
fsWrite("MCP工作区/config.txt", content)

# ❌ 错误
executePwsh("New-Item -Path 'MCP工作区/config.txt' -Value $content")
```

### 3. 错误处理

```python
# MCP工具返回的错误信息格式
if "[ERROR]" in result:
    # 处理错误
    pass
elif "[SUCCESS]" in result:
    # 处理成功
    pass
```

## 工具选择决策树

```
需要执行操作
    │
    ├─ 文件操作?
    │   ├─ 创建/写入 → fsWrite
    │   ├─ 读取 → readFile
    │   └─ 删除 → deleteFile
    │
    ├─ 需要MCP工具?
    │   ├─ 是 → 使用MCP工具
    │   │       不要混用executePwsh
    │   └─ 否 → 可以使用executePwsh
    │
    └─ 需要命令执行?
        ├─ 与MCP工具无关 → executePwsh
        └─ 与MCP工具相关 → 分阶段执行
```

## 常见错误及解决方案

### 错误1: JSON解析失败

```
ERROR: Invalid JSON: expected value at line 1 column 1
```

**原因**: executePwsh与MCP工具混用

**解决**: 移除executePwsh调用,使用fsWrite等替代

### 错误2: 文件路径找不到

```
[ERROR] 文件不存在 - file.txt
```

**原因**: 使用了相对路径

**解决**: 使用绝对路径

### 错误3: 编码错误

```
'gbk' codec can't encode character
```

**原因**: MCP服务器使用了特殊Unicode字符

**解决**: 使用修复版本的MCP服务器(mcp_server_fixed.py)

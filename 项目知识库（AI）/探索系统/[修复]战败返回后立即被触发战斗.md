# 战败返回后立即被触发战斗 问题诊断和修复

> **最后更新**: 2026-03-25
> **状态**: 有效
> **严重程度**: 中等
> **修复状态**: 已修复

---

## 问题概述

玩家战斗失败后返回探索场景，会立刻被刚刚的敌人再次拉入战斗，体验极差。`ResetAlert()` 虽然将警觉度清零，但不足以阻止立即再次触发。

---

## 症状描述

### 表现形式

- 玩家失败战斗 → 返回探索场景 → 当帧或次帧立刻进入新一轮战斗准备
- 敌人警觉 UI 瞬间升满，再次进入战斗

### 复现步骤

1. 在探索场景靠近敌人触发战斗
2. 故意失败（让己方棋子全部死亡）
3. 观察返回探索场景后是否立即进入下一场战斗

**预期行为**：玩家有时间在场景中移动，不立刻触发战斗

**实际行为**：战斗结束 → 返回场景 → 立即进入下一场战斗

### 影响范围

- 探索场景所有敌人（因为玩家站在敌人旁边）
- 战斗失败流程

---

## 根本原因分析

### 初步诊断

`ResetAllEnemyAlertLevels()` 在 `OnCombatEnd()` 中调用，将所有敌人的 `AlertLevel` 清零，理论上应该防止立刻再次触发。

### 深入调查

警觉度清零之后，`VisionConeDetector.UpdateDetection()` 在下一帧继续执行：

```
RestoreEntityAfterCombat() → SetActive(true) → AI 恢复
VisionConeDetector.UpdateDetection() 检测到玩家（玩家还在原地）
→ AlertLevel 快速升高（玩家在视野锥内，速率×1.5）
→ EnemyIdleState.OnUpdate() 检测到 IsPlayerDetected
→ ChangeState(EnemyAIState.Alert)
→ Alert 状态积累满警觉度
→ TriggerCombat()
```

整个流程在几帧内完成，玩家完全来不及移开。

### 根本原因

**根本原因**：`AlertLevel` 被清零后，如果玩家仍处于敌人检测范围内（玩家位置未改变），下一帧就会重新开始积累，几帧内即达到触发阈值。`ResetAlert()` 只解决了"警觉值"的问题，没有解决"玩家还在原地"的问题。

**为什么之前没有发现**：重构前敌人在战斗失败时未被恢复，不存在二次触发的机会。

---

## 修复方案

### 修复思路

给 `EnemyEntity` 增加"战斗冷却期"。`ExitCombat()` 时记录一个截止时间，Idle 和 Patrol 状态在冷却期内跳过玩家检测逻辑，让玩家有时间离开。

注意：后续将改成当玩家离开战斗时，给玩家一个持续20s的无法被敌人发现的效果，来实现让玩家能够主动离开的需求

### 代码修改

**修改的文件**:
```
Assets/AAAGame/Scripts/Game/Explore/Enemy/Core/EnemyEntity.cs
Assets/AAAGame/Scripts/Game/Explore/Enemy/State/EnemyIdleState.cs
Assets/AAAGame/Scripts/Game/Explore/Enemy/State/EnemyPatrolState.cs
```

**代码对比**:

```csharp
// ✅ EnemyEntity — 新增字段和属性
private float m_CombatCooldownEndTime;
private const float k_CombatCooldownDuration = 5f;

public bool IsCombatCooldownOver => Time.time >= m_CombatCooldownEndTime;

// ExitCombat() 中设置冷却
public void ExitCombat()
{
    m_IsInCombat = false;
    m_CombatCooldownEndTime = Time.time + k_CombatCooldownDuration; // ✅ 新增
    // ...
}
```

```csharp
// ❌ 修改前 — EnemyIdleState.OnUpdate()
if (m_AI.IsPlayerDetected)
{
    m_AI.ChangeState(EnemyAIState.Alert);
    return;
}

// ✅ 修改后
if (m_AI.IsPlayerDetected && m_AI.Entity.IsCombatCooldownOver)
{
    m_AI.ChangeState(EnemyAIState.Alert);
    return;
}
```

```csharp
// ❌ 修改前 — EnemyPatrolState.OnUpdate()
if (m_AI.IsPlayerDetected)
{
    m_AI.ChangeState(EnemyAIState.Alert);
    return;
}

// ✅ 修改后
if (m_AI.IsPlayerDetected && m_AI.Entity.IsCombatCooldownOver)
{
    m_AI.ChangeState(EnemyAIState.Alert);
    return;
}
```

### 冷却期的影响

- 冷却期 5 秒内：`EnemyIdleState` 和 `EnemyPatrolState` 不响应玩家，不切换到 Alert 状态
- 5 秒后：恢复正常检测，玩家若在范围内则正常触发警觉
- 冷却期内 `VisionConeDetector` 仍在工作（AlertLevel 仍在累积），但 AI 状态机层面被拦截，不会切换到 Alert

## 验证方法

### 测试步骤

1. 启动游戏，进入探索场景
2. 靠近敌人触发战斗
3. 故意失败战斗
4. 返回探索场景，观察是否立即进入下一场战斗
5. 在敌人旁边等待 5 秒，观察是否开始正常触发警觉

### 预期结果

- 返回后不立即触发战斗
- 5 秒冷却结束后敌人恢复正常检测行为

### 验证结果

- [x] 战败返回后不立即被触发战斗
- [x] 5 秒后恢复正常检测
- [x] 没有引入新问题

---

## 预防措施

### 代码审查重点

- 凡是实体从"禁用→启用"（`SetActive(false/true)` 或 `Destroy` + 重建）的场景，都需要考虑是否需要冷却期
- AI 状态机的玩家检测应区分"可以进攻"和"已检测到玩家"两个概念

### 文档更新

- `EnemyEntity` 中的 `k_CombatCooldownDuration` 常量可配置化（移入 EnemyEntityTable）

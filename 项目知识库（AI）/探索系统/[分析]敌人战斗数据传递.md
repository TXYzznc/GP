### 2. 保存敌人引用

**位置**：`EnemyEntityManager.TriggerCombat()`

```csharp
public void TriggerCombat(EnemyEntity entity)
{
    // ⭐ 保存当前触发战斗的敌人实体
    m_CurrentCombatEnemy = entity;
    m_IsInCombat = true;

    // 让敌人进入战斗状态
    entity.EnterCombat();

    DebugEx.LogModule("EnemyEntityManager", 
        $"触发战斗: {entity.Config.Name}, BattleConfigId={entity.Config.BattleConfigId}");

    // 切换到战斗准备状态
    SwitchToCombatPreparation();
}
```

**关键点**：
- `m_CurrentCombatEnemy` 保存了触发战斗的敌人实体引用
- 这个引用在整个战斗过程中都可以访问
- 敌人实体包含 `Config`（EnemyEntityTable），其中有 `BattleConfigId`

### 4. 加载敌人波次配置

**位置**：`CombatPreparationState.LoadEnemyWaveConfig()`

```csharp
private void LoadEnemyWaveConfig()
{
    // ⭐ 从 EnemyEntityManager 获取当前触发战斗的敌人
    var enemyManager = EnemyEntityManager.Instance;
    if (enemyManager != null && enemyManager.CurrentCombatEnemy != null)
    {
        // ⭐ 获取 BattleConfigId（对应 EnemyTable 的 ID）
        int battleConfigId = enemyManager.CurrentCombatEnemy.Config.BattleConfigId;
        
        // ⭐ 加载敌人波次配置
        EnemySpawnManager.Instance.LoadWaveFromConfig(battleConfigId);

        DebugEx.LogModule("CombatPreparationState",
            $"从敌人实体加载战斗配置: {enemyManager.CurrentCombatEnemy.Config.Name}, BattleConfigId={battleConfigId}");
    }
    else
    {
        // 如果没有敌人实体，使用默认配置
        DebugEx.WarningModule("CombatPreparationState", "未找到触发战斗的敌人，使用默认配置");
        int waveId = 1;
        EnemySpawnManager.Instance.LoadWaveFromConfig(waveId);
    }
}
```

**关键点**：
- 通过 `EnemyEntityManager.CurrentCombatEnemy` 获取敌人实体
- 从敌人实体的 `Config.BattleConfigId` 获取战斗配置ID
- 这个 `BattleConfigId` 对应 `EnemyTable` 的 ID

---

### 5. 从配置表加载敌人数据

**位置**：`EnemySpawnManager.LoadWaveFromConfig()`

```csharp
public void LoadWaveFromConfig(int waveId)
{
    LoadFromEnemyTable(waveId);
}

private void LoadFromEnemyTable(int enemyId)
{
    // ⭐ 从 EnemyTable 读取配置
    var dataTable = GF.DataTable.GetDataTable<EnemyTable>();
    if (dataTable == null)
    {
        DebugEx.ErrorModule("EnemySpawnManager", "EnemyTable 未加载");
        return;
    }

    var enemyData = dataTable.GetDataRow(enemyId);
    if (enemyData == null)
    {
        DebugEx.ErrorModule("EnemySpawnManager", $"未找到敌人配置: ID={enemyId}");
        return;
    }

    // ⭐ 创建波次配置
    m_CurrentWave = new EnemyWaveConfig
    {
        WaveId = enemyId,
        EnemyChessIds = new List<int>(enemyData.ChessIds),  // ⭐ 棋子ID列表
        FormationType = enemyData.FormationType,            // 阵型类型
        Spacing = enemyData.Spacing                         // 间距
    };

    DebugEx.LogModule("EnemySpawnManager",
        $"从 EnemyTable 加载配置: {enemyData.EnemyName}, 棋子数量={enemyData.ChessIds.Length}");
}
```

**关键点**：
- `EnemyTable` 包含敌人的战斗配置
- `ChessIds` 是敌人出战的棋子ID数组
- `FormationType` 和 `Spacing` 定义了敌人的站位

## 配置表结构

### EnemyEntityTable（敌人实体配置）

**文件**：`Assets/AAAGame/DataTable/EnemyEntityTable.txt`

```
EnemyID  Name          BattleConfigId  ...
1        后羿敌人测试   1               ...
```

**字段说明**：
- `EnemyID`：敌人实体ID
- `Name`：敌人名称
- `BattleConfigId`：关联的战斗配置ID（对应 EnemyTable 的 ID）

---

### EnemyTable（敌人战斗配置）

**文件**：`Assets/AAAGame/DataTable/EnemyTable.txt`

```
ID  EnemyName  ChessIds      FormationType  Spacing  ...
1   后羿       [1001,1002]   1              2.0      ...
```

**字段说明**：
- `ID`：战斗配置ID（对应 EnemyEntityTable 的 BattleConfigId）
- `EnemyName`：敌人名称
- `ChessIds`：敌人出战的棋子ID数组
- `FormationType`：阵型类型（1=横排，2=竖排，3=矩形）
- `Spacing`：棋子间距（米）
- `WaveCount`：波次数量
- `DifficultyMultiplier`：难度倍率
- `RewardMultiplier`：奖励倍率

## 数据传递的关键点

### 1. 敌人实体不会被销毁

**重要**：敌人实体在触发战斗后**不会立即销毁**！

```csharp
// EnemyEntityManager.TriggerCombat()
m_CurrentCombatEnemy = entity;  // ⭐ 保存引用，不销毁对象
entity.EnterCombat();           // 只是进入战斗状态
```

**销毁时机**：
- 战斗胜利后：`EnemyEntityManager.OnCombatEnd(true)` 中销毁
- 战斗失败后：敌人返回巡逻状态，不销毁

---

### 2. 数据传递是通过引用

```
EnemyEntity (GameObject)
    ↓ (引用保存)
EnemyEntityManager.m_CurrentCombatEnemy
    ↓ (读取配置)
EnemyEntityTable.BattleConfigId
    ↓ (查询配置表)
EnemyTable.ChessIds
    ↓ (生成棋子)
敌人棋子实例
```

---

### 3. 配置表的关联关系

```
EnemyEntityTable (探索场景的敌人)
    ├─ EnemyID: 1
    └─ BattleConfigId: 1  ──┐
                            │
                            ↓
EnemyTable (战斗配置)       │
    ├─ ID: 1  ←─────────────┘
    └─ ChessIds: [1001, 1002]
```

## 潜在问题和注意事项

### 问题 1：敌人实体被提前销毁

**症状**：`CurrentCombatEnemy` 为 null，无法加载战斗配置

**原因**：
- 场景切换时敌人被销毁
- 敌人实体没有设置 `DontDestroyOnLoad`

**解决方案**：
- 确保 `EnemyEntityManager` 在切换到战斗准备状态前保存了敌人引用
- 或者在触发战斗时立即保存 `BattleConfigId`，而不是保存实体引用

---

### 问题 2：配置表关联错误

**症状**：找不到战斗配置，或者生成了错误的敌人

**原因**：
- `EnemyEntityTable.BattleConfigId` 与 `EnemyTable.ID` 不匹配
- 配置表数据错误

**解决方案**：
- 检查配置表数据的一致性
- 添加配置验证逻辑

---

### 问题 3：战斗结束后敌人状态错误

**症状**：战斗结束后敌人还在战斗状态

**原因**：
- `OnCombatEnd()` 没有正确处理敌人状态

**解决方案**：
```csharp
public void OnCombatEnd(bool playerWin)
{
    if (playerWin && m_CurrentCombatEnemy != null)
    {
        // 胜利：销毁敌人
        UnregisterEntity(m_CurrentCombatEnemy);
        Destroy(m_CurrentCombatEnemy.gameObject);
    }
    else if (!playerWin && m_CurrentCombatEnemy != null)
    {
        // 失败：让敌人返回巡逻
        m_CurrentCombatEnemy.ExitCombat();
    }

    m_CurrentCombatEnemy = null;
    m_IsInCombat = false;
}
```

## 优化建议

### 建议 1：保存配置ID而不是实体引用

**当前方案**：保存 `EnemyEntity` 引用

**问题**：
- 实体可能被销毁
- 依赖 GameObject 的生命周期

**优化方案**：
```csharp
// EnemyEntityManager
private int m_CurrentBattleConfigId;  // 只保存配置ID

public void TriggerCombat(EnemyEntity entity)
{
    m_CurrentBattleConfigId = entity.Config.BattleConfigId;  // 保存ID
    m_CurrentCombatEnemy = entity;  // 仍然保存引用（用于战斗结束处理）
    
    // ...
}
```

---

### 建议 2：添加战斗数据验证

```csharp
private void LoadEnemyWaveConfig()
{
    var enemyManager = EnemyEntityManager.Instance;
    if (enemyManager != null && enemyManager.CurrentCombatEnemy != null)
    {
        int battleConfigId = enemyManager.CurrentCombatEnemy.Config.BattleConfigId;
        
        // ⭐ 验证配置是否存在
        var dataTable = GF.DataTable.GetDataTable<EnemyTable>();
        if (dataTable != null && dataTable.GetDataRow(battleConfigId) != null)
        {
            EnemySpawnManager.Instance.LoadWaveFromConfig(battleConfigId);
        }
        else
        {
            DebugEx.ErrorModule("CombatPreparationState", 
                $"战斗配置不存在: BattleConfigId={battleConfigId}");
            // 使用默认配置
            EnemySpawnManager.Instance.LoadWaveFromConfig(1);
        }
    }
}
```

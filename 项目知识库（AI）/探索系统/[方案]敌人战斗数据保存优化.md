## 设计思路

### 当前问题

**现状**：
- `EnemyEntityManager` 保存 `CurrentCombatEnemy` 引用（GameObject）
- 战斗准备阶段需要从这个引用读取 `BattleConfigId`
- 但敌人实体需要在战斗准备阶段销毁

**冲突**：
- 如果提前销毁敌人实体，引用会变成 null
- 如果不销毁，战斗场景中会有多余的敌人实体

---

### 解决方案

**核心思路**：保存战斗配置数据，而不是保存实体引用

```csharp
// 不保存实体引用
private EnemyEntity m_CurrentCombatEnemy;  // ❌

// 保存战斗配置数据
private EnemyCombatData m_CurrentCombatData;  // ✅
```

## 实现方案

### 1. 创建战斗数据结构

**新增类**：在 `EnemyEntityManager.cs` 中添加

```csharp
/// <summary>
/// 敌人战斗数据
/// 保存触发战斗时的敌人配置信息
/// </summary>
public class EnemyCombatData
{
    /// <summary>敌人实体配置ID</summary>
    public int EntityConfigId { get; set; }

    /// <summary>敌人名称</summary>
    public string EnemyName { get; set; }

    /// <summary>战斗配置ID（对应 EnemyTable 的 ID）</summary>
    public int BattleConfigId { get; set; }

    /// <summary>敌人类型</summary>
    public EnemyType EnemyType { get; set; }

    /// <summary>奖励等级</summary>
    public int RewardTier { get; set; }

    /// <summary>净化后获得的卡牌ID（如果可净化）</summary>
    public int PurificationCardId { get; set; }

    /// <summary>触发战斗的位置</summary>
    public Vector3 TriggerPosition { get; set; }

    /// <summary>
    /// 从敌人实体创建战斗数据
    /// </summary>
    public static EnemyCombatData FromEntity(EnemyEntity entity)
    {
        if (entity == null || entity.Config == null)
            return null;

        return new EnemyCombatData
        {
            EntityConfigId = entity.EntityConfigId,
            EnemyName = entity.Config.Name,
            BattleConfigId = entity.Config.BattleConfigId,
            EnemyType = entity.EnemyType,
            RewardTier = entity.Config.RewardTier,
            PurificationCardId = entity.Config.PurificationCardId,
            TriggerPosition = entity.transform.position
        };
    }
}
```

public void TriggerCombat(EnemyEntity entity)
{
    if (entity == null)
    {
        DebugEx.ErrorModule("EnemyEntityManager", "触发战斗失败：敌人实体为空");
        return;
    }

    if (m_IsInCombat)
    {
        DebugEx.WarningModule("EnemyEntityManager", "已经在战斗中，忽略触发请求");
        return;
    }

    // ⭐ 保存战斗数据（不依赖实体引用）
    m_CurrentCombatData = EnemyCombatData.FromEntity(entity);
    if (m_CurrentCombatData == null)
    {
        DebugEx.ErrorModule("EnemyEntityManager", "创建战斗数据失败");
        return;
    }

    // 临时保存实体引用（用于销毁）
    m_CurrentCombatEnemy = entity;
    m_IsInCombat = true;

    // 让敌人进入战斗状态
    entity.EnterCombat();

    DebugEx.LogModule("EnemyEntityManager", 
        $"触发战斗: {m_CurrentCombatData.EnemyName}, BattleConfigId={m_CurrentCombatData.BattleConfigId}");

/// 销毁当前战斗的敌人实体（战斗准备阶段调用）

public void DestroyCurrentCombatEnemy()
{
    if (m_CurrentCombatEnemy != null)
    {
        DebugEx.LogModule("EnemyEntityManager", 
            $"销毁敌人实体: {m_CurrentCombatEnemy.Config.Name}");

        // 从列表中移除
        UnregisterEntity(m_CurrentCombatEnemy);

        // 销毁 GameObject
        Destroy(m_CurrentCombatEnemy.gameObject);

        // 清除引用
        m_CurrentCombatEnemy = null;
    }
}

## 数据流向（优化后）

```
1. 触发战斗
   EnemyEntity → EnemyEntityManager.TriggerCombat()
   ↓
2. 保存战斗数据
   EnemyCombatData.FromEntity(entity)
   ↓ 保存到
   EnemyEntityManager.m_CurrentCombatData
   ↓
3. 切换到战斗准备状态
   CombatPreparationState.OnEnter()
   ↓
4. 生成战斗场地
   SpawnBattleArena()
   ↓
5. 销毁敌人实体
   EnemyEntityManager.DestroyCurrentCombatEnemy()
   ↓
6. 加载战斗配置
   LoadEnemyWaveConfig() 使用 CurrentCombatData.BattleConfigId
   ↓
7. 生成敌人棋子
   EnemySpawnManager.SpawnWaveAsync()
```

## 优势

### 1. 数据独立性
- 战斗数据不依赖 GameObject 的生命周期
- 敌人实体可以随时销毁，不影响战斗流程

### 2. 清晰的职责分离
- `EnemyEntity`：探索场景中的敌人实体
- `EnemyCombatData`：战斗配置数据
- 战斗棋子：战斗场景中的敌人表现

### 3. 便于扩展
- 可以轻松添加更多战斗相关数据
- 支持群体战斗（多个敌人的数据）
- 支持战斗回放、战斗记录等功能

# 敌人棋子数据重构 系统设计

> **最后更新**: 2026-03-25
> **状态**: 有效
> **标签**: #enemy #chess #combat #explore

---

## 问题陈述

**重构前的问题**：
- 敌人棋子数据走 `GlobalChessManager`，与玩家棋子共用一套管理，数据互相污染
- 同一场景中两个同类型的敌人（相同 ChessId）会共享数据，HP 不独立
- 玩家战斗失败后，敌人实体被销毁重建，无法保留当前 HP（每次都满血）
- 战斗结束后不区分胜负，敌人实体统一销毁，失败方无法恢复

---

## 解决方案概述

为每个 `EnemyEntity` 分配 GUID，以 `"{guid}_{slotIndex}"` 为 key 在 `EnemyChessDataManager` 中独立存储每个棋子的 HP 数据。战斗结束时根据胜负决定是否销毁敌人实体，HP 回写分发到对应管理器（玩家→`GlobalChessManager`，敌人→`EnemyChessDataManager`）。

## 系统架构

### 整体架构图

```
探索场景
┌─────────────────────────────────────────────┐
│  EnemyEntity (guid=AAA)                     │
│  ├── 棋子槽0: ChessId=9, key="AAA_0"        │
│  └── 棋子槽1: ChessId=9, key="AAA_1"        │
│                                             │
│  EnemyEntity (guid=BBB) ← 同类型，独立数据   │
│  ├── 棋子槽0: ChessId=9, key="BBB_0"        │
│  └── 棋子槽1: ChessId=9, key="BBB_1"        │
└─────────────────────────────────────────────┘
            ↓ 战斗触发
┌─────────────────────────────────────────────┐
│  EnemyChessDataManager (纯C#单例)            │
│  Dictionary<string, EnemyChessState>        │
│  "AAA_0" → {HP=87, MaxHP=100}               │
│  "AAA_1" → {HP=0,  MaxHP=100, IsDead=true}  │
└─────────────────────────────────────────────┘
            ↓ 战斗中棋子读取
┌─────────────────────────────────────────────┐
│  BattleChessManager                         │
│  BattleChessData: Camp + EnemyKey           │
│  WriteBackHp: Camp==1 → EnemyChessDataManager│
│              Camp==0 → GlobalChessManager   │
└─────────────────────────────────────────────┘
```

## 设计细节

### 1. GUID 隔离独立实例

**设计目标**：同场景中两个使用相同 ChessId 的敌人，棋子数据完全独立。

**实现方式**：`EnemyEntity.Awake()` 生成一次性 GUID，以 `"{guid}_{slotIndex}"` 为唯一 key。

```csharp
// EnemyEntity.Awake()
m_EntityGuid = Guid.NewGuid().ToString();

// EnemyChessDataManager key
public static string BuildKey(string entityGuid, int slotIndex)
    => $"{entityGuid}_{slotIndex}";
```

**权衡**：GUID 字符串略有开销，但棋子数量有限（通常 2-4 个），可接受。

---

### 2. 注册时跳过已有数据（HP 持久化核心）

**设计目标**：第二次战斗时不覆盖上次战斗留下的 HP。

**实现方式**：`Register()` 中检查 key 是否已存在，已存在则跳过。

```csharp
public void Register(string entityGuid, int slotIndex, int chessId, double maxHp)
{
    string key = BuildKey(entityGuid, slotIndex);
    if (m_Data.ContainsKey(key))
        return; // 已有历史数据，保留不覆盖
    m_Data[key] = new EnemyChessState(chessId, maxHp);
}
```

---

### 3. 战斗结束分流：隐藏 vs 销毁

**设计目标**：玩家失败后敌人保留，HP 数据也保留；玩家胜利后敌人销毁，数据清理。

**实现方式**：`EnemyEntityManager.OnCombatEnd(playerWin)` 分支处理。

```csharp
public void OnCombatEnd(bool playerWin)
{
    ResetAllEnemyAlertLevels(); // 两种结果都重置警觉度

    foreach (var entity in m_CurrentCombatEntities)
    {
        if (playerWin)
            Destroy(entity.gameObject); // OnDestroy 自动清理 EnemyChessDataManager
        else
            RestoreEntityAfterCombat(entity); // SetActive(true) + 恢复 AI
    }
    // ...
}
```

---

### 4. HP 回写按 Camp 分发

**设计目标**：玩家棋子 HP 走 `GlobalChessManager`，敌人棋子 HP 走 `EnemyChessDataManager`。

**实现方式**：`BattleChessData` 扩展 `Camp`（0=玩家，1=敌人）和 `EnemyKey` 字段，`BattleChessManager.WriteBackHp()` 按 Camp 分发。

```csharp
private void WriteBackHp(int chessId, double hp, int camp)
{
    if (camp == 1)
    {
        if (m_BattleDataDict.TryGetValue(chessId, out var data)
            && !string.IsNullOrEmpty(data.EnemyKey))
            EnemyChessDataManager.Instance.UpdateHp(data.EnemyKey, hp);
    }
    else
    {
        GlobalChessManager.Instance.UpdateChessHP(chessId, hp);
    }
}
```

---

### 5. 生成棋子时继承历史 HP

**设计目标**：第二次战斗时，敌人棋子从上次战斗留下的 HP 值开始。

**实现方式**：`EnemySpawnManager.SpawnWaveAsync()` 在每个棋子生成后调用 `ApplyHistoricalHp()`。

```csharp
private void ApplyHistoricalHp(ChessEntity entity, int chessId, int slotIndex)
{
    if (string.IsNullOrEmpty(m_CurrentEnemyGuid)) return;
    string key = EnemyChessDataManager.BuildKey(m_CurrentEnemyGuid, slotIndex);
    BattleChessManager.Instance.SetEnemyKeyForChess(chessId, key);
    var state = EnemyChessDataManager.Instance.GetState(key);
    if (state != null) entity.Attribute.SetHp(state.CurrentHp);
}
```

## 数据流

```
【初始化】
EnemyEntity.Awake() → 生成 GUID
EnemyEntity.Initialize() → RegisterChessData()
  → 读 EnemyTable.ChessIds
  → 读 SummonChessTable.MaxHp
  → EnemyChessDataManager.Register(guid, slotIndex, chessId, maxHp)
     └─ 若 key 已存在则跳过（保留历史 HP）

【进入战斗】
CombatTriggerManager.TriggerCombat()
  → EnemyEntityManager.TriggerCombat(entity)
  → HideEntityForCombat() → SetActive(false)
  → CombatPreparationState
  → EnemySpawnManager.LoadFromEnemyTable(battleConfigId, enemyGuid)
  → SpawnWaveAsync() → ApplyHistoricalHp() 继承 HP

【战斗中】
ChessEntity 受伤 → BattleChessManager 记录
战斗结束 → BattleChessManager.OnBattleEnd()
  → WriteBackHp(chessId, hp, camp)
     Camp==1 → EnemyChessDataManager.UpdateHp(key, hp)
     Camp==0 → GlobalChessManager.UpdateChessHP(chessId, hp)

【战斗结束】
EnemyEntityManager.OnCombatEnd(playerWin)
  → ResetAllEnemyAlertLevels()
  → playerWin=true  → Destroy(entity.gameObject) → OnDestroy 清理 EnemyChessDataManager
  → playerWin=false → RestoreEntityAfterCombat() → SetActive(true) + ExitCombat()
```

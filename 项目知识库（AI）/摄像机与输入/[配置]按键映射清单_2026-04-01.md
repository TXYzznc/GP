## 一、游戏内正式按键

| 按键 | 功能 | 触发属性 / 处理位置 | 备注 |
|------|------|-------------------|------|
| **W / A / S / D** | 角色移动 | `PlayerInputManager.Move`（Vector2）| 通过 `KeyboardInputSource.GetMove()` |
| **鼠标移动** | 视角旋转 | `PlayerInputManager.MouseDelta` | 锁定光标时生效 |
| **鼠标滚轮** | 调整摄像机距离 | `PlayerInputManager.ScrollDelta` | 第三人称缩放 |
| **Shift（左/右）** | 冲刺 | `PlayerInputManager.SprintDown` | GetKeyDown，单次触发 |
| **鼠标右键** | 右键交互（拾取/使用） | `PlayerInputManager.RightMouseButtonDown` | `WorldItemPickup` 使用 |
| **Space** | 主动触发战斗 | `CombatOpportunityDetector.Update()` | 靠近敌人时出现交互提示后按下 |
| **Tab** | 开/关背包 | `PlayerInputManager.InventoryToggleTriggered` → `ExplorationState.ToggleInventory()` | 同时切换光标锁定 |
| **G** | 开/关仓库 | `PlayerInputManager.WarehouseToggleTriggered` → `ExplorationState.ToggleWarehouse()` | 同时切换光标锁定 |
| **I** | 切换视角模式（第三人称↔俯视） | `PlayerInputManager.ViewModeSwitchTriggered` | `ThirdPersonCamera` 响应 |
| **Q** | 召唤师技能槽 1 | `PlayerInputManager.summonerSkillDown[1]` | |
| **E** | 召唤师技能槽 2 | `PlayerInputManager.summonerSkillDown[2]` | |
| **R** | 召唤师技能槽 3 | `PlayerInputManager.summonerSkillDown[3]` | |
| **1** | 快捷栏格子 1 / 技能槽 1 | `PlayerInputManager.GetHotbarKeyDown(1)` / `KeyboardInputSource.GetSkillDown(1)` | 注意：背包快捷栏和技能槽共用数字键，需确认上下文 |
| **2** | 快捷栏格子 2 / 技能槽 2 | `PlayerInputManager.GetHotbarKeyDown(2)` / `KeyboardInputSource.GetSkillDown(2)` | 同上 |
| **3** | 快捷栏格子 3 / 技能槽 3 | `PlayerInputManager.GetHotbarKeyDown(3)` / `KeyboardInputSource.GetSkillDown(3)` | 同上 |
| **4** | 快捷栏格子 4 | `PlayerInputManager.GetHotbarKeyDown(4)` | |
| **5** | 快捷栏格子 5 | `PlayerInputManager.GetHotbarKeyDown(5)` | |
| **Escape** | 关闭最顶层 UI | `UIFormBase.OnUpdate()`（`isOnEscape=true` 时生效）| GF.UI.GetTopUIFormId() 匹配 |

## 三、已知冲突 / 注意事项

| 冲突点 | 说明 |
|--------|------|
| **数字键 1-3 双重绑定** | 同时映射到背包快捷栏（`GetHotbarKeyDown`）和技能槽（`GetSkillDown`）。目前背包快捷栏在 `InventoryUI.OnUpdate` 处理，技能在 `PlayerSkillManager` 处理，需确认两者是否会同时响应。 |
| **R 键冲突** | `GameTestManager` 的 R 键测试与 `ProjectileTestController` 的 R 键重置均直接读 `Input.GetKeyDown`，未走 PlayerInputManager，可能同时触发。 |
| **Space 键冲突** | `CombatOpportunityDetector`、`ChessTestInput`、`ProjectileTestController` 均直接读 Space 键，未走 PlayerInputManager。 |

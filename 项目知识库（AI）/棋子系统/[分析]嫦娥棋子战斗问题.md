## 问题分析

### 根本原因

通过深入分析日志、代码和配置表，发现了问题的真正根源：

**后羿被动技能导致的攻击范围增加**：

#### 攻击范围数值分析

**配置表基础数值**：
- 后羿：`AtkRange = 5`
- 嫦娥：`AtkRange = 4`

**运行时实际数值**：
- 后羿：35.00（5 + 30 = 35）
- 嫦娥：4.00（无修改）

#### 后羿被动技能：日月长弓

**技能机制**：
1. **被动技能**：`HouyiPassive`（ID=11）
2. **触发条件**：白天时自动激活
3. **效果**：添加"日月长弓"Buff（ID=5）
4. **Buff效果**：
   - 攻击力 +15%
   - **攻击范围 +30**

**代码实现**：
```csharp
// HouyiPassive.cs 中的逻辑
if (m_IsDay && !m_DayBuffApplied)
{
    // 白天：添加日月长弓Buff（伤害+15%，射程+30）
    m_Ctx.BuffManager.AddBuff(5, m_Ctx.Owner, m_Ctx.Attribute);
    m_DayBuffApplied = true;
}

// StatModBuff.cs 中的Buff定义
public static StatModBuff CreateSunsetBow()
{
    var buff = new StatModBuff();
    buff.SetMods(
        new StatMod(StatType.AtkDamage, 0.15, true),  // 攻击力+15%
        new StatMod(StatType.AtkRange, 30, false)     // 攻击范围+30
    );
    return buff;
}
```

### 嫦娥战斗问题的真相

**实际情况**：
- 嫦娥攻击范围：4.00
- 后羿攻击范围：35.00（含被动加成）
- 两者距离：约11.74米
- **结果**：嫦娥无法攻击到后羿，但后羿可以轻松攻击嫦娥

**系统工作正常**：
- 阵营关系正确：嫦娥（阵营0）vs 后羿（阵营1）= 敌对
- AI逻辑正确：正确跳过超出攻击范围的敌人
- 被动技能正常：后羿的日月长弓被动按设计工作

## 技术分析

### 敌人识别逻辑

在 `FSMRangedAI.FindTarget()` 方法中：

```csharp
// 检查阵营关系
CampRelation relation = CampRelationService.GetRelation(myCamp, chess.Camp);
if (relation != CampRelation.Enemy)
    continue;
```

这个逻辑是正确的，应该能识别敌对关系。

### 阵营关系判断

在 `CampRelationService.GetRelation()` 方法中：

```csharp
public static CampRelation GetRelation(int campA, int campB)
{
    // 同阵营
    if (campA == campB)
    {
        return CampRelation.Ally;
    }

    // 检查是否为敌对关系
    if (s_EnemyRelations.TryGetValue(campA, out var enemies))
    {
        if (enemies.Contains(campB))
        {
            return CampRelation.Enemy;
        }
    }

    // 默认为中立
    return CampRelation.Neutral;
}
```

按照配置，阵营0和阵营1应该互为敌人。

## 解决方案

### 方案1：调整嫦娥攻击范围（推荐）

**问题定位**：嫦娥的基础攻击范围4太小，无法与拥有被动加成的后羿抗衡

**建议修改**：在 `Assets/AAAGame/DataTable/SummonChessTable.txt` 中：
- 将嫦娥的 `AtkRange` 从 4 调整为 15-20
- 这样即使后羿有30点被动加成，嫦娥也能有一定的战斗能力

### 方案2：为嫦娥添加被动技能

**设计思路**：给嫦娥也添加一个增加攻击范围的被动技能
- 可以设计为"月华之力"：夜晚时攻击范围+20
- 与后羿的"日月长弓"形成昼夜对比

### 方案3：调整后羿被动技能

**平衡调整**：降低日月长弓的攻击范围加成
- 将攻击范围加成从+30调整为+15或+20
- 在 `StatModBuff.CreateSunsetBow()` 中修改数值

### 方案4：临时测试修正

如需快速验证，可临时修改 `ChessAttribute.Initialize()`：

```csharp
// 临时平衡调整

{
    m_AtkRange = 15.0; // 临时提升嫦娥攻击范围
    DebugEx.Warning("ChessAttribute", $"临时修正嫦娥攻击范围：{config.AtkRange} → {m_AtkRange}");
}
else
{
    m_AtkRange = config.AtkRange;
}
```

## 问题总结

**实际问题**：后羿拥有"日月长弓"被动技能，白天时攻击范围增加30点（5+30=35），而嫦娥没有类似被动，攻击范围仅为4点，导致战斗不平衡

**误判过程**：
1. 最初以为是阵营关系问题 → 验证后发现阵营关系正确
2. 然后以为是配置表数值问题 → 发现配置表数值正常
3. 最终发现是被动技能机制导致的数值差异

**技术细节**：
- 后羿被动：`HouyiPassive` → 添加Buff ID=5 → `StatModBuff.CreateSunsetBow()` → 攻击范围+30
- 系统设计：通过Buff系统动态修改属性，这是正常的游戏机制
- 平衡问题：嫦娥缺乏对应的被动技能来平衡后羿的优势

**解决方向**：游戏平衡调整，而非Bug修复
**影响范围**：主要影响嫦娥vs后羿的战斗平衡性

## 一、攻击类型说明

| 类型值 | 枚举名 | 说明 | 适用场景 |
|--------|--------|------|----------|
| 0 | Instant | 瞬发命中 | 自动战斗普攻、锁定类技能 |
| 1 | Melee | 近战碰撞 | 挥砍、刺击等近战攻击 |
| 2 | Projectile | 投射物 | 弓箭、火球、飞刀等远程攻击 |
| 3 | AOE | 范围检测 | AOE技能、爆炸、范围治疗 |
| 4 | Raycast | 射线检测 | 激光、狙击、穿透攻击 |

## 二、配置表字段

### 2.1 SummonChessTable（棋子表）

需要新增以下字段：

| 字段名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| AttackHitType | int | 0 | 普攻命中类型（见攻击类型说明） |
| ProjectilePrefabId | int | 0 | 投射物预制体ID（AttackHitType=2时使用） |

**配置示例：**

| Id | Name | AIType | AttackHitType | ProjectilePrefabId |
|----|------|--------|---------------|-------------------|
| 1001 | 剑士 | 1 | 1 | 0 |
| 1002 | 战士 | 1 | 1 | 0 |
| 2001 | 弓箭手 | 2 | 2 | 101 |
| 2002 | 法师 | 2 | 2 | 102 |
| 3001 | 稻草人 | 3 | 0 | 0 |

### 2.1 SummonChessSkillTable（技能表）

**现有字段说明：**

| 字段名 | 类型 | 说明 |
|--------|------|------|
| ID | int | 技能唯一ID |
| Name | string | 技能名称 |
| SkillType | int | 技能类型：1=被动 2=普攻效果 3=主动技能 4=大招 |
| DamageType | int | 伤害类型：0=无伤害 1=物理 2=魔法 3=真实 |
| DamageCoeff | double | 伤害系数（基于攻击力/法强的百分比，如1.5=150%） |
| BaseDamage | double | 基础固定伤害值 |
| MpCost | double | 法力消耗 |
| Cooldown | double | 冷却时间（秒） |
| CastRange | double | 施法/生效范围 |
| AreaRadius | double | AOE半径（0=单体） |
| Duration | double | 持续时间（引导类技能） |
| HitCount | int | 命中次数（多段伤害） |
| BuffIds | int[] | 释放时附加给目标的BuffID数组 |
| SelfBuffIds | int[] | 释放时给自身附加的BuffID数组 |
| ProjectileSpeed | double | 投射物速度（0=即时命中） |
| IconPath | int | 技能图标资源ID |
| EffectPath | int | 技能特效资源ID（释放时播放） |
| HitEffectPath | int | 受击特效资源ID（命中时播放） |
| Desc | string | 技能描述 |

**命中类型推断规则：**
由于配置表没有显式的 `HitType` 字段，系统根据以下规则自动推断：

| 条件 | 推断类型 |
|------|----------|
| `ProjectileSpeed > 0` | Projectile（投射物） |
| `AreaRadius > 0` | AOE（范围） |
| `SkillType == 2`（普攻效果）且 `ProjectileSpeed == 0` | 根据棋子 AIType 决定（近战/瞬发） |
| 其他情况 | Instant（瞬发） |

**配置示例：**

| SkillId | SkillName | SkillType | HitType | AreaRadius | HitCount |
|---------|-----------|-----------|---------|------------|----------|
| 3001 | 火焰爆发 | 3 | 3 | 3.0 | 0 |
| 3002 | 穿透射线 | 3 | 4 | 0 | 3 |
| 3003 | 冲锋斩 | 3 | 1 | 0 | 1 |
| 3004 | 火球术 | 3 | 2 | 0 | 1 |

## 三、预制体配置

### 3.1 近战棋子预制体配置

近战棋子需要配置武器碰撞器：

**步骤：**
1. 打开棋子预制体
2. 找到武器骨骼节点（如 `Weapon` 或 `RightHand`）
3. 创建子物体 `WeaponCollider`
4. 添加碰撞器组件（推荐 `BoxCollider` 或 `CapsuleCollider`）
5. 设置碰撞器为 **Trigger** 模式
6. 添加 `ChessWeaponCollider` 组件
7. 调整碰撞器大小覆盖武器范围

**预制体结构示例：**
Chess_Warrior (预制体根节点) ├── Model │ ├── Armature │ │ ├── Hips │ │ │ └── ... │ │ └── RightHand │ │ └── Weapon │ │ └── WeaponCollider ← 添加在这里 │ │ ├── BoxCollider (isTrigger=true) │ │ └── ChessWeaponCollider │ └── Mesh └── 


**ChessWeaponCollider 组件说明：**
- 无需手动配置参数
- 运行时由 `MeleeHitDetector` 自动控制开关
- 碰撞器默认禁用，仅在攻击时启用

### 3.2 投射物预制体配置

远程棋子需要配置投射物预制体：

**步骤：**
1. 创建投射物预制体（如 `Arrow.prefab`）
2. 添加模型/特效
3. 添加碰撞器组件，设为 **Trigger** 模式
4. 添加 `ChessProjectile` 组件
5. 配置组件参数
6. 保存到 `Resources/Prefabs/Combat/` 目录

**预制体结构示例：**
Arrow (预制体根节点) ├── Model (箭矢模型) ├── Trail (拖尾特效，可选) ├── SphereCollider (isTrigger=true) └── ChessProjectile

**ChessProjectile 组件参数：**

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| Speed | float | 10 | 投射物飞行速度 |
| MaxLifetime | float | 5 | 最大存活时间（秒） |
| DestroyOnHit | bool | true | 命中后是否销毁 |
| IsPiercing | bool | false | 是否穿透（可命中多个目标） |

**投射物预制体命名规则：**
- 放置路径：`Resources/Prefabs/Combat/Projectile_{Id}.prefab`
- 或使用默认投射物：`Resources/Prefabs/Combat/DefaultProjectile.prefab`

### 3.3 投射物预制体ID对照表

| ProjectilePrefabId | 预制体路径 | 说明 |
|--------------------|-----------|------|
| 0 | DefaultProjectile | 默认投射物 |
| 101 | Projectile_101 | 普通箭矢 |
| 102 | Projectile_102 | 火球 |
| 103 | Projectile_103 | 冰锥 |
| 104 | Projectile_104 | 飞刀 |

## 四、动画配置

### 4.1 Animator Controller 配置

**添加参数：**
| 参数名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| AttackSpeed | Float | 1 | 攻击动画播放速度倍率 |
| IsMoving | Bool | false | 是否移动中 |
| Attack | Trigger | - | 触发攻击动画 |
| Skill1 | Trigger | - | 触发技能1动画 |
| Skill2 | Trigger | - | 触发技能2动画 |
| Death | Trigger | - | 触发死亡动画 |

**Attack 状态配置：**
1. 选中 Attack 状态
2. 在 Inspector 中找到 `Speed` 属性
3. 点击右侧齿轮图标，选择 `Parameter`
4. 选择 `AttackSpeed` 参数

### 4.2 Animation Event 配置

在攻击动画 Clip 中添加 Animation Event：

**普攻动画事件：**
| 帧位置 | 方法名 | 参数 | 说明 |
|--------|--------|------|------|
| 出手帧 | AnimEvent_AttackExecute | 无 | 触发攻击执行 |
| 挥砍结束帧 | AnimEvent_MeleeAttackEnd | 无 | 关闭武器碰撞（仅近战） |
| 最后一帧 | AnimEvent_AnimationComplete | "Attack" | 标记动画结束 |

**技能1动画事件：**
| 帧位置 | 方法名 | 参数 | 说明 |
|--------|--------|------|------|
| 释放帧 | AnimEvent_Skill1Execute | 无 | 触发技能1执行 |
| 最后一帧 | AnimEvent_AnimationComplete | "Skill1" | 标记动画结束 |

**技能2动画事件：**
| 帧位置 | 方法名 | 参数 | 说明 |
|--------|--------|------|------|
| 释放帧 | AnimEvent_Skill2Execute | 无 | 触发技能2执行 |
| 最后一帧 | AnimEvent_AnimationComplete | "Skill2" | 标记动画结束 |

**添加 Animation Event 步骤：**
1. 在 Project 窗口选中动画 Clip
2. 打开 Animation 窗口
3. 将时间轴拖到目标帧
4. 点击 `Add Event` 按钮
5. 在 Inspector 中选择方法名
6. 如有参数，填写参数值

### 4.3 ChessAnimationEventReceiver 组件

**重要：** 需要手动挂载此组件才能在编辑器中配置 Animation Event。

**挂载位置：** Animator 组件所在的 GameObject

**挂载步骤：**
1. 打开棋子预制体
2. 找到有 Animator 组件的 GameObject
3. 添加 `ChessAnimationEventReceiver` 组件
4. 保存预制体

## 五、Layer 配置

确保项目中有以下 Layer：

| Layer 名称 | 用途 |
|-----------|------|
| Chess | 棋子层（所有棋子） |
| Enemy | 敌人层（可选） |
| Player | 玩家层（可选） |
| Projectile | 投射物层（可选） |

**Physics 碰撞矩阵配置：**
- Chess 与 Chess 可碰撞
- Projectile 与 Chess 可碰撞
- 其他根据需要配置

## 七、常见问题

### Q1: 近战攻击没有命中效果
**可能原因：**
1. 没有添加 `ChessWeaponCollider` 组件
2. 碰撞器没有设为 Trigger
3. 没有添加 `AnimEvent_AttackExecute` 动画事件
4. Layer 配置不正确

### Q2: 投射物没有生成
**可能原因：**
1. `ProjectilePrefabId` 配置错误
2. 投射物预制体路径不正确
3. 投射物预制体没有 `ChessProjectile` 组件

### Q3: AOE 技能没有命中目标
**可能原因：**
1. `AreaRadius` 配置为 0
2. 目标 Layer 不在检测范围内
3. 目标阵营配置错误

### Q4: 动画事件没有触发
**可能原因：**
1. 没有挂载 `ChessAnimationEventReceiver` 组件
2. Animation Event 方法名拼写错误
3. 动画 Clip 没有正确配置事件

## 八、调试技巧

### 8.1 开启调试日志
系统默认输出调试日志，可在 Console 窗口查看：
- `[ChessCombatController]` - 战斗控制器日志
- `[HitDetector]` - 命中检测器日志
- `[ChessProjectile]` - 投射物日志

### 8.2 可视化调试
在 Scene 视图中：
- 武器碰撞器会显示绿色线框（启用时）
- 可使用 `Debug.DrawRay` 查看射线检测
- 可使用 `Gizmos.DrawWireSphere` 查看 AOE 范围

### 8.3 测试命令
在开发模式下，可使用 `ChessTestInput` 组件测试：
- 按 `1` 键：测试普攻
- 按 `2` 键：测试技能1
- 按 `3` 键：测试技能2

物品系统使用 4 个配置表来管理所有物品相关数据。配置表解析类由工具自动生成，位于 `Assets/AAAGame/Scripts/DataTable/` 目录。

## 配置表列表

### 1. ItemTable.txt - 物品配置表

**文件路径**：`Assets/AAAGame/DataTable/ItemTable.txt`

**字段说明**：

| 字段名 | 类型 | 说明 |
|:---|:---|:---|
| Id | int | 物品ID（分段：1xxxx消耗品，2xxxx任务道具，3xxxx宝物，4xxxx装备） |
| Name | string | 物品名称 |
| Type | int | 物品类型（1=消耗品，2=任务道具，3=宝物，4=装备） |
| Quality | int | 品质等级（1=普通，2=优秀，3=稀有，4=史诗，5=传说） |
| Description | string | 物品描述 |
| IconPath | string | 缩略图路径 |
| DetailIconPath | string | 详细图路径（为空则使用IconPath） |
| CanStack | int | 是否可堆叠（0=否，1=是） |
| MaxStackCount | int | 最大堆叠数量 |
| CanUse | int | 是否可使用（0=否，1=是） |
| UseEffectId | int | 使用效果ID（关联SpecialEffectTable） |
| CanEquip | int | 是否可装备（0=否，1=是） |
| EquipType | int | 装备类型（0=无，1=武器，2=防具，3=饰品，4=宝物） |
| SpecialEffectId | int | 特殊效果ID（关联SpecialEffectTable） |
| AffixPoolIds | string | 词条池ID列表（逗号分隔，关联AffixTable） |
| AffixCount | string | 词条数量范围（格式：最小-最大，如"1-3"） |
| SynergyIds | string | 羁绊ID列表（逗号分隔，关联SynergyTable） |
| BaseAttributes | string | 基础属性（JSON格式，如`{"Attack":50,"CritRate":10}`） |
| SellPrice | int | 售价 |

**示例数据**：
```
10001	钱包	1	2	使用后随机获得100-500金币	Items/Consumable/Wallet		1	99	1	1001	0	0	0			{}	50
30001	八尺琼勾玉	3	5	传说中的三神器之一	Items/Treasure/Magatama		0	1	0	0	1	4	3001	2001,2002,2003	1-3	3001	{}	1000
40001	石中剑	4	4	传说中的圣剑	Items/Equipment/Excalibur		0	1	0	0	1	1	4001		{}	500
```

### 2. SpecialEffectTable.txt - 特殊效果配置表

**文件路径**：`Assets/AAAGame/DataTable/SpecialEffectTable.txt`

**字段说明**：

| 字段名 | 类型 | 说明 |
|:---|:---|:---|
| Id | int | 效果ID |
| Name | string | 效果名称 |
| Description | string | 效果描述 |
| EffectType | int | 效果类型（1=消耗品效果，2=装备特效，3=宝物特效，4=羁绊效果） |
| EffectParams | string | 效果参数（JSON格式） |

**EffectParams JSON 格式示例**：

```json
// 随机金币
{"type":"AddGold","min":100,"max":500}

// 恢复生命
{"type":"RestoreHP","value":500}

// 死亡免疫
{"type":"DeathImmune","chance":0.3,"healPercent":0.5}

// 无视防御
{"type":"IgnoreDefense","percent":0.3}

// 双倍伤害
{"type":"DoubleDamage","chance":0.15}
```

**示例数据**：
```
1001	随机金币	随机获得100-500金币	1	{"type":"AddGold","min":100,"max":500}
3001	神力护佑	受到致命伤害时免疫	3	{"type":"DeathImmune","chance":0.3,"healPercent":0.5}
4001	王者之力	攻击时有15%概率造成双倍伤害	2	{"type":"DoubleDamage","chance":0.15}
```

### 3. AffixTable.txt - 词条配置表

**文件路径**：`Assets/AAAGame/DataTable/AffixTable.txt`

**字段说明**：

| 字段名 | 类型 | 说明 |
|:---|:---|:---|
| Id | int | 词条ID |
| Name | string | 词条名称 |
| Description | string | 词条描述（{0}为数值占位符） |
| AffixType | int | 词条类型（1=属性加成，2=特殊效果） |
| AttributeType | int | 属性类型（1=攻击力，2=生命值，3=暴击率，4=攻击速度，5=移动速度，6=防御力，7=魔法强度，0=全属性） |
| ValueType | int | 数值类型（1=固定值，2=百分比） |
| ValueRange | string | 数值范围（格式：最小-最大，如"10-50"） |
| Weight | int | 权重（随机抽取时使用，数值越大越容易抽到） |

**示例数据**：
```
2001	攻击力加成	攻击力+{0}	1	1	1	10-50	100
2003	暴击率加成	暴击率+{0}%	1	3	2	5-15	80
2010	全属性加成	全属性+{0}%	1	0	2	3-8	30
```

## 配置表使用流程

### 1. 编辑配置表

直接编辑 `.txt` 文件，使用 Tab 分隔字段。

### 2. 生成解析类

使用项目的配置表生成工具自动生成解析类：
- `ItemTable.cs`
- `SpecialEffectTable.cs`
- `AffixTable.cs`
- `SynergyTable.cs`

### 3. 加载配置表

在游戏启动时通过 GameFramework 加载配置表：

```csharp
// 在 ProcedureLaunch 或类似的启动流程中
GF.DataTable.LoadDataTable<ItemTable>("ItemTable");
GF.DataTable.LoadDataTable<SpecialEffectTable>("SpecialEffectTable");
GF.DataTable.LoadDataTable<AffixTable>("AffixTable");
GF.DataTable.LoadDataTable<SynergyTable>("SynergyTable");
```

### 4. 初始化 ItemManager

配置表加载完成后，调用 ItemManager 的加载方法：

```csharp
ItemManager.Instance.LoadItemTable();
ItemManager.Instance.LoadSpecialEffectTable();
ItemManager.Instance.LoadAffixTable();
ItemManager.Instance.LoadSynergyTable();
```

## 注意事项

1. **不要手动修改解析类**：ItemTable.cs 等文件由工具自动生成，手动修改会被覆盖
2. **字段顺序必须一致**：配置表文本文件的列顺序必须与解析类的 ParseDataRow 方法一致
3. **JSON 格式验证**：BaseAttributes 和 EffectParams 字段使用 JSON 格式，确保格式正确
4. **ID 分段规则**：
   - 1xxxx：消耗品
   - 2xxxx：任务道具
   - 3xxxx：宝物
   - 4xxxx：装备
   - 词条ID：2xxx
   - 效果ID：1xxx（消耗品）、3xxx（宝物）、4xxx（装备）

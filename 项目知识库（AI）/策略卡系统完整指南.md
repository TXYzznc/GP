# 策略卡系统完整指南

## 📋 系统概述

策略卡系统是战斗UI的核心组件，负责：
- 卡牌的扇形排列显示
- 卡牌的拖拽和交互
- 卡牌效果的执行
- 卡牌的选中反馈

**核心特性：**
- ✅ 从左到右依次发牌的进场动画
- ✅ 运行时参数调整（半径、角度等）
- ✅ 流畅的拖拽让位动画
- ✅ 卡牌选中状态管理
- ✅ 详情UI的动态显示

---

## 🏗️ 系统架构

```
CardManager (管理器)
    ├─ 维护卡牌列表
    ├─ 发送卡牌事件
    └─ 战斗初始化/清理

CombatUI (战斗UI)
    ├─ 订阅战斗事件
    ├─ 初始化 CardManager
    ├─ 创建卡牌槽UI
    └─ 管理卡牌移除

CardSlotContainer (卡牌容器)
    ├─ 扇形位置计算
    ├─ 进场动画播放
    ├─ 拖拽交互处理
    └─ 卡牌重排动画

CardSlotItem (单个卡牌项)
    ├─ 卡牌数据绑定
    ├─ 选中/悬停动画
    ├─ 拖拽预览
    └─ 卡牌效果执行

CardData (卡牌数据)
    └─ 包装 CardTable 配置
```

## 📝 配置方法

### 1️⃣ 编辑配置表

**文件位置:** `Assets/AAAGame/DataTable/CardTable.txt`

```
配置表结构：
|  Id   |  Name   |  Desc   |  IconId  |  SpiritCost  | ...
|  int  | string  | string  |   int    |   float      | ...
```

**关键字段说明：**

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| **Id** | int | 卡牌ID（1001-1012） | 1001 |
| **Name** | string | 卡牌名称 | 神圣庇护 |
| **Desc** | string | 卡牌描述 | 为所有友方单位提供护盾 |
| **IconId** | int | 图标资源ID | 1201 |
| **SpiritCost** | float | 灵力消耗 | 30 |
| **TargetType** | int | 目标类型<br>1=自身<br>2=友方单体<br>3=友方全体<br>4=敌方单体<br>5=敌方全体<br>6=全场 | 3 |
| **DamageType** | int | 伤害类型<br>0=无 1=物理 2=魔法 3=真实 | 0 |
| **DamageCoeff** | float | 伤害系数 | 2.5 |
| **BaseDamage** | float | 基础固定伤害 | 150 |
| **InstantBuffs** | string | 释放时施加的Buff<br>格式: `buffId:duration` | 5001:5 |
| **HitBuffs** | string | 命中时施加的Buff | 5002:3 |
| **EffectId** | int | 技能特效ID | 30101 |
| **Rarity** | int | 稀有度<br>1=普通 2=稀有 3=史诗 4=传说 | 2 |

**完整示例行：**
```
1001	神圣庇护	为所有友方单位提供护盾，吸收伤害	1201	{"effectPrefab":20101}	30	3	0	0	0	0	0	5001:5		{"shieldAmount":100,"duration":5}	30101	0	1.5	2		众神之光笼罩战场，守护者的誓言永不磨灭
```

## 🎮 使用工作流程

### 战斗启动流程

```
1. 战斗进入 (CombatEnterEventArgs)
    ↓
2. CombatUI.OnCombatEnter() 触发
    ├─ CardManager.InitializeForCombat()  // 随机加载8张卡
    └─ RefreshCardSlots()  // 创建卡牌槽UI
    ↓
3. 对每张卡执行 CreateCardSlot()
    ├─ 实例化 CardSlotItem
    ├─ SetData(CardData) 绑定数据
    └─ container.AddCardAsync(slotItem)  // 播放进场动画
    ↓
4. CardSlotContainer.AddCardAsync()
    ├─ 计算延迟时间 (卡索引 * 0.1秒)
    ├─ 播放进场动画 (淡入+移动)
    └─ 最后一张卡完成后执行重排
```

### 卡牌交互流程

**选中流程：**
```
鼠标点击卡牌
    ↓
CardSlotItem.OnCardClicked()
    ↓
SelectCard()
    ├─ 更新选中状态
    ├─ 移动卡牌 (基准位置 + 20Y偏移)
    ├─ 播放脉冲动画
    └─ 显示 DetailInfoUI
```

**拖拽流程：**
```
OnBeginDrag()
    ├─ 取消选中状态
    ├─ 创建拖拽预览
    └─ 通知容器拖拽开始

OnDrag()
    ├─ 更新预览位置
    ├─ 计算插入位置
    └─ 更新其他卡的让位偏移

OnEndDrag()
    ├─ 判断释放区域
    ├─ 执行卡牌效果 或 返回卡槽
    └─ 删除卡牌 → 触发重排
```

### 卡牌移除流程

```
CardSlotItem.ExecuteCardEffect()
    ├─ 播放闪光效果
    ├─ 创建 CardEffectExecutor 执行效果
    ├─ 播放销毁动画
    └─ CardManager.RemoveCard()
        ↓
    OnCardRemoved 事件触发
        ↓
    CombatUI.OnCardRemoved()
        ├─ 查找UI并销毁
        └─ container.RemoveCard()
            ├─ 更新卡牌列表
            └─ 执行重排动画
```

## 🎨 参数配置详解

### CardSlotContainer 配置

在Prefab或Inspector中调整以下参数实时预览：

```csharp
[Header("扇形排列")]
[SerializeField] private float m_FanRadius = 800f;           // 圆弧半径
[SerializeField] private float m_MaxFanAngle = 50f;          // 最大展开角度（度）
[SerializeField] private Vector2 m_CircleCenter = Vector2.zero;  // 圆心偏移
[SerializeField] private float m_CircleCenterOffsetY = -100f;    // 圆心Y偏移

[Header("动效时长")]
[SerializeField] private float m_EnterDuration = 0.35f;      // 进场动画时长
[SerializeField] private float m_CardDealInterval = 0.1f;    // 卡牌发牌间隔
[SerializeField] private float m_RearrangeDuration = 0.25f;  // 补位动画时长
[SerializeField] private Ease m_RearrangeEase = Ease.OutCubic;
```

**参数调整建议：**

| 参数 | 默认值 | 调整范围 | 说明 |
|------|--------|--------|------|
| FanRadius | 800f | 600-1000 | 越大卡牌展开越开 |
| MaxFanAngle | 50f | 30-80 | 越大展开角度越广 |
| CircleCenterOffsetY | -100f | -200-0 | 负值将圆心下移 |
| EnterDuration | 0.35f | 0.2-0.5 | 进场动画速度 |
| CardDealInterval | 0.1f | 0.05-0.2 | 卡牌发牌延迟 |
| RearrangeDuration | 0.25f | 0.1-0.4 | 补位动画速度 |

## 🐛 常见问题排查

### ❌ 卡牌不显示

**原因：** CardTable 未加载或卡牌ID范围错误

**解决：**
1. 确保 `CardTable.bytes` 存在于 `StreamingAssets/DataTable/`
2. 检查 `CardManager.InitializeForCombat()` 中的ID范围 (1001-1012)
3. 查看日志确认 DataTable 加载成功

### ❌ 卡牌位置跳跃

**原因：** 多次战斗时状态未正确重置

**解决：**
1. 战斗结束时调用 `container.ClearState()`
2. 战斗结束时调用 `CardManager.Clear()`
3. 重新初始化时调用 `InitializeForCombat()`

### ❌ 拖拽预览显示错误

**原因：** Canvas 层级或 RectTransform 设置不正确

**解决：**
1. 确保拖拽预览的父物体是 Canvas
2. 检查 raycastTarget 是否为 false
3. 验证 ScreenPointToLocalPointInRectangle 的转换正确

### ❌ 动画卡顿

**原因：** DOTween 动画未正确清理或 Ease 设置不当

**解决：**
1. 在 `OnDestroy()` 中调用 `Kill()` 清理所有Tween
2. 调整 `Ease` 参数（OutQuad/OutCubic 更平滑）
3. 检查卡牌数量是否过多（>12张）

## 🎯 总结

**三个核心规则：**

1. **配置更新规则** → 修改 `.txt` → 运行 `DataTableGenerator` → 自动生成 `.cs`
2. **状态管理规则** → 战斗进入初始化 → 战斗结束清理 → 重复战斗时重置
3. **时序管理规则** → 发牌延迟 → 进场动画完成 → 最后一张卡时重排

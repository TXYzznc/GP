# 出战预设系统设计

## 概述

出战预设（Battle Loadout Preset）允许玩家在局外预先配置带入局内的**棋子阵容**和**策略卡组**，支持多套预设方案的保存和快速切换。

## 数据基础

### 已有数据结构

```csharp
// PlayerAccountData.cs 中已定义
[Serializable]
public class DeckData
{
    public string DeckName;              // 预设名称
    public List<int> UnitCardIds;        // 棋子ID列表（SummonChessTable.Id）
    public List<int> StrategyCardIds;    // 策略卡ID列表（CardTable.Id）
}
```

**存档字段**（PlayerSaveData）：
- `SavedDecks` — JSON字符串，存储 `List<DeckData>`
- `CurrentDeckIndex` — 当前选中的预设索引

### 相关数据源
- `ChessUnlockManager.GetUnlockedChess()` — 已解锁棋子
- `PlayerSaveData.OwnedStrategyCardIds` — 已拥有策略卡
- `BattleLoadoutProvider` — 出战资源提供者（需扩展以支持预设）

## 功能设计

### 核心功能

| 功能 | 说明 |
|------|------|
| 查看预设列表 | 最多 **5 个**预设槽位，显示名称和简要信息 |
| 创建预设 | 选择空槽位，输入名称，配置棋子和策略卡 |
| 编辑预设 | 修改已有预设的棋子/策略卡配置 |
| 删除预设 | 清空某个预设槽位 |
| 设为默认 | 将某个预设标记为默认出战方案 |
| 快速应用 | 战斗准备阶段可一键应用预设阵容 |

### 预设约束规则

| 约束 | 值 | 来源 |
|------|-----|------|
| 最大预设数量 | 5 | 固定 |
| 每预设最大棋子数 | 由统御值上限决定 | PlayerDataTable（等级相关） |
| 每预设最大策略卡数 | 8 | 战斗系统（CardManager初始化数量） |
| 棋子来源 | 已解锁棋子 | ChessUnlockManager |
| 策略卡来源 | 已拥有策略卡 | PlayerSaveData.OwnedStrategyCardIds |

### 预设编辑流程

```
点击"出战预设"按钮
    ↓
打开 BattlePresetUI
    ↓
左侧显示预设列表（5个槽位）
    ├─ 有数据的槽位：显示名称 + 棋子/卡牌数量摘要
    └─ 空槽位：显示"+"创建新预设
    ↓
选择/创建预设 → 进入编辑模式
    ↓
上方：棋子选择区
    ├─ 已选棋子栏（可拖拽排序/移除）
    └─ 可选棋子池（来自已解锁，点击添加）
    ↓
下方：策略卡选择区
    ├─ 已选策略卡栏（可点击移除）
    └─ 可选策略卡池（来自已拥有，点击添加）
    ↓
底部：[保存] [重置] [删除预设]
```

### 与战斗准备的集成

扩展 `BattleLoadoutProvider`，在进入 `CombatPreparationState` 时：
1. 如果有默认预设 → 自动预选预设中的棋子
2. `CombatPreparationUI` 新增"应用预设"按钮 → 快速替换当前选择

## 需要用户手动完成的步骤

1. **创建 Prefab** — `Assets/AAAGame/Prefabs/UI/BattlePresetUI.prefab`
2. **UITable 注册** — 在 `UITable.txt` 中添加 BattlePresetUI 记录
3. **UIViews 枚举** — 添加 `BattlePresetUI` 项
4. **生成 Variables** — 运行 UI Variables 生成工具
5. **创建子项 Prefab** — PresetSlotItem、ChessSelectItem、CardSelectItem
6. **生成 UIItemVariables** — 运行生成工具

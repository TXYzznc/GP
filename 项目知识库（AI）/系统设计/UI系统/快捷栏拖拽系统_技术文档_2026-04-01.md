# 快捷栏拖拽系统实现

> **最后更新**: 2026-04-01
> **状态**: 有效
> **版本**: 1.0

---

**完成日期**: 2026-04-01

## 功能概述

实现了完整的拖拽系统，支持背包、仓库、快捷栏、装备栏四个容器间的物品交互。

## 核心组件

### 1. SlotContainerType 枚举
```csharp
public enum SlotContainerType
{
    Inventory,  // 背包
    Warehouse,  // 仓库
    Equip,      // 装备栏
    FastBar,    // 快捷栏（新增）
}
```

### 2. FastBarManager（新增）
管理快捷栏数据的独立容器管理器。

**关键方法：**
- `Initialize()` - 初始化 5 个快捷栏槽位
- `StoreItemToSlot(ItemBase item, int count, int targetSlotIndex)` - 存入物品到指定格子
- `SwapSlots(int fromSlotIndex, int toSlotIndex)` - 交换两个格子
- `ClearSlot(int slotIndex)` - 清空格子
- `GetSlot(int slotIndex)` - 获取指定格子
- `GetAllSlots()` - 获取所有格子

**事件：**
- `OnItemStored(int slotIndex, InventorySlot slot)` - 物品存入时触发
- `OnSlotCleared(int slotIndex)` - 格子清空时触发

### 3. InventoryUI 集成

- OnOpen 时自动初始化 FastBarManager
- `m_FastSlots` 容器类型设置为 `SlotContainerType.FastBar`
- `RefreshHotbar()` 从 FastBarManager 读取数据而非旧的 m_HotbarBoundSlots

**公共接口：**
```csharp
public InventorySlot GetFastBarSlot(int hotbarIndex)
```

### 4. InventoryDragHandler 拖拽逻辑

支持 9 种拖拽组合（对称设计）：

| 源 \ 目标 | Inventory | Warehouse | FastBar |
|---------|-----------|-----------|---------|
| **Inventory** | MoveItem | StoreItemToSlot | StoreItemToSlot |
| **Warehouse** | 3-Case处理 | SwapSlots | StoreItemToSlot |
| **FastBar** | 3-Case处理 | StoreItemToSlot | SwapSlots |

**处理方法：**

1. `HandleInventoryToInventory` - 背包↔背包交换
2. `HandleInventoryToWarehouse` - 背包→仓库存入
3. `HandleInventoryToFastBar` - 背包→快捷栏存入
4. `HandleWarehouseToInventory` - 仓库→背包（空/堆叠/交换）
5. `HandleWarehouseToWarehouse` - 仓库↔仓库交换
6. `HandleWarehouseToFastBar` - 仓库→快捷栏存入
7. `HandleFastBarToInventory` - 快捷栏→背包（空/堆叠/交换）
8. `HandleFastBarToWarehouse` - 快捷栏→仓库存入
9. `HandleFastBarToFastBar` - 快捷栏↔快捷栏交换

## 拖拽流程

```
OnBeginDrag
  ↓ 创建拖拽图标（80x80, 70% 透明）
  ↓
OnDrag
  ↓ 更新图标位置（跟随鼠标）
  ↓
OnEndDrag
  ↓ 射线检测获取目标格子
  ↓ 检验有效性（源≠目标）
  ↓ 调用 HandleDrop(src, dst)
  ↓ 根据容器类型组合调用对应处理方法
  ↓ 销毁拖拽图标
```

## 数据流向

### 背包→快捷栏
```
背包格子 → 获取 ItemBase
         → FastBarManager.StoreItemToSlot()
         → 从背包移除
         → UI自动刷新
```

### 快捷栏→背包/仓库
```
快捷栏格子 → 获取 ItemStack
           → 创建新 ItemBase 实例
           → 放入目标容器
           → 清空快捷栏源格子
           → UI自动刷新
```

## 时序保证

- 所有操作原子性：先操作数据，后清理资源
- 异常安全：操作失败时源数据保持不变
- 事件驱动：容器变化时自动触发事件，UI 监听事件刷新显示

## 关键细节

1. **FastBar 数据独立**: FastBarManager 独立管理快捷栏数据，不依赖 InventoryManager
2. **类型兼容性**: 所有 CreateItem() 调用返回 ItemBase，与 StoreItemToSlot 参数对齐
3. **对称设计**: 除了源/目标容器类型不同外，逻辑流程相同
4. **交换策略**:
   - 相同容器：使用 SwapSlots() 交换 SlotIndex/ItemStack
   - 不同容器：清空目标后 SetItem/SetItemStack，返回物品到源

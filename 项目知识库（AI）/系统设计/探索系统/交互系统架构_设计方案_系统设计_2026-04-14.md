# 交互系统架构设计

> 最后更新：2026-04-10  
> 状态：✅ 框架已实现，待用户配置 Prefab 和 UITable

## 系统概述

通用环境交互框架，支持玩家在整个游戏过程中与各种可交互对象（宝箱、NPC、资源点、传送门等）进行交互。交互系统与游戏阶段无关，随玩家角色生命周期存在。

**设计目标**：低耦合、无冗余、扩展友好。

## 架构

```
PlayerInputManager (F键输入)
    ↓
InteractionDetector (挂在玩家身上，检测+优先级选择)
    ↓ 候选对象
IInteractable (接口，可交互对象实现)
    ↓ 交互提示
InteractionPromptUI (屏幕提示 "[F] 交互")
    ↓ 触发交互
PlayerInteraction (播放交互动画，锁定移动)
```

## 优先级评分算法

```
Score = Priority × 10 + (1 - normalizedDistance) × 5 + facingBonus × 3
```

- `normalizedDistance` = distance / detectionRadius (0~1)
- `facingBonus` = Dot(playerForward, dirToTarget) > 0.5 ? 1 : 0
- 所有权重通过 SerializeField 暴露，可在 Inspector 调整

## 交互流程

1. 玩家靠近 → InteractionDetector.OnTriggerEnter → 加入候选列表
2. 每帧评分 → 选出最佳目标 → InteractionPromptUI 显示 "[F] 提示文本"
3. 按 F 键 → 检查 CanInteract + 未在交互中
4. 如需动画 → PlayerInteraction.TriggerInteractWithCallback → 动画完成 → OnInteract
5. 无需动画 → 直接 OnInteract
6. 离开范围 → 移出候选 → UI 隐藏

## 扩展方式

继承 `InteractableBase` 即可：

```csharp
public class TreasureChestInteractable : InteractableBase
{
    private bool isOpened = false;
    public override bool CanInteract(GameObject player) => !isOpened;
    public override void OnInteract(GameObject player)
    {
        isOpened = true;
        // 播放开箱动画、发放奖励...
    }
}
```

## 生命周期

- `InteractionDetector` 挂在玩家角色 Prefab 上，随角色存在而存在
- `InteractionPromptUI` 由 `InteractionDetector.Start()` 打开，`OnDestroy()` 关闭
- 系统与游戏阶段（探索/战斗等）无关，全程可用

## 用户手动步骤

1. 在玩家角色 Prefab 上挂载 `InteractionDetector` 组件
2. 创建 `InteractionPromptUI` 的 Prefab（含 Text + CanvasGroup）
3. 在 UITable.xlsx 中注册 `InteractionPromptUI`，运行 DataTableGenerator
4. 取消 InteractionDetector 中 `OpenPromptUI()` 的注释
5. 为可交互对象设置专用 Layer（如 "Interactable"），并在 InteractionDetector 中配置 interactableLayer

## 问题概述

敌人探索系统在实现广播机制时出现了设计问题和死循环bug，导致敌人在 Chase 和 Broadcast 状态之间疯狂切换，无法正常进入战斗。

## 问题1：广播死循环

### 问题表现
敌人在 Chase 和 Broadcast 状态之间无限循环切换：

```
Chase → Broadcast → Chase → Broadcast → ...（无限循环）
```

### 根本原因
**死循环流程**：
1. Chase 状态：检测到 `distanceToPlayer <= CombatDistance` → 切换到 Broadcast
2. Broadcast 状态：`OnEnter()` 广播一次 → `OnUpdate()` 检测到 `distanceToPlayer <= ChaseDistance * 0.8` → 切回 Chase
3. Chase 状态：下一帧又检测到 `distanceToPlayer <= CombatDistance` → 再次切换到 Broadcast
4. 无限循环...

### 设计问题
- **错误设计**：广播被设计成一个独立的状态（`EnemyBroadcastState`）
- **正确设计**：广播应该是 Chase 状态下的一个行为，而不是独立状态
- 广播本质上是一个"瞬间动作"，不应该占用一个状态

## 问题2：广播方法参数不匹配

### 问题描述
`EnemyBroadcastState` 调用 `BroadcastPlayerPosition()` 时缺少 `broadcastRange` 参数

### 修复方案
```csharp
// 修复前
EnemyGroupManager.Instance?.BroadcastPlayerPosition(
    m_AI.Entity, 
    m_AI.PlayerTransform.position
);

// 修复后
EnemyGroupManager.Instance?.BroadcastPlayerPosition(
    m_AI.Entity, 
    m_AI.PlayerTransform.position,
    m_AI.Entity.Config.BroadcastDistance  // ✅ 使用配置表中的广播距离
```

## 最终解决方案

### 重构方案

1. **删除 Broadcast 状态**：不再使用 `EnemyBroadcastState`
2. **在 Chase 状态中添加广播逻辑**：
   - 添加广播标志 `m_HasBroadcasted`，确保只广播一次
   - 在追击过程中，当接近玩家时触发广播
   - 广播后继续追击，直到进入战斗距离

### 修改后的状态流程

```
Idle → Alert → Chase（包含广播行为）→ Combat
```

### 关键代码逻辑

```csharp
// 在 EnemyChaseState 中
private bool m_HasBroadcasted = false;

public override void OnUpdate()
{
    float distanceToPlayer = Vector3.Distance(
        m_AI.Entity.CachedTransform.position, 
        m_AI.PlayerTransform.position
    );
    
    // 广播逻辑（只执行一次）
    if (!m_HasBroadcasted && 
        m_AI.Entity.Config.IsElite && 
        distanceToPlayer <= m_AI.Entity.Config.BroadcastDistance)
    {
        BroadcastPlayerPosition();
        m_HasBroadcasted = true;
    }
    
    // 进入战斗距离
    if (distanceToPlayer <= m_AI.Entity.Config.CombatDistance)
    {
        m_AI.ChangeState(EnemyStateType.Combat);
        return;
    }
    
    // 继续追击...
}

public override void OnExit()
{
    m_HasBroadcasted = false;  // 重置广播标志
}
```

## 经验总结

### 设计原则
1. **状态应该是持续的**：如果一个行为是瞬间完成的，不应该设计成状态
2. **避免状态切换死循环**：状态切换条件要互斥，避免来回切换
3. **广播是行为，不是状态**：类似的瞬间行为应该作为状态内的逻辑，而不是独立状态

### 调试技巧
1. 使用详细的日志输出状态切换
2. 检查状态切换条件是否互斥
3. 注意状态切换的时机和频率

- 用户要求战斗模式下相机位置应该完全固定，不支持缩放

## 修复方案

### 1. 禁用战斗模式滚轮缩放
- 修改 `HandleZoomInput()` 方法，在战斗模式下直接返回
- 确保战斗模式下完全忽略鼠标滚轮输入

### 2. 保留FOV动态调整
- 保持 `HandleCombatMode()` 中的 `HandleDynamicFOV()` 调用
- 用于从战斗前状态平缓过渡到战斗状态

### 3. 清理冗余代码
- 移除未使用的战斗模式缩放字段：`combatMinDistance`、`combatMaxDistance`
- 移除未使用的方法：`UpdateCombatCameraZoom()`
- 修复 `LateUpdate()` 中重复调用 `HandleDynamicFOV()` 的问题

## 修改内容

### HandleZoomInput 方法
```csharp
private void HandleZoomInput()
{
    // ⭐ 战斗模式下不处理缩放输入
    if (viewMode == CameraViewMode.Combat)
    {
        DebugEx.LogModule("ThirdPersonCamera", "战斗模式：忽略缩放输入");
        return;
    }
    // ... 其他逻辑保持不变
}
```

### HandleCombatMode 方法
```csharp
private void HandleCombatMode()
{
    // ⭐ 战斗模式下相机位置固定，不处理缩放输入
    // 只处理动态FOV
    HandleDynamicFOV();
    
    DebugEx.LogModule("ThirdPersonCamera", "战斗模式：相机位置固定，不支持缩放");
}
```

### LateUpdate 方法优化
- 移除战斗模式分支中重复的 `HandleDynamicFOV()` 调用

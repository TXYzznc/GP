## 创建的文件

### 1. 钩子配置文件
**文件**: `.kiro/hooks/update-skill-index.kiro.hook`

**类型**: userTriggered (手动触发)

**功能**:
- 扫描 `.kiro/skills/` 目录下所有 SKILL.md 文件

1. 通过 Agent Hooks 面板的按钮

## 技术方案

### 为什么选择 userTriggered?

**问题**: 如果使用 `fileEdited` 事件监听 SKILL.md 文件的修改:
```
SKILL.md 被修改
    ↓
触发钩子
    ↓
钩子修改 skill-auto-indexer.kiro.hook
    ↓
如果钩子也监听 .kiro.hook 文件...
    ↓
再次触发钩子 (循环!)
```

**解决方案**: 使用 `userTriggered` 类型:
- 用户完全控制执行时机
- 避免自动触发导致的循环
- 可以在修改多个 SKILL.md 后一次性更新
- 更加可预测和可控

### 钩子工作流程

```
用户触发钩子
    ↓
Agent 接收提示词
    ↓
扫描 .kiro/skills/**/SKILL.md
    ↓
提取 frontmatter 信息
    ├─ name
    ├─ description
    └─ keywords
    ↓
分析关键词
    ├─ 从 description 提取
    ├─ 从 keywords 提取
    └─ 智能推断相关词
    ↓
生成触发词映射
    ↓
更新 skill-auto-indexer.kiro.hook
    ├─ 保持 GitNexus 部分不变
    └─ 更新"其他技能触发词"
    ↓
生成更新报告
    ├─ 扫描到的技能数量
    ├─ 新增的技能
    ├─ 更新的关键词
    └─ 索引统计
```

## 使用示例

### 示例 1: 添加新技能后更新索引

```bash
# 1. 创建新技能
mkdir .kiro/skills/my-new-skill
cat > .kiro/skills/my-new-skill/SKILL.md << EOF
---
name: my-new-skill
description: 这是一个新技能,用于处理特定任务
keywords: 任务, 处理, 自动化
---

# 新技能

详细文档...
EOF

# 2. 触发更新 (在 Kiro 中)
# 方法 A: 点击 Agent Hooks 面板的"更新 SKILL 索引"按钮
# 方法 B: 在对话中输入"请更新技能索引"

# 3. 验证结果
# 检查 skill-auto-indexer.kiro.hook 是否包含新技能
```

### 示例 2: 修改技能描述后更新

```bash
# 1. 修改 SKILL.md
# 编辑 .kiro/skills/txt-to-xlsx-converter/SKILL.md
# 更新 description 或 keywords

# 2. 触发更新
# 在 Kiro 中触发"更新 SKILL 索引"钩子

# 3. 测试
# 在对话中输入相关关键词,验证推荐是否正确
```

## 优势

1. **避免循环触发**: 使用手动触发,完全避免循环修改的风险
2. **灵活控制**: 用户可以在修改多个技能后一次性更新
3. **可预测性**: 明确的触发时机,不会意外执行
4. **易于调试**: 可以随时触发,方便测试和验证
5. **性能友好**: 不会在每次文件保存时自动执行

- 日志中出现大量重复的 `[DefaultSkillReleaseStrategy] 后羿 决策: 使用技能1`
- 棋子卡住无法正常战斗，既不攻击也不释放技能

**触发条件：**
1. 棋子处于攻击状态（`ChessAIState.Attacking`）
2. 棋子正在执行攻击动作（`m_IsAttacking = true`）
3. 技能冷却完成，满足释放条件
4. 玩家操作棋子移动过程中技能冷却到了

## 根本原因分析

### 1. 技能决策循环
- `TickAttacking` 方法每帧都会调用 `ShouldUseSkill()`
- 当技能条件满足时，系统标记 `m_ShouldUseSkillAfterAttack = true`
- 但由于棋子仍在攻击中，状态不会切换
- 下一帧继续调用 `ShouldUseSkill()`，形成无限循环

### 2. 缺乏防抖机制
- 没有机制防止在短时间内重复进行技能决策
- 每帧都输出决策日志，造成日志刷屏

### 3. 攻击完成回调可能缺失
- 某些情况下 `OnAttackComplete()` 可能没有被正确调用
- 导致 `m_IsAttacking` 一直为 `true`，无法切换到技能状态

## 修复方案

### 1. 添加技能决策防抖机制

**新增字段：**
```csharp
/// <summary>技能决策防抖计时器</summary>
protected float m_SkillDecisionCooldown;

/// <summary>上次技能决策的时间戳</summary>
protected float m_LastSkillDecisionTime;

/// <summary>技能决策防抖间隔（秒）</summary>
protected const float SKILL_DECISION_COOLDOWN = 0.2f;
```

**防抖检查方法：**
```csharp
protected virtual bool CanMakeSkillDecision()
{
    // 如果防抖计时器还在运行，不允许决策
    if (m_SkillDecisionCooldown > 0)
        return false;

    // 如果已经标记了攻击后使用技能，不再重复决策
    if (m_ShouldUseSkillAfterAttack)
        return false;

    return true;
}
```

### 2. 优化 TickAttacking 逻辑

**修改前：**
```csharp
if (m_CurrentState == ChessAIState.Attacking && ShouldUseSkill())
```

**修改后：**
```csharp
if (m_CurrentState == ChessAIState.Attacking && CanMakeSkillDecision())
{
    if (ShouldUseSkill())
    {
        // 记录技能决策时间，启动防抖
        m_LastSkillDecisionTime = Time.time;
        m_SkillDecisionCooldown = SKILL_DECISION_COOLDOWN;
        // ... 后续逻辑
    }
}
```

### 3. 增强攻击完成处理

在 `OnAttackComplete()` 中添加：
```csharp
// 重置技能决策防抖，允许新的技能决策
m_SkillDecisionCooldown = 0f;
```

## 修复效果

### 预期改进：
1. **消除无限循环**：防抖机制确保技能决策不会在短时间内重复执行
2. **减少日志刷屏**：每0.2秒最多进行一次技能决策
3. **提高性能**：减少不必要的技能条件检查
4. **保持功能完整**：不影响正常的技能释放流程

### 兼容性：
- 所有使用 `ChessAIBase` 的棋子都会受益于这个修复
- 不会影响现有的技能释放逻辑
- 保持向后兼容性

## 测试建议

1. **重现问题场景**：
   - 让后羿进入攻击状态
   - 在攻击过程中等待技能冷却完成
   - 观察是否还会出现循环决策

2. **验证正常流程**：
   - 确认技能仍能正常释放
   - 验证攻击后技能切换正常工作
   - 检查其他棋子不受影响

3. **性能测试**：
   - 观察CPU使用率是否有改善
   - 检查日志输出频率是否正常

## 一、优化目标

### 1.1 核心问题
1. **飘字系统** - 每帧计算距离和缩放，存在性能浪费
2. **对象池管理** - 缺少预热机制，运行时频繁实例化
3. **棋子管理** - 频繁的列表遍历和死亡检查
4. **场景查找** - 使用 FindObjectsOfType 造成性能开销

### 1.2 优化指标
- 减少每帧 CPU 计算量
- 降低 GC 分配频率
- 优化对象池使用效率
- 消除不必要的场景查找

## 二、优化方案详解

### 2.1 飘字系统优化 (DamagePopupItem.cs)

#### 问题分析
```csharp
// ❌ 当前问题：每帧都在计算距离和缩放
private async UniTaskVoid PlayAnimationAsync()
{
    while (m_IsPlaying && m_ElapsedTime < duration)
    {
        FaceCamera(m_Camera);  // 每帧调用
        float distanceScale = CalculateDistanceScale();  // 每帧计算距离
        // ...
    }
}
```

#### 优化方案
1. **降低更新频率** - 距离缩放每 3-5 帧更新一次
2. **缓存计算结果** - 避免重复的 Vector3.Distance 调用
3. **移除未使用字段** - 删除 m_RectTransform（编译器警告）

#### 预期收益
- CPU 开销降低 40-60%（针对飘字动画）
- 减少每帧的数学计算

### 2.2 对象池优化 (DamageFloatingTextManager.cs)

#### 问题分析
```csharp
// ❌ 当前问题：对象池没有预热，首次使用时才创建
private DamagePopupItem GetPopupFromPool()
{
    if (m_PopupPool.Count > 0)
    {
        return m_PopupPool.Dequeue();
    }
    else
    {
        // 运行时创建，造成卡顿
        GameObject popupObj = Instantiate(m_PopupPrefab, m_PopupParent);
    }
}
```

#### 优化方案
1. **对象池预热** - 初始化时预创建 10-20 个对象
2. **容量管理** - 设置最大容量，避免无限增长
3. **字段优化** - 将 SerializeField 标记为 readonly

#### 预期收益
- 消除战斗中的实例化卡顿
- 减少 GC 压力
- 提升首次显示飘字的响应速度

---

### 2.3 棋子管理优化 (CombatChessManager.cs)

#### 问题分析
```csharp
// ❌ 当前问题：每 2 秒遍历所有棋子检查死亡状态
private void Update()
{
    m_CleanupTimer -= Time.deltaTime;
    if (m_CleanupTimer <= 0f)
    {
        CleanupDeadChess();  // 遍历所有列表
        m_CleanupTimer = CLEANUP_INTERVAL;
    }
}
```

#### 优化方案
1. **事件驱动清理** - 棋子死亡时主动通知，而非定时轮询
2. **延迟清理** - 使用脏标记，仅在需要时清理
3. **字段优化** - 将字典和列表标记为 readonly

#### 预期收益
- 减少不必要的列表遍历
- 降低 Update 中的 CPU 开销
- 提升大规模战斗的性能

---

### 2.4 角色管理优化 (PlayerCharacterManager.cs)

#### 问题分析
```csharp
// ❌ 当前问题：使用 FindObjectsOfType 查找所有 AudioListener
AudioListener[] listeners = FindObjectsOfType<AudioListener>();
foreach (var listener in listeners)
{
    if (listener.gameObject != cameraObj)
    {
        Destroy(listener);
    }
}
```

#### 优化方案
1. **缓存 AudioListener** - 使用静态列表管理，避免场景查找
2. **移除未使用的 using** - 清理 `using GameFramework.Event;`
3. **简化对象创建** - 使用 `new()` 简化语法

#### 预期收益
- 消除 FindObjectsOfType 的性能开销（O(n) 场景遍历）
- 减少代码冗余
- 提升摄像机初始化速度

## 三、实施计划

### 阶段 1: 飘字系统优化
- **文件**: `DamagePopupItem.cs`
- **任务**:
  1. 降低距离缩放更新频率（每 3 帧更新一次）
  2. 缓存摄像机距离计算结果
  3. 移除未使用的 m_RectTransform 字段
  4. 移除未使用的 OnDisable 方法

### 阶段 2: 对象池优化
- **文件**: `DamageFloatingTextManager.cs`
- **任务**:
  1. 添加对象池预热机制（初始化 15 个对象）
  2. 添加最大容量限制（50 个对象）
  3. 将 SerializeField 标记为 readonly
  4. 移除未使用的 using System;

### 阶段 3: 棋子管理优化
- **文件**: `CombatChessManager.cs`
- **任务**:
  1. 将定时清理改为事件驱动（可选优化）
  2. 将字典和列表字段标记为 readonly
  3. 简化 new 表达式语法

### 阶段 4: 角色管理优化
- **文件**: `PlayerCharacterManager.cs`
- **任务**:
  1. 优化 AudioListener 查找逻辑
  2. 移除未使用的 using GameFramework.Event
  3. 简化 new 表达式语法

## 七、总结

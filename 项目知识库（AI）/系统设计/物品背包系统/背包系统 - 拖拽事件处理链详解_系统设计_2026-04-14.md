# 背包系统 - 拖拽事件处理链详解

## 一、拖拽事件的三个阶段

### 完整流程图

```
用户按下鼠标并移动（拖拽）
  │
  ├─ Stage 1: OnBeginDrag（拖拽开始）
  │  ├─ 检测源格子是否有物品
  │  ├─ 创建拖拽图标（跟随鼠标）
  │  └─ 保存源格子信息
  │
  ├─ Stage 2: OnDrag（拖拽过程中每帧触发）
  │  ├─ 获取鼠标当前位置
  │  └─ 更新拖拽图标位置
  │
  └─ Stage 3: OnEndDrag（拖拽结束）
     ├─ 射线检测目标格子
     ├─ 执行容器间转移逻辑
     └─ 清理拖拽状态
```

---

## 二、Stage 1: OnBeginDrag - 拖拽开始

### 流程详解

```
用户按住鼠标并开始移动
  │
  └─> EventSystem 检测拖拽事件
      │
      └─> InventoryDragHandler.OnBeginDrag(PointerEventData)
          │
          ├─ [检查 1] 获取源格子 InventorySlotUI
          │  ├─ 尝试 GetComponent<InventorySlotUI>()
          │  └─ 如果失败，尝试 GetComponentInParent<InventorySlotUI>()
          │     （支持 DragHandler 在 InventorySlotUI 或 InventoryItemUI 上）
          │
          ├─ [检查 2] 获取物品 UI InventoryItemUI
          │  └─ InventorySlotUI.GetItemUI()
          │
          ├─ [检查 3] 检查格子是否有物品
          │  └─ itemUI.HasItem() == true?
          │
          ├─ [操作 1] 记录源格子
          │  └─ m_SourceSlot = inventorySlotUI
          │
          ├─ [操作 2] 创建拖拽图标
          │  └─ CreateDragIcon()
          │     ├─ 新建 GameObject: "DragIcon"
          │     ├─ 添加 Image 组件（raycastTarget=false，不拦截点击）
          │     ├─ 复制源物品的图标 Sprite
          │     ├─ 设置透明度 0.7
          │     └─ 父物体设为最高层 Canvas
          │
          └─ [返回] 准备进入 OnDrag 阶段
```

### 代码对应

```csharp
public void OnBeginDrag(PointerEventData eventData)
{
    // [检查 1] 获取源格子
    m_SourceSlot = GetComponent<InventorySlotUI>();
    if (m_SourceSlot == null)
        m_SourceSlot = GetComponentInParent<InventorySlotUI>();

    if (m_SourceSlot == null)
        return;  // ❌ 中止：找不到源格子

    // [检查 2 & 3] 检查物品是否存在
    var itemUI = m_SourceSlot.GetItemUI();
    if (itemUI == null || !itemUI.HasItem())
    {
        m_SourceSlot = null;
        return;  // ❌ 中止：格子为空
    }

    // [操作 1 & 2] 开始拖拽
    CreateDragIcon();  // 显示拖拽图标
}
```

### 常见失败场景

| 场景 | 原因 | 症状 |
|------|------|------|
| ❌ 找不到源格子 | DragHandler 未在正确的物体上 | OnBeginDrag 立即返回 |
| ❌ 格子为空 | itemUI.HasItem() = false | 无法开始拖拽 |
| ❌ ItemUI 为 null | 预制体层级错误 | OnBeginDrag 返回 |
| ❌ 图标不显示 | Canvas 为 null 或 raycastTarget=true | 拖拽可以，但看不到图标 |

## 三、Stage 2: OnDrag - 拖拽进行中

### 流程详解

```
用户移动鼠标（每帧触发）
  │
  └─> EventSystem 触发 OnDrag 事件
      │
      └─> InventoryDragHandler.OnDrag(PointerEventData)
          │
          ├─ [检查] 拖拽图标是否存在?
          │  └─ m_DragIcon != null?
          │
          ├─ [转换] 屏幕坐标 → Canvas 本地坐标
          │  ├─ eventData.position（屏幕坐标）
          │  │
          │  └─ RectTransformUtility.ScreenPointToLocalPointInRectangle()
          │     │
          │     └─ localPoint（Canvas 内的本地坐标）
          │
          └─ [更新] 更新图标位置
             └─ m_DragIcon.rectTransform.anchoredPosition = localPoint
```

### 关键：坐标转换

```csharp
public void OnDrag(PointerEventData eventData)
{
    if (m_DragIcon != null)
    {
        var canvasRT = m_TopCanvas.GetComponent<RectTransform>();

        // ⚠️ 重要：屏幕坐标→Canvas本地坐标
        if (RectTransformUtility.ScreenPointToLocalPointInRectangle(
            canvasRT,
            eventData.position,      // 输入：屏幕坐标
            m_TopCanvas.worldCamera,
            out var localPoint))     // 输出：本地坐标
        {
            // 更新图标位置（在 Canvas 内的相对位置）
            m_DragIcon.rectTransform.anchoredPosition = localPoint;
        }
    }
}
```

### 性能注意

- ✅ OnDrag 每帧调用一次，只是更新 RectTransform 位置
- ❌ 不要在 OnDrag 中创建新的 GameObject
- ❌ 不要在 OnDrag 中做射线检测（那是 OnEndDrag 的工作）

## 四、Stage 3: OnEndDrag - 拖拽结束（核心逻辑）

### 流程详解

```
用户释放鼠标
  │
  └─> EventSystem 触发 OnEndDrag 事件
      │
      └─> InventoryDragHandler.OnEndDrag(PointerEventData)
          │
          ├─ [检查] 源格子是否有效?
          │  └─ m_SourceSlot != null?
          │     └─ 如果为 null → 清理并返回
          │
          ├─ [查询] 射线检测目标格子
          │  └─ GetTargetSlot(eventData.position)
          │     │
          │     ├─ 创建 PointerEventData（鼠标最后位置）
          │     ├─ EventSystem.RaycastAll() → 返回 results 列表
          │     │
          │     └─ 遍历 results，寻找第一个 InventorySlotUI
          │        ├─ 检查每个对象
          │        ├─ 调用 GetComponent<InventorySlotUI>()
          │        └─ 返回第一个非 null 的结果
          │
          ├─ [验证] 检查拖放目标有效性
          │  ├─ targetSlot == null? → 拖拽取消
          │  ├─ targetSlot == sourceSlot? → 拖拽取消
          │  └─ 否则 → 执行拖放
          │
          ├─ [执行] HandleDrop(srcSlot, dstSlot)
          │  │
          │  ├─ 获取源格子容器信息
          │  │  ├─ srcType, srcIndex
          │  │  └─ src.SlotContainer（实际的容器对象）
          │  │
          │  ├─ 获取目标格子容器信息
          │  │  ├─ dstType, dstIndex
          │  │  └─ dst.SlotContainer（实际的容器对象）
          │  │
          │  ├─ [核心调用] 执行容器间转移
          │  │  └─ src.SlotContainer.TryMoveToContainer(
          │  │      srcIndex,          // 源格子索引
          │  │      dst.SlotContainer, // 目标容器
          │  │      dstIndex           // 目标格子索引
          │  │    )
          │  │    └─ 返回 success（bool）
          │  │
          │  └─ [完成] 根据结果记录日志
          │
          └─ [清理] CleanupDrag()
             ├─ 销毁 DragIcon GameObject
             └─ 清空 m_SourceSlot 缓存
```

### 代码对应

```csharp
public void OnEndDrag(PointerEventData eventData)
{
    // [检查] 源格子有效性
    if (m_SourceSlot == null)
    {
        CleanupDrag();
        return;
    }

    // [查询] 射线检测目标格子
    var targetSlot = GetTargetSlot(eventData.position);

    // [验证] 检查拖放目标
    if (targetSlot == null)
    {
        // 拖到空白处，取消拖拽
    }
    else if (targetSlot == m_SourceSlot)
    {
        // 拖到自己，取消拖拽
    }
    else
    {
        // [执行] 拖放操作
        HandleDrop(m_SourceSlot, targetSlot);
    }

    // [清理] 清理拖拽状态
    CleanupDrag();
}

private void HandleDrop(InventorySlotUI src, InventorySlotUI dst)
{
    // [核心] 调用源容器的交互方法
    bool success = src.SlotContainer.TryMoveToContainer(
        src.SlotIndex,
        dst.SlotContainer,
        dst.SlotIndex
    );

    if (success)
    {
        // ✅ 拖放成功
    }
    else
    {
        // ❌ 拖放失败（容器不兼容或操作被拒绝）
    }
}
```

## 五、射线检测（GetTargetSlot）

### 工作原理

```
释放鼠标时的鼠标位置
  │
  └─> ScreenPoint (屏幕坐标)
      │
      └─> EventSystem.RaycastAll()
          │
          └─> 返回 results 列表
              │
              └─> results = [
                      UI元素1 (距离最远),
                      UI元素2,
                      ...
                      UI元素N (距离最近/最上面)
                  ]
              │
              └─> 遍历 results，寻找 InventorySlotUI
                  │
                  └─> 返回第一个找到的 InventorySlotUI
```

### 代码

```csharp
private InventorySlotUI GetTargetSlot(Vector2 position)
{
    var results = new List<RaycastResult>();

    // 射线检测：返回指定位置的所有可交互 UI 元素
    EventSystem.current.RaycastAll(
        new PointerEventData(EventSystem.current) { position = position },
        results
    );

    // 遍历结果，寻找第一个 InventorySlotUI
    foreach (var r in results)
    {
        var slot = r.gameObject.GetComponent<InventorySlotUI>();
        if (slot != null)
        {
            return slot;  // ✅ 找到目标
        }
    }

    return null;  // ❌ 未找到有效的目标格子
}
```

### 影响射线检测的因素

```
✓ raycastTarget = true      → 参与射线检测
✗ raycastTarget = false     → 被跳过
✗ gameObject.activeSelf     → 必须激活
✗ Image.color.a == 0        → 透明度为 0 也会被检测（默认行为）
✗ CanvasGroup.blocksRaycast → 必须为 true
✗ RectTransformUtility      → 必须在 RectTransform 范围内
```

## 六、容器间交互（TryMoveToContainer）

### 交互流程

```
src.SlotContainer.TryMoveToContainer(srcIndex, dst.SlotContainer, dstIndex)
  │
  ├─ [双向检查] 确保容器兼容
  │  ├─ src.CanInteractWith(dst.Type)?
  │  └─ dst.CanInteractWith(src.Type)?
  │
  ├─ [验证] 源格子和目标格子都有效
  │  ├─ src.GetSlot(srcIndex) != null?
  │  └─ dst.GetSlot(dstIndex) != null?
  │
  ├─ [执行] ExecuteMove()
  │  └─ 根据源容器类型，调用具体实现
  │     │
  │     ├─ InventorySlotContainerImpl
  │     │  ├─ MoveToInventory()（背包内移动）
  │     │  ├─ MoveToWarehouse()（转移到仓库）
  │     │  └─ MoveToFastBar()（转移到快捷栏）
  │     │
  │     ├─ WarehouseSlotContainerImpl
  │     │  ├─ MoveToInventory()（取出到背包）
  │     │  └─ MoveToWarehouse()（仓库内交换）
  │     │
  │     └─ FastBarSlotContainerImpl
  │        ├─ MoveToInventory()（取出到背包）
  │        └─ MoveToFastBar()（快捷栏内交换）
  │
  └─ [返回] success (bool)
```

### 容器兼容性矩阵

```
源 \ 目标    | Inventory | Warehouse | FastBar | Equip
------------|-----------|-----------|---------|-------
Inventory   | ✅        | ✅        | ✅      | ✅
Warehouse   | ✅        | ✅        | ❌      | ❌
FastBar     | ✅        | ❌        | ✅      | ❌
Equip       | ✅        | ❌        | ❌      | ✅
```

### 代码实例：InventorySlotContainerImpl

```csharp
public override bool CanInteractWith(SlotContainerType otherContainerType)
{
    return otherContainerType == SlotContainerType.Inventory ||
           otherContainerType == SlotContainerType.Warehouse ||
           otherContainerType == SlotContainerType.FastBar;
}

protected override bool ExecuteMove(int fromSlotIndex, ISlotContainer targetContainer, int targetSlotIndex)
{
    // 获取源物品
    var fromSlot = GetSlot(fromSlotIndex);
    if (fromSlot == null || fromSlot.IsEmpty)
        return false;

    var itemId = fromSlot.ItemId;
    var count = fromSlot.Count;

    // 根据目标容器类型，选择处理方法
    bool success = targetContainer switch
    {
        InventorySlotContainerImpl inv => MoveToInventory(inv, fromSlotIndex, targetSlotIndex),
        WarehouseSlotContainerImpl wh => MoveToWarehouse(wh, itemId, count, targetSlotIndex),
        FastBarSlotContainerImpl fb => MoveToFastBar(fb, fromSlot, targetSlotIndex),
        _ => false
    };

    // 如果转移成功，清空源格子
    if (success && targetContainer is not InventorySlotContainerImpl &&
        (targetSlot.IsEmpty || targetSlot.ItemId == itemId))
    {
        // 只有转移到其他容器时才清空源格子
        InventoryManager.Instance.RemoveItem(itemId, count);
    }

    return success;
}
```

## 七、DragHandler 的挂载位置

### 方案 1：挂在 InventorySlotUI 上

```
✅ 优点：
  ├─ 直接获取格子信息
  └─ raycastTarget 检查清晰

❌ 缺点：
  └─ 如果格子为空（没有 InventoryItemUI），无法拖拽

结构：
  InventorySlotUI（← DragHandler 在这里）
  ├─ Image (varBg, raycastTarget=true)
  ├─ Image (varHighLightImg)
  └─ InventoryItemUI（当有物品时）
     └─ Image (varItemImg)
```

### 方案 2：挂在 InventoryItemUI 上

```
✅ 优点：
  ├─ 只有有物品才能拖拽
  └─ 自动检测物品存在

❌ 缺点：
  └─ 需要用 GetComponentInParent 获取格子

结构：
  InventorySlotUI
  ├─ Image (varBg)
  ├─ Image (varHighLightImg)
  └─ InventoryItemUI（← DragHandler 在这里）
     ├─ Image (varItemImg, raycastTarget=true)
     └─ Text (varCountText)
```

### 推荐配置

```
✅ 推荐：方案 1（挂在 InventorySlotUI）

原因：
├─ 代码已经支持两种方式（OnBeginDrag 中有 try-catch）
├─ InventorySlotUI 更稳定（总是存在）
├─ 逻辑更清晰（格子负责拖拽，物品只显示）
└─ 如果格子为空，OnBeginDrag 中会检测并中止

配置：
  InventorySlotUI
  ├─ Image (varBg)
  │  └─ raycastTarget: ✓ true（必须，用于接收拖拽）
  ├─ Image (varHighLightImg)
  │  └─ raycastTarget: ✗ false（装饰品）
  ├─ InventoryDragHandler ← 脚本挂在这里
  └─ InventoryItemUI
     └─ Image (varItemImg)
        └─ raycastTarget: 任意（不影响）
```

## 八、常见问题排查

### 问题 1：无法开始拖拽

**症状**：点击并拖动物品，OnBeginDrag 不触发或立即返回

**排查步骤**：
```csharp
// Step 1: 检查 DragHandler 是否存在和启用
var dragHandler = inventorySlotUI.GetComponent<InventoryDragHandler>();
Debug.Log($"DragHandler 存在: {dragHandler != null}");
Debug.Log($"DragHandler 启用: {dragHandler?.enabled}");

// Step 2: 检查 Image 的 raycastTarget
var bgImage = inventorySlotUI.GetComponent<Image>();
Debug.Log($"varBg raycastTarget: {bgImage?.raycastTarget}");

// Step 3: 检查物品是否存在
var itemUI = inventorySlotUI.GetItemUI();
Debug.Log($"ItemUI 存在: {itemUI != null}");
Debug.Log($"ItemUI 有物品: {itemUI?.HasItem()}");

// Step 4: 检查 Canvas
var canvas = FindObjectOfType<Canvas>();
Debug.Log($"Canvas 存在: {canvas != null}");
Debug.Log($"Canvas worldCamera: {canvas?.worldCamera}");
```

### 问题 2：拖拽图标不显示

**症状**：可以拖拽，但没有图标跟随鼠标

**原因和解决**：
```
1. Canvas 为 null
   ├─ 原因：FindTopCanvas() 失败
   └─ 解决：检查 Canvas 是否存在，是否有 worldCamera

2. 图标 Image 的 raycastTarget = true
   ├─ 原因：代码第 186 行已设置为 false，但检查一下
   └─ 解决：m_DragIcon.raycastTarget = false;

3. 图标 Sprite 为空
   ├─ 原因：itemUI.GetComponentInChildren<Image>() 失败
   └─ 解决：确保 InventoryItemUI 内有 Image 组件
```

### 问题 3：无法放下物品（目标格子为 null）

**症状**：可以拖拽，但无论放到哪里都失败

**原因**：
```
GetTargetSlot() 返回 null 的原因：

1. 射线检测没有命中任何格子
   ├─ 检查：鼠标释放时是否在格子上方
   └─ 解决：确保目标格子的 raycastTarget = true

2. 射线检测命中了其他 UI，但不是 InventorySlotUI
   ├─ 原因：可能被高亮层或其他 Image 拦截
   └─ 解决：确保所有装饰品的 raycastTarget = false

3. 目标格子不存在或已销毁
   ├─ 原因：动态生成的格子可能未正确初始化
   └─ 解决：检查格子的生命周期
```

**调试方法**：
```csharp
// 在 OnEndDrag 中添加日志
private void OnEndDrag(PointerEventData eventData)
{
    var targetSlot = GetTargetSlot(eventData.position);

    if (targetSlot == null)
    {
        // 打印射线检测结果
        var results = new List<RaycastResult>();
        EventSystem.current.RaycastAll(
            new PointerEventData(EventSystem.current) { position = eventData.position },
            results
        );

        Debug.Log($"射线检测命中数量: {results.Count}");
        foreach (var r in results)
        {
            Debug.Log($"  - {r.gameObject.name} (InventorySlotUI: {r.gameObject.GetComponent<InventorySlotUI>() != null})");
        }
    }
}
```

### 问题 4：拖放成功但物品未消失

**症状**：物品从 A 格子拖到 B 格子，但 A 格子的物品还在

**原因**：
```
ExecuteMove 返回 true，但源格子没有清空

可能的原因：
1. targetContainer is InventorySlotContainerImpl（背包内移动）
   └─ 这种情况不应该清空，因为 MoveItem 已处理

2. targetSlot 不为空且不是同种物品（交换失败）
   └─ 此时不应该清空

3. 事件系统未及时刷新 UI
   └─ 需要等待事件链完成后刷新
```

**解决方案**：
- 检查 InventoryManager 的 OnInventoryChanged 事件是否触发
- 检查 InventoryUI 的刷新是否及时

## 九、拖拽事件的完整数据流

```
                    用户操作                           系统响应
                       ↓                                 ↓
        用户按住鼠标并移动 ──→ EventSystem 检测拖拽
                                    │
                    ┌───────────────┼───────────────┐
                    ↓               ↓               ↓
            OnBeginDrag        OnDrag(每帧)      OnEndDrag
              (触发1次)       (触发多次)         (触发1次)
                │               │                 │
                ├─ 获取源格子    ├─ 获取鼠标位置   ├─ 射线检测目标
                ├─ 检查物品      ├─ 坐标转换       ├─ 验证目标有效
                ├─ 创建图标      └─ 更新图标位置   ├─ 执行 HandleDrop
                └─ 保存状态                       │   └─ TryMoveToContainer
                                                  │      └─ InventoryManager/
                                                  │         WarehouseManager
                                                  │         / FastBarManager
                                                  └─ 清理拖拽状态
                                                     └─ 销毁图标


                                 事件链
                                   ↓
        InventoryManager.OnInventoryChanged?
                                   ↓
                    InventoryUI.RefreshAll()
                                   ↓
                           UI 更新完成
```

## 十、性能考虑

### OnBeginDrag（执行一次）
- ✅ GetComponent/GetComponentInParent：低消耗
- ✅ CreateDragIcon：创建一个 GameObject，可接受
- ⚠️ FindTopCanvas：遍历所有 Canvas，大项目可能昂贵（但只执行一次）

### OnDrag（每帧执行）
- ✅ ScreenPointToLocalPointInRectangle：高效的坐标转换
- ✅ rectTransform.anchoredPosition = localPoint：简单赋值
- ❌ 不要在这里做射线检测或创建对象

### OnEndDrag（执行一次）
- ⚠️ EventSystem.RaycastAll()：遍历所有 UI 元素
  - 最坏情况：O(n)，其中 n 是场景中的 UI 元素数量
  - 对大型 UI 系统可能有影响
  - 解决：缓存结果，避免频繁调用
- ✅ GetComponent<InventorySlotUI>()：快速查询（哈希表）
- ✅ TryMoveToContainer()：逻辑复杂但执行次数少

### 优化建议

```csharp
// 优化：缓存 Canvas
private Canvas m_TopCanvas;

private void Awake()
{
    // 只查找一次，存储引用
    m_TopCanvas = FindTopCanvas();
}

// 优化：减少 RaycastAll 调用
// OnEndDrag 中已经是最少调用次数（只在释放鼠标时调用）

// 优化：避免在 OnDrag 中创建 GC
public void OnDrag(PointerEventData eventData)
{
    if (m_DragIcon != null)
    {
        // 不创建新对象
        var canvasRT = m_TopCanvas.GetComponent<RectTransform>();
        RectTransformUtility.ScreenPointToLocalPointInRectangle(
            canvasRT,
            eventData.position,
            m_TopCanvas.worldCamera,
            out var localPoint  // out 参数，不分配新内存
        );

        m_DragIcon.rectTransform.anchoredPosition = localPoint;
    }
}
```

| 阶段 | 方法 | 调用频率 | 职责 | 关键检查 |

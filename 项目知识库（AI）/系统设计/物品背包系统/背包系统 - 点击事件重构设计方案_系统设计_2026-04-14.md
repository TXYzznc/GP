# 背包系统 - 点击事件重构设计方案

## 一、设计目标

参考 `InventoryDragHandler` 的实现模式，统一点击事件的处理流程。

**现状问题**：
- ❌ InventoryItemUI 直接处理点击逻辑（职责过重）
- ❌ 需要从 InventoryItemUI 获取格子，再获取详情 UI（链路复杂）
- ❌ 难以在 InventorySlotUI 层面统一管理

**目标状态**：
- ✅ 创建 InventoryClickHandler 统一监听点击事件
- ✅ 在 InventorySlotUI 层面进行事件管理（参考 DragHandler 模式）
- ✅ InventoryClickHandler 只负责事件接收和初步分发
- ✅ InventorySlotUI 负责处理具体的点击逻辑

---

## 二、架构对比

### 现有拖拽架构（参考）

```
拖拽流程：
  InventoryDragHandler（监听拖拽）
    ├─ OnBeginDrag / OnDrag / OnEndDrag（3个接口）
    ├─ 获取 InventorySlotUI
    └─ 调用 src.SlotContainer.TryMoveToContainer()
       └─ 容器管理转移逻辑

Prefab 结构：
  InventorySlotUI
  ├─ Image (varBg, raycastTarget=true)
  ├─ InventoryDragHandler ← 挂在这里
  └─ InventoryItemUI
     └─ Image (varItemImg)

特点：
  ✅ Handler 只负责事件监听和基本验证
  ✅ InventorySlotUI 管理格子信息（SlotIndex, ContainerType）
  ✅ 容器对象负责实际业务逻辑
```

### 提议的点击架构（统一模式）

```
点击流程：
  InventoryClickHandler（监听点击）
    ├─ OnPointerClick（1个接口）
    ├─ 判断左/右键
    ├─ 获取 InventorySlotUI
    ├─ 获取物品数据
    └─ 调用 InventorySlotUI.OnLeftClick / OnRightClick
       ├─ 左键 → InventorySlotUI.OnLeftClick()
       │        └─ 显示物品详情
       └─ 右键 → InventorySlotUI.OnRightClick()
                 └─ 显示上下文菜单

Prefab 结构：
  InventorySlotUI
  ├─ Image (varBg, raycastTarget=true)
  ├─ InventoryClickHandler ← 新增，挂在这里
  ├─ InventoryDragHandler
  └─ InventoryItemUI
     └─ Image (varItemImg)

特点：
  ✅ ClickHandler 只负责事件监听和分发（类似 DragHandler）
  ✅ InventorySlotUI 提供公共接口处理点击
  ✅ 不再依赖 InventoryItemUI（直接在 SlotUI 层处理）
  ✅ 左右键分离，易于扩展
```

## 三、详细设计方案

### ① InventoryClickHandler 脚本

**文件位置**：`Assets/AAAGame/Scripts/UI/Components/InventoryClickHandler.cs`

**职责**：
1. 实现 IPointerClickHandler 接口
2. 接收点击事件
3. 验证源格子和物品存在
4. 分发左键/右键事件给 InventorySlotUI

**伪代码**：
```csharp
public class InventoryClickHandler : MonoBehaviour, IPointerClickHandler
{
    // ========== 字段 ==========
    private InventorySlotUI m_SourceSlot;

    // ========== Unity 生命周期 ==========
    private void Awake()
    {
        EnsureRaycastable();
    }

    // ========== 事件接口 ==========
    public void OnPointerClick(PointerEventData eventData)
    {
        // 判断左/右键
        if (eventData.button == PointerEventData.InputButton.Left)
        {
            HandleLeftClick();
        }
        else if (eventData.button == PointerEventData.InputButton.Right)
        {
            HandleRightClick(eventData.position);
        }
    }

    // ========== 点击处理 ==========
    private void HandleLeftClick()
    {
        // Step 1: 获取源格子
        m_SourceSlot = GetComponent<InventorySlotUI>();
        if (m_SourceSlot == null)
            m_SourceSlot = GetComponentInParent<InventorySlotUI>();

        if (m_SourceSlot == null)
        {
            DebugEx.Warning("InventoryClickHandler", "未找到 InventorySlotUI");
            return;
        }

        // Step 2: 检查物品
        var itemUI = m_SourceSlot.GetItemUI();
        if (itemUI == null || !itemUI.HasItem())
        {
            return;  // 格子为空，不处理
        }

        // Step 3: 分发给 InventorySlotUI 处理
        m_SourceSlot.OnLeftClick();
    }

    private void HandleRightClick(Vector2 position)
    {
        // 同样的步骤，只是分发的是右键事件
        m_SourceSlot = GetComponent<InventorySlotUI>();
        if (m_SourceSlot == null)
            m_SourceSlot = GetComponentInParent<InventorySlotUI>();

        if (m_SourceSlot == null)
            return;

        var itemUI = m_SourceSlot.GetItemUI();
        if (itemUI == null || !itemUI.HasItem())
            return;

        // 传递鼠标位置，便于菜单定位
        m_SourceSlot.OnRightClick(position);
    }

    // ========== 工具方法 ==========
    private void EnsureRaycastable()
    {
        if (TryGetComponent(out Image img))
        {
            img.raycastTarget = true;
            DebugEx.Log("InventoryClickHandler", "确保 raycastTarget = true");
        }
    }
}
```

**优点**：
- ✅ 代码量小（只负责事件监听和分发）
- ✅ 参考 DragHandler 的模式，保持一致性
- ✅ 易于扩展（后续可以添加其他指针事件）

### ② InventorySlotUI 扩展接口

**文件位置**：`Assets/AAAGame/Scripts/UI/Item/InventorySlotUI.cs`

**新增方法**（在现有代码后添加）：

```csharp
public partial class InventorySlotUI : UIItemBase, IPointerEnterHandler, IPointerExitHandler
{
    // ========== 点击事件处理 ==========

    /// <summary>
    /// 处理左键点击（显示物品详情）
    /// </summary>
    public void OnLeftClick()
    {
        var itemUI = GetItemUI();
        if (itemUI == null || !itemUI.HasItem())
            return;

        var itemStack = itemUI.GetItemStack();
        if (itemStack == null || itemStack.IsEmpty)
            return;

        DebugEx.Log("InventorySlotUI", $"左键点击 格子={SlotIndex} 物品={itemStack.Item.Name}");

        // 显示物品详情面板
        ShowItemDetailPanel(itemStack);
    }

    /// <summary>
    /// 处理右键点击（显示上下文菜单）
    /// </summary>
    public void OnRightClick(Vector2 mousePosition)
    {
        var itemUI = GetItemUI();
        if (itemUI == null || !itemUI.HasItem())
            return;

        var itemStack = itemUI.GetItemStack();
        if (itemStack == null || itemStack.IsEmpty)
            return;

        DebugEx.Log("InventorySlotUI", $"右键点击 格子={SlotIndex} 物品={itemStack.Item.Name}");

        // 显示上下文菜单
        ShowContextMenu(itemStack, SlotIndex, mousePosition);
    }

    /// <summary>
    /// 显示物品详情面板
    /// </summary>
    private void ShowItemDetailPanel(ItemStack itemStack)
    {
        // 获取 InventoryUI
        var inventoryUI = GetComponentInParent<InventoryUI>();
        if (inventoryUI != null)
        {
            inventoryUI.ShowItemDetail(itemStack);
            DebugEx.Success("InventorySlotUI", $"显示物品详情: {itemStack.Item.Name}");
        }
    }

    /// <summary>
    /// 显示上下文菜单
    /// </summary>
    private void ShowContextMenu(ItemStack itemStack, int slotIndex, Vector2 position)
    {
        // 获取 InventoryUI
        var inventoryUI = GetComponentInParent<InventoryUI>();
        if (inventoryUI == null)
        {
            DebugEx.Warning("InventorySlotUI", "无法获取 InventoryUI");
            return;
        }

        // 获取菜单
        var contextMenu = inventoryUI.GetItemContextMenu();
        if (contextMenu != null)
        {
            contextMenu.ShowContextMenu(itemStack, slotIndex, position);
            DebugEx.Success("InventorySlotUI", $"显示上下文菜单: {itemStack.Item.Name}");
        }
        else
        {
            DebugEx.Error("InventorySlotUI", "无法获取 ItemContextMenu");
        }
    }
}
```

**职责**：
- ✅ 提供 `OnLeftClick()` 和 `OnRightClick()` 公共接口
- ✅ 处理物品详情和菜单的显示逻辑
- ✅ 保持简洁，避免过度设计

## 四、流程对比

### 改造前（混乱）

```
用户左键点击
  │
  └─> InventoryItemUI.OnPointerClick(左键)
      │
      └─> InventoryItemUI.OnItemClick()
          │
          └─> InventoryItemUI.ShowItemDetailPanel()
              │
              └─> GetComponentInParent<InventoryUI>()
                  │
                  └─ InventoryUI.ShowItemDetail()
```

**问题**：
- InventoryItemUI 混合了点击事件处理和业务逻辑
- 需要从 ItemUI → SlotUI → InventoryUI 的三层获取
- 难以在 SlotUI 层面统一管理

### 改造后（清晰）

```
用户左键点击
  │
  └─> InventoryClickHandler.OnPointerClick(左键)
      │
      ├─ 获取 InventorySlotUI
      ├─ 检查物品存在
      │
      └─> InventorySlotUI.OnLeftClick()
          │
          ├─ 获取 ItemStack
          │
          └─> InventorySlotUI.ShowItemDetailPanel(itemStack)
              │
              └─ GetComponentInParent<InventoryUI>()
                  │
                  └─ InventoryUI.ShowItemDetail(itemStack)
```

**优点**：
- ✅ ClickHandler 只负责事件和基本验证
- ✅ InventorySlotUI 是统一的事件入口和管理者
- ✅ InventoryItemUI 只负责数据显示
- ✅ 职责清晰，易于维护

## 十、总结

这个设计方案的核心思想是：
1. **参考拖拽的成功模式** - Handler 只负责事件监听和分发
2. **统一在 SlotUI 层管理** - 所有点击逻辑在 InventorySlotUI 中处理
3. **清晰的职责划分** - ClickHandler（事件）、SlotUI（管理）、ItemUI（显示）
4. **易于扩展和维护** - 后续添加新的指针事件或修改逻辑都很简单

这样做既统一了事件处理方式，又提高了代码的可维护性和可读性。

```csharp
public void OnPointerEnter(PointerEventData eventData)
{
    if (varHighLightImg != null && varHighLightImg.gameObject.activeSelf == false)
    {
        varHighLightImg.gameObject.SetActive(true);
    }
}

public void OnPointerExit(PointerEventData eventData)
{
    if (varHighLightImg != null && varHighLightImg.gameObject.activeSelf == true)
    {
        varHighLightImg.gameObject.SetActive(false);
    }
}
```

## 🐛 常见问题排查

### 问题 1：菜单按钮还是为 null

**表现**：右键点击后显示菜单但看不到按钮

**排查**：
1. 检查 Console 日志：
   ```
   按钮初始化完成：UseBtn=False, SplitBtn=False, DiscardBtn=False
   ```
2. **解决方案**：
   - 确保 ItemContextMenu 下确实有 3 个 Button 子物体
   - 确保按钮顺序：第 1 个 = UseBtn，第 2 个 = SplitBtn，第 3 个 = DiscardBtn
   - 如果顺序不对，手动在 Prefab 中重新排序

### 问题 2：菜单显示在错误的位置

**表现**：菜单出现在屏幕的奇怪位置，超出边界

**排查**：
1. 检查 Console 日志：
   ```
   菜单位置设置为: (-50, 100)  // 应该是合理的坐标
   ```
2. **解决方案**：
   - 确保 Canvas 的 `worldCamera` 不为 null
   - 确保 ItemContextMenu 的 RectTransform 的 Anchors 设置为 (0.5, 0.5)（中心点）
   - 检查 Canvas 的 `Render Mode`：
     - Screen Space - Overlay: worldCamera = null
     - Screen Space - Camera: worldCamera = 正确的 Camera
     - World Space: 不适合菜单

### 问题 3：高亮不显示或闪烁

**表现**：高亮效果看不见或闪烁不定

**排查**：
1. 检查 varHighLightImg 的初始状态：
   ```
   Inspector: HighLight 对象是否激活？
   ```
2. **解决方案**：
   - 确保 varHighLightImg GameObject 存在于 Prefab 中
   - 确保 varHighLightImg 的 Canvas Group 没有禁用交互
   - 确保 varHighLightImg 的颜色不是完全透明 (alpha = 0)
   - 检查 Sort Order：高亮应该在物品下方、背景上方

### 问题 4：点击菜单外没有关闭菜单

**现象**：菜单显示后，点击其他地方菜单不会自动关闭

**解决方案**（可选功能，需补充代码）：
```csharp
// 在 InventoryUI.OnUpdate() 中添加
if (Input.GetMouseButtonDown(0))
{
    var contextMenu = GetItemContextMenu();
    if (contextMenu != null && contextMenu.gameObject.activeSelf)
    {
        contextMenu.HideContextMenu();
    }
}
```

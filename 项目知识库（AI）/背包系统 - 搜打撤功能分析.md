# 背包系统 - 搜打撤功能分析

## 一、功能概述

搜打撤背包系统包含三个核心功能模块：

| 功能 | 含义 | 实现位置 | 主要类 |
|------|------|--------|--------|
| **搜** | 搜索/查看物品详情 | DetailInfoUI | InventoryUI、ItemContextMenu |
| **打** | 打开/编辑背包，物品操作 | InventoryUI | InventoryUI、InventorySlotUI、容器系统 |
| **撤** | 撤销/移除物品 | ItemContextMenu | InventoryManager、ItemContextMenu |

## 五、关键流程详解

### 5.1 物品添加流程

```
AddItem(itemId, count)
  │
  ├─ 获取物品配置
  │
  ├─ 如果可堆叠：TryStackItem()
  │   └─ 查找有相同 ItemId 且未满的格子
  │       └─ 增加 Count，直到堆叠满或物品全部放入
  │
  └─ 创建新物品实例
      └─ 查找空格子 FindEmptySlot()
          └─ 放入新物品堆叠
```

### 5.2 物品移动流程

```
MoveItem(fromSlot, toSlot)
  │
  ├─ 目标格子为空？
  │   ├─ 是：直接转移内容，清空源格子
  │   └─ 否：
  │       ├─ 同 ItemId 且可堆叠？
  │       │   └─ 是：合并，增加目标数量
  │       └─ 互换：交换两个格子的内容
  │
  └─ 触发 OnInventoryChanged 事件
```

### 5.3 背包与仓库转移流程

```
背包 → 仓库：
  1. InventorySlotContainerImpl.ExecuteMove()
  2. 调用 WarehouseManager.StoreItemToSlot()
  3. 转移成功后，调用 InventoryManager.RemoveItem()
  4. 触发两个管理器的事件，刷新 UI

仓库 → 背包：
  1. WarehouseSlotContainerImpl.ExecuteMove()
  2. 调用 InventoryManager.AddItem()
  3. 转移成功后，调用 WarehouseManager.RetrieveItem()
  4. 触发两个管理器的事件，刷新 UI
```

## 六、整理算法（SortInventory）

### 6.1 排序规则

```
Step 1: 收集所有非空堆叠
Step 2: 堆叠合并
  - 同 ItemId 且可堆叠的合并到同一堆
Step 3: 多级排序
  1. 按 ItemType 升序（0=消耗品, 1=装备, 2=宝物, 3=任务道具）
  2. 按 Quality 降序（品质高的排前面）
  3. 按 ItemId 升序（相同类型和品质的按ID排序）
Step 4: 重新填回格子
```

## 九、常见问题与解决方案

### 9.1 格子不刷新

**原因**：未触发 `OnInventoryChanged` 事件

**解决**：
```csharp
// ✅ 正确：操作后触发事件
bool success = InventoryManager.Instance.AddItem(itemId, count);
if (success)
    InventoryManager.Instance.OnInventoryChanged?.Invoke();
```

### 9.2 物品移动失败

**原因**：容器交互规则检查失败

**排查**：
1. 检查 `CanInteractWith()` 方法
2. 确认双向规则都通过
3. 查看 DebugEx 日志中的警告信息

### 9.3 快捷栏与背包不同步

**原因**：未订阅快捷栏事件

**解决**：
```csharp
m_FastBarManager.OnItemStored += OnFastBarChanged;
m_FastBarManager.OnItemRetrieved += OnFastBarChanged;
m_FastBarManager.OnSlotCleared += OnFastBarSlotCleared;
```

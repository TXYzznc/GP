# 图13：事件驱动通信流程

**位置**: 第3章 系统架构  
**章节**: 3.3 系统通信  
**类型**: 序列图  
**用途**: 说明事件驱动的通信机制

## Graphviz DOT 代码

```dot
digraph G {
    rankdir=LR;
    splines=orthogonal;
    nodesep=0.7;
    ranksep=1.0;
    
    // 节点定义
    Publisher [shape=box, label="事件发布者", style=filled, fillcolor="#FFE4B5"];
    EventMgr [shape=box, label="事件管理器", style=filled, fillcolor="#B0C4DE"];
    Subscriber1 [shape=box, label="订阅者 1", style=filled, fillcolor="#E8F4F8"];
    Subscriber2 [shape=box, label="订阅者 2", style=filled, fillcolor="#E8F4F8"];
    Subscriber3 [shape=box, label="订阅者 3", style=filled, fillcolor="#E8F4F8"];
    
    // 订阅阶段
    Subscriber1 -> EventMgr [label="订阅事件 A"];
    Subscriber2 -> EventMgr [label="订阅事件 A"];
    Subscriber3 -> EventMgr [label="订阅事件 B"];
    EventMgr -> Subscriber1 [label="订阅成功", style=dashed];
    EventMgr -> Subscriber2 [label="订阅成功", style=dashed];
    EventMgr -> Subscriber3 [label="订阅成功", style=dashed];
    
    // 发布阶段
    Publisher -> EventMgr [label="发布事件 A"];
    EventMgr -> Subscriber1 [label="回调事件 A"];
    EventMgr -> Subscriber2 [label="回调事件 A"];
    Subscriber1 -> EventMgr [label="处理完成"];
    Subscriber2 -> EventMgr [label="处理完成"];
    EventMgr -> Publisher [label="发布完成", style=dashed];
    
    // 取消订阅阶段
    Subscriber1 -> EventMgr [label="取消订阅事件 A"];
    EventMgr -> Subscriber1 [label="取消成功", style=dashed];
    
    // 后续发布
    Publisher -> EventMgr [label="发布事件 A"];

    EventMgr -> Publisher [label="发布完成", style=dashed];

```

## 说明

事件驱动通信的完整流程：

1. **订阅阶段**
   - 各个系统向事件管理器注册事件监听
   - 事件管理器维护订阅者列表

2. **发布阶段**
   - 事件发布者向事件管理器发布事件
   - 事件管理器查找所有订阅者
   - 依次调用订阅者的回调函数
   - 订阅者处理事件

3. **取消订阅阶段**
   - 订阅者可以随时取消订阅
   - 事件管理器更新订阅者列表

4. **解耦优势**
   - 发布者和订阅者完全解耦
   - 支持一对多的通信模式
   - 易于扩展和维护

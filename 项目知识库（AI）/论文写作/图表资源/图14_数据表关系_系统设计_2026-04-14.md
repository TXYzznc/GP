# 图14：数据表关系图（ERD）

**位置**: 第3章 系统架构  
**章节**: 3.5 数据设计  
**类型**: ERD（实体关系图）  
**用途**: 展示数据模型的设计

## Graphviz DOT 代码

```dot
digraph G {
    rankdir=LR;
    splines=orthogonal;
    nodesep=0.7;
    ranksep=1.0;
    
    // 数据表节点
    BUFF_TABLE [shape=record, label="{BUFF_TABLE|id (PK)|name|duration|value|effect_type}", style=filled, fillcolor="#FFE4B5"];
    SKILL_TABLE [shape=record, label="{SKILL_TABLE|id (PK)|name|cost|damage|cooldown}", style=filled, fillcolor="#FFE4B5"];
    CARD_TABLE [shape=record, label="{CARD_TABLE|id (PK)|name|rarity|cost|effect}", style=filled, fillcolor="#FFE4B5"];
    ITEM_TABLE [shape=record, label="{ITEM_TABLE|id (PK)|name|type|rarity|max_stack}", style=filled, fillcolor="#FFE4B5"];
    ENEMY_TABLE [shape=record, label="{ENEMY_TABLE|id (PK)|name|hp|attack|defense}", style=filled, fillcolor="#FFE4B5"];
    SYNERGY_TABLE [shape=record, label="{SYNERGY_TABLE|id (PK)|name|bonus_type|bonus_value}", style=filled, fillcolor="#FFE4B5"];
    LEVEL_TABLE [shape=record, label="{LEVEL_TABLE|id (PK)|name|difficulty}", style=filled, fillcolor="#FFE4B5"];
    RESOURCE_CONFIG [shape=record, label="{RESOURCE_CONFIG|id (PK)|resource_id (FK)|config}", style=filled, fillcolor="#FFE4B5"];
    
    // 实体节点
    ENTITY [shape=record, label="{ENTITY|id (PK)|type|hp|attack|defense}", style=filled, fillcolor="#87CEEB"];
    PLAYER [shape=record, label="{PLAYER|id (PK)|name|level|exp}", style=filled, fillcolor="#87CEEB"];
    INVENTORY [shape=record, label="{INVENTORY|id (PK)|player_id (FK)|capacity}", style=filled, fillcolor="#87CEEB"];
    CHESS [shape=record, label="{CHESS|id (PK)|synergy_id (FK)|name}", style=filled, fillcolor="#87CEEB"];
    SCENE [shape=record, label="{SCENE|id (PK)|name|level_id (FK)}", style=filled, fillcolor="#87CEEB"];
    RESOURCE [shape=record, label="{RESOURCE|id (PK)|name|path}", style=filled, fillcolor="#87CEEB"];
    
    // 关系定义
    BUFF_TABLE -> ENTITY [label="applies"];
    SKILL_TABLE -> ENTITY [label="uses"];
    CARD_TABLE -> PLAYER [label="owns"];
    ITEM_TABLE -> INVENTORY [label="contains"];
    ENEMY_TABLE -> ENTITY [label="defines"];
    SYNERGY_TABLE -> CHESS [label="affects"];
    LEVEL_TABLE -> SCENE [label="defines"];
    RESOURCE_CONFIG -> RESOURCE [label="configures"];
}
```

## 说明

主要配置表之间的关系：

1. **Buff 表** - 定义所有 Buff 的属性和效果
2. **技能表** - 定义所有技能的属性和消耗
3. **卡牌表** - 定义所有卡牌的属性和效果
4. **物品表** - 定义所有物品的属性和堆叠规则
5. **敌人表** - 定义所有敌人的属性和行为
6. **协同表** - 定义棋子之间的协同效果
7. **关卡表** - 定义游戏关卡的配置
8. **资源配置表** - 定义资源的加载和管理配置

这些表通过外键关系相互关联，形成完整的游戏数据模型。

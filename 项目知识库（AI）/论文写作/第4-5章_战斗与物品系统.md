# 第 4-5 章 战斗系统与物品背包系统

---

## 第 4 章 战斗系统的设计与实现

### 4.1 战斗系统架构

战斗系统是游戏的核心系统之一，负责管理战斗流程、棋子管理、Buff 系统、AI 决策等。系统架构如下：

```
战斗系统
├── 战斗流程管理（CombatProcedure）
│   ├── 准备阶段
│   ├── 战斗阶段
│   └── 结算阶段
├── 棋子管理（BattleChessManager）
│   ├── 棋子数据（ChessData）
│   ├── 棋子实体（ChessEntity）
│   └── 棋子 AI（ChessAI）
├── Buff 系统（BuffManager）
│   ├── Buff 数据（BuffData）
│   ├── Buff 效果（BuffEffect）
│   └── Buff 配置表（BuffTable）
├── 战斗 UI（CombatUI）
│   ├── 棋子显示
│   ├── Buff 显示
│   └── 技能 UI
└── 战斗事件系统
    ├── 战斗开始事件
    ├── 棋子死亡事件
    └── 战斗结束事件
```

### 4.2 棋子系统设计

**4.2.1 棋子数据结构**

每个棋子包含以下数据：
- **基础属性**：生命值、攻击力、防御力、速度等
- **状态**：活跃、眩晕、冰冻、死亡等
- **技能**：可以使用的技能列表
- **装备**：穿戴的装备和 Buff

**4.2.2 棋子管理**

BattleChessManager 负责管理所有棋子：
- 维护棋子列表
- 处理棋子的生命周期
- 提供棋子查询接口
- 发送棋子相关事件

**4.2.3 棋子 AI**

棋子 AI 负责自动决策：
- 选择攻击目标
- 选择使用的技能
- 决定移动方向
- 支持多种 AI 策略（激进、保守、平衡等）

### 4.3 Buff 系统实现

**4.3.1 Buff 数据结构**

Buff 包含以下信息：
- **ID**：唯一标识
- **类型**：增益或减益
- **效果**：属性修改、状态改变等
- **持续时间**：永久或有限时间
- **层数**：可以叠加的 Buff 层数

**4.3.2 Buff 效果**

Buff 可以产生以下效果：
- **属性修改**：增加或减少属性值
- **状态改变**：改变棋子状态（眩晕、冰冻等）
- **特殊效果**：触发特殊机制（反伤、吸血等）

**4.3.3 Buff 配置表**

BuffTable 包含 50+ 个 Buff 的配置，包括：
- 伤害增加、防御增加等属性 Buff
- 眩晕、冰冻等状态 Buff
- 反伤、吸血等特殊 Buff

### 4.4 战斗流程管理

**4.4.1 战斗阶段**

战斗分为三个阶段：

1. **准备阶段**
   - 玩家部署棋子
   - 玩家选择装备和技能
   - 玩家选择策略卡

2. **战斗阶段**
   - 棋子自动攻击
   - 玩家可以使用技能和卡牌
   - 敌人 AI 自动决策
   - 持续到一方全灭

3. **结算阶段**
   - 计算战斗结果
   - 分配奖励
   - 保存战斗数据

**4.4.2 战斗流程控制**

使用状态机管理战斗流程：
```
准备阶段 → 战斗阶段 → 结算阶段 → 返回
```

### 4.5 AI 决策系统

**4.5.1 AI 决策流程**

```
获取当前状态
    ↓
评估可用行动
    ↓
计算每个行动的价值
    ↓
选择价值最高的行动
    ↓
执行行动
```

**4.5.2 AI 策略**

支持多种 AI 策略：
- **激进策略**：优先攻击敌人
- **保守策略**：优先防守和治疗
- **平衡策略**：根据情况灵活选择
- **特殊策略**：针对特定敌人的策略

### 4.6 性能优化方案

**4.6.1 对象池**

- 预先创建棋子对象，重复使用
- 减少 GC 分配
- 提高战斗性能

**4.6.2 缓存**

- 缓存频繁查询的数据
- 减少重复计算
- 提高查询速度

**4.6.3 异步加载**

- 使用 UniTask 进行异步加载
- 不阻塞主线程
- 提高游戏响应速度

## 第 5 章 物品与背包系统的设计与实现

### 5.1 物品系统架构

物品系统负责管理游戏中的所有物品。系统架构如下：

```
物品系统
├── 物品管理（ItemManager）
│   ├── 物品数据（ItemData）
│   ├── 物品配置表（ItemTable）
│   └── 物品效果（ItemEffect）
├── 背包容器（InventoryContainer）
│   ├── 容器数据（ContainerData）
│   ├── 容器 UI（InventoryUI）
│   └── 拖拽交互（DragDropSystem）
├── 物品搜索（ItemSearchSystem）
│   ├── 搜索规则
│   ├── 搜索结果
│   └── 物品生成
└── 数据持久化（SaveSystem）
    ├── 物品存档
    ├── 容器状态
    └── 负重管理
```

### 5.2 背包容器设计

**5.2.1 容器类型**

游戏包含多种容器：
- **背包**：主要物品存储容器
- **装备栏**：装备物品容器
- **快捷栏**：快速使用物品容器
- **掉落物**：地面上的物品容器

**5.2.2 容器数据结构**

每个容器包含：
- **容量**：最多能存储的物品数量
- **物品列表**：当前存储的物品
- **负重**：当前负重和最大负重
- **状态**：打开、关闭、锁定等

**5.2.3 容器管理**

InventoryContainer 负责：
- 添加物品
- 移除物品
- 查询物品
- 管理容量和负重

### 5.3 拖拽交互实现

**5.3.1 拖拽流程**

```
鼠标按下
    ↓
创建拖拽预览
    ↓
鼠标移动
    ↓
更新预览位置
    ↓
检测吸附区域
    ↓
鼠标释放
    ↓
处理放置结果
```

**5.3.2 拖拽检测**

- 使用射线检测检测吸附区域
- 限制检测频率（0.1 秒/次）
- 支持多个吸附区域

**5.3.3 放置处理**

- 检测放置位置是否有效
- 如果有效，执行物品移动
- 如果无效，返回原位置

### 5.4 物品配置表管理

**5.4.1 配置表结构**

ItemTable 包含以下字段：
- **ID**：物品唯一标识
- **名称**：物品名称
- **描述**：物品描述
- **类型**：武器、防具、消耗品等
- **稀有度**：普通、稀有、传说等
- **属性**：攻击力、防御力等
- **效果**：使用物品的效果

**5.4.2 配置表使用**

代码中通过 ID 查询配置表：
```csharp
ItemTable itemTable = ItemTableManager.GetItemById(itemId);
if (itemTable != null)
{
    // 使用配置数据
}
```

### 5.5 UI 交互优化

**5.5.1 UI 设计**

- 清晰的物品显示
- 直观的拖拽交互
- 实时的负重显示
- 物品详情提示

**5.5.2 交互优化**

- 快速的响应速度
- 流畅的动画效果
- 清晰的视觉反馈
- 易于理解的操作

### 5.6 数据持久化

**5.6.1 存档设计**

存档包含以下信息：
- 背包中的物品列表
- 装备栏的装备
- 快捷栏的物品
- 负重信息

**5.6.2 存档加载**

游戏启动时：
1. 读取存档文件
2. 恢复背包状态
3. 恢复装备状态
4. 恢复快捷栏状态

**5.6.3 存档保存**

游戏运行中定期保存：
- 物品变化时保存
- 场景切换时保存
- 定期自动保存

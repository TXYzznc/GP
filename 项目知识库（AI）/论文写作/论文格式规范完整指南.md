## 1. 论文结构规范

### 1.1 章节比重要求

| 章节 | 页数要求 | 说明 |
|------|----------|------|
| 第1章 绪论 | 5-6页 | 研究背景、意义、内容、方法 |
| 第2章 相关工作 | 7-8页 | 文献综述、技术背景 |
| 第3章 需求分析 | 5-6页 | 需求分析、用例图等 |
| 第4章 系统设计 | 12-14页 | 架构设计、类图、E-R图等 |
| 第5章 系统实现 | 14-16页 | 核心功能实现、流程图等 |
| 第6章 系统测试 | 4-5页 | 测试用例、测试结果 |
| 第7章 总结与展望 | 1-2页 | 工作总结、未来展望 |
| 参考文献 | 3页左右 | 30-40篇参考文献 |
| **总计** | **至少60页** | 致谢前 |

### 1.2 章节结构要求

- **2-6章开头**：必须有3-4行的本章简述
- **2-6章结尾**：必须有本章小结
- **关键概念及定义**：全文统一

### 1.3 核心章节内容

| 章节 | 核心内容 | 必需图表 |
|------|----------|----------|
| 第3章 需求分析 | 功能需求、非功能需求 | 用例图 |
| 第4章 系统设计 | 架构设计、数据库设计 | 类图、E-R图 |
| 第5章 系统实现 | 核心功能实现 | 流程图 |
| 第6章 系统测试 | 测试用例、测试结果 | 测试用例表、界面截图 |

## 2. 字体与段落格式

### 2.1 字体规范

| 内容类型 | 中文字体 | 英文字体 | 字号 | 说明 |
|----------|----------|----------|------|------|
| 一级标题（章） | 黑体 | - | 小二（18pt，size=36） | 居中 |
| 二级标题（节） | 黑体 | - | 四号（14pt，size=28） | 左对齐 |
| 三级标题 | 黑体 | - | 小四（12pt，size=24） | 左对齐 |
| 四级标题 | 黑体 | - | 小四（12pt，size=24） | 左对齐，不建议使用 |
| 正文 | 宋体 | Times New Roman | 小四（12pt，size=24） | 两端对齐 |
| 摘要正文 | 宋体 | Times New Roman | 小四（12pt，size=24） | 两端对齐 |
| 关键词 | 宋体 | Times New Roman | 小四（12pt，size=24） | 顶格 |
| 图表标题 | 宋体 | Times New Roman | 五号（10.5pt，size=21） | 居中 |
| 参考文献 | 宋体 | Times New Roman | 小四（12pt，size=24） | 悬挂缩进 |
| 目录标题 | 黑体 | - | 小二（18pt，size=36） | 居中 |
| 页眉 | 宋体 | Times New Roman | 五号（10.5pt，size=21） | 居中 |
| 页码 | 宋体 | Times New Roman | 五号（10.5pt，size=21） | 居中 |

### 2.2 字号对照表

| 字号名称 | 字号数值 | size属性值（半磅） |
|----------|----------|-------------------|
| 小二 | 18pt | 36 |
| 三号 | 16pt | 32 |
| 四号 | 14pt | 28 |
| 小四 | 12pt | 24 |
| 五号 | 10.5pt | 21 |

### 2.3 段落格式

| 内容类型 | 行距 | 对齐方式 | 缩进 |
|----------|------|----------|------|
| 正文 | 固定值 23磅（line=460） | 两端对齐 | 首行缩进2字符 |
| 标题 | 单倍行距 | 按级别 | 无缩进 |
| 图表标题 | 单倍行距 | 居中 | 无缩进 |
| 参考文献 | 固定值 23磅 | 两端对齐 | 悬挂缩进 |

### 2.4 字体使用注意事项

- **英文字体统一为新罗马（Times New Roman）**
- **缩写规范**：首次出现全称，括号内标注缩写，如 "Virtual Reality, VR"
- **标点符号**：正文不应存在英文标点，统一使用中文标点

## 4. 章节编号规范

### 4.1 编号层级

```
第1章 绪论                    （一级标题，黑体小二居中）
  1.1 研究背景                 （二级标题，黑体四号左对齐）
    1.1.1 研究背景概述          （三级标题，黑体小四左对齐）
      (1) 具体内容              （四级标题，不建议使用）
      (a) 更细分内容            （五级标题，不建议使用）
```

### 4.2 编号规则

- **章节编号**：1, 1.1, 1.1.1, (1), (a)
- **不应出现四级标题**：尽量控制在三级以内
- **标题中尽可能不出现标点**

### 4.3 常见错误

- 章节编号错乱
- 标题层级过多
- 标题中使用标点符号

## 5. 图表规范

### 5.1 图表命名

- **图名位置**：图下方居中
- **表名位置**：表上方居中
- **编号格式**：按章节编号，如"图3-1"、"表4-2"
- **名称要求**：准确、简洁、能说明图表内容

### 5.2 图表引用

- **所有图表须在正文中引用**
- 引用格式："如图3-1所示"、"如表4-2所示"
- 图表应紧跟在首次引用的段落后

### 5.3 图表格式要求

#### 图片要求
- **不建议用深色背景**：打印看不清
- **非自绘图必须加引用**：标注来源
- **分辨率要求**：清晰可辨

#### 表格要求
- **标准三线表**：顶线、栏目线、底线
- **跨页需要续表**：标注"续表X-X"
- **表格宽度**：不超过页面宽度

### 5.4 图表常见错误

- 名称不准确
- 编号标错乱
- 未在文中引用
- 深色背景打印不清
- 非自绘图未加引用
- 表格不是三线表

## 6. 参考文献规范

### 6.1 数量要求

- **总数**：30-40篇
- **近两年文献比例**：应占一定比例
- **中英文献比例**：应保持平衡

### 6.2 格式要求

- **文中引用**：使用上标标号，如[1]、[2-5]
- **编号顺序**：按引用顺序编号
- **格式统一**：遵循学校规定的参考文献格式

### 6.3 参考文献类型标识

| 类型 | 标识 | 示例 |
|------|------|------|
| 专著 | M | [1] 作者. 书名[M]. 出版地: 出版社, 年份. |
| 期刊文章 | J | [2] 作者. 文章名[J]. 期刊名, 年, 卷(期): 页码. |
| 学位论文 | D | [3] 作者. 论文名[D]. 学校, 年份. |
| 会议论文 | C | [4] 作者. 文章名[C]//会议名. 年份: 页码. |
| 电子文献 | EB/OL | [5] 作者. 文章名[EB/OL]. 网址, 访问日期. |

## 7. 摘要与关键词

### 7.1 摘要要求

- **篇幅**：1页左右
- **内容**：交代主要工作及贡献
- **语言**：精炼、准确
- **中英文对应**：中文摘要与英文摘要内容对应

### 7.2 摘要结构

1. **研究背景与目的**（1-2句）
2. **研究方法**（1-2句）
3. **主要工作与成果**（核心内容，3-5句）
4. **结论与意义**（1-2句）

### 7.3 关键词要求

- **数量**：4-6个
- **顺序**：从宏观到具体
- **体现核心技术**
- **中英文对应**

### 7.4 关键词格式

```
关键词：关键词1；关键词2；关键词3；关键词4
Keywords: Keyword1; Keyword2; Keyword3; Keyword4
```

## 10. 公式规范

### 10.1 公式格式

- **公式本身**：居中
- **编号**：居右，格式为"(X-X)"
- **编号顺序**：按章节编号，如"(3-1)"

### 10.2 公式说明

- **关键参数须文中说明**
- **编号不错乱**

### 10.3 公式示例

```
E = mc²                                    (3-1)

其中，E表示能量，m表示质量，c表示光速。
```

## 13. 必需图表类型

### 13.1 图表类型总览

| 图表类型 | 适用章节 | 用途说明 |
|----------|----------|----------|
| 用例图 | 第3章 需求分析 | 描述系统功能需求 |
| 类图 | 第4章 系统设计 | 描述系统静态结构 |
| E-R图 | 第4章 系统设计 | 描述数据库设计 |
| 时序图 | 第4-5章 | 描述对象交互 |
| 活动图 | 第3-5章 | 描述业务流程 |
| 程序流程图 | 第5章 系统实现 | 描述程序逻辑 |
| 功能层次图 | 第4章 系统设计 | 描述系统模块结构 |
| 测试用例表 | 第6章 系统测试 | 描述测试内容 |
| 界面截图 | 第5-6章 | 展示系统界面 |

### 13.2 用例图规范

**组成要素**：
- 参与者（Actor）
- 用例（Use Case）
- 关系（关联、包含、扩展、泛化）

**绘制要点**：
- 所有用例都应该和某一个参与者有交互关系
- 每个用例要辅以用例说明
- 用例之间存在3种关系：
  - 包含关系（<<include>>）：复杂用例分解为小用例
  - 扩展关系（<<extend>>）：为基用例附加功能
  - 泛化关系：用例的不同实现手段

### 13.3 类图规范

**组成要素**：
- 类（属性、方法）
- 关系（泛化、关联、聚集、合成、依赖）

**绘制要点**：
- 尽量在一张类图中描述超过一种关系
- 尽量将类中的属性和方法画出来
- 常见错误：类之间的关系画得不准确

**类之间关系**：
- 泛化：父类与子类
- 关联：一个类是另一个类的数据成员
- 聚集：整体与部分，空心菱形
- 合成：强聚集，实心菱形
- 依赖：数据成员类型或参数类型

### 13.4 E-R图规范

**组成要素**：
- 实体
- 属性
- 联系

**绘制要点**：
- 一般不会单独画一个实体
- 对于大一点的ER图，实体的属性可以不画
- 针对多对多的联系，一般将联系的属性画出来

**常见错误**：
- 将实体间联系的内涵弄混
- 两个实体直接连线

### 13.5 时序图规范

**组成要素**：
- 对象
- 消息（简单消息、同步消息、异步消息）
- 生命线

**绘制要点**：
- 消息线上要标注消息
- 表示对象的方框内应是对象，不是类
- 通常使用简单消息即可

### 13.6 活动图规范

**组成要素**：
- 活动
- 分支
- 并发
- 泳道（可选）

**绘制要点**：
- 可按不同角色或模块画泳道图
- 描述一个过程或者操作的步骤

### 13.7 程序流程图规范

**组成要素**：
- 开始/结束
- 处理
- 判断
- 流程线

**绘制要点**：
- 描述程序运行具体步骤
- 多条件case的画法要规范

### 13.8 功能层次图规范

**组成要素**：
- 功能模块
- 模块间联系

**绘制要点**：
- 模块间联系的线一般没有箭头
- 位于同一层次上的表示模块的矩形尽量大小一致
- 模块的层次划分要与需求分析、系统实现部分相一致

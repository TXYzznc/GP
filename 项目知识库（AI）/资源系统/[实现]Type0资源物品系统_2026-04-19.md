# Type 0 资源物品系统实现总结

**完成时间**: 2026-04-19  
**状态**: ✅ 完成

## 系统概述

Type 0 资源物品是一种特殊的物品类型，用于表示游戏中的资源（金币、起源石、灵石）。这些物品在探索中作为宝箱掉落物显示，但不进入背包系统，而是直接转换为全局资源。

## 资源物品配置

### ItemTable.txt 中的配置

| ID | 名称 | Type | Quality | Icon | MaxStackCount | 说明 |
|---|---|---|---|---|---|---|
| 999 | 金币 | 0 | 1 | 1101 | 999999999 | 基础货币 |
| 9999 | 起源石 | 0 | 4 | 1102 | 999999999 | 高级货币 |
| 99999 | 灵石 | 0 | 3 | 1103 | 999999999 | 战斗资源 |

## 核心实现

### 1. 宝箱容器处理 (TreasureBoxSlotContainerImpl.cs)

#### ReceiveItemFromTreasureBox 方法
- **功能**: 接收来自宝箱的物品
- **Type 0 处理**: 直接调用 `HandleResourceItem()` 转换为全局资源，不存入宝箱
- **日志**: `[资源直接添加] ID={itemId}, 数量={count}`

```csharp
// 检查是否为 Type 0 资源项
if (itemRow != null && itemRow.Type == 0)
{
    HandleResourceItem(itemId, count);
    return true;
}
```

#### MoveToInventory 方法
- **功能**: 处理从宝箱拖拽物品到背包的操作
- **Type 0 处理**: 检测到 Type 0 物品时，直接调用 `HandleResourceItem()` 而不是放入背包
- **日志**: `[宝箱→资源] 资源项转换: ID={itemId}, 数量={count}`

```csharp
if (itemRow != null && itemRow.Type == 0)
{
    HandleResourceItem(itemId, count);
    return true;
}
```

#### TakeAll 方法
- **功能**: 一键拿走宝箱中的所有物品
- **Type 0 处理**: 检测 Type 0 物品，调用 `HandleResourceItem()` 添加到全局资源
- **其他物品**: 放入背包（如果背包有空间）

```csharp
if (itemRow != null && itemRow.Type == 0)
{
    HandleResourceItem(item.ItemId, item.Count);
    m_Slots[i] = null;
}
```

#### HandleResourceItem 方法
- **功能**: 根据资源 ID 添加到对应的全局资源
- **实现**:
  - ID 999 → `AddGold()`
  - ID 9999 → `AddOriginStone()`
  - ID 99999 → `AddSpiritStone()`

### 2. 背包 UI 过滤 (InventoryUI.cs)

#### RefreshInventory 方法
- **功能**: 刷新背包物品显示
- **Type 0 处理**: 检测到 Type 0 物品时，跳过显示（不添加到背包 UI）
- **日志**: `过滤资源项: slot[{i}] = {name} (Type 0)`

```csharp
if (itemRow != null && itemRow.Type == 0)
{
    slotUI.SetData(null);
    continue;
}
```

#### RefreshEquipSlots 方法
- **功能**: 刷新装备栏显示
- **Type 0 处理**: 只显示 Type 4 装备，自动过滤其他类型（包括 Type 0）

### 3. 宝箱 UI 显示 (TreasureBoxUI.cs)

#### RefreshSlots 方法
- **功能**: 刷新宝箱格子显示
- **Type 0 处理**: 正常显示（带图标），不做特殊处理
- **说明**: Type 0 物品在宝箱中显示，只在拿走时才转换为资源

### 4. 物品菜单处理 (ItemContextMenu.cs)

#### RefreshMenuOptions 方法
- **功能**: 根据物品类型显示不同的菜单选项
- **Type 0 处理**: 隐藏菜单背景，不显示任何操作按钮
- **日志**: `资源物品，隐藏菜单背景`

```csharp
case 0: // 资源
    if (varMenuBg != null)
        varMenuBg.SetActive(false);
    break;
```

## 工作流程

### 探索中获得资源物品

1. **宝箱生成**: 宝箱中包含 Type 0 资源物品（ID 999/9999/99999）
2. **UI 显示**: 宝箱 UI 显示资源物品，带有对应的图标
3. **玩家操作**:
   - **点击"拿走"**: 调用 `TakeAll()` → 检测 Type 0 → 调用 `HandleResourceItem()` → 添加到全局资源
   - **拖拽到背包**: 调用 `MoveToInventory()` → 检测 Type 0 → 调用 `HandleResourceItem()` → 添加到全局资源
   - **右键菜单**: 菜单背景隐藏，无操作选项

### 资源添加流程

```
Type 0 物品 (ID 999/9999/99999)
    ↓
HandleResourceItem(itemId, count)
    ↓
根据 ID 调用对应方法:
  - 999 → PlayerAccountDataManager.AddGold()
  - 9999 → PlayerAccountDataManager.AddOriginStone()
  - 99999 → PlayerAccountDataManager.AddSpiritStone()
    ↓
全局资源更新 → CurrencyUI 刷新显示
```

## 关键设计决策

### 1. 为什么 Type 0 物品不进入背包？
- **资源管理分离**: 全局资源和背包物品使用不同的管理系统
- **简化 UI**: 避免在背包中显示资源物品，保持背包专注于装备和消耗品
- **性能优化**: 减少背包容量占用

### 2. 为什么 Type 0 物品在宝箱中显示？
- **用户反馈**: 玩家需要看到获得了什么资源
- **一致性**: 所有掉落物都在宝箱中显示，然后根据类型处理
- **灵活性**: 支持未来扩展（如条件性获取）

### 3. 为什么隐藏 Type 0 物品的菜单？
- **无操作**: 资源物品无法使用、拆分或丢弃
- **简化 UX**: 避免显示无用的菜单选项
- **清晰意图**: 隐藏菜单表示"这是自动处理的"

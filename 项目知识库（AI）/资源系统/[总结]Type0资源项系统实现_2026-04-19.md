# Type 0 资源项系统实现总结

**完成时间**: 2026-04-19  
**任务状态**: ✅ 完成

## 任务概述

实现 Type 0 资源项（金币、起源石、灵石）的完整系统，使其在宝箱中显示但不进入背包，而是直接转换为全局资源。

## 核心设计

### 资源分类

| 资源类型 | ItemID | 类型 | 说明 |
|---------|--------|------|------|
| 金币 | 999 | Type 0 | 基础货币 |
| 起源石 | 9999 | Type 0 | 高级货币 |
| 灵石 | 99999 | Type 0 | 战斗资源 |

### 系统流程

```
宝箱打开 → 生成物品列表 → 显示在TreasureBoxUI
                          ↓
                    玩家点击"全部拿走"
                          ↓
                    TakeAll() 处理
                    ↙         ↘
            Type 0 资源项      其他物品
                ↓              ↓
            直接添加到      放入背包
            全局资源
```

## 实现细节

### 1. 数据模型更新

**文件**: `Assets/AAAGame/Scripts/DataModel/PlayerAccountData.cs`

添加灵石字段到 `PlayerSaveData`:
```csharp
/// <summary>
/// 灵石（战斗资源，用于棋子进阶）
/// </summary>
public int SpiritStone;
```

### 2. 资源管理器扩展

**文件**: `Assets/AAAGame/Scripts/Manager/PlayerAccountDataManager.cs`

新增方法:
- `AddOriginStone(int amount)` - 添加起源石
- `ConsumeOriginStone(int amount)` - 消耗起源石
- `AddSpiritStone(int amount)` - 添加灵石
- `ConsumeSpiritStone(int amount)` - 消耗灵石

所有方法都包含日志输出，便于调试。

### 3. 宝箱容器改造

**文件**: `Assets/AAAGame/Scripts/UI/Components/TreasureBoxSlotContainerImpl.cs`

修改 `TakeAll()` 方法:
- 检查物品类型（通过 ItemTable 的 Type 字段）
- Type 0 资源项调用 `HandleResourceItem()` 直接添加到全局资源
- 其他物品正常放入背包

新增 `HandleResourceItem()` 方法:
- 根据资源ID (999/9999/99999) 调用对应的资源添加方法
- 包含完整的日志记录

### 4. 背包UI过滤

**文件**: `Assets/AAAGame/Scripts/UI/InventoryUI.cs`

修改 `RefreshInventory()`:
- 检查物品类型，过滤 Type 0 资源项
- 资源项不显示在背包格子中

`RefreshEquipSlots()` 已有类型检查，自动过滤 Type 0 项。

## 关键特性

✅ **资源自动转换**: 宝箱中的资源项自动转换为全局资源  
✅ **背包隔离**: Type 0 资源项永远不进入背包  
✅ **UI一致性**: 宝箱UI显示资源项，但收集时特殊处理  
✅ **日志完整**: 所有操作都有 DebugEx 日志输出  
✅ **类型安全**: 通过 ItemTable 的 Type 字段进行类型检查

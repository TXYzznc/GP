### 场景 1: 宝箱中获得资源物品

```
1. 宝箱生成 → 包含 Type 0 资源物品 (ID 999/9999/99999)
2. TreasureBoxUI.RefreshSlots() → 显示资源物品（带图标）
3. 玩家点击"拿走" → TreasureBoxSlotContainerImpl.TakeAll()
4. 检测 Type 0 → HandleResourceItem(itemId, count)

   - 999 → PlayerAccountDataManager.AddGold()

6. 全局资源更新 → CurrencyUI 刷新显示
```

```
5. 日志: [宝箱→资源] 资源项转换
```

## 关键设计决策

### 1. 为什么在 InventoryManager.AddItem() 中添加 Type 0 检测？
- **防卫性编程**: 即使其他地方没有正确处理 Type 0，也能防止资源进入背包
- **集中管理**: 所有进入背包的物品都经过这个方法，统一检测
- **日志记录**: 如果发现 Type 0 物品试图进入背包，会有警告日志

### 2. 为什么 Type 0 物品在宝箱中显示？
- **用户反馈**: 玩家需要看到获得了什么资源
- **一致性**: 所有掉落物都在宝箱中显示
- **灵活性**: 支持未来扩展（如条件性获取）

### 3. 为什么隐藏 Type 0 物品的菜单？
- **无操作**: 资源物品无法使用、拆分或丢弃
- **简化 UX**: 避免显示无用的菜单选项
- **清晰意图**: 隐藏菜单表示"这是自动处理的"

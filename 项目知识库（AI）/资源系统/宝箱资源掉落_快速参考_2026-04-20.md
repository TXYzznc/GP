## 📊 掉落机制详解

### 金币掉落
```
if (Random.value < CoinsProbability)
{
    coins = Random.Range(MinCoins, MaxCoins + 1)
    PlayerAccountDataManager.AddGold(coins)
}
```

**特点**:
- 概率判断：`Random.value` 返回 [0, 1) 的随机数
- 金币数量：[MinCoins, MaxCoins] 均匀分布
- 保存到存档：自动保存

**示例**:
- `CoinsProbability: 0.5` → 50% 概率掉落
- `MinCoins: 100, MaxCoins: 200` → 掉落 100-200 之间的金币

### 灵石掉落（连续抽奖）
```
totalStone = MinMagicaStone  // 先获得保底

while (Random.value < MagicaStoneProbability)
{
    totalStone++
    if (totalStone >= MaxMagicaStone) break
}

PlayerRuntimeDataManager.AddSpiritStone(totalStone)
```

**特点**:
- 保底机制：最少获得 MinMagicaStone 个
- 连续抽奖：每次有 MagicaStoneProbability 概率继续
- 上限保护：不会超过 MaxMagicaStone 个
- 不保存到存档：仅在战斗中存在

**示例**:
- `MinMagicaStone: 1, MaxMagicaStone: 5, MagicaStoneProbability: 0.5`
  - 最少获得 1 个
  - 每次有 50% 概率继续抽奖
  - 最多获得 5 个
  - 期望值约为 1 + 0.5 + 0.25 + 0.125 + 0.0625 ≈ 1.94 个

## ⚠️ 常见问题

### Q: 灵石没有显示？
**A**: 灵石是战斗资源，仅在战斗中显示。确保：
- 在战斗场景中打开宝箱
- PlayerRuntimeDataManager 已初始化
- UI 已订阅灵石变化事件

### Q: 金币没有保存？
**A**: 金币应该自动保存。检查：
- PlayerAccountDataManager 是否已初始化
- 是否调用了 `SaveArchive()`
- 配置表中 CoinsProbability 是否 > 0

### Q: 掉落数量不对？
**A**: 检查配置表中的 Min/Max 值：
- MinCoins > MaxCoins → 会导致异常
- MinMagicaStone > MaxMagicaStone → 会导致异常
- 确保数值都是正整数

### Q: 概率不生效？
**A**: 概率值范围是 [0, 1]：
- 0 = 0% 概率（不掉落）
- 0.5 = 50% 概率
- 1.0 = 100% 概率（必掉落）

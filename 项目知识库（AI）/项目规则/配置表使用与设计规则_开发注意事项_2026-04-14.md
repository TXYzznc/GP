# 配置表使用与设计规则

> **最后更新**: 2026-03-23
> **状态**: 有效
> **工作流**: Excel → TXT → Bytes

---

## 概述

本项目使用 GameFramework 的 DataTable 系统进行配置表管理。配置表采用 Excel → TXT → Bytes 的工作流程，支持热更新和高效的二进制读取。

**核心设计原则**：
- 配置表中不应直接填入资源路径，而是填入资源 ID
- 所有资源路径统一在 `ResourceConfigTable` 中管理
- 使用 `ResourceIds` 常量类引用资源
- 通过 `ResourceExtension` 扩展类加载资源

---

## 配置表设计规范

### 资源引用规范（重要）

#### 核心原则

**❌ 禁止在配置表中直接填入资源路径**

配置表中不应该包含资源路径字段，而应该使用资源 ID 字段。

**原因**：
1. 资源路径变更时，只需修改 `ResourceConfigTable`，无需修改所有配置表
2. 统一管理资源路径，避免路径错误和重复
3. 支持资源的动态加载和热更新
4. 便于资源的批量管理和优化

#### 错误示例

```
❌ 错误：ItemTable 配置表中直接填入资源路径

| ID | Name   | IconPath                  | PrefabPath                |
|----|--------|---------------------------|---------------------------|
| 1  | 金币   | UI/Items/Gold.png         | Items/GoldItem            |
| 2  | 灵石   | UI/Items/MagicalStone.png | Items/MagicalStoneItem    |
```

**问题**：
- 资源路径分散在多个配置表中，难以统一管理
- 路径变更时需要修改所有相关配置表
- 容易出现路径错误和拼写错误
- 无法统一管理资源类型和加载方式

#### 正确示例

```
✅ 正确：ItemTable 配置表中使用资源 ID

| ID | Name   | IconId | PrefabId |
|----|--------|--------|----------|
| 1  | 金币   | 1101   | 2201     |
| 2  | 灵石   | 1102   | 2202     |
```

**优点**：
- 资源路径统一在 `ResourceConfigTable` 中管理
- 使用 `ResourceIds` 常量引用，避免硬编码
- 路径变更只需修改 `ResourceConfigTable`
- 支持资源的统一加载和管理

#### 资源 ID 命名规范

在配置表中，资源 ID 字段应该遵循以下命名规范：

- **图标资源**：`IconId`（单个）或 `IconIds`（数组）
- **预制体资源**：`PrefabId`（单个）或 `PrefabIds`（数组）
- **特效资源**：`EffectId`（单个）或 `EffectIds`（数组）
- **材质资源**：`MaterialId`（单个）或 `MaterialIds`（数组）
- **纹理资源**：`TextureId`（单个）或 `TextureIds`（数组）
- **配置资源**：`ConfigId`（单个）或 `ConfigIds`（数组）

**示例**：

```
| ID | Name     | IconId | PrefabId | EffectIds    |
|----|----------|--------|----------|--------------|
| 1  | 火球术   | 1501   | 2501     | 2401|2402    |
| 2  | 冰霜箭   | 1502   | 2502     | 2403         |
```

#### 代码中使用资源 ID

```csharp
using GameExtension;
using Cysharp.Threading.Tasks;

public class ItemManager : MonoBehaviour
{
    private async void LoadItemData(int itemId)
    {
        // 从 ItemTable 获取物品数据
        var itemTable = GF.DataTable.GetDataTable<ItemTable>();
        var itemData = itemTable.GetDataRow(itemId);
        
        if (itemData == null)
        {
            DebugEx.Error("ItemManager", $"物品数据不存在: ID={itemId}");
            return;
        }
        
        DebugEx.Log("ItemManager", $"开始加载物品: {itemData.Name}");
        
        // ✅ 正确：使用 IconId 字段（资源 ID）加载图标
        Sprite icon = await ResourceExtension.LoadSpriteAsync(itemData.IconId);
        if (icon != null)
        {
            DebugEx.Success("ItemManager", $"物品图标加载成功: {itemData.Name}");
            m_IconImage.sprite = icon;
        }
        
        // ✅ 正确：使用 PrefabId 字段（资源 ID）加载预制体
        GameObject prefab = await ResourceExtension.LoadPrefabAsync(itemData.PrefabId);
        if (prefab != null)
        {
            DebugEx.Success("ItemManager", $"物品预制体加载成功: {itemData.Name}");
        }
    }
}
```

## 第一步：创建 Excel 配置表

### 1. Excel 表格格式

Excel 配置表必须遵循以下格式：

| 行号 | 内容 | 说明 |
|:---|:---|:---|
| 第 1 行 | 字段注释 | 中文说明，方便策划理解 |
| 第 2 行 | 字段名称 | 英文字段名，必须以大写字母开头 |
| 第 3 行 | 字段类型 | 数据类型（int, string, float 等） |
| 第 4 行起 | 数据内容 | 实际配置数据 |

**示例**：

```
| ID注释 | 名称注释 | 等级注释 | 攻击力注释 |
| Id     | Name     | Level    | Attack     |
| int    | string   | int      | float      |
| 1      | 战士     | 10       | 50.5       |
| 2      | 法师     | 15       | 30.0       |
```

### 2. 字段命名规范

- **必须以大写字母开头**：`Name`、`Level`、`Attack`
- **使用 PascalCase**：`MaxHealth`、`MoveSpeed`
- **禁止使用特殊字符**：不能包含空格、中文、符号
- **ID 字段必须存在**：第一个字段必须是 `Id`（int 类型）

### 3. 支持的数据类型

#### 基础类型
- `int`、`float`、`double`、`bool`、`string`
- `long`、`byte`、`char`

#### Unity 类型
- `vector2`、`vector3`、`vector4`
- `color`、`quaternion`、`rect`

#### 数组类型
- `int[]`、`float[]`、`string[]`
- `vector3[]`、`color[]`

#### 二维数组
- `int[][]`、`float[][]`、`string[][]`

#### 枚举类型
- `enum`（需要在代码中定义枚举）

**数组格式示例**：
```
// 一维数组：使用 | 分隔
1|2|3|4|5

// 二维数组：使用 | 分隔行，, 分隔列
1,2,3|4,5,6|7,8,9

// Vector3 数组
0,1,0|1,0,0|0,0,1
```

## 第二步：转换为 TXT 格式

### 1. 使用 txt-to-xlsx-converter 工具

项目提供了专用的配置表转换工具，支持 AI 生成 TXT 配置表。

**工具位置**：`.kiro/skills/txt-to-xlsx-converter/`

**使用方法**：

```python
# 1. 生成 TXT 配置表到 MCP 工作区
fsWrite("MCP工作区/配置表/TableName.txt", content)

# 2. 启动 TCP 服务器（首次使用）
controlPwshProcess(
    action="start",
    command="python mcp_server_tcp.py",
    cwd=".kiro/skills/txt-to-xlsx-converter/Tool/Server"
)

# 3. 执行转换
controlPwshProcess(
    action="start",
    command="python .kiro/skills/txt-to-xlsx-converter/Tool/Core/txt_converter.py -f 配置表/TableName.txt"
)
```

### 2. TXT 格式规范

TXT 配置表使用 Tab 分隔符（\t），格式如下：

```
#	TableName													
#	ID		Field1	Field2	Field3
#	int		type1	type2	type3
#	ID编号	备注	说明1	说明2	说明3
	1		value1	value2	value3
	2		value1	value2	value3
```

**关键规则**：
- 使用 Tab 分隔符（\t）
- 第 3 列固定为空
- 数据行第 1 列为空
- 元数据行第 1 列是 `#`
- ID 列在第 2 列

### 3. 手动创建 TXT（可选）

如果不使用转换工具，也可以手动创建 TXT 文件：

1. 将 Excel 另存为 `.txt` 格式
2. 确保使用 UTF-8 编码
3. 检查 Tab 分隔符是否正确
4. 保存到 `Assets/AAAGame/DataTable/` 目录

```csharp
public partial class TestTable
```

## 配置表最佳实践

### 1. 命名规范

- **表名**：使用 PascalCase，如 `ItemTable`、`EnemyTable`
- **字段名**：使用 PascalCase，如 `MaxHealth`、`MoveSpeed`
- **文件名**：与表名一致，如 `ItemTable.xlsx`

### 2. 数据组织

- **单一职责**：一个配置表只管理一类数据
- **避免冗余**：相关数据通过 ID 关联，不要重复存储
- **合理分表**：数据量大时考虑分表，如 `Enemy1Table`、`Enemy2Table`

### 3. 性能优化

- **使用二进制格式**：`useBytes: true`，加载速度更快
- **预加载常用表**：在 `AppConfigs` 中配置自动加载
- **缓存查询结果**：频繁查询的数据缓存到字典中

```csharp
// 缓存示例
private Dictionary<string, TestTable> m_NameCache;

private void BuildCache()
{
    var testTable = GF.DataTable.GetDataTable<TestTable>();
    m_NameCache = new Dictionary<string, TestTable>();
    
    foreach (var row in testTable)
    {
        m_NameCache[row.Name] = row;
    }
    
    DebugEx.Success("ConfigCache", "配置表缓存构建完成");
}

private TestTable GetByName(string name)
{
    if (m_NameCache.TryGetValue(name, out var row))
    {
        return row;
    }
    
    DebugEx.Warning("ConfigCache", $"未找到名称为 {name} 的配置");
    return null;
}
```

### 4. 数据验证

在加载配置表后，建议进行数据验证：

```csharp
private void ValidateTestTable()
{
    var testTable = GF.DataTable.GetDataTable<TestTable>();
    
    foreach (var row in testTable)
    {
        // 检查必填字段
        if (string.IsNullOrEmpty(row.Name))
        {
            DebugEx.Error("ConfigValidator", $"ID={row.Id} 的 Name 字段为空");
        }
        
        // 检查数值范围
        if (row.Level < 1 || row.Level > 100)
        {
            DebugEx.Warning("ConfigValidator", $"ID={row.Id} 的 Level={row.Level} 超出范围");
        }
    }
    
    DebugEx.Success("ConfigValidator", "TestTable 数据验证完成");
}
```

## 常见问题

### Q1: 配置表生成失败怎么办？

**排查步骤**：
1. 检查 Excel 格式是否正确（字段名、类型）
2. 确认字段名符合命名规范（大写字母开头）
3. 检查数据类型是否支持
4. 查看 Console 错误日志

### Q2: 配置表加载失败怎么办？

**排查步骤**：
1. 确认配置表文件存在（.bytes 或 .txt）
2. 检查配置表是否在 `AppConfigs` 中配置
3. 确认配置表路径正确
4. 查看 Console 错误日志

### Q3: 如何调试配置表数据？

**调试方法**：
1. 使用文本格式加载（`useBytes: false`）
2. 在 `ParseDataRow` 方法中添加断点
3. 使用 `DebugEx.Log` 输出数据内容
4. 检查 .txt 文件内容是否正确

### Q4: 配置表数据不对怎么办？

**解决方案**：
1. 检查 Excel 文件内容
2. 重新生成配置表（删除 .bytes 和 .cs 文件）
3. 清除缓存重新加载
4. 使用数据验证方法检查

## 禁止事项

### ❌ 禁止硬编码配置数据

```csharp
// ❌ 禁止硬编码
int maxLevel = 100;
string weaponName = "神剑";

// ✅ 正确做法：从配置表读取
var config = GF.DataTable.GetDataTable<GameConfig>().GetDataRow(1);
int maxLevel = config.MaxLevel;

var weapon = GF.DataTable.GetDataTable<ItemTable>().GetDataRow(1001);
string weaponName = weapon.Name;
```

### ❌ 禁止在配置表中填入资源路径

```csharp
// ❌ 禁止：在配置表中填入资源路径
| ID | Name   | IconPath              | PrefabPath        |
|----|--------|-----------------------|-------------------|
| 1  | 金币   | UI/Items/Gold.png     | Items/GoldItem    |

// ✅ 正确：在配置表中填入资源 ID
| ID | Name   | IconId | PrefabId |
|----|--------|--------|----------|
| 1  | 金币   | 1101   | 2201     |
```

**原因**：
- 资源路径应该统一在 `ResourceConfigTable` 中管理
- 使用资源 ID 可以避免路径错误和重复
- 便于资源的统一加载和管理

### ❌ 禁止在运行时修改配置表数据

```csharp
// ❌ 禁止修改配置表数据
var row = testTable.GetDataRow(1);
// row.Level = 20; // 配置表数据是只读的

// ✅ 正确做法：使用运行时数据
public class PlayerData
{
    public int Level { get; set; }

```

### ❌ 禁止频繁查询配置表

```csharp
// ❌ 禁止在 Update 中查询配置表
void Update()
{
    var row = testTable.GetDataRow(1); // 每帧查询，性能差
```

### ❌ 禁止硬编码资源 ID

```csharp
// ❌ 禁止：硬编码资源 ID
Sprite icon = await ResourceExtension.LoadSpriteAsync(1101);

// ✅ 正确：使用 ResourceIds 常量
```

## 总结

1. **遵循格式规范**：Excel 表格必须符合 4 行格式要求
2. **使用工具生成**：通过 DataTableGenerator 自动生成代码
3. **二进制格式优先**：使用 `.bytes` 格式提升性能
4. **合理组织数据**：单一职责，避免冗余
5. **缓存查询结果**：频繁使用的数据缓存到内存
6. **添加数据验证**：加载后验证数据有效性
7. **使用扩展类**：为复杂配置表添加自定义方法
8. **添加日志输出**：使用 `DebugEx` 记录关键操作
9. **使用资源 ID**：配置表中不填路径，只填资源 ID
10. **统一资源管理**：所有资源路径在 `ResourceConfigTable` 中管理

遵循这些规则，可以确保配置表系统高效、稳定、易维护。
